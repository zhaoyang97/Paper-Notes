# From Narrow to Panoramic Vision: Attention-Guided Cold-Start Reshapes Multimodal Reasoning

**会议**: ICLR 2026  
**arXiv**: [2603.03825](https://arxiv.org/abs/2603.03825)  
**代码**: https://github.com/lrlbbzl/Qwen-AVAR (有)  
**领域**: 多模态VLM / LLM推理  
**关键词**: 视觉注意力, 多模态推理, cold-start, 注意力引导训练, GRPO

## 一句话总结
发现多模态 LLM 的推理性能与视觉注意力分数（VAS）高度相关（r=0.96），提出 AVAR 框架通过视觉锚定数据合成、注意力引导训练目标和视觉锚定奖励塑造三个阶段提升 VAS，在 77 个基准上平均提升 7%。

## 研究背景与动机

1. **领域现状**：多模态 LLM（如 Qwen2.5-VL）在推理任务上取得显著进展，但研究发现它们的推理过程往往"懒于看图"——模型倾向于关注系统 token 而非视觉 token。
2. **现有痛点**：多模态 cold-start 训练后，模型的 Visual Attention Score（VAS）并未提升，甚至下降。VAS 衡量推理 token 对视觉 token 的注意力占比，低 VAS 意味着模型在推理时没有充分利用图像信息。
3. **核心矛盾**："Lazy Attention Localization"——模型学会了用文本描述和系统指令中的线索"偷懒"推理，而不是真正去看图像中的关键信息。
4. **本文要解决什么？** 如何在 cold-start 和 RL 阶段强制模型增加对视觉 token 的注意力？
5. **切入角度**：发现 VAS 与推理性能正相关（r=0.96），直接优化 VAS 作为训练信号。
6. **核心idea一句话**：通过注意力层面的显式监督——增强图像注意力 + 抑制系统注意力——迫使模型"回头看图"。

## 方法详解

### 整体框架
三阶段训练：(1) 用视觉锚定反射数据（VARD）进行 cold-start SFT，(2) 在 SFT 中加入注意力引导训练目标（AGTO），(3) 在 GRPO 强化学习中加入视觉锚定奖励（VARS）。

### 关键设计

1. **视觉锚定反射数据合成（VARD）**:
   - 做什么：合成包含"回头看图"指令的高质量训练数据
   - 核心思路：三阶段合成——Stage 1 用 Gemini 2.5-Pro 生成高精度视觉描述，Stage 2 用 Qwen3-235B 生成带反思的推理链，Stage 3 用 Qwen3-32B 插入视觉锚定指令（如"回头看三角形的位置"）。
   - 设计动机：数据层面引导模型在推理中显式引用图像，类比于 Chain-of-Thought 的视觉版本。

2. **注意力引导训练目标（AGTO）**:
   - 做什么：在 SFT 损失基础上增加两个注意力正则项
   - 核心思路：(a) 图像增强损失 L_enhance-img = 对所有层和注意力头，最大化推理 token 对图像 token 的平均注意力的对数；(b) 系统抑制损失 L_suppress-sys = 最小化推理 token 对系统 token 的平均注意力的对数。总损失 = L_LM + 0.15 * L_enhance + 0.15 * L_suppress。
   - 设计动机：直接在注意力图上施加约束，比间接通过数据增强更直接有效。Cold-start 后 VAS 从 7.5 提升到 13.8。

3. **视觉锚定奖励塑造（VARS）**:
   - 做什么：在 GRPO 强化学习中，将 VAS 作为额外奖励信号
   - 核心思路：对于正确回答，奖励 = r_accuracy + 0.3 * r_visual + 0.1 * r_format。r_visual 等于推理 token 对图像 token 与系统 token 的注意力比值。只在回答正确时给视觉奖励，避免强化错误的视觉关注模式。
   - 设计动机：SFT 阶段只能让模型模仿训练数据中的注意力模式，RL 阶段通过奖励信号让模型自主探索更好的视觉注意力策略。

## 实验关键数据

### 主实验（77 个基准平均）

| 模型 | MathVision | MMMU | HallusionBench | 总平均 |
|------|-----------|------|---------------|-------|
| Qwen2.5-VL-7B | 25.2% | 58.1% | 50.7% | 49.1% |
| **AVAR-Thinker** | **37.4%** | **63.8%** | **59.5%** | **56.1%** |
| 提升 | +12.2% | +5.7% | +8.8% | +7.0% |

### 消融实验

| 组件 | VAS | 性能 |
|------|-----|------|
| Baseline | 7.5 | 49.1% |
| +VARD 数据 | 10.1 | 51.0% |
| +AGTO 注意力引导 | 13.8 | 52.6% |
| **+VARS 奖励塑造** | **18.9** | **56.1%** |

### 关键发现
- VAS 与推理性能的 Pearson 相关系数高达 0.96，说明视觉注意力是多模态推理的关键瓶颈
- VARS（RL 阶段）贡献最大（+3.5%），说明 RL 比 SFT 更适合优化注意力模式
- 在数学推理和幻觉检测上提升最显著，这些任务最依赖视觉信息
- 三个组件互补且递进，缺一不可

## 亮点与洞察
- **VAS 作为可优化指标**：将抽象的"模型是否在看图"量化为具体可微的 VAS 指标，并直接优化它。这种"先度量再优化"的思路值得借鉴。
- **注意力层面的显式约束**：大多数方法只在输入/输出层面优化，本文直接在注意力图上施加约束，更直接。
- **三阶段递进设计**：数据(VARD) -> 训练目标(AGTO) -> 奖励(VARS)，从浅到深地引导模型增加视觉注意力。

## 局限性 / 可改进方向
- 注意力引导损失对所有层和头施加相同约束，不同层/头的最优注意力分布可能不同
- VARD 数据依赖 Gemini 2.5-Pro 和 Qwen3-235B，合成成本较高
- 只在 Qwen2.5-VL-7B 上验证，对更大模型的效果未知
- VAS 指标假定更高的视觉注意力总是更好，但某些纯文本推理步骤可能不需要视觉注意力

## 相关工作与启发
- **vs Qwen2.5-VL**: 本文以其为基座模型，证明了 cold-start 训练的不足
- **vs GRPO**: 本文在标准 GRPO 上增加了视觉注意力奖励，是 RLVF 的有趣探索
