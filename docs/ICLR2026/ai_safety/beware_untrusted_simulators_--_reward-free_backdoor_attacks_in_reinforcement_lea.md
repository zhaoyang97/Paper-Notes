# Beware Untrusted Simulators -- Reward-Free Backdoor Attacks in Reinforcement Learning

**会议**: ICLR 2026  
**arXiv**: [2602.05089](https://arxiv.org/abs/2602.05089)  
**代码**: 无  
**领域**: AI安全 / RL安全  
**关键词**: backdoor attack, reinforcement learning, simulator security, reward-free, supply chain attack  

## 一句话总结
提出 Daze 攻击——恶意模拟器开发者无需访问或修改智能体的奖励函数，仅通过操控状态转移来植入后门：智能体在触发状态下不执行目标动作时被迫执行随机动作（"眩晕"），从而在理论上保证攻击成功且隐蔽，并首次在真实机器人硬件上演示了 RL 后门攻击。

## 研究背景与动机
1. **领域现状**：RL 训练严重依赖模拟器（MuJoCo、PyBullet 等），研究者和工程师常使用第三方模拟器或云计算服务。模拟器在 RL 供应链中是一个受信任但未被充分审视的组件。
2. **现有痛点**：已有 RL 后门攻击（TrojDRL、SleeperNets、Q-Incept）需要对训练流程的深度控制——尤其需要读写智能体的奖励函数。但很多模拟器（如 MuJoCo、PyBullet）只接收动作、返回状态，奖励由外部计算，攻击者无法接触。
3. **核心矛盾**：所有已知 RL 后门攻击都依赖奖励操控来引导目标动作成为最优——如果攻击者既不能观察也不能修改奖励（模拟器场景），传统方法完全失效。
4. **本文要解决什么？** 在仅能修改状态返回值（不能碰奖励）的极度受限威胁模型下，如何高效、隐蔽地植入后门？
5. **切入角度**：在几乎所有有意义的 MDP 中，均匀随机采样动作都是次优的。攻击者可以利用这一普适性质——当智能体在触发状态下不执行目标动作时，强制其执行随机动作，使其收到更低的回报。
6. **核心idea一句话**：用"不执行目标动作→被迫随机行动→低回报"的因果链替代直接奖励操控，通过 transition manipulation 而非 reward manipulation 实现后门。

## 方法详解

### 整体框架
Daze 作为模拟器 wrapper 实现。正常情况下，模拟器按原始转移函数 $T$ 运行（概率 $1-\beta$）；以概率 $\beta$ 进入触发状态 $\delta(s)$。在触发状态：若智能体执行目标动作 $a^+$，执行"null transition"——按良性状态下的策略采样动作转移，智能体无感知代价；若不执行 $a^+$，进入"眩晕状态" $\phi(s)$，被强制执行随机动作若干步。

### 关键设计

1. **Daze 机制（无奖励后门）**：
   - 做什么：在不碰奖励的前提下让目标动作成为触发状态下的最优动作。
   - 核心思路：利用 Assumption 1（随机策略几乎对所有 MDP 都是次优的）。在触发状态中，执行 $a^+$ → 正常转移（无代价）；不执行 $a^+$ → 进入眩晕状态（$k \cdot \mathcal{L}_{adv}(a, a^+)$ 步随机动作），期间奖励由真实物理计算但动作由攻击者随机采样 → 低回报。
   - 设计动机：智能体为最大化回报，自然学会在触发状态执行 $a^+$。奖励完全由外部正常计算，攻击者零接触。

2. **理论保证**：
   - Theorem 1：在 $M'$ 中的最优策略 $\pi^*$ 必然满足 $\mathcal{L}_{adv}(\pi^+(δ(s)), a^+) = 0$（攻击必然成功）。
   - Theorem 2：$M'$ 的最优策略在良性状态下也是 $M$ 的最优策略（隐蔽性保证）。
   - 直觉：因为最优策略总是执行 $a^+$，从不进入眩晕状态，所以在良性状态下的行为与未中毒完全相同。

3. **连续动作空间适配**：
   - 做什么：将离散域的"精确匹配 $a^+$"推广为连续域的"$\ell_\infty$ 距离小于阈值 $\tau$"。
   - 眩晕时长与误差成比例：$\lceil k \cdot \mathcal{L}_{adv}(a, a^+) \rceil$ 步，提供更密的梯度信号。

### 实现
作为环境 wrapper（Algorithm 1），3 个逻辑分支：Benign Routine → Trigger Routine → Daze Routine。只需攻击者选择触发函数 $\delta$、眩晕函数 $\phi$、目标动作 $a^+$、中毒率 $\beta$。

### 连续动作空间（MuJoCo + 自定义机器人环境）

|------|------|-----|---------------|-----------|
| HalfCheetah | **Daze** | **92.4%** | **1627.8** | 1736.3 |
| | TrojDRL-C | 20.3% | 604.4 | |
| | SleeperNets-C | 3.3% | 1511.3 | |
| Hopper | **Daze** | **94.1%** | **2321.5** | 2085.4 |
| | TrojDRL-C | 25.6% | 984.9 | |
| Intersection(真实机器人) | **Daze** | **92.3%** | **226.7** | 238.4 |

### 离散动作空间（Atari）
| 环境 | Daze | Q-Incept | TrojDRL | SleeperNets |
|------|------|----------|---------|-------------|
| Q*bert | 99.3% | 100% | 88.2% | 100% |
| Frogger | 99.9% | 99.2% | 95.7% | 100% |
| Pacman | 99.4% | 100% | - | - |
| Breakout | 97.6% | 100% | - | - |

### 关键发现
- Daze 在连续和离散动作空间中均达到 >92% ASR，同时 benign return 与未中毒模型几乎无差异。
- TrojDRL-C 和 SleeperNets-C 在连续域中 ASR 显著低于 Daze（3-65%），说明传统奖励操控方法难以扩展到连续域。
- 首次在真实 Turtlebot2 和 Fetch 机器人上成功触发后门行为——被触发的 Turtlebot 直线加速导致碰撞，Fetch 机器人急转弯导致物品掉落。
- 仅 1% 中毒率 + 仅 0.3% 额外眩晕步（Hopper），攻击极其隐蔽。

## 亮点与洞察
- **最受限威胁模型下的最强攻击**：Daze 在无奖励访问的极度受限场景下，反而超越了需要完全奖励控制的先前方法（连续域）。这一反直觉结果的原因是：transition manipulation 比 reward manipulation 更直接地影响策略学习。
- **真实硬件演示的冲击力**：首次在实体机器人上展示 RL 后门攻击——训练时在模拟器中被植入后门的策略，在 sim-to-real 迁移后仍然有效，证明这不仅是理论威胁。
- **理论简洁而有力**：核心思想极为简单（随机 < 最优），却推导出了严格的攻击成功和隐蔽性保证。

## 局限性 / 可改进方向
- Assumption 1 要求随机策略非最优——在完全对称环境（如左右等价的迷宫）中可能违反，虽然实际影响小。
- 触发函数 $\delta$ 的设计仍需人工选择；论文使用人工标志位或棋盘格图案，更隐蔽的触发设计有待研究。
- 仅在 PPO、TD3、DQN 上测试，未涉及 SAC 或 model-based RL。
- 防御方面仅提出问题和动机，未提出具体检测方法。

## 相关工作与启发
- **vs TrojDRL (Kiourti et al., 2019)**：需要读写奖励访问，在连续域 ASR 仅 20-65%，且降低 benign return。Daze 不碰奖励却达 92%+ ASR。
- **vs SleeperNets (Rathbun et al., 2024)**：需要 RAM 级外循环访问，威胁模型更强但更不现实。在连续域 ASR 极低（0-7.5%）。
- **vs Q-Incept (Rathbun et al., 2025)**：离散域表现接近，但 Q-Incept 无法扩展到连续域。
- 对 RL 安全的启示：仅信任奖励函数（externally computed）不够——模拟器的 transition 同样可能被恶意操控，需要对 RL 训练流程的所有组件进行安全审查。
