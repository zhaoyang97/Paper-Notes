# CARE: Towards Clinical Accountability in Multi-Modal Medical Reasoning with an Evidence-Grounded Agentic Framework

**会议**: ICLR 2026  
**arXiv**: [2603.01607](https://arxiv.org/abs/2603.01607)  
**代码**: https://xypb.github.io/CARE-Project-Page/ (有)  
**领域**: 医学图像 / 多模态VLM / Agent  
**关键词**: 医学VQA, 证据引导, Agent框架, 分割定位, 临床可解释性

## 一句话总结
提出 CARE Agent 框架，将医学 VQA 分解为实体提议、指称分割和证据引导推理三个专家模块，通过 GPT-5 作为动态协调器，在医学 VQA 基准上以 77.54% 准确率超越 32B 模型。

## 研究背景与动机

1. **领域现状**：多模态大模型在医学 VQA 上取得进展，但缺乏临床可追溯性——模型给出答案但不能精确指出"证据在哪里"。
2. **现有痛点**：端到端 VLM 是黑盒，临床医生无法验证模型"看了什么"。现有方法的推理过程不可审计，无法满足医疗场景的问责要求。
3. **核心矛盾**：更大的端到端模型虽然准确率更高，但可解释性更差；小模型可解释但准确率不足。
4. **本文要解决什么？** 如何在提升准确率的同时，让每个推理步骤都有可追溯的视觉证据？
5. **切入角度**：将 VQA 拆分为可审计的子步骤——先找实体，再分割定位，最后带着证据推理。
6. **核心idea一句话**：分而治之——用三个专家模块产生可追溯的证据链，用 Agent 协调器动态编排。

## 方法详解

### 整体框架
三阶段管道：(1) InternVL3-2B 提议相关医学实体，(2) SA-Med-2D 为每个实体生成分割掩码，(3) 根据分割结果构建三种视觉证据（裁剪、掩码、全局指标）送入 VQA 模型推理。GPT-5 作为协调器动态规划工具调用和迭代审查。

### 关键设计

1. **实体提议（RLVR 微调）**:
   - 做什么：从问题和图像中提议相关的医学实体
   - 核心思路：用 InternVL3-2B 微调，训练时用 Kuhn-Munkres 匹配计算预测实体与GT实体的相似度奖励，DAPO 优化。
   - 设计动机：小模型（2B）足以做实体提议，且推理速度快。RLVR 比 SFT 更好地学习开放式实体提议。

2. **指称分割**:
   - 做什么：给定文本描述的实体，在医学图像中分割出对应区域
   - 核心思路：基于 SA-Med-2D，增加生物医学文本编码器。输出置信度 C(M) = 1 - Entropy(M)/log(2)，低置信度掩码被过滤。
   - 设计动机：分割掩码是证据链的关键——它精确指出模型在"看哪里"。

3. **证据引导 VQA**:
   - 做什么：利用三种视觉证据增强推理
   - 三种证据：(a) 裁剪——放大实体区域的细节，(b) 二值掩码——叠加在原图上标出区域，(c) 全局指标——如面积比、亮度等数值特征。
   - 设计动机：不同类型的问题需要不同粒度的证据，三种互补。

4. **动态协调器（GPT-5）**:
   - 做什么：规划工具调用序列，迭代审查 CoT 和答案
   - CoT-答案审查：+5.1% 改进，说明迭代精炼对医学推理很重要。

## 实验关键数据

### 主实验（4 个医学 VQA 基准）

| 方法 | 参数量 | OMVQA | VQA-RAD | SLAKE | Overall |
|------|-------|-------|---------|-------|---------|
| Lingshu-32B | 32B | 83.97 | 64.75 | 82.25 | 72.29% |
| CARE-Flow-B | 10B | 96.17 | 63.64 | 83.21 | 74.91% |
| **CARE-Coord-B** | **10B** | **97.97** | **68.29** | **83.11** | **77.54%** |

### 消融实验
- 视觉证据组合：全部三种 > 任何子集（+2.5%）
- CoT 审查：+5.1%
- RLVR vs SFT：RLVR +9.6%

### 关键发现
- 10B 的 CARE 超越 32B 的端到端模型，证明了模块化的优势
- GPT-5 协调器比微调的 InternVL3-8B 协调器好 5%+
- 实体提议的 RLVR 训练显著优于 SFT（+9.6%）

## 亮点与洞察
- **临床可追溯性**：每个推理步骤都有视觉证据支撑，满足医疗场景的可审计需求。
- **小模型+工具 > 大模型**：10B 参数的模块化系统超越 32B 端到端模型，证明了 agent 范式在医疗 AI 中的潜力。

## 局限性 / 可改进方向
- 依赖 GPT-5 作为协调器，部署成本高且有 API 延迟
- 分割模型的质量直接影响证据quality，对罕见病变可能分割不准
- 三阶段管道的错误会累积

## 相关工作与启发
- **vs Lingshu-32B**: 端到端大模型，本文用更小模型 + agent 超越之
- **vs SA-Med-2D**: 提供分割能力，本文将其集成为证据生成组件
