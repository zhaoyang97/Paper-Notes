# Gradient-Sign Masking for Task Vector Transport Across Pre-Trained Models

**会议**: ICLR 2026  
**arXiv**: [2510.09658](https://arxiv.org/abs/2510.09658)  
**代码**: [GitHub](https://github.com/fillo-rinaldi/GradFix)  
**领域**: 自监督学习 / 模型合并 / 迁移学习  
**关键词**: task vector, model merging, gradient masking, foundation model, transfer learning

## 一句话总结
提出 GradFix 方法，利用目标模型的梯度符号作为掩码过滤源模型的任务向量，仅保留与目标损失景观对齐的分量，在少样本下实现跨预训练模型的任务知识迁移，并提供一阶下降保证。

## 研究背景与动机

1. **领域现状**：深度学习范式已转向微调大型预训练模型。Task Arithmetic 表明任务向量可以通过线性操作组合来诱导新能力。
2. **现有痛点**：当基础模型更新版本时，之前的微调工作无法直接复用。直接将旧模型的任务向量加到新模型上效果差，因为参数空间不对齐。
3. **核心矛盾**：任务向量包含有价值的任务信息，但跨预训练模型迁移时与目标损失几何不对齐。
4. **本文要解决什么？** 高效识别并过滤任务向量中对目标模型有害的分量。
5. **切入角度**：借鉴 SignSGD 的洞察——梯度的符号是下降方向的鲁棒代理。
6. **核心idea一句话**：用目标模型的梯度符号做掩码，保留任务向量中与目标反梯度方向一致的分量。

## 方法详解

### 整体框架
给定源模型的任务向量 $\tau_A = \theta_A^{ft} - \theta_A$ 和目标模型 $\theta_B$，计算 $\theta_B$ 上少量样本的梯度，构建符号掩码，过滤 $\tau_A$ 后应用到 $\theta_B$。

### 关键设计

1. **梯度符号掩码**:
   - 做什么：逐坐标过滤任务向量
   - 核心思路：$m_i = \mathbb{1}\{\text{sign}(\tau_{A,i}) = \text{sign}(-g_i)\}$
   - 一阶保证：$\mathbf{g}^\top \delta^A = -\alpha \sum_i m_i |g_i||\tau_{A,i}| \leq 0$

2. **少样本梯度估计**:
   - 做什么：用极少量标注样本估计梯度符号
   - 核心思路：对每个坐标用多数投票法
   - 理论保证（Hoeffding 不等式）：准确率随样本数指数收敛

### 训练策略
不需要额外微调——只需几步梯度计算，构建掩码，一步应用。

## 实验关键数据

### 主实验：跨预训练模型迁移

| 方法 | 平均效果 | 说明 |
|------|---------|------|
| $\theta_B$ zero-shot | 基线 | 无微调 |
| $\theta_B + \tau_A$（直接加） | 常降低 | 有害分量 |
| TransFusion | 中等 | 排列对齐 |
| **GradFix** | **显著提升** | 梯度掩码过滤 |
| $\theta_B$ fine-tune (上界) | 最佳 | 全量微调 |

### 消融实验

| 配置 | 效果 |
|------|------|
| 无掩码 | 常降低性能 |
| 均值替代多数投票 | 较差 |
| 增加样本数 | 持续提升至饱和 |
| 多任务合并 | 也有提升 |

### 关键发现
- 极少量样本（每类几个）即可估计可靠的梯度符号
- 在视觉（CLIP）和语言（T5/FLAN-T5）基准上均有效
- GradFix 同样提升多任务和多源模型合并性能

## 亮点与洞察
- 理论优雅——一阶下降保证推导简洁明了
- 极度轻量——不需要任何微调，计算成本几乎可忽略
- 多数投票法从分布式训练文献中借鉴，跨领域启发

## 局限性 / 可改进方向
- 只保留符号对齐分量，丢弃了所有不对齐的
- 缩放因子 $\alpha$ 需要验证集调参
- 假设源和目标架构完全相同

## 相关工作与启发
- **vs TransFusion**: 用排列对齐做 rebasin，更复杂但效果不如 GradFix
- **vs TIES-Merging**: 处理符号冲突但在同一预训练模型内
- **vs SignSGD**: 在分布式训练中用符号压缩，本文迁移到迁移学习
