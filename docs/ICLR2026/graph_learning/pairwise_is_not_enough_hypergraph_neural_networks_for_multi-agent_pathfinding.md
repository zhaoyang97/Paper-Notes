# Pairwise is Not Enough: Hypergraph Neural Networks for Multi-Agent Pathfinding

**会议**: ICLR2026  
**arXiv**: [2602.06733](https://arxiv.org/abs/2602.06733)  
**代码**: [GitHub](https://github.com/proroklab/HMAGAT)  
**领域**: llm_agent / multi-agent pathfinding  
**关键词**: MAPF, 超图神经网络, 注意力机制, 模仿学习, 群体交互  

## 一句话总结
提出 HMAGAT，用有向超图注意力网络替代 GNN 的成对消息传递来建模多智能体路径规划中的群体交互，仅用 1M 参数和 1% 训练数据即超越 85M 参数的 SOTA 模型。

## 背景与动机
1. 多智能体路径规划 (MAPF) 要求多个智能体无碰撞地到达各自目标，最优求解是 NP-hard
2. 现有学习方法（GNN、Transformer）仅建模成对 (pairwise) 交互，无法捕获多智能体同时交互的群体动态
3. 在高密度场景中，GNN 的注意力稀释问题尤为严重：大量无关智能体稀释了关键交互的注意力权重
4. MAPF 本质上是群体问题——只有建模全局联合状态空间才能达到最优性和完备性
5. 超图 (hypergraph) 天然适合建模群体交互，但尚未被应用于复杂、强耦合的 MAPF 场景
6. 现有超图方法仅处理小规模 (~10 agents) 问题，能否扩展到大规模高耦合场景是开放问题

## 方法详解
**HMAGAT 架构**：CNN 编码器 → 超图注意力网络 (HGNN) 层 × 3 → MLP 解码器

- **有向超图设计**：单头多尾结构，多个尾节点（影响者）→ 单个头节点（被影响者）
- **双层注意力机制**：(1) 尾节点 → 超边的注意力聚合生成超边表示；(2) 超边 → 头节点的注意力聚合更新节点特征
- **超图生成策略**：
  - Lloyd 超图：基于 Voronoi 划分 + 软边界实现重叠分组
  - k-means 超图：随机点扩散 + 聚类，复杂度 O(k|V|)，适合大图
  - 最短距离超图：基于智能体间最短路径距离构建

**训练流程**：
- 用 lacam3 专家求解器在 21K 实例上收集示范轨迹（MAPF-GPT 用 3.75M 实例）
- 交叉熵损失 + DAgger 在线专家 + 后训练质量提升
- RL 温度采样模块动态调整 softmax 温度 τ∈[0.5, 1.0]

## 实验关键数据
| 指标 | HMAGAT (1M) | MAPF-GPT (85M) |
|------|------------|----------------|
| 参数量 | ~1M (1.2%) | 85M |
| 训练数据 | 21K (~1%) | 3.75M |
| Dense Warehouse 成功率 | **75%+** | <11% |
| ost003d 大地图 | 可扩展 | 无法扩展 |

- HGNN vs GNN 直接对比：Dense Warehouse 成功率 39.8% vs 2.3%
- 消融：每个组件（HGNN, 后训练, 温度采样）均带来提升
- 注意力分析：HGNN 在高注意力区间保持更多智能体，避免 GNN 的注意力稀释

## 亮点
- **归纳偏置 > 数据量/参数量**：1% 参数 + 1% 数据胜过 85M 大模型，说明正确的结构先验至关重要
- 注意力稀释问题的形式化分析和实验验证
- 手工构造场景展示 GNN 的群体交互建模失败和 HGNN 的修复

## 局限性 / 可改进方向
- 超图生成策略依赖预设的通信半径和颜色数量，未自适应学习
- 仅在网格环境上评估，未拓展到连续空间或更复杂拓扑
- 模仿学习仍依赖专家求解器质量

## 与相关工作的对比
- vs MAGAT (GNN)：用超图层替代图层，保留其他组件，直接对比显示超图显著优于图
- vs MAPF-GPT：GPT 式大模型在大地图上无法扩展，HMAGAT 以极少资源取得竞争性能
- vs 已有超图方法：首次将超图应用于大规模强耦合 MAPF（非小规模简单场景）
