# Select, Read, and Write: A Multi-Agent Framework of Full-Text-based Related Work Generation

**会议**: ACL 2025  
**arXiv**: [2505.19647](https://arxiv.org/abs/2505.19647)  
**代码**: https://github.com/1190200817/Full_Text_RWG (有)  
**领域**: LLM Agent / 学术写作  
**关键词**: 相关工作生成, 多Agent框架, 全文阅读, 图感知选择, 工作记忆

## 一句话总结
提出 Select-Read-Write 三 Agent 协同框架，通过图感知的阅读顺序决策和共享工作记忆机制，实现基于论文全文（而非摘要）的 Related Work 自动生成，在 Llama3-8B / Claude-3-Haiku / GPT-4o 三个基座模型上均取得一致提升，Citation Graph 策略效果最优。

## 研究背景与动机

1. **领域现状**：自动 Related Work 生成（RWG）是学术写作辅助的重要任务，现有方法因上下文窗口限制，大多仅使用摘要、Introduction/Conclusion 或检索片段作为输入。
2. **现有痛点**：
   - **信息不足导致幻觉**：仅用摘要无法捕捉方法细节和论文间深层关系，模型频繁产生误解和幻觉
   - **论文间关系缺失**：现有方法对每篇参考文献独立描述，缺乏论文间的对比和关联分析，生成的 RWS 结构松散
   - **长文本处理瓶颈**：多篇全文总 token 数动辄 70K+，即使 128K 窗口的 LLM 直接塞入全文效果也会下降
3. **核心矛盾**：需要全文级别的深度理解来避免幻觉，但全文输入又超出模型有效处理范围；需要建模论文间关系，但现有隐式图结构方法效果有限。
4. **切入角度**：将阅读过程拆解为"选择-阅读-写作"三个独立 Agent，通过迭代阅读+共享记忆解决上下文窗口限制；通过显式图结构约束阅读顺序来捕捉论文间关系。
5. **核心idea一句话**：用图约束的多 Agent 迭代阅读框架替代一次性全文输入，在控制记忆大小的同时保留全文信息和论文间关系。

## 方法详解

### 整体框架
输入为 citing paper $C$ 和一组参考文献 $\mathcal{R} = \{R_1, ..., R_N\}$，每篇论文包含完整章节 $R_i = \{s_{i,1}, ..., s_{i,L_i}\}$。框架包含三个 Agent：

- **Selector**：根据所有论文摘要、当前工作记忆 $M_{t-1}$ 和阅读历史 $H_{t-1}$，决定下一步阅读哪篇论文的哪个章节 $(R_t, s_t)$
- **Reader**：消化选定章节内容，更新共享工作记忆 $M_t$（JSON 格式，上限 4096 token）
- **Writer**：迭代结束后，基于最终记忆 $M_T$ 和阅读历史 $H_T$ 生成 Related Work Section

Selector 和 Reader 交替迭代工作，直到 Selector 输出终止信号 `<End>`。阅读历史 $H$ 记录已读 (论文ID, 章节名) 序列以防止重复阅读。

### 关键设计

1. **工作记忆机制（Working Memory）**:
   - 做什么：维护一个 JSON 格式的共享记忆，存储阅读过程中提取的关键信息
   - 核心思路：设置显式大小约束（4096 token），每一步 Reader 更新记忆时需要重新整理内容，丢弃不相关信息，保留任务相关的关键知识
   - 设计动机：解决多篇全文总长度远超上下文窗口的问题，将"一次性全文输入"转化为"迭代阅读+记忆压缩"，类似人类做笔记的阅读方式

2. **图感知选择器（Graph-Aware Selector）**:
   - 做什么：利用论文间关系图约束阅读顺序，使模型在相关论文间跳转阅读
   - 核心思路：构建两种图——**共现图**（$G_{co}$：两篇论文在某篇参考文献的 RWS 中被共同引用则连边）和**引用图**（$G_{ci}$：论文间存在引用关系则连边）。Selector 在当前论文的一跳子图 $G_{t-1}$ 内做决策，要么继续读当前论文，要么跳转到图中相邻论文
   - 设计动机：图结构显式编码了论文间关系，约束阅读顺序使模型在处理相关论文时能自然建立对比和关联，而非孤立理解每篇论文。引用图比共现图效果更好，因为共现图连通度过高，约束力不足

3. **五种选择器策略对比设计**:
   - 做什么：提供从简单到复杂的 5 种阅读策略——Sequential Reading (SR)、Random Reading (RR)、Vanilla LLM-Based、Graph-Co、Graph-Ci
   - 核心思路：SR 按论文 ID 顺序逐章阅读；RR 打乱顺序减少位置偏差；Vanilla 用 LLM 自主决策阅读顺序；Graph-Co/Ci 在图约束下决策
   - 设计动机：系统对比不同阅读策略的效果，验证图约束的必要性。实验证明性能排序为 SR < RR < Vanilla < Graph-Co < Graph-Ci

### 训练策略
无需训练，纯 prompt-based 多 Agent 框架，直接使用现有 LLM 作为三个 Agent 的基座模型。Writer 通过 prompt 指导（避免孤立描述、要解释论文间关系、相似研究归组）+ in-context learning 示例来提升生成质量。

## 实验关键数据

### 主实验
在 OARelatedWork 数据集上测试（开放领域，测试集平均输入 70K token）。评估使用图指标（边数/度/聚类系数）和 LLM 评估（Coverage/Logic/Relevance，5分制）。

| 模型配置 | Avg. Edges | Clustering Coeff. | Overall (LLM) | 相比基线提升 |
|----------|-----------|-------------------|---------------|------------|
| GPT-4o (Abstract) | 1.180 | 0.057 | 3.69 | baseline |
| GPT-4o + GO (Retrieval) | 1.611 | 0.096 | 3.75 | +1.6% |
| GPT-4o (Full, 128K) | 1.244 | 0.136 | 3.68 | -0.3% |
| **GPT-4o Graph-Ci (Ours)** | **2.125** | **0.128** | **3.87*** | **+3.2%** |
| Claude-3-Haiku + GO | 2.308 | 0.100 | 3.52 | baseline |
| **Claude-3-Haiku Graph-Ci** | **3.240** | **0.231** | **3.61*** | **+2.6%** |
| Llama3-8B + GO | 1.511 | 0.054 | 3.29 | baseline |
| **Llama3-8B Graph-Ci** | **1.410** | **0.154** | **3.44*** | **+4.6%** |

### 消融实验（选择器策略对比）

| 策略 | 性能趋势 | 说明 |
|------|---------|------|
| Sequential Reading (SR) | 最低 | 固定顺序引入位置偏差 |
| Random Reading (RR) | > SR | 打乱顺序缓解偏差 |
| Vanilla LLM | > RR | LLM 自主决策更智能 |
| Graph-Co | > Vanilla | 共现图约束捕捉隐式关系 |
| Graph-Ci | **最优** | 引用图约束最直接有效 |

### 关键发现
- **全文 > 摘要**：全文输入模型在所有基座和指标上均优于摘要输入模型，验证了全文 RWG 的必要性
- **直接塞入全文反而差**：Claude-3-Haiku 直接输入全文 Overall 下降 9.3%，说明长窗口 LLM 的有效利用仍是问题
- **图约束一致有效**：Graph-Ci 在三个不同能力层级的基座模型上都取得统计显著的提升（p<0.05）
- **人工评估验证**：5名 AI 方向研究生的 pairwise 评估中，Graph-Ci 在 Logic 维度 win rate 最高（61-68%），与自动指标一致
- **弱模型对策略不敏感**：Llama3-8B 的五种策略差异很小，可能因其基础能力不足导致对阅读顺序不敏感

## 亮点与洞察
- **将人类阅读行为建模为多 Agent 系统**：Select-Read-Write 对应学术阅读中"选论文-精读-写综述"的流程，工作记忆类似人类做笔记，非常直觉
- **图约束是关键差异化因素**：不是简单的多 Agent 对话，而是通过论文关系图引导 Agent 在相关论文间"游走"阅读，使生成的 RWS 自然包含论文间对比
- **实验设计严谨**：纯 prompt 框架搭配开源 Llama3 保证可复现性，5种策略的 ablation + 人工评估验证了每个设计选择

## 局限性 / 可改进方向
- **与人类水平仍有大差距**：人类 RWS 平均边数 9.48 vs 最佳模型 3.24，模型仍倾向于过度展开不重要的参考文献
- **需要预提供参考文献集**：用户需手动检索、筛选论文，框架不包含自动论文检索模块
- **记忆容量受限**：4096 token 的工作记忆在处理大量参考文献时可能丢失关键信息，压缩过程中的信息损失未被量化
- **基座模型能力决定上限**：弱模型（Llama3-8B）即使用最优策略也不如中等模型的 baseline，框架对模型能力有较高要求

## 相关工作与启发
- **vs Li & Ouyang (2025)**：同为 LLM 时代的 RWG 方法，但 Li & Ouyang 仍基于摘要 + prompt engineering，本文通过全文阅读和图约束实现更深层的论文关系建模
- **vs RAG-based 方法**：检索方法（Greedy Oracle）选择关键句子输入，但可能遗漏上下文信息；本文的迭代阅读+记忆更新策略更灵活
- **vs 长上下文 LLM**：直接用 128K/200K 窗口塞入全文效果反而下降，说明有效的长文本处理需要结构化的阅读策略，而非盲目扩大窗口
