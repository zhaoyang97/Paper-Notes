# A Multi-Agent Framework for Mitigating Dialect Biases in Privacy Policy Question-Answering Systems

**会议**: ACL 2025  
**arXiv**: [2506.02998](https://arxiv.org/abs/2506.02998)  
**代码**: 待确认  
**领域**: LLM Agent / 公平性  
**关键词**: 方言偏差, 隐私政策QA, 多Agent框架, 标准美式英语, AAVE, 公平性, 零样本  

## 一句话总结

提出多 Agent 协作框架缓解隐私政策 QA 中的方言偏差——Dialect Agent 将方言查询翻译为标准美式英语（SAE）并验证意图保留，Privacy Policy Agent 利用领域专长生成答案，两者迭代协商至达成一致。在 PrivacyQA 和 PolicyQA 上将 GPT-4o-mini 零样本准确率从 0.394 提升至 0.601，方言间最大 F1 差距降低 82%。

## 背景与动机

隐私政策的复杂法律语言限制了用户理解自身数据权利。现有隐私 QA 系统对语言多样性漠不关心——非标准英语方言（如 AAVE、奇卡诺英语、原住民英语等）的使用者在这些系统中性能显著更差。EPIC 指出边缘化社区在数据收集和隐私滥用中受害最重。如果特定群体无法有效地提问保护隐私，他们就面临更大风险。

已有方言 NLP 研究（如 Multi-VALUE）聚焦通用任务，但未探索方言偏差如何在隐私 QA 这样的领域特定应用中表现。

## 核心问题

如何在不收集方言特定训练数据和不重训模型的前提下，缓解 LLM 在隐私政策 QA 中对非标准英语方言的性能差距？

## 方法详解

### 整体框架

两个专用 Agent 迭代协作：Dialect Agent + Privacy Policy Agent，参考人本设计（Human-Centered Design）原则。

### 关键设计

1. **Step 1 - Dialect Agent 翻译**：接收方言查询 + 方言语言学特征简介（语音/语法/词汇/文化层面），将查询翻译为 SAE 同时保留原始意图和文化细微差异。

2. **Step 2a - Privacy Policy Agent 回答**：接收翻译后的 SAE 查询 + 隐私政策片段，作为领域专家生成初始回答和理由。

3. **Step 2b/2c - Dialect Agent 评估+协商**：Dialect Agent 评估回答是否充分捕获原始方言查询的意图。若满意则接受（2c）；若发现不准或遗漏方言特有细微差异，则反馈给 Privacy Policy Agent 修正（2b）。最多迭代 2 次。

4. **无需重训/微调**：纯基于 prompt 的 Agent 协作，适用于任何 LLM。

### 训练策略

无训练。few-shot 版本每个 Agent 使用 8 个覆盖多方言/多题型/多场景的示例。

## 实验关键数据

### PrivacyQA（GPT-4o-mini）

| 方法 | SAE F1 | AAVE F1 | 平均 F1↑ | 最大差距↓ |
|------|--------|---------|---------|---------|
| Zero-shot | .394 | .344 | .335 | .093 |
| Few-shot | .605 | .573 | .565 | .058 |
| **Multi-agent-zero (ours)** | .601 | .588 | **.587** | **.025** |
| **Multi-agent-few (ours)** | .611 | .595 | **.598** | **.019** |

- 平均 F1 提升：0.335 → 0.587（多Agent零样本 vs 原始零样本）
- **方言间最大差距降低 82%**（.093 → .019，多Agent + few-shot）
- 零样本多Agent超越原始 few-shot 基线

### PolicyQA

| 方法 | 平均 F1↑ | 最大差距↓ |
|------|---------|---------|
| Zero-shot | .352 | 高 |
| **Multi-agent-zero** | **.464** | **显著降低** |

### 跨模型验证

- 在 Llama 3.1 上同样有效：零样本 .368 → 多Agent .469，最大差距从 .144 降至 .044

## 亮点

- **首次研究隐私 QA 的方言偏差**：将公平性问题引入领域特定 QA
- **多Agent协作缓解偏差**：不同于传统方言适应（需要方言特定训练），本方法仅需方言语言学描述即可
- **效果显著**：偏差降低 82%，同时整体准确率大幅提升
- **无需重训**：即插即用到任何 LLM，扩展性好

## 局限性 / 可改进方向

- **方言信息依赖外部知识**：需要为每种方言提供语言学特征描述
- **仅测试英语方言**：其他语言的方言差异可能更复杂
- **Agent 协商增加延迟**：最多 2 轮迭代，生产环境下可能太慢
- **评估数据集规模有限**：PrivacyQA 1750 题，PolicyQA 714 题
- **方言生成依赖 Multi-VALUE 规则转换**：合成方言可能不完全反映真实使用者的表达

## 与相关工作的对比

- **vs Multi-VALUE/DADA/TADA**：这些需要方言特定训练/微调；本方法仅需 prompt 描述
- **vs 一般 LLM 方言研究**：此前未有在隐私 QA 领域的方言偏差研究
- **vs LongAgent 等多Agent QA**：LongAgent 处理长文档，本文处理方言公平性——不同目标

## 启发与关联

- "给 LLM 提供关于用户群体的元知识"是缓解偏差的通用策略——不限于方言
- Dialect Agent 的翻译+验证双重角色设计值得借鉴——不只是翻译，还要验证回答是否保留了原始意图
- 隐私 QA 的可及性对边缘化社区有直接社会影响
