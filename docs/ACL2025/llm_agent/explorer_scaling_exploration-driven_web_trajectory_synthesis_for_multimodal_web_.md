# Explorer: Scaling Exploration-Driven Web Trajectory Synthesis for Multimodal Web Agents

**会议**: ACL 2025  
**arXiv**: [2502.11357](https://arxiv.org/abs/2502.11357)  
**代码**: https://osu-nlp-group.github.io/Explorer/ (有)  
**领域**: LLM Agent  
**关键词**: Web Agent, 轨迹合成, 多模态, 数据扩展, 多智能体

## 一句话总结
提出 Explorer——一个可扩展的多智能体 pipeline，通过自主网页探索和逐步精炼来合成大规模多模态 web 轨迹数据集（94K 成功轨迹，49K+ URL，720K 截图），训练的 Explorer-7B 在 Mind2Web-Live、MiniWob++ 等 benchmark 上达到甚至超过 GPT-4 水平。

## 研究背景与动机

1. **领域现状**：训练多模态 web agent 需要大规模高质量的轨迹级数据（任务→动作序列→截图），但人工标注成本极高，现有合成方法要么规模有限，要么多样性不足。
2. **现有痛点**：(a) 人工标注的轨迹数据（如 Mind2Web）规模小、覆盖网站少；(b) 已有的合成方法（如 AgentTrek）依赖教程式知识指导，受限于预定义任务模板；(c) 多数方法仅支持文本模态，忽略了视觉截图在 GUI 交互中的关键作用。
3. **核心矛盾**：Web 环境极其多样（49K+ URLs），如何在不依赖人工模板的情况下自动生成高质量、多样化的任务-轨迹对？
4. **本文要解决什么？** 设计一个无需人工任务模板、可大规模合成多模态 web 轨迹的自动化 pipeline。
5. **切入角度**：自底向上——先让 agent 在网页上自由探索执行动作，再从执行历史中提炼出高层任务描述，用验证器过滤不一致的轨迹。
6. **核心 idea 一句话**：用 4 个 LLM agent（提议→精炼→摘要→验证）自主探索网页并从低层动作自底向上合成高质量任务轨迹数据。

## 方法详解

### 整体框架
输入是大量网站 URL（来自 similarweb Top 100 + Tranco 列表），输出是 <任务描述, 动作序列, 截图序列> 三元组。4 个 GPT-4o 驱动的 agent 协作：Task Proposer 在首页提出抽象任务并执行第一步动作→Task Refiner 逐步精炼任务描述并预测下一步动作→Task Summarizer 从完整轨迹生成高层任务描述→Task Verifier 验证轨迹是否成功完成任务。

### 关键设计

1. **Task Proposer Agent**:
   - 做什么：给定网站首页（截图+accessibility tree），提出多样化的高层抽象任务，并执行第一个动作
   - 核心思路：不给定固定任务模板，让 LLM 根据网页内容自主提出合理任务。遇到 CAPTCHA、登录、支付等页面时自动终止
   - 设计动机：自底向上的探索比自顶向下的模板驱动更能覆盖真实网站的多样交互

2. **Task Refiner Agent**:
   - 做什么：在每步执行后，根据当前状态精炼任务描述，并预测下一步动作
   - 核心思路：接收初始/上一轮精炼的任务描述 + 动作历史，输出更新后的任务描述和下一步动作。确保精炼后的任务与已有动作历史一致
   - 设计动机：任务描述在探索过程中逐步具体化，避免一开始就过度具体化导致探索空间受限

3. **Task Summarizer**:
   - 做什么：处理完整的动作和截图历史，输出纯高层任务描述（只描述"做什么"不描述"怎么做"）
   - 设计动机：Refiner 输出的描述可能包含过程细节，Summarizer 提供干净的任务定义

4. **Task Verifier**:
   - 做什么：评估轨迹是否成功完成了任务
   - 核心思路：综合任务描述、动作历史、截图和最终页面 markdown，判断是否成功。不一致的轨迹被丢弃
   - 设计动机：自主探索不可避免会产生不连贯的轨迹，验证器是质量保障的关键；与人类判断的一致率为 81%

### 数据集规模
- 175K 总轨迹，94K 成功轨迹（通过验证）
- 49,494 个唯一 URL，720K 截图，33M 网页元素
- 平均每条轨迹 7.7 步，每个截图 46.3 个元素
- 成本仅 **$0.28/成功轨迹**，50 小时 60 并行进程完成

### 训练策略
- 在合成数据集上微调 Phi-3.5V (4B) 和 Qwen2-VL-7B
- 输入格式包含截图 + accessibility tree + 任务描述

## 实验关键数据

### 主实验

**Mind2Web-Live (83 tasks)**

| 模型 | Avg Step SR | Full Task SR |
|------|------------|-------------|
| GPT-4o | 58.5% | 25.3% |
| Qwen2-VL-7B (base) | 40.2% | 14.5% |
| **Explorer-7B** | **45.3%** | **19.3%** |
| Phi-3.5V (base) | 28.5% | 2.4% |
| **Explorer-4B** | **44.0%** | **18.1%** |

**MiniWob++ (46 tasks, zero-shot)**

| 模型 | Accuracy |
|------|---------|
| GPT-4 | 53.04% |
| **Explorer-7B** | **53.26%** |
| AgentTrek-7B | 45.28% |
| **Explorer-4B** | **46.74%** |

Explorer-7B 在 MiniWob++ 上匹敌甚至超过 GPT-4（53.26% vs 53.04%）。

### 消融实验

| 配置 | Full Task SR (Mind2Web-Live) |
|------|---------------------------|
| Explorer-4B (multimodal) | 18.1% |
| Phi-3-mini (text-only) | 13.3% |
| LLaVA-Mistral-7B | 4.8% |
| 25% data → 100% data | 逐步一致提升 |

### 关键发现
- **数据规模是关键驱动力**：25%→100% 数据量，所有指标持续提升
- **视觉模态至关重要**：去掉截图后 Full Task SR 从 18.1% 降至 13.3%（-4.8pp）
- **与 AgentTrek 对比**：同样 7B backbone，Explorer (54.3%) > AgentTrek (53.2%) 在 Multimodal-Mind2Web 上，说明自主探索合成比教程驱动合成更有效
- **小模型也能强**：Explorer-4B 已超过 AgentTrek-7B 和多个 7B 模型

## 亮点与洞察
- **自底向上的轨迹合成范式**：不预设任务模板，从探索中涌现任务。这种"先做再总结"的数据生成策略可迁移到其他需要大规模交互数据的场景（如 app 测试、机器人操作）
- **$0.28/轨迹的成本效率**：远低于人工标注（通常 $5-20/轨迹），使大规模数据合成变得可行
- **4 个 agent 的角色分工清晰**：Proposer→Refiner→Summarizer→Verifier 的流水线设计，每个环节可独立优化和替换

## 局限性 / 可改进方向
- **验证器准确率仅 81%**：约 19% 的判断与人类不一致，会引入噪声数据
- **依赖 GPT-4o 做合成**：closed-source API 成本仍不可忽视（175K 轨迹总成本估计 ~$26K+）
- **主要失败模式是 grounding error**：agent 在精炼阶段的接地动作与自然语言描述不匹配
- **登录限制**：大量需要认证的网站无法探索，数据覆盖有缺口
- 可改进：(a) 可以训练开源 LLM 替代 GPT-4o 做合成以降低成本；(b) 可以引入自我纠正机制处理 grounding error

## 相关工作与启发
- **vs AgentTrek**: AgentTrek 依赖教程知识指导任务生成，Explorer 采用自主探索，数据多样性更高
- **vs Mind2Web**: Mind2Web 是人工标注的静态数据集，Explorer 是自动合成的且可无限扩展
- **vs SeeAct**: SeeAct 使用 GPT-4 做推理但不做训练，Explorer 合成数据用于微调开源模型
