# Natural Language Processing in Support of Evidence-based Medicine: A Scoping Review

**会议**: ACL2025
**arXiv**: [2505.22280](https://arxiv.org/abs/2505.22280)
**代码**: [GitHub](https://github.com/bionlplab/awesome-nlp-in-ebm)
**领域**: llm_nlp
**关键词**: evidence-based medicine, NLP survey, clinical NLP, PICO extraction, clinical trial, systematic review

## 一句话总结

对 129 篇研究进行范围综述，系统梳理了 NLP 技术在循证医学（EBM）五步流程（Ask-Acquire-Appraise-Apply-Assess）中的应用，覆盖证据检索、PICO 提取、质量评估、证据合成、摘要生成、临床试验匹配等任务，指出了 LLM 时代的新机遇与挑战。

## 研究背景与动机

1. **EBM 的核心地位**：循证医学是现代医疗的基石，强调基于最佳可用科学证据做出临床决策，整合临床专业知识、患者价值观和最新研究数据。

2. **文献爆炸式增长**：医学文献数量庞大且增长迅速，人工筛选、评估和综合证据的成本极高，亟需自动化工具辅助。

3. **NLP 技术的演进**：从早期统计机器学习和 RNN/LSTM 到 Transformer 和 LLM，NLP 已在医学信息处理中取得显著进展，但缺乏系统性综述。

4. **LLM 带来新机遇**：GPT、LLaMA 等大模型在复杂任务（证据评估与合成、临床问答、临床试验识别）上展现出新能力，但也带来幻觉、可归因性等新挑战。

5. **覆盖面不足**：现有综述未全面覆盖 NLP 在 EBM 各环节的应用，且未充分反映 LLM 时代的最新进展。

6. **标准化需求**：缺乏跨 EBM 步骤的统一 NLP 任务分类框架和标准数据集，阻碍了方法的对比和进步。

## 方法详解

### 综述方法

遵循 PRISMA 指南，检索 PubMed、IEEE Xplore、ACM Digital Library、ACL Anthology 四个数据库，时间跨度 2019-2024。从初始 601 篇论文中经去重（-8）、标题摘要筛选（-386）、全文筛选（-88）后纳入 129 篇。两位标注者交叉验证，分歧由第三人裁决。

### EBM 五步 NLP 任务映射

| EBM 步骤 | 描述 | NLP 任务 |
|----------|------|----------|
| **Ask** | 搜索与筛选研究 | 问答、信息检索 |
| **Acquire** | 收集数据 | 命名实体识别与归一化、关系提取 |
| **Appraise** | 检查相关性、有效性 | 质量评估、证据排序与筛选、证据合成、证据摘要 |
| **Apply & Assess** | 实践应用与评估 | 临床试验识别与设计、问答、专科应用 |

### 各环节关键技术演进

**Ask（信息检索）**：从关键词搜索 → BioBERT 嵌入 → 生成式 AI（如 Bing AI、ChatGPT 加速系统文献搜索）。

**Acquire（实体提取）**：规则方法 → RNN/LSTM 序列分类 → Transformer 主导（SciBERT、PubMedBERT 用于干预提取；SrBERT 用于文献筛选）→ 知识图谱（图节点表示实体，边表示关系）。

**Appraise（质量评估与合成）**：自动化质量评估工具 → BioBERT/BlueBERT 微调分类 → BM25+BERT 重排 → 证据合成（复现 meta 分析）→ LLM 摘要（RAG+GPT-4 回答临床试验资格问题）。

**Apply & Assess**：专科应用（肿瘤学 CTM 准确率 92%、心脏病学混合模型准确率 87.3%）→ 临床试验匹配（AutoTrial、TrialGPT）→ 药物再利用 → 临床问答。

## 实验关键数据

### 文献统计

- 纳入论文：129 篇（2019-2024）
- 年度分布：2023 年达峰值
- 最常见任务：实体提取、分类、评估（基础性任务）
- 新兴任务：问答、质量评估（近年增长）

### 关键系统性能

| 系统 | 任务 | 性能 |
|------|------|------|
| CTM (Alexander 2020) | 肺癌临床试验筛选 | 准确率 92% |
| 混合模型 (Tun 2023) | 心衰患者资格评估 | 准确率 87.3%（4万患者） |
| TextRank | 证据摘要提取 | 优于 LexRank 和 LSA |
| SciBERT/BioBERT 重排 | 临床试验搜索 | 显著提升鲁棒性 |
| TriSum | 结构化理由摘要 | 双评分选择+课程学习 |

### 基准数据集分析

- 最常涉及任务：证据检索、证据提取、临床试验识别
- 显著缺口：证据合成与评估、问答领域缺乏专用医学数据集（如 CNN-DailyMail 用于摘要但非医学领域）
- 主要数据来源：PubMed 学术论文和 ClinicalTrials.gov

## 亮点与洞察

1. **EBM 5A 框架映射**：首次系统地将 NLP 任务映射到 EBM 的五步流程，提供了清晰的研究全景图
2. **技术演进梳理**：清楚呈现了从规则方法 → 统计ML → RNN → Transformer → LLM 的演进路径及各阶段的代表性工作
3. **实用资源**：开源了 awesome-nlp-in-ebm 资源库，含论文列表、数据集索引和元数据
4. **LLM 的双面性分析**：既肯定了 LLM 在复杂任务上的潜力，也指出了幻觉、缺乏来源归因和提示敏感性等关键局限

## 局限性

1. 仅覆盖英文论文，可能遗漏其他语言的重要研究
2. 时间范围限于 2019-2024，可能遗漏了奠基性早期工作
3. 检索数据库有限，可能存在遗漏
4. 综述性质决定了无法对各方法进行直接实验对比
5. 部分专科领域（泌尿科、肝病科）的 NLP 应用尚未覆盖
6. 对 LLM 时代（2023-2024）的覆盖受发表周期限制可能不够全面

## 相关工作与启发

- **与其他医学 NLP 综述的区分**：本综述独特之处在于以 EBM 五步流程为组织框架，而非以 NLP 技术或疾病领域为维度
- **未来方向**：整合真实世界数据（移动设备、社交媒体、基因组学）；少样本学习应对罕见病数据稀缺；构建证据合成/评估专用基准数据集；提升 NLP 模型的可解释性和公平性
- **启发**：EBM 是一个 NLP 技术高度集成的应用场景，涉及从 IE、IR、QA 到生成和推理的全栈能力，可作为评估通用 NLP 系统的综合测试床
