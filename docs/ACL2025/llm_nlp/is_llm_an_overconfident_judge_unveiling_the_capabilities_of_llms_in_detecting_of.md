# Is LLM an Overconfident Judge? Unveiling the Capabilities of LLMs in Detecting Offensive Language with Annotation Disagreement

**会议**: ACL 2025  
**arXiv**: [2502.06207](https://arxiv.org/abs/2502.06207)  
**代码**: [github.com/DUT-lujunyu/Disagreement](https://github.com/DUT-lujunyu/Disagreement)  
**领域**: LLM评估 / 内容审核  
**关键词**: 攻击性语言检测, 标注分歧, LLM裁判, 过度自信, 人机对齐, 置信度校准

## 一句话总结

系统评估了多个 LLM 在攻击性语言检测中面对标注分歧时的表现，发现 LLM 在标注者高度一致的样本上表现优异（GPT-4o F1 85.24%）但在低一致度样本上骤降至 57.06%，且模型对不确定样本表现出严重的过度自信；进一步通过 few-shot 和指令微调实验证明，在训练中引入分歧样本可同时提升检测准确率和人-AI 对齐度。

## 研究背景与动机

1. **领域现状**：LLM-as-a-Judge 范式日益流行，越来越多的研究和系统用 LLM 代替人工标注来评判文本内容。在攻击性语言检测（offensive language detection）领域，LLM 已被广泛应用于自动化内容审核。
2. **现有痛点**：现有评估几乎清一色采用二分类设定——用多数投票聚合标注后得到一个"金标准"标签，然后在此基础上评测 LLM。这种做法忽视了一个核心事实：人类标注者对同一文本的判断经常存在分歧，尤其是涉及文化背景、语言模糊性和个人感知差异的主观性任务。多数投票会丢失标注分歧的信息。
3. **核心矛盾**：LLM 的输出是确定性的（尤其在 temperature=0 时），而人类对攻击性语言的感知本质上是主观且不确定的。一个理想的裁判模型应当在标注者一致时给出高置信判断，在标注者分歧大时表达不确定性——但 LLM 是否具备这种"知道自己不知道"的能力，此前没有系统研究。
4. **本文要解决什么**：(RQ1) LLM 在不同标注一致度水平上的检测准确率如何？是否存在系统性的过度自信？(RQ2) 分歧样本在 few-shot 学习和指令微调中如何影响模型的决策行为？
5. **切入角度**：利用 MD-Agreement 数据集（10,753 条推文，每条 5 人标注）按标注一致度分层（A++ 全一致 / A+ 轻微分歧 / A0 弱一致），在同一数据上分层评估 LLM 的分类准确率和置信度，揭示"一致度-性能"的系统性关系。
6. **核心 idea 一句话**：按标注一致度分层剖析 LLM 裁判能力，揭示其在模糊样本上的过度自信缺陷，并证明分歧感知训练（disagreement-aware training）可有效缓解该问题。

## 方法详解

### 整体框架

本文不是提出一个新模型，而是设计了一套系统的评估框架来回答两个研究问题。整个研究分三个阶段：

1. **Zero-shot 分类性能评估**：在 12 个 LLM（6 闭源 + 6 开源）上进行 zero-shot 攻击性语言检测，按标注一致度分层报告准确率和 F1。
2. **置信度-一致度对齐分析**：通过 self-consistency 方法（5 个温度设置下多次采样取平均）估计 LLM 置信度，用 MSE 和 Spearman 相关系数衡量与人类标注一致度的对齐程度。
3. **分歧样本对学习的影响**：分别在 few-shot（GPT-4o）和指令微调（LLaMa3-8B）中，系统比较不同一致度级别的训练样本对模型性能和校准度的影响。

### 关键设计一：标注一致度分层评估框架

- **做什么**：将 MD-Agreement 数据集按 5 人标注的一致程度分为三档——A++（5/5 一致）、A+（4/5 一致）、A0（3/5 一致），每档再按攻击性 / 非攻击性细分为 O++/O+/O0 和 N++/N+/N0。
- **核心思路**：传统评估只报告一个总体 F1 值，掩盖了模型在不同难度样本上的差异化表现。分层评估将"标注确定性"作为一个新维度，使得性能下降的模式可量化、可对比。同时引入软标签（5 人硬标签的均值，范围 [0,1]）作为人类不确定性的连续度量。
- **设计动机**：人类标注分歧不是噪声而是样本固有属性（这一点已被 SemEval-2023 Task 11 独立验证），因此评估 LLM 时必须区分"容易案例"和"困难案例"。

### 关键设计二：Self-Consistency 置信度估计

- **做什么**：对每个样本，在 temperature={0, 0.25, 0.5, 0.75, 1} 下分别采样模型输出，将硬预测（0/1）在 5 次采样中取平均，得到 [0,1] 范围的置信度得分。
- **核心思路**：temperature=0 时 LLM 总是给出同一个确定性答案，无法观察其内在不确定性。通过引入温度扰动让模型"暴露"对不同样本的确定程度——如果模型在所有温度下都给相同答案，说明它很确定；如果答案随温度变化翻转，说明它不确定。
- **设计动机**：对于黑盒闭源模型（如 GPT-4o），无法获取内部 logits 或 token 概率，self-consistency 是最直接可行的不确定性估计方法。选用 MSE（衡量绝对偏差）和 Spearman ρ（衡量排序相关性）两个互补指标来全面评估对齐度。

### 关键设计三：分歧感知训练实验

- **做什么**：在 few-shot 和指令微调两种学习范式中，系统比较使用不同一致度级别训练样本的效果——单一级别（A++/A+/A0）和混合级别（A++/+ / A++/0 / A+/0 / A++/+/0）。
- **核心思路**：Few-shot 实验中，为每个一致度级别构造一对正负样本（一条攻击性 + 一条非攻击性）作为 prompt 示例；指令微调中，从每个级别抽取等量（1800条）样本进行微调。通过对比不同配置在 test set 上的准确率和 MSE，确定最优的训练数据组合策略。
- **设计动机**：如果 LLM 的过度自信源于训练数据中缺乏模糊样本，那么在学习过程中引入分歧样本应该能帮助模型建立更细腻的决策边界。

| 训练配置 | Test Acc | A++ Acc | A0 Acc | A0 MSE↓ |
|---------|----------|---------|--------|---------|
| Zero-shot | 70.92 | 85.22 | 54.09 | 0.229 |

| w/ A0 | 73.99 | 86.53 | 56.31 | **0.167** |

| w/ A++/+/0 | **84.23** | **95.98** | **64.37** | 0.215 |

### 关键发现
- **LLM 不知道自己不知道**：所有模型在 A0 上 Cohen's κ 仍 >0.75（高自一致性），但决策方向与人类判断严重偏离——这正是"过度自信"的操作性定义。
- **扩大模型对模糊案例帮助有限**：LLaMa3-8B→70B 在 A++ 上 F1 提升 10.64%，在 A0 上仅提升 3.11%。
- **LLM 偏向将不确定样本判为攻击性**：低一致度非攻击性样本（N0）准确率仅 45.77%，模型在不确定时倾向"宁可误判为攻击"。
- **分歧样本是最有价值的训练数据**：A0 样本在微调中将 MSE 从 0.229 降至 0.167（校准度最佳），三级混合训练达到最佳综合性能 84.23%。
- **A+ 是单一级别的最优选择**：在微调中取得准确率和校准度的最佳平衡。

## 亮点与洞察

- **分层评估方法论价值大于具体数字**：按标注一致度分层评估 LLM-as-Judge 是通用框架，可迁移到情感分析、幽默检测等所有主观标注任务。未来所有 LLM 裁判评估都应报告分层结果而非单一总分。
- **"过度自信"的精确刻画**：通过 self-consistency + 混淆矩阵直观展示了 GPT-4o 即使在软标签 0.4~0.6 的模糊区域仍给出极端置信度，这不是模型大小问题而是训练范式问题。
- **分歧样本的"课程学习"效应**：混合不同一致度级别的训练数据类似于课程学习——先学简单案例建立基础边界，再引入困难样本细化决策面，这一策略可迁移到其他标注不确定性高的任务。

## 局限性 / 可改进方向

- **单一数据集 + 单一任务**：仅在 MD-Agreement 上实验，未验证在其他主观任务上的泛化性。
- **置信度估计方法粗糙**：self-consistency 依赖温度采样，对开源模型可用 logit-based 方法做更精确校准。
- **标注者只有 5 人**：一致度分辨率有限（只有 3/5、4/5、5/5 三档），更多标注者可提供更细粒度的分歧光谱。

## 相关工作与启发

- **vs 传统攻击性语言检测**：传统方法在多数投票标签上训练和评测，完全忽略标注分歧。本文首次将标注分歧作为核心评估维度引入 LLM 评测。
- **vs LLM-as-Judge 研究**：现有 LLM 裁判评估关注模型间排名一致性，本文关注模型与人类不确定性的对齐——更深层的可靠性问题。
- **vs 不确定性估计文献**：LLM 不确定性研究多关注事实性问答，本文关注主观任务中的不确定性——"正确答案"本身就是模糊的。
