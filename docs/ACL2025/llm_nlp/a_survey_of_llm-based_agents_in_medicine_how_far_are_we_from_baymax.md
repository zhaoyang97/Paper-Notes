# A Survey of LLM-based Agents in Medicine: How Far Are We from Baymax?

**会议**: ACL 2025  
**arXiv**: [2502.11211](https://arxiv.org/abs/2502.11211)  
**代码**: 无  
**领域**: LLM Agent / 医学AI  
**关键词**: LLM医学Agent, 临床决策, 多Agent协作, 医学推理, 架构综述  

## 一句话总结

系统综述 LLM-based Agent 在医学中的应用——分析 Agent 架构（系统配置/临床规划/医学推理/外部能力增强）、应用场景（临床决策/文档/训练模拟/服务优化）和评估框架，覆盖 60 篇研究，识别幻觉管理、多模态整合、部署障碍和伦理问题等关键挑战。

## 背景与动机

LLM-based Agent 通过整合外部知识检索、任务规划和工具调用，超越了纯文本处理的 LLM，可在真实世界医疗应用中自主运作。医疗领域有独特考量：多模态数据整合、临床协作、精确性和可靠性要求、透明性和可追溯性。

## 核心内容

### Agent 架构四大组件

1. **系统配置（Profile）**：
   - 功能模块化：按任务分组（数据分析/诊断推理）
   - 角色专业化：模拟真实医学角色（影像科/外科等）
   - 科室组织化：按医学学科划分知识边界

2. **临床规划（Clinical Planning）**：
   - 单 Agent 自主处理 → 顺序任务链 → 协作专家（多科会诊）→ 迭代自进化
   - 关键特征：MDAgents 框架的自适应规划架构

3. **医学推理（Medical Reasoning）**：
   - 结构化推理（CoT 在医学场景的应用）
   - 基于知识图谱的推理增强
   - 实时反馈驱动的推理校正

4. **外部能力增强**：
   - RAG（检索增强生成）+ 医学知识库
   - 工具调用（PACS 系统、实验室接口）
   - 多模态数据处理

### 四种 Agent 范式

- **单 Agent**：自主处理简单任务
- **顺序任务链**：多步骤流水线
- **协作专家**：多 Agent 多科室协作
- **迭代进化**：基于历史案例持续改进

### 应用场景

- 临床决策支持（诊断/治疗方案推荐）
- 医疗文档（病历生成/ICD编码）
- 训练模拟（模拟患者/临床技能训练）
- 服务优化（预约调度/资源分配）

### 关键挑战

- 幻觉管理：医疗场景下幻觉的代价极高
- 多模态整合：影像、实验室、文本的融合
- 部署障碍：隐私、合规、系统集成
- 伦理考量：责任归属、知情同意

## 亮点

- **Baymax 隐喻切入点好**：从理想化医疗AI引出当前差距
- **架构分析系统**：四组件+四范式的框架清晰完整
- **覆盖实际部署挑战**：不仅是技术综述，还讨论伦理和制度障碍
- **60 篇研究的系统回顾**：经过筛选的高质量文献集

## 局限性 / 可改进方向

- **截止 2024 年**：最新的推理增强 Agent（如 o1-medical）可能未覆盖
- **缺少定量比较**：不同 Agent 架构在同一基准上的效果对比缺失
- **偏重英文文献**：非英语医疗AI应用覆盖不足
- **对小型/开源模型的讨论有限**：实际部署中大模型API成本是重要考量

## 启发与关联

- 协作专家范式（多科会诊模拟）是医疗AI最有前景的方向
- 迭代自进化架构使 Agent 能从历史案例持续学习——接近真实医生的成长路径
- 距离 Baymax 的主要差距不在技术而在可靠性和信任——幻觉和可解释性是瓶颈
