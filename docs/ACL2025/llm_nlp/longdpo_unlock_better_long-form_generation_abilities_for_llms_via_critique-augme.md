# LongDPO: Unlock Better Long-form Generation Abilities for LLMs via Critique-augmented Stepwise Information

**会议**: ACL 2025  
**arXiv**: [2502.02095](https://arxiv.org/abs/2502.02095)  
**代码**: https://github.com/pingbowen23/LongDPO (有)  
**领域**: LLM NLP  
**关键词**: 长文本生成, stepwise DPO, MCTS, 偏好学习, critique增强

## 一句话总结
提出 LongDPO，通过 MCTS 收集步级偏好对、全局记忆池维护事实一致性、critique 增强低质量候选，再用步级 DPO 进行细粒度优化，在 LongBench-Write 上显著提升长文本生成质量，同时保持通用能力。

## 研究背景与动机

1. **领域现状**：LLM 在长上下文输入方面已取得显著进展（如 GPT-4o 支持 128K），但在长文本**生成**方面依然薄弱——多数模型难以生成超过 2000 词的高质量内容。
2. **现有痛点**：已有方法如 Suri 和 LongWriter 主要采用**结果级监督**（outcome supervision），即对完整的长文本直接做 DPO。但长文本更容易出现逻辑不一致、事实编造、未满足指令要求等问题，结果级监督难以定位具体问题所在。
3. **核心矛盾**：长文本的质量问题是**逐步累积**的——前面的错误会传播到后续内容，但结果级 DPO 只能在整体层面提供粗糙的反馈信号，奖励边际不明显，学习效率低。
4. **本文要解决什么？** (a) 构建步级（step-level）偏好数据来提供细粒度监督；(b) 在搜索过程中维护事实一致性；(c) 提升低奖励候选的质量。
5. **切入角度**：过程监督（process supervision）在数学推理中已被证明优于结果监督；长文本生成可以自然分解为多个步骤，也适合过程监督。
6. **核心 idea 一句话**：用 MCTS 搜索长文本的步级偏好对，配合全局记忆池检查一致性和 critique 增强，再用步级 DPO 训练。

## 方法详解

### 整体框架
LongDPO 包含两大部分：(1) **步级偏好数据构建**——用 MCTS 以步为单位扩展生成树，每层提取一对 chosen/rejected，全局记忆池过滤不一致节点，critique 增强低奖励 chosen；(2) **步级 DPO 训练**——将长文本 $y$ 分解为 $s_1 \oplus s_2 \oplus \cdots \oplus s_t$，在每一步独立做 DPO 学习。

### 关键设计

1. **MCTS 收集步级偏好对**：
   - 做什么：将长文本生成过程建模为树搜索，每一层（步）生成多个候选片段，用奖励模型评估质量，选取最优和随机一个分别作为 chosen/rejected
   - 核心思路：树的第 $t$ 层节点 $s_{t+1} = \pi_\theta(q \oplus s_1 \oplus \cdots \oplus s_t)$，节点选择用 UCB 公式平衡探索与利用：$\text{UCB}_i = \alpha \sqrt{2 \ln(N_i / (1 + n_i))} + v_i$。每步用 7 个原则（事实性、连贯性等）评估，取平均奖励
   - 设计动机：搜索比直接采样能找到更高质量的偏好对，且步级分解让奖励信号更精准

2. **全局记忆池（Global Memory Pool）**：
   - 做什么：在 MCTS 选择阶段过滤与前文事实矛盾的候选节点
   - 核心思路：每选定一个节点，从中提取事实内容存入记忆池 $M_t$。下一步选择时，将候选节点切分为 128 词的片段，与记忆池中的每条事实计算嵌入相似度（用 gte-Qwen2-1.5B），相似度 $\geq 0.8$ 的片段与对应事实做一致性检查，不一致则跳过
   - 设计动机：长文本生成中事实不一致是最常见的质量问题，记忆池在搜索阶段就排除矛盾候选，而非事后修补

3. **Critique 增强生成**：
   - 做什么：对奖励低于阈值 $\eta = 2.5$ 的 chosen 候选，用外部 critique 模型生成改进建议，然后重新生成
   - 核心思路：为每个低奖励 $s_{win}$，找到在某个原则下表现更好的兄弟节点 $s_{sib}$，构建三元组 (原则, 兄弟节点, 待改进节点)。critique 模型生成分析、理由、相关文本、置信度和写作建议，将建议注入重新生成：$s_{win\_new} = \pi_\theta(q \oplus s_1 \oplus \cdots \oplus s_t \oplus z_\lambda)$
   - 设计动机：单纯扩大搜索空间效率低且收益有限，用 critique 直接指导改进更高效；外部 critique 比自我反思更可靠

4. **步级 DPO 训练**：
   - 做什么：将收集到的步级偏好对用于细粒度的 DPO 学习
   - 核心公式：$\mathcal{L}_{LongDPO} = -\mathbb{E}[\log \sigma(\beta \log \frac{\pi_\theta(s_w|q'))}{\pi_{ref}(s_w|q')} - \beta \log \frac{\pi_\theta(s_l|q')}{\pi_{ref}(s_l|q')})]$，其中 $q' = q \oplus s_{1 \sim i}$
   - 与传统 DPO 的区别：传统 DPO 对整个长文本做一次偏好学习，LongDPO 在每个步骤独立做偏好学习，奖励信号更清晰，避免长序列中信号被稀释

## 实验关键数据

### 主实验（LongBench-Write-en）

| 模型 | [0,500) $S_l$/$S_q$ | [2k,4k) $S_l$/$S_q$ | [4k,20k) $S_l$/$S_q$ | Avg $S_l$/$S_q$ |
|---|---|---|---|---|
| LongWriter-Llama | 88.1/86.0 | 89.1/88.3 | 80.8/79.2 | 83.1/85.1 |
| + DPO | 90.9/85.8 | 90.0/90.5 | 81.1/80.9 | 85.6/85.7 |
| + **LongDPO** | **90.7/86.3** | **93.4/90.5** | **88.3/85.1** | **87.4/88.3** |
| LongWriter-Qwen | 90.8/88.0 | 84.2/84.8 | 58.7/78.1 | 79.5/85.1 |
| + DPO | 86.3/88.2 | 89.3/84.1 | 60.9/78.8 | 81.3/- |
| + **LongDPO** | **91.6/88.6** | **92.1/88.3** | **81.2/84.9** | **87.7/87.6** |

- LongDPO 在 Llama 上整体提升 +4.3/+3.2（$S_l$/$S_q$），在长区间 [4k,20k) 提升最显著：+7.5/$S_l$
- 在 Qwen 上，长区间 [4k,20k) 从 58.7 提升到 81.2（+22.5），极大改善了超长生成能力

### 消融实验

| 配置 | LongBench-Write Avg $S_l$ | 说明 |
|---|---|---|
| LongDPO (full) | 87.38 | 完整方法 |
| w/o Memory Pool | 85.91 (↓1.47) | 去掉一致性检查，事实错误累积 |
| w/o Critique | 86.12 (↓1.26) | 不做 critique 增强，低奖励 chosen 未改进 |
| w/o Stepwise (→ outcome DPO) | 85.55 (↓1.83) | 退化为传统结果级 DPO |
| LongDPO w/ self-critique | 86.43 (↓0.95) | 用自我反思替代外部 critique，效果降低 |

### 关键发现
- **步级监督是最关键贡献**：去掉步级学习（变成传统 DPO）掉 1.83 分，说明过程监督在长文本生成中确实优于结果监督
- **记忆池对长区间贡献最大**：在 [4k,20k) 区间，去掉记忆池后性能下降更明显，因为越长的文本越容易出现事实不一致
- **外部 critique 优于自我反思**：印证了 Qi et al. (2024) 的发现——模型自我反思的改进不稳定
- **通用能力基本无损**：在 TruthfulQA 等通用 benchmark 上性能保持或略有提升

## 亮点与洞察
- **MCTS + 记忆池的组合设计很巧妙**：MCTS 提供搜索能力，记忆池解决长文本特有的事实一致性问题，两者互补。记忆池随搜索过程增量更新，是一种轻量级的"长期记忆"
- **Critique 增强只针对低奖励候选**：不是对所有候选都做 critique（成本太高），而是只对 $r \leq 2.5$ 的进行改进，兼顾效率和效果
- **过程监督从推理迁移到生成**：之前过程监督主要用在数学推理（MCTS + PRM），这篇论文展示了它在开放式长文本生成中的有效性，扩展了过程监督的应用范围

## 局限性 / 可改进方向
- **MCTS 搜索成本高**：每个 query 需要多轮扩展和评估，数据构建成本显著高于直接采样
- **依赖外部奖励模型**：用 LLM 做 7 原则评估，奖励模型本身的偏差会传播到偏好数据
- **步的划分方式未优化**：似乎使用固定长度划分步骤，对不同类型的长文本任务可能不是最优的
- **实验模型规模有限**：主要在 LongWriter-Llama/Qwen（均为 7B-9B 级别）上验证，更大模型的效果未知
- 改进方向：自适应步长划分；轻量化的奖励模型替代 LLM judge；与 LongReward 结合

## 相关工作与启发
- **vs LongWriter (Bai et al. 2024)**：LongWriter 用 agent pipeline 分解任务构建数据 + 结果级 DPO；LongDPO 用 MCTS 构建步级数据 + 步级 DPO，监督更细粒度
- **vs Suri (Pham et al. 2024)**：Suri 对同一回复创建不同指令做结果级偏好优化；LongDPO 在步级收集偏好对
- **vs LongReward (Zhang et al. 2024)**：LongReward 需要额外参考文档作为输入，适用范围受限；LongDPO 不需要参考文档，更通用
- 这篇论文证明了过程监督在长文本生成中的价值，可以考虑将类似方法应用到长代码生成、长论文写作等任务
