# Only a Little to the Left: A Theory-grounded Measure of Political Bias in LLMs

**会议**: ACL 2025  
**arXiv**: [2503.16148](https://arxiv.org/abs/2503.16148)  
**代码**: [https://github.com/MaFa211/theory_grounded_pol_bias](https://github.com/MaFa211/theory_grounded_pol_bias)  
**领域**: AI安全  
**关键词**: 政治偏见, LLM偏见测量, Political Compass, 调查设计, 提示敏感性  

## 一句话总结
基于政治科学理论构建科学有效的LLM政治偏见测量方法，对11个模型用多种提示变体收集88,110条回复，发现指令微调模型普遍偏左，但偏见度量对提示高度敏感，且Political Compass Test会夸大某些模型的偏见。

## 研究背景与动机
1. **领域现状**：LLM被广泛用于角色模拟、信息搜索等，其政治偏见会影响这些应用的公正性。研究者常用Political Compass Test (PCT)等调查来评估偏见。
2. **现有痛点**：(a) PCT不是科学有效的调查工具；(b) 不同提示技术导致结论分歧；(c) 大多数研究使用限制性答案格式。
3. **核心矛盾**：现有偏见测量既不符合调查设计原则，也未充分考虑提示敏感性，导致偏见估计不可靠。
4. **本文要解决什么？** 构建基于政治科学理论、考虑提示敏感性的LLM政治偏见测量方法。
5. **切入角度**：遵循调查设计原则，测试多种提示变体，自动分类开放式回答。
6. **核心idea一句话**：用科学有效的调查工具替代PCT，在大量提示变体下收集回复，发现偏见估计对提示高度敏感但指令微调模型总体偏左。

## 方法详解

### 整体框架
1. 基于政治科学理论设计调查问卷（替代PCT）
2. 对11个模型（开源+商业）用多种提示变体收集回复
3. 自动分类开放式回复的政治立场
4. 计算跨提示变体的政治偏见profile

### 关键设计
1. **理论导向的调查设计**：基于政治科学的维度（而非PCT的非标准维度）构建问题
2. **提示敏感性分析**：系统测试不同提示模板、指令格式等变体
3. **区分指令微调vs非指令微调**：分析后训练对偏见的影响
4. **开放式回答分类**：不限制答案格式，用分类器自动判断政治立场

## 实验关键数据
- 11个模型（GPT系列、LLaMA等），88,110条回复
- PCT倾向于夸大某些模型（如GPT-3.5）的偏见
- 指令微调模型普遍比base模型更偏左
- 偏见度量对提示变化不稳定

### 关键发现
- 指令微调引入左倾偏见——可能与RLHF的价值对齐有关
- 偏见测量的高度不稳定性表明需要更稳健的评估方法
- PCT不是可靠的偏见测量工具

## 亮点与洞察
- 首次将政治科学调查设计原则引入LLM偏见评估
- 揭示了PCT作为偏见测量工具的不可靠性

## 局限性 / 可改进方向
- 调查设计基于西方政治理论框架，可能不适用于其他政治文化
- 自动立场分类可能引入额外偏差
