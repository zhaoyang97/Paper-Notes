# Answer When Needed, Forget When Not: Language Models Pretend to Forget via In-Context Knowledge Unlearning

**会议**: ACL 2025  
**arXiv**: [2410.00382](https://arxiv.org/abs/2410.00382)  
**代码**: [GitHub](https://github.com/seele1917/test-time-in-context-unlearning)  
**领域**: LLM隐私 / 知识遗忘  
**关键词**: 上下文遗忘, 测试时遗忘, 遗忘token, 选择性遗忘, LLM内部机制  

## 一句话总结

提出"上下文知识遗忘"（In-Context Knowledge Unlearning）——通过引入遗忘 token（`<<UNL>>...<<UNL>>`）使 LLM 在推理时根据查询上下文选择性地"遗忘"特定知识，微调后 LLM 在 TOFU/AGE/RWKU 上达到 95% 遗忘准确率同时保留 80% 无关知识。更深入的内部分析发现 LLM 在中间层仍生成正确答案，仅在最后一层决定"假装遗忘"。

## 背景与动机

LLM 部署需要选择性信息处理——例如企业环境中对授权用户提供机密信息而对外部用户隐藏。现有知识遗忘方法的局限：

| 方法 | 测试时遗忘 | 无幻觉输出 |
|------|---------|---------|
| 梯度上升 | ✗ | ✗ |
| ROME/MEMIT | ✗ | ✓ |
| ICUL | ✓ | ✗（输出错误答案） |
| **本方法** | **✓** | **✓**（输出"forgot"） |

关键区别：ICUL 通过翻转标签实现遗忘但产生幻觉（输出错误答案）；本方法输出"forgot"表示真正的遗忘。

## 核心问题

能否让 LLM 在推理时根据上下文标记选择性遗忘特定知识，既不产生幻觉也不影响保留知识？

## 方法详解

### 遗忘 token

- 用 `<<UNL>>` 和 `<</UNL>>` 包裹需遗忘的信息
- 例如：`<<UNL>>Paris<</UNL>>`——模型应忽略关于"Paris"的知识
- 微调后模型识别这些特殊 token 并调整输出分布

### 损失函数

$\mathcal{L} = L_{forget} + L_{retain}$

- **$L_{forget}$**：对包含遗忘 token 的查询，训练模型输出"forgot"
- **$L_{retain}$**：对不含遗忘 token 的查询，保持正常回答能力

### 微调策略

使用 LoRA 或全参微调调整 θ 以识别遗忘 token 并做出相应响应。

### 内部机制分析

- 分析微调后 LLM 各层的隐藏状态
- **关键发现："LLM 假装遗忘"**——中间层仍生成正确预测并一路保持到倒数第二层，仅在最后一层切换为"forgot"
- 模型并非真正删除知识，而是学会在最后一步压制已知答案

## 实验关键数据

### 遗忘能力（Forget Accuracy）

| 模型 | TOFU | AGE | RWKU |
|------|------|-----|------|
| Llama2-7B | ~95% | 高 | 高 |
| Llama2-13B | ~93% | 高 | 高 |
| Mistral-7B | ~94% | 高 | 高 |

### 保留能力（Retain Accuracy）

- 保留 ~80% 的无关知识——遗忘不会严重损害通用能力
- 显著优于基线（梯度上升、ICUL 等）在 ID 和 OOD 场景下

### "假装遗忘"现象

- 中间层正确答案概率高
- 最后一层急剧切换为"forgot"
- 证明遗忘是一种后期决策，而非真正的知识删除

## 亮点

- **测试时选择性遗忘**：根据上下文动态决定遗忘什么——比全局遗忘灵活得多
- **无幻觉设计**：输出"forgot"而非错误答案——比 ICUL 更安全
- **"假装遗忘"的发现极具深度**：揭示了 LLM 遗忘的内部机制——知识仍在但被压制
- **同时支持域内和域外遗忘**：泛化到训练遗忘数据之外的相关查询

## 局限性 / 可改进方向

- **遗忘 token 可被绕过**：攻击者若知道机制，可能不使用遗忘 token 直接提问
- **"假装遗忘"意味着知识仍可提取**：中间层仍包含正确答案——可能被探针攻击恢复
- **仅测试3个数据集**：更广泛的知识类型（如多模态知识）未验证
- **遗忘粒度有限**：目前以实体级为主，更细粒度（如特定属性）需探索

## 与相关工作的对比

- **vs ROME/MEMIT**：永久编辑参数，不可测试时切换；本方法可按上下文灵活切换
- **vs ICUL**：输出错误答案（幻觉）；本方法输出"forgot"（无幻觉）
- **vs 差分隐私/联邦学习**：从数据层面保护隐私；本方法从模型输出层面选择性遗忘

## 启发与关联

- "假装遗忘"提示安全研究需关注模型内部状态而非仅关注输出——表面遗忘不等于真正遗忘
- 遗忘 token 的设计思路可推广到其他条件控制场景（如情感控制 token、风格控制 token）
- 企业级 LLM 部署需要分层访问控制——上下文遗忘是一种轻量级解决方案
