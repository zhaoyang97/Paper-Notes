# Ad-hoc Concept Forming in the Game Codenames as a Means for Evaluating Large Language Models

**会议**: ACL 2025  
**arXiv**: [2502.11707](https://arxiv.org/abs/2502.11707)  
**代码**: [GitHub](https://github.com/clembench/clembench)  
**领域**: LLM评估 / 游戏基准  
**关键词**: Codenames, LLM评测, 临时概念形成, 语义关联, 合作推理, 歧义词  

## 一句话总结

以桌游 Codenames 作为 LLM 评测工具——LLM 分别扮演线索给出者（Spymaster）和猜测者（Field Operative），通过控制词频/歧义性/具体性/风险等级/对手速度等变量系统评估 LLM 的临时概念形成、语义关联、合作推理和语用能力，发现 o3-mini 和 Claude-3.5 领先但所有模型在高风险和抽象词条件下均显著退化。

## 背景与动机

LLM 面临"评估危机"：传统参考答案评估模式不适合交互式使用，训练数据污染使测试集可靠性下降。游戏作为交互式评估环境可解决这两个问题——容易生成新实例避免污染，且可扩展到测试新现象。Codenames 尤其适合：需要跨目标词形成临时概念（ad-hoc concept）、预测合作者行为（心智理论）、权衡风险（避免刺客词），涉及同义词/反义词/谐音/文化引用等多维语义关系。

## 核心问题

LLM 在需要创造性语义关联和合作推理的 Codenames 游戏中表现如何？哪些因素（词频/歧义/具体性/风险/对手速度）影响其表现？

## 方法详解

### 游戏实现

- 基于 clembench 框架，GameMaster 协调游戏流程
- LLM 扮演同一队的 Spymaster（给线索词+目标数）和 Field Operative（猜词）
- 对手为确定性模拟（每回合揭示 n 个词），消除对手随机性

### 实验设计（7 组变量，130 个棋盘实例）

1. **风险等级**：5 个刺客词（高风险） vs 0 个（低风险）
2. **词语关联**：同类词归同队（简单） vs 散布各组（困难）
3. **对手速度**：每轮揭示 0/1/2 词
4. **词频**：高频词 vs 低频词（SUBTLEX-US）
5. **歧义性**：多义词（homonyms） vs 单义词（monosemes）
6. **具体性**：具体概念 vs 抽象概念（Brysbaert 评级）

### 评估指标

- **clemscore**：played 比例 × 胜率（平衡遵循指令和完成任务）
- 灵敏度（揭示己方词比例）、效率（每轮揭示2词为上限）

### 评估模型

- 商业：o3-mini、GPT-4o、Claude-3.5、Gemini-2.0-Flash
- 开源：Llama-3.1/3.3、Qwen2/2.5、Deepseek-v3/r1

## 实验关键数据

| 模型 | clemscore↑ | Played%↑ | 胜率↑ |
|------|-----------|---------|------|
| o3-mini | 49.2 | 100.0 | 49.2 |
| Claude-3.5 | 46.9 | 93.8 | 50.0 |
| GPT-4o | 45.4 | 93.8 | 48.4 |
| Deepseek-r1 | 45.4 | 85.4 | 53.2 |
| Gemini-2.0 | 较低 | 较低 | 较低 |

### 关键发现

- 高风险（5刺客）显著降低所有模型性能——但模型未学会降低每轮目标数来规避风险
- 低频词和抽象词对性能影响最大——与人类表现模式一致
- 歧义词（多义词）反而更容易找到跨词关联——与假设一致
- 小模型（8B）在格式遵从上严重不足（大量 Aborted）
- Deepseek-r1 胜率最高但 played 率低——推理强但指令遵从差

## 亮点

- **实验设计精巧**：7 组控制变量系统分析不同语言/认知因素的影响
- **确定性对手消除噪声**：比 LLM vs LLM 设置更可控可复现
- **与心理语言学发现一致**：低频/抽象/高风险的影响模式与人类认知研究对应
- **开源 clembench 框架**：可扩展添加新游戏评估

## 局限性 / 可改进方向

- **仅英语**：未测试多语言版 Codenames
- **确定性对手过于简单**：真正的合作需要双方都是 LLM
- **clemscore 受格式遵从影响大**：某些模型仅因格式问题排名低
- **10 个棋盘/实验**：统计功效有限

## 与相关工作的对比

- **vs BigBench Codenames**：BigBench 仅含静态评测；本文实现完整交互式游戏
- **vs Stephenson et al. 2024**：用 LLM vs LLM；本文用确定性对手更可控
- **vs 标准 NLU 基准**：Codenames 测试创造性语义关联+合作推理，不可通过记忆解决

## 启发与关联

- 游戏基准是 LLM 评估的未来方向——天然抗污染、可扩展、测试交互能力
- "临时概念形成"能力是 LLM 创造力的核心指标之一
- 风险感知不足暗示 LLM 在需要代价权衡的场景中仍有盲区
