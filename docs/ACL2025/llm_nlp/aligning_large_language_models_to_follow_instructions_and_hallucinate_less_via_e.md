# NOVA: Aligning LLMs to Follow Instructions and Hallucinate Less via Effective Data Filtering

**会议**: ACL 2025  
**arXiv**: [2502.07340](https://arxiv.org/abs/2502.07340)  
**代码**: [GitHub](https://github.com/S1s-Z/NOVA)  
**领域**: LLM对齐 / 幻觉缓解  
**关键词**: 指令微调, 幻觉, 数据过滤, 内部一致性探测, 语义等价识别, 知识熟悉度  

## 一句话总结

提出 NOVA 框架——通过过滤掉 LLM "不熟悉"的指令数据来同时减少幻觉和保持指令遵从能力——Internal Consistency Probing（ICP）通过多次自生成回复的隐藏状态一致性评估 LLM 对指令的熟悉度，Semantic Equivalence Identification（SEI）通过语义聚类+投票评估 LLM 对目标回复的熟悉度，再用质量奖励模型确保数据质量。大幅减少幻觉同时保持指令遵从。

## 背景与动机

指令微调面临一个困境：(1) LLM 需要学习遵从指令（需要高质量数据）；(2) 但高质量数据可能包含 LLM 预训练阶段未学过的知识，导致过度自信和幻觉增加。已有研究证实：在包含不熟悉知识的数据上微调会鼓励幻觉。RL 方法（如 DPO）可减少幻觉但可能削弱指令遵从，且需额外数据和 API 成本。

## 核心问题

如何在指令微调阶段直接过滤掉"不熟悉"数据，使 LLM 既学会遵从指令又减少幻觉——而无需额外的 RL 阶段？

## 方法详解

### ICP（Internal Consistency Probing）

- 给定指令 q，让 LLM 生成 K 个回复
- 提取每个回复最后层最后 token 的隐藏状态作为句子嵌入
- 将嵌入视为多变量高斯分布，计算**微分熵**（differential entropy）
- 低微分熵 = 隐藏状态高度一致 = LLM 对该指令很熟悉
- 公式：$F_{ins}(q) = \frac{1}{2} \sum_{i=1}^{d} \lambda_i + G$（协方差矩阵特征值之和）

### SEI（Semantic Equivalence Identification）

- 对生成的 K 个回复用语义聚类（训练好的模型分类同义回复）
- 判断目标回复 r 属于哪个语义簇
- 如果 r 落入最大簇 → LLM 对目标回复内容熟悉
- 基于投票策略确定匹配关系

### 质量奖励模型

- 仅靠熟悉度不够——还需确保数据质量（指令复杂度、回复流畅性等）
- 专家对齐的奖励模型过滤低质量样本

### 训练流程

$F_{ins}(q)$ + $F_{res}(r)$ + Quality Score → 综合过滤 → 选中数据用于指令微调

## 实验关键数据

### 指令遵从 vs 幻觉（图1核心结果）

- **MT-Bench**（指令遵从）：NOVA 保持高分
- **LongFact**（幻觉）：NOVA 大幅降低幻觉率
- 同时达到两个目标——不像 RL 方法那样以指令遵从换幻觉减少

### 对比实验

| 方法 | 指令遵从 | 幻觉率 |
|------|---------|-------|
| 全数据微调 | 高 | 高（不熟悉数据多） |
| 质量过滤（SOTA） | 更高 | **更高**（高质量=更多专家知识=更不熟悉） |
| **NOVA** | **高** | **显著更低** |

### 关键洞察

- 高质量数据≠低幻觉数据——高质量数据可能含更多 LLM 不熟悉的专家级知识
- ICP 的微分熵准确度量了 LLM 的知识边界

## 亮点

- **揭示"高质量数据反而增加幻觉"的反直觉发现**——质量和熟悉度是两个维度
- **ICP 的微分熵度量优雅**——利用 LLM 自己的隐藏状态判断自己知不知道
- **无需 RL 的高效方案**——仅过滤数据即可减少幻觉，无需额外偏好数据和训练阶段
- **SEI 的语义聚类+投票**——比简单的 ROUGE/困惑度更好地捕捉语义一致性

## 局限性 / 可改进方向

- **需要生成 K 个回复**：ICP 和 SEI 的计算开销与 K 成正比
- **高斯假设可能不成立**：隐藏状态分布可能非高斯
- **过滤可能丢弃有价值的数据**：某些"不熟悉"但有用的数据也被排除
- **仅在指令微调阶段**：预训练阶段的数据质量问题未涉及

## 与相关工作的对比

- **vs DPO/PPO 去幻觉**：需额外 RL 阶段+偏好数据+可能损害指令遵从；NOVA 仅需数据过滤
- **vs 现有数据过滤（IFD/Deita 等）**：仅关注质量不考虑熟悉度；NOVA 同时考虑两个维度
- **vs 知识编辑去幻觉**：事后修补；NOVA 从源头（训练数据）解决

## 启发与关联

- "不要教 LLM 不知道的东西"是减少幻觉的简单但深刻的洞察
- 微分熵度量知识边界的思路可推广到主动学习（选择 LLM 最需要学的数据）
- 质量×熟悉度的二维数据选择框架是通用方法论
