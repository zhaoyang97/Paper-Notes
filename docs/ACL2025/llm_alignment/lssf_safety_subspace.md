# LSSF: Safety Alignment via Low-Rank Safety Subspace Fusion

**会议**: ACL 2025
**arXiv**: [2602.00038](https://arxiv.org/abs/2602.00038)
**代码**: 无
**机构**: 华东师范大学 & 蚂蚁集团
**领域**: LLM对齐 / 安全性
**关键词**: safety alignment, low-rank subspace, SVD, post-hoc safety, singular value entropy, task arithmetic

## 一句话总结

LSSF 发现 LLM 的安全信息集中在低秩子空间中，利用 SVD 构建低秩投影矩阵提取安全向量的主成分，并通过安全奇异值熵自适应确定每层保留秩，最终将低秩安全主成分通过线性算术融合到微调后模型中，实现免训练的后验安全恢复，同时最大化保持下游任务性能。

## 研究背景与动机

### 现状

经过 RLHF/DPO 等安全对齐训练的 LLM（如 Llama3.1、Qwen2.5），在进行下游任务微调后，即使使用完全良性的数据，安全对齐能力也会显著退化。仅需几十个有害样本就能让对齐后的模型"越狱"，在 AdvBench 等评估中拒绝率从 ~0.9 骤降至 ~0.1。

### 痛点

1. **安全脆弱性**：微调过程中安全参数与任务参数纠缠，任务相关的梯度更新会无差别地干扰编码安全知识的参数方向。
2. **已有方法的代价**：在微调阶段进行安全对齐（如 VLGuard 混入安全数据、Lisa 近端正则化）不仅增加训练复杂度和计算开销，还可能抑制模型的通用能力。
3. **全秩安全向量的副作用**：RESTA 等后验方法直接将安全向量（aligned - base 的权重差）加到微调模型上，但安全向量中包含抑制通用能力的成分，导致下游任务性能显著下降。

### 核心矛盾

如何在微调后恢复安全对齐，同时不损害下游任务性能？全秩安全向量包含噪声和能力抑制成分，直接融合效果差。

### 切入角度

已有研究表明 LLM 中安全区域在秩层面是稀疏和隔离的（Sun et al., 2023; Wei et al., 2024）。安全行为（拒绝有害请求）相对语言理解和推理来说是简单的行为模式，其参数信息应集中在低维子空间。

### 核心 Idea

提取安全向量的**低秩主成分**而非使用全秩安全向量，通过投影矩阵过滤掉与通用能力相关的冗余方向，仅保留真正的安全方向偏移进行融合。同时用奇异值熵自适应确定每层的保留秩，避免手动调参。

## 方法详解

### 整体框架

LSSF 框架分三步：

1. **Step I**：获取不安全模型——用少量有毒数据对安全对齐模型做 SFT，得到去除安全防护的模型
2. **Step II**：提取安全向量——计算安全模型与不安全模型之间的参数差值 $\delta_{\text{safe}}$
3. **Step III**：低秩安全主成分融合——构建低秩投影矩阵，提取安全向量的主成分，通过线性算术融合到下游微调模型

### 安全向量定义

安全向量是对齐模型与未对齐模型之间的参数差：

$$\theta_{\text{safe}} = \theta_{\text{unsafe}} + \delta_{\text{safe}}$$

其中 $\delta_{\text{safe}}$ 包含两个成分：(1) 低秩安全方向偏移（可通过低秩分解提取）；(2) 通用能力抑制成分（需要过滤掉的噪声）。

实际操作中，通过对安全模型 $\theta_{\text{safe}}$ 用 500 个有害 QA 对做 3 个 epoch 的 SFT 来获取反向安全向量 $-\delta_{\text{safe}}$。

### 低秩正交分解

对安全模型 $\theta_{\text{base}}$ 的激活进行 SVD 分解。构建校准数据集 $\mathcal{D}_{\text{anchor}}$（128 个有害 query + 安全拒绝回复），对每层线性层权重矩阵 $W \in \mathbb{R}^{d_{\text{out}} \times d_{\text{in}}}$ 收集输入激活 $\hat{X}$，目标是找到秩 $r$ 的低秩矩阵 $\hat{W}$ 最小化输出误差：

$$\hat{W} = \arg\min_{\text{rank}(\hat{W}) \leq r} \|W\hat{X} - \hat{W}\hat{X}\|_F^2$$

对激活矩阵 $Z = W\hat{X}$ 做标准化消除神经元敏感性差异：

$$\tilde{z}_{ij} = \frac{z_{ij} - \mu_j}{\delta_j}$$

再对标准化后的 $\tilde{Z}$ 做 SVD：$USV^\top \approx \tilde{Z}$。使用 Halko et al. (2011) 的随机化算法加速大矩阵 SVD 计算。

### 安全奇异值熵（Safety Singular Value Entropy）

不同层编码安全信息的密度不同，不能统一设定保留秩。LSSF 受 Shannon 熵启发，定义安全奇异值熵来量化信息密度：

首先定义每个主成分的信息贡献：

$$p_i = \frac{\sigma_i^2}{\sum_{j=1}^{n} \sigma_j^2}$$

计算前 $\rho$ 个秩的奇异值熵：

$$H_\rho = -\sum_{i=1}^{\rho} \frac{\sigma_i^2}{\sum_{j=1}^{n} \sigma_j^2} \log\left(\frac{\sigma_i^2}{\sum_{j=1}^{n} \sigma_j^2}\right)$$

通过信息保留阈值 $\eta$ 确定保留秩 $r$：

$$\frac{H_r}{H_n} > \eta$$

- $\eta$ 小 → 保留较少主成分（更激进的低秩近似，信息损失多）
- $\eta$ 大 → 保留较多主成分（更保守，信息损失少但可能引入噪声）
- 实验表明 $\eta > 0.8$ 时安全拒绝率接近 1.0

### 低秩投影矩阵与缩放

构建正交投影矩阵 $P^{(r)} = U^{(r)}(U^{(r)})^\top$，为平衡效用和安全性，对奇异向量施加缩放：

$$\alpha_i = 1 + (\alpha_1 - 1) \times \frac{\sigma_i - \sigma_r}{\sigma_1 - \sigma_r}$$

较大奇异值对应的安全方向获得更大权重。最终投影矩阵为 $P'^{(r)} = U'^{(r)}(U'^{(r)})^\top$。

### 线性融合

将低秩安全主成分融合到下游微调模型：

$$\theta_{\text{DST}}' = \theta_{\text{safe}} + \delta_{\text{DST}} + \alpha P'^{(r)} \delta_{\text{safe}}$$

核心思想是 $P'^{(r)} \delta_{\text{safe}}$ 只包含安全方向的低秩偏移，与下游任务方向 $\tau_{\text{DST}}$ 近似正交，因此融合后下游性能几乎不受影响。

### 训练策略

**完全免训练**。无损失函数、无梯度优化。超参数仅两个：$\eta$（奇异值熵阈值，通常 0.8-0.9）和 $\alpha_1$（缩放因子），通过在安全验证集上 grid search 确定。

关键发现：直接融合全秩安全向量（SafetyJ）导致 BBH 从 68.29 暴跌至 20.03、IFEval 从 81.65 降至 35.01，而 LSSF 的低秩主成分融合后 BBH=68.52、IFEval=81.88，甚至略高于原始模型。这强力证明了低秩安全子空间与通用能力方向的隔离性。

### 消融实验关键结论

- **$\eta$ 阈值的影响**：$\eta > 0.8$ 时安全拒绝率趋近 1.0，BLEU 仅轻微下降
- **奇异向量权重 $\alpha$ 的影响**：在 $\eta=0.5$ 设定下，$\alpha > 2.5$ 时安全性接近安全模型
- **校准数据量鲁棒性**：方法对校准样本数量不敏感，128 样本即足够

## 亮点与洞察

1. **低秩假设的实证验证**：安全向量的奇异值快速衰减，前几个主成分即包含绝大部分安全信息。通过可视化（Figure 3）可清晰看到低秩主成分是从全局网络中提取的结构化子网络，而非简单的参数缩放。
2. **安全子空间的稳定性**：该低秩安全子空间在微调过程中保持稳定，不随下游任务的变化而漂移，这是方法有效的核心前提。
3. **自适应秩确定机制**：安全奇异值熵巧妙借鉴信息论，自动为不同层分配不同保留秩——注意力层与 FFN 层的安全信息分布特征不同，统一秩要么信息不足要么引入噪声。
4. **完全免训练**：作为后验方法，LSSF 不修改任何训练流程，对 API 服务商场景（用户自行微调后恢复安全性）非常实用。
5. **下游性能保持能力突出**：在全参数微调的 Medical QA 任务上，BLEU 仅下降 ~0.02，远优于 RESTA（下降 ~0.10）和 Lisa（下降 ~0.13），同时达到相同的安全水平。

## 局限性 / 可改进方向

1. **模型规模限制**：仅在 7B/8B 规模验证（附录中测了 3B、14B），未测试 70B+ 或 405B 级别模型，大规模模型的低秩特性可能不同。
2. **安全向量获取依赖有毒数据**：需要用有毒数据集做 SFT 来获取 $-\delta_{\text{safe}}$，这在实际部署中有伦理和安全风险。
3. **超参敏感性分析不够充分**：$\eta$ 和 $\alpha$ 的交互效应未被系统探索，仅分别做了单变量消融。
4. **未涵盖多模态和 MoE 架构**：方法理论上架构无关，但未在 VLM 或 MoE 模型上验证。
5. **安全评估的局限**：仅使用拒绝率作为安全指标，未评估过度拒绝（over-refusal）问题——低秩安全主成分是否会导致模型对正常请求也过度谨慎。
6. **与 LoRA 适配器的交互**：在 LoRA 微调场景下，安全向量提取和融合是否可以直接在适配器空间进行，而非全参数空间？

## 相关工作与启发

- **RESTA (Bhardwaj et al., ACL 2024)**：直接用全秩安全向量做线性算术，是 LSSF 的直接改进对象。LSSF 的核心改进在于低秩投影过滤掉了抑制通用能力的成分。
- **ActSVD (Wei et al., 2024)**：通过激活矩阵的 SVD 证明安全信息的低秩性和独立性，是 LSSF 低秩假设的理论基础。
- **Task Arithmetic (Ilharco et al., 2022)**：任务向量的加减可以增强或减弱特定能力，LSSF 将此扩展到安全领域。
- **DARE (Yu et al., 2023)**：通过随机丢弃冗余参数做模型融合，LSSF 的对比实验表明低秩投影优于随机裁剪。
- **Lisa (Huang et al., 2024)**、**VLGuard (Zong et al., 2024)**：微调阶段的安全对齐方法，增加训练复杂度且效果不稳定。
- **启发**：低秩子空间的思路可推广到其他能力维度——如多语言能力、指令遵循能力等是否也存在类似的低秩隔离子空间？
