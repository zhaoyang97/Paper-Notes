# Chain-of-Jailbreak Attack for Image Generation Models via Editing Step by Step

**会议**: ACL 2025  
**arXiv**: [2410.03869](https://arxiv.org/abs/2410.03869)  
**代码**: https://github.com/Jarviswang94/Chain-of-Jailbreak (有)  
**领域**: LLM对齐/安全  
**关键词**: 越狱攻击, 图像生成安全, 逐步编辑, 安全护栏, Think-Twice防御

## 一句话总结
提出 Chain-of-Jailbreak（CoJ）攻击，将无法直接绕过安全护栏的恶意 query 分解为多步编辑子 query（删然后插、插然后删、改然后改回），在 GPT-4V/4o/Gemini 上达到 60%+ 越狱成功率；同时提出 Think-Twice Prompting 防御，拦截 95%+ 的 CoJ 攻击。

## 研究背景与动机

1. **领域现状**：文本生成图像模型（Stable Diffusion、DALL-E 3、Gemini）已广泛应用，产业界部署了多层安全护栏（数据过滤、SFT、RLHF、输入/输出审核）来阻止有害内容生成。
2. **现有痛点**：现有安全护栏主要检测单次 prompt 的恶意性——如果用户一次性提交恶意 query，会被拒绝。但**多轮交互式的图像编辑模式**未被充分考虑，模型可能在逐步编辑过程中生成有害内容。
3. **核心矛盾**：图像生成模型越来越支持多轮编辑（如"在图中插入X"、"删除Y"），每一步子操作单独看都无害，但组合起来可以合成恶意内容——这是一个**组合安全漏洞**。
4. **本文要解决什么？** (a) 形式化多步编辑越狱攻击方法；(b) 构建系统化的评测基准；(c) 提出有效防御方案。
5. **切入角度**：受 Levenshtein 编辑距离启发——将目标恶意文本/图像看作终态，通过多步编辑操作（插入、删除、替换）从安全初态到达终态，每步的子 query 都设计为安全护栏可放行的。
6. **核心 idea 一句话**：将恶意 query 分解为多步"无害"编辑操作序列，逐步引导模型生成有害内容。

## 方法详解

### 整体框架
CoJ 攻击分两步：(1) 将原始恶意 query 分解为一系列子 query，每个子 query 单独看似无害；(2) 逐步提交子 query，引导模型通过图像编辑逐步合成恶意内容。防御方面，提出 Think-Twice Prompting——在生成前让模型先描述将要生成的图像。

### 关键设计

1. **三种编辑操作组合**：
   - **Delete-then-Insert**：先删除恶意关键词生成安全版本，再要求插入被删除的词。如"GPT4 will __ the world" → 插入 "destroy"
   - **Insert-then-Delete**：先插入消解词生成安全版本，再要求删除。如"GPT4 will **not** destroy the world" → 删除 "not"
   - **Change-then-Change-back**：先替换恶意词为安全词，再改回。如"GPT4 will **help** the world" → 把 "help" 改为 "destroy"
   - 设计动机：模拟 Levenshtein 编辑距离的三种基本操作，确保最终编辑距离为 0（完整恢复恶意内容）

2. **三种编辑元素**：
   - **Word 级**：编辑整个单词（如插入/删除 "destroy"），覆盖最常见的文本恶意内容
   - **Character 级**：编辑单个字符（如逐个插入"G""P""T"），适用于缩写、敏感词
   - **Image 级**：编辑图像元素而非文本（如"把花改成武器"），**19.3% 的攻击完全不涉及文字**
   - 设计动机：全面覆盖不同粒度的恶意内容，特别是 image 级操作展示了纯视觉越狱的可能性

3. **CoJ-Bench 评测基准**：
   - 9 类安全场景：辱骂、色情、违法犯罪、仇恨言论、偏见歧视、人身伤害、暴力、虐童、虐待动物
   - 每个场景至少 15 个种子恶意 query（共 150 个），经人工确认均被模型直接拒绝
   - 用 LLM 辅助分解每个种子 query 为 3×3=9 种组合（3 编辑操作 × 3 编辑元素）
   - 评估指标：越狱成功率（ASR）+ 人工标注

4. **Think-Twice Prompting 防御**：
   - 做什么：在模型执行编辑操作前，先让它"想象并描述"即将生成的完整图像
   - 核心思路：通过让模型提前"预览"最终结果，触发安全护栏对最终内容的检测。类似于 Chain-of-Thought 让模型先推理再回答，Think-Twice 让模型先描述再生成
   - 效果：拦截 95%+ 的 CoJ 攻击
   - 设计动机：每步子 query 单独无害，但最终组合结果有害——让模型提前合成预览触发安全检测

## 实验关键数据

### 主实验（越狱成功率 ASR）

| 模型 | 直接攻击 | 其他越狱方法 | **CoJ 攻击** |
|---|---|---|---|
| GPT-4V | <5% | ~14% | **60%+** |
| GPT-4o | <5% | ~14% | **60%+** |
| Gemini 1.5 | <5% | ~14% | ~55% |
| Gemini 1.5 Pro | <5% | ~14% | ~50% |

- CoJ 攻击在所有模型上的 ASR 显著高于直接攻击和其他越狱方法（14% → 60%+）

### 消融实验（不同编辑操作/元素的效果）

| 编辑操作 | Word 级 ASR | Character 级 ASR | Image 级 ASR |
|---|---|---|---|
| Delete-then-Insert | 较高 | 中等 | 较高 |
| Insert-then-Delete | 最高 | 中等 | 中等 |
| Change-then-Change | 中等 | 中等 | 较高 |

### 防御实验

| 防御方法 | 防御成功率 |
|---|---|
| 无防御 | ~0%（60%+ 被攻破） |
| 安全提示增强 | ~30-40% |
| **Think-Twice Prompting** | **95%+** |

### 关键发现
- **Insert-then-Delete 是最有效的攻击操作**：因为初始生成看起来完全安全（包含否定词），删除操作看起来也无害
- **Image 级编辑最难防御**：纯视觉内容的恶意性更难被文本安全护栏检测
- **GPT-4V 比 GPT-4o 更容易被攻破**：可能是 4o 的多模态安全对齐更强
- **Think-Twice 防御极其有效**：因为它迫使模型在安全检查中考虑最终合成结果，而非仅检查当前子 query

## 亮点与洞察
- **Levenshtein 距离的巧妙类比**：将越狱攻击形式化为编辑距离操作，既提供了系统化的攻击分类框架，又使得攻击方法可以组合和扩展
- **攻防一体的研究设计**：不仅提出攻击（CoJ），还同时提出防御（Think-Twice），且防御方法简洁高效。这种负责任的安全研究范式值得推广
- **Image 级编辑的安全盲区**：揭示了纯视觉编辑可以绕过文本安全护栏的重要漏洞，提醒业界需要更强的多模态安全检测

## 局限性 / 可改进方向
- **攻击假设较强**：需要多轮交互和图像编辑能力，不适用于单轮 text-to-image 模型
- **CoJ-Bench 规模有限**：150 个种子 query，可能不覆盖所有安全场景
- **Think-Twice 防御增加延迟**：每次生成前多一轮"预览"描述，增加了推理成本
- **未考虑自适应攻击**：攻击者如果知道 Think-Twice 防御，可能设计绕过"描述"阶段的方法
- 改进方向：将 CoJ 扩展到视频生成模型；研究自适应攻击对 Think-Twice 的绕过；将防御集成到模型训练阶段

## 相关工作与启发
- **vs 传统文本越狱（GCG、AutoDAN 等）**：传统方法针对单次 prompt 做对抗扰动；CoJ 利用多轮编辑的组合漏洞，是全新的攻击维度
- **vs 视觉越狱（Shayegani et al. 2023）**：视觉越狱通过图像输入注入恶意信息；CoJ 通过逐步编辑绕过，不需要对抗性图像输入
- **Think-Twice vs Safety Prompt**：安全提示只是在系统 prompt 中加入安全约束；Think-Twice 让模型主动"预演"最终结果，触发更深层的安全检查
