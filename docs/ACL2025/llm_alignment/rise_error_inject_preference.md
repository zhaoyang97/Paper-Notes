# RISE: Subtle Errors in Reasoning: Preference Learning via Error-injected Self-editing

**会议**: ACL 2025
**arXiv**: [2410.06638](https://arxiv.org/abs/2410.06638)
**机构**: 香港理工大学 & 华为诺亚方舟实验室
**领域**: LLM对齐 / 数学推理
**关键词**: math reasoning, error injection, preference learning, DPO, hard negatives, subtle errors

## 一句话总结

RISE 发现 LLM 约 75% 的数学错误是微妙的步内错误（计算错误、替代错误、遗漏项），提出让 LLM 自编辑（REPLACE/SWAP/DELETE）向正确解注入预定义微妙错误来构造难负样本，配合步级+序列级+自适应 NLL 三合一的错误感知 DPO 训练，仅用约 4.5K 样本在 Qwen2-7B 上 GSM8K +3.0%、MATH +7.9%，且泛化到逻辑推理和代码生成。

## 研究背景与动机

1. **LLM 数学推理的现状**：LLM 已在基本算术到竞赛级数学问题上展现出较强能力，但仍频繁犯错。关键问题在于——这些错误到底是什么类型的？只有搞清楚这一点，才能针对性优化。
2. **错误分布的关键洞察**：作者对 Qwen2-7B 在 MATH 数据集上的错误进行了细致分类（用 GPT-4o 检测+人工验证 92% 准确率），发现约 **75% 的错误是步内微妙错误**——包括数值计算错误（~60%）、数字/符号替代错误、计算项遗漏等。只有约 25% 是步间逻辑跳跃或问题理解错误。这彻底颠覆了"LLM 不会推理"的笼统判断：LLM 的推理链路大体正确，问题出在执行层面的小失误。
3. **现有偏好学习的痛点**：Step-DPO 等方法通过 rejection sampling 构造正负样本对来训练，但这种方式有两个核心问题：(a) 正负样本差异过大，模型学到的是表面特征而非推理精度；(b) 无法针对性地覆盖特定类型的微妙错误。
4. **Hard Negative 的重要性**：机器学习中经典的 hard negative mining 思想——与正样本只有微小差异的负样本才能真正逼迫模型学习精细区分。在偏好学习中，这意味着需要"只差几个 token"的错误解。
5. **核心矛盾**：手动为每道题构造微妙错误解需要数学专业知识且不可扩展，而随机采样得到的错误解又太"粗糙"。
6. **RISE 的切入角度**：利用 LLM 自身作为编辑器——LLM 理解数学推理结构，可以被 prompt 引导去恰好修改关键 token 引入预定义的微妙错误，再用编辑距离过滤保证"微妙"程度，从而自动化大规模构造高质量难负样本。

## 方法详解

### 整体框架

RISE 分三个阶段：

1. **采样与配对**：给定数学问题 $x$，用原始模型 $M$ 采样 $K$ 个多步解 $\{\hat{y}\}$，根据最终答案正确性选出一个正确解 $\hat{y}^+$ 和一个错误解 $\hat{y}^-$，构成全序列对（Full-solution pair）。
2. **错误注入自编辑**：将正确解的每个推理步骤 $\hat{y}_i^+$ 交给同一个 LLM 执行编辑，按预定义错误类型注入微妙错误，生成 $\hat{y}_i^-$，经 Levenshtein 距离过滤后构成步级对（Self-edited pair）。
3. **错误感知 DPO 训练**：将步级对和全序列对混合，用三合一损失训练。

### 关键设计

**1. 五类预定义微妙错误**

作者从模型生成的错误解中归纳出五种常见步内错误：

- **(1) 数值计算错误**：如 $3 \times 7 = 24$（占比最大，约 60%）
- **(2) 数值/符号替代错误**：如将已知的 $x=5$ 错误替代为 $x=3$
- **(3) 计算项遗漏**：如 $a + b + c$ 被省略为 $a + b$
- **(4) 计算顺序错误**：如先乘后除变成先除后乘
- **(5) 运算符号错误**：如 $+$ 变成 $-$

前三类最为常见，是 RISE 主要针对的目标。

**2. 三种编辑操作**

自编辑 prompt 包含三种操作类型：

- **REPLACE**：将步骤中的某个数字/变量替换为另一个值（如 $12 \to 15$）
- **SWAP**：交换操作数位置（如 $a - b \to b - a$）
- **DELETE**：删除某个计算项或步骤片段

每次编辑只修改约 **10 个 token**，确保修改足够微妙。编辑 prompt 包含完整上下文：问题 $x$、前序步骤 $\hat{y}_{<i}$、当前正确步骤 $\hat{y}_i^+$、以及编辑类型 $e$。

**3. Levenshtein 距离过滤**

为保证自编辑样本的质量，对生成的步级对用编辑距离过滤：

$$\mathcal{D}_{\text{Edit}}^{\pm} = \{(\hat{y}_i^+, \hat{y}_i^-) \mid LD(\hat{y}_i^+, \hat{y}_i^-) < \alpha\}$$

其中 $LD$ 为 Levenshtein 距离函数，$\alpha$ 为阈值。距离太大的说明 LLM 编辑时改动过多，不再是"微妙"错误，直接丢弃。

**4. 自适应 NLL 稳定训练**

由于自编辑对的正负样本高度相似（只差几个 token），标准 DPO 容易导致正确解的生成概率也被压低（概率崩塌）。RISE 引入自适应 NLL 损失：当 policy model 对正确解的生成概率 **低于** reference model 时（即 $r < 0$），激活 NLL 来"托底"：

$$r = \log \frac{\pi_\theta(\hat{y}_i^+ | x; \hat{y}_{<i}^+)}{\pi_M(\hat{y}_i^+ | x; \hat{y}_{<i}^+)}$$

这个设计非常巧妙——只在需要时介入，避免过度正则化。

### 损失函数

总损失由三部分组成：

$$\mathcal{L} = \mathcal{L}_{\text{DPO-Edit}} + \mathcal{L}_{\text{DPO-Full}} + \lambda \cdot \mathbb{1}_{[r<0]} \cdot \mathcal{L}_{NLL}$$

- $\mathcal{L}_{\text{DPO-Edit}}$：步级 DPO，在自编辑的 $(y_i^+, y_i^-)$ 对上以前序步骤为条件进行偏好学习
- $\mathcal{L}_{\text{DPO-Full}}$：序列级 DPO，在完整的正确/错误解对上进行标准偏好学习
- $\mathcal{L}_{NLL}$：正确解的负对数似然损失，仅在 $r < 0$ 时激活，权重为 $\lambda$

### 训练细节

- 训练数据：约 9K 数学问题（来自 MetaMath + AQuA），只用问题不用原始解
- 采样次数 $K=5$，自编辑对数 $N=1$（Qwen2）或 $N=3$（Llama-3.1）
- 最终训练集约 4.5K 有效样本（需要至少一个正确+一个错误解的问题才可用）
- 基座模型：Qwen2-7B/72B-Instruct、Llama-3.1-8B/70B-Instruct

### 消融实验

| 消融项 | GSM8K (Qwen2) | MATH (Qwen2) | GSM8K (Llama) | MATH (Llama) |
|--------|---------------|--------------|---------------|--------------|
| 完整 RISE | 88.4 | 59.9 | 87.9 | 51.0 |
| w/o 自编辑对 | 88.3 | 58.2 (-1.7) | 86.8 (-1.1) | 49.9 (-1.1) |
| w/o 全序列对 | 88.0 | 58.1 (-1.8) | 86.6 (-1.3) | 50.3 (-0.7) |
| w/o NLL 损失 | 88.2 | 59.4 (-0.5) | 87.4 (-0.5) | 50.7 (-0.3) |

分析：(1) 自编辑对和全序列对各自贡献相近、互为补充，两者结合才能达到最优；(2) Llama 系列对自编辑对的依赖更强（移除后 GSM8K 掉 1.1%），可能因为 Llama 的解更长、步内错误更多；(3) NLL 的贡献虽然看起来小（约 0.3-0.5%），但它是训练稳定性的关键保障。

## 亮点与洞察

1. **错误分类驱动的方法设计**：不是盲目地"给更多数据"，而是先精确分析错误分布（75% 步内微妙错误），再针对性设计解决方案。这种数据驱动的研究范式值得借鉴。
2. **自编辑 = 自我博弈的一种形式**：让 LLM 既生成正确解又注入错误，类似于 self-play——模型既要学会"做对"，也要学会"什么样的错是容易犯的"。用同一个模型编辑比用外部模型更合理，因为它最了解自己的 error-prone token。
3. **极致的数据效率**：仅 4.5K 样本就在 MATH 上提升 7.7%，远优于需要几十万样本的 SFT 方法，且不需要 GPT-4 标注（Step-DPO 依赖 GPT-4）。
4. **自适应 NLL 的精巧设计**：$\mathbb{1}_{[r<0]}$ 指示函数让 NLL 只在正确解概率低于参考模型时才激活，既避免了概率崩塌又不会过度正则化，是一个简洁优雅的工程技巧。
5. **跨领域泛化的解释**：数学推理中学到的"避免微妙错误"能力可以迁移到逻辑推理和代码生成，暗示这些任务共享某种"精确执行"的底层能力。

## 局限性 / 可改进方向

1. **错误类型覆盖有限**：仅针对 5 种数学推理中的微妙错误，更广泛的推理错误（如概念混淆、公式选择错误）和知识密集型任务（法律、医学）中的错误尚未覆盖。
2. **竞赛级问题上表现平淡**：在 AIME24 上无提升（4/30 → 4/30），因为竞赛级问题的失败不是微妙执行错误，而是更深层的方法层面缺失。RISE 的能力边界在于——它只能修补"会做但做错"的问题，无法帮助"不会做"的问题。
3. **采样次数的反直觉结果**：更多采样 ≠ 更好效果。当 $K$ 增大时，会引入"极端"问题（模型总对或总错），反而影响训练。这表明需要更好的问题筛选策略。
4. **编辑质量依赖模型能力**：用小模型编辑可能注入不够自然的错误，用大模型编辑成本高。未来可探索可控的 token 级扰动（如直接在 embedding 空间做 perturbation）替代 prompt-based editing。
5. **缺少 iterative 训练**：只做了一轮 RISE 训练。如果用训练后的模型重新采样+自编辑进行多轮迭代，可能进一步提升。

## 相关工作与启发

- **Step-DPO (Lai et al., 2024)**：步级偏好学习的先驱，但需要 GPT-4 进行步级标注。RISE 省去了这一昂贵环节。
- **SCDPO (Lu et al., 2024b)**：也做步级 DPO，但用正确性标注过滤步骤。RISE 的自编辑方式更直接、更可控。
- **Hard negative mining**：对比学习中的经典思路，RISE 将其成功引入偏好学习领域。
- **Self-play/self-edit**：与 self-play fine-tuning（SPIN）思路相通——都是让模型与自身的变体对抗，但 RISE 更有针对性（预定义错误类型 vs 随机变体）。
- **对后续研究的启发**：(1) 其他任务（代码、翻译）是否也能分析错误分布后做定向错误注入？(2) 能否让模型自动发现自己的 error pattern 然后自适应注入？(3) 与 process reward model 结合——用 PRM 来验证自编辑样本的质量。

### 关键发现

- 微妙错误的难负样本比 rejection sampling 得到的"容易"负样本有效得多——MATH 提升从 +1.6%（DPO）到 +7.9%（RISE）
- 三种错误类型（REPLACE/SWAP/DELETE）的效果有差异，但组合使用效果最佳，说明多样性的微妙错误能覆盖更全面的推理弱点
- 步级 DPO-Edit 和序列级 DPO-Full 互补而非冗余——步级提供精确的错误定位信号，序列级提供全局质量判断
- RISE 训练的模型在逻辑推理（LogiQA +2.1%）和代码生成（HumanEval +1.8%）上也有提升，说明"辨别微妙错误"的能力可迁移
- 仅 4.5K 训练样本即可获得如此显著的提升，说明数据质量远比数据量重要

## 亮点与洞察

- **错误分类驱动方法设计**：RISE 的一切从"75% 的错误是微妙步内错误"这个经验发现出发——数据分析先行，方法设计后行。这种研究范式值得借鉴
- **自编辑的巧妙性**：不人工写错误，而是让 LLM 自己在正确解上做可控编辑——模型最了解什么样的错误"看起来合理"
- **Levenshtein 距离作为微妙度的度量**：简单但有效的质量控制，确保负样本不会退化为 random sampling
- **极高的数据效率**：4.5K 样本，MATH +7.9%——说明好的偏好数据构造策略可以用很小的数据量撬动很大的性能提升

## 局限性 / 可改进方向

- 错误分类体系基于人工分析，可能遗漏了某些错误模式
- LLM 自编辑的质量依赖于基础模型的编辑能力——较弱的模型可能生成不自然的微妙错误
- 仅在数学推理上深入验证，逻辑推理和代码的泛化实验较浅
- 未与更新的偏好学习方法（如 KTO、ORPO）结合验证
- Levenshtein 距离阈值是人工设定的，自动化阈值选择可以进一步研究
- 错误注入集中在数值和操作层面，对于更抽象的推理错误（如概念混淆）覆盖不足

## 相关工作与启发

- **vs rejection sampling DPO**：标准方法通过多次采样获取正/负样本，但负样本通常与正样本差异太大，模型简单区分即可——RISE 的难负样本逼模型学到更细粒度的推理能力
- **vs Step-DPO**：Step-DPO 在步级做偏好学习但负样本仍来自采样，RISE 用构造的微妙错误提供更精准的步级偏好信号
- **vs 数学验证器（ORM/PRM）**：验证器从外部判断正确性，RISE 从训练数据角度让生成器自身学会辨别微妙错误——两者互补
- **启发**：错误注入的思路可以扩展到其他需要"精细辨别力"的场景——如事实核查（注入微妙的事实错误）、代码 debug（注入微妙 bug）
