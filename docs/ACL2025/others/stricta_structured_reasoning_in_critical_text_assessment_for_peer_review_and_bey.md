# STRICTA: Structured Reasoning in Critical Text Assessment for Peer Review and Beyond

**会议**: ACL 2025
**arXiv**: [2409.05367](https://arxiv.org/abs/2409.05367)
**代码**: [GitHub](https://github.com/UKPLab/acl2025-stricta)
**领域**: LLM推理
**关键词**: structured reasoning, peer review, causal model, text assessment, human-AI collaboration

## 一句话总结
提出 STRICTA 框架，基于结构因果模型（SCM）将文本评审建模为显式的逐步推理图（workflow），在生物医学论文评审中收集 40+ 位专家的 4000+ 推理步骤数据集，发现先验知识差异是专家分歧主因、写作风格对最终评审有因果影响，LLM 存在错误传播但人类监督可有效缓解。

## 研究背景与动机

1. **领域现状**：关键文本评审（同行评审、事实核查、作文评分）是专家活动核心。现有自动化方法将评审视为黑箱，不揭示专家如何得出判断。
2. **现有痛点**：(a) 缺乏形式化模型捕捉专家推理过程；(b) 现有事实核查解释仅面向短声明，不处理主观性；(c) LLM 在评审中的能力边界不明。
3. **核心矛盾**：要实现透明的人-AI 协作评审，必须先理解"专家是如何推理得出评判的"。
4. **本文要解决什么**：(a) 形式化评审推理过程；(b) 提供实证数据集研究人类推理模式；(c) 评估 LLM 在结构化推理中的表现。
5. **切入角度**：用 SCM 表示评审推理——每个推理步骤是变量，步骤间依赖是因果边，背景变量代表主观性。
6. **核心 idea**：将"阅读→思考→打分"分解为有因果依赖的结构化 workflow，支持因果分析和细粒度人-AI 协作。

## 方法详解

### 整体框架
STRICTA 定义为 SCM $\mathcal{M} = (U, V, F, P_\mathcal{M})$：$V$ 含输入节点 $I$（文本）、推理步骤 $C$、最终判决 $T$；$F$ 是结构方程；$U$ 是背景变量（主观性来源）。

### 关键设计

1. **三步框架实例化**
   - **Step 1: 设计 SCM 结构**：通过专家访谈确定 45 个互联推理步骤，三类活动：read（阅读）、extract（提取）、infer（推理）
   - **Step 2: 填充数据**：40+ 位专家按 workflow 评审 22 篇论文，产出 4371 个推理步骤答案
   - **Step 3: 分析**：高斯过程分类器估计布尔节点结构方程，支持 ACE 和反事实查询

2. **因果分析**
   - **ACE 估计**：结论与研究问题对齐度（step33）对最终评判影响最大 ACE=0.37；写作清晰度 ACE=0.20——**写作风格对评审结果有因果影响**（非仅关联）
   - **反事实分析**：60% 负面评判仅需改善"图文对齐"即可翻转
   - 背景知识步骤变异最大 → 先验知识是专家分歧主因

3. **LLM 辅助推理**
   - 将 workflow 作为 LLM program 逐步执行（溯因推理：给定论文+判决，重建中间推理）
   - LLM 事实一致性接近人类，但布尔决策差距大（倾向"重述"而非聚焦关键论据）
   - **人类监督消除错误传播**：io-condition 下所有 LLM 性能显著提升

## 实验关键数据

### 因果效应 (ACE)

| 推理步骤 | ACE | 说明 |
|---------|-----|------|
| 结论与研究问题对齐 (step33) | **0.37** | 影响最大 |
| 结论相关性 (step46) | 0.20 | |
| 写作清晰度 (step48) | 0.20 | 风格有因果效应 |
| 是否方法论文 (step4) | 0.02 | 几乎无影响 |

### LLM 性能（溯因推理）

| 模型 | BERT-F1↑ | Boolean F1↑ |
|------|----------|-------------|
| Human baseline | 0.55 | **0.71** |
| GPT-4o | 0.48 | 0.52 |
| GPT-3.5 | 0.46 | 0.48 |
| Mixtral | 0.45 | 0.45 |
| LLama3-8B | 0.42 | 0.40 |

### 关键发现
- **先验知识是分歧主因**：infer-knowledge 步骤语义相似度最低（~0.45），extract 最高（~0.65）
- **写作风格有因果效应**——好的写作直接因果提升评审结果，不仅是间接关联
- **"改图可翻转"**：60% 负面评审可通过仅 1 个节点干预（图文对齐）翻转
- **LLM "重述"倾向**：事实覆盖高但判断准确率低——权衡因素与人类不同
- **人类监督关键**：逐步校正下 LLM 性能显著提升，STRICTA 天然支持有效人-AI 协作

## 亮点与洞察
- **因果框架 + NLP 创新结合**：首次将 SCM 系统用于文本评审——不仅"哪些特征重要"，而是"改变什么会改变结果"
- **反事实分析的实用价值**：可告诉作者"如果改善图文对齐，评审会翻转"——比笼统建议更有指导性
- **"写作风格有因果效应"的学术启示**：实证证据表明审稿人确实被写作质量因果性地影响了科学判断
- **LLM program 架构**：workflow 作为执行脚手架天然防止"跳步"和"幻觉"

## 局限性 / 可改进方向
- **仅限生物医学**：其他领域评审推理结构可能不同
- **数据规模有限**：22 篇论文/93 次执行，对 SCM 估计偏小
- **未与既有评审辅助系统对比**
- **LLM 评估指标不足**：自动指标与人类判断相关性中等

## 相关工作与启发
- **vs 自动评审生成 (D'Arcy et al., 2024)**：他们生成完整报告，本文关注推理过程——更深层
- **vs CoT prompting**：CoT 是自由形式推理链，STRICTA 是有因果约束的结构化推理——更可控
- **启发**：SCM+LLM program 范式可推广到医学诊断、法律论证等需要结构化判断的任务

## 评分
- 新颖性: ⭐⭐⭐⭐⭐ SCM+文本评审结合全新，形式化框架有理论深度，数据集独一无二
- 实验充分度: ⭐⭐⭐⭐ 40+专家+4 LLM+因果分析+协作模拟，但数据量偏小
- 写作质量: ⭐⭐⭐⭐⭐ Definition→Instantiation→Analysis→LLM 逻辑链完整
- 价值: ⭐⭐⭐⭐⭐ 对 AI+同行评审有开创贡献，因果发现对学术出版有实际意义
