# Ewe: Improving Factuality with Explicit Working Memory

**会议**: ACL 2025  
**arXiv**: [2412.18069](https://arxiv.org/abs/2412.18069)  
**代码**: 无（Meta FAIR）  
**领域**: 其他  
**关键词**: factuality, working memory, RAG, fact-checking, hallucination

## 一句话总结
提出 Ewe（Explicit Working Memory），用显式工作记忆增强 LLM 的事实性长文本生成——记忆由多个 KV cache 单元组成，实时接收检索知识和事实核查反馈，错误句子被删除重生，在 4 个事实生成基准上 VeriScore 提升 2-6 分且不损失回答有用性。

## 研究背景与动机
RAG 减少幻觉但仅提供单次静态知识。迭代 RAG（FLARE/Self-RAG）改善但仍受限于传统 RAG 设计——知识仅作为输入字符串的一部分。Ewe 用并行 KV cache 记忆单元存储多源反馈（检索+事实核查），可动态追加/删除/更新，更灵活高效。

## 方法详解
(1) 初始检索 k 个段落 → 编码为 k 个记忆单元（各自 KV cache）。(2) 生成过程中定期暂停：检索新段落更新记忆 + 事实核查当前句子。(3) 若有事实错误 → 将正确信息作为新记忆单元追加 + 删除错误句子 + 从上一步重新生成。(4) 注意力聚合：生成 token 同时 attend k+1 个源（k 个记忆+纯上下文），按注意力归一化加权平均。FIFO 更新策略。

## 实验关键数据
| 模型 | VeriScore 提升 | 有用性 |
|------|:---:|:---:|
| Llama-3.1-70B (基线) | - | 基线 |
| + RAG | +少量 | 保持 |
| + FLARE/DRAGIN | +中等 | 保持 |
| **+ Ewe** | **+2~6分** | **保持** |

人类评估确认 VeriScore 提升确实转化为事实性改善。

## 亮点与洞察
- **工作记忆设计**极其灵活：多个 KV cache 单元并行编码不同来源知识，可独立更新/追加/删除。
- **实时事实核查反馈**区别于传统 RAG——不只检索新知识，还验证已生成内容并纠错。
- **退化为已有方法的特例**：k=1 无停顿=RAG，k=1 逐句停顿+低概率触发=FLARE——Ewe 是更通用的框架。

## 局限性 / 可改进方向
- 每步需多次前向传播（检索+事实核查+重生成），推理成本高。FIFO 记忆更新策略可能不最优。
