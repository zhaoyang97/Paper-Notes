# An Effective Incorporating Heterogeneous Knowledge Curriculum Learning for Sequence Labeling

**会议**: ACL 2025  
**arXiv**: [2402.13534](https://arxiv.org/abs/2402.13534)  
**领域**: 序列标注 / 课程学习  
**关键词**: 课程学习, 序列标注, 贝叶斯不确定性, 中文分词, 命名实体识别

## 一句话总结

提出面向序列标注任务的双阶段课程学习（DCL）框架，通过数据级和模型级两阶段的由易到难训练策略以及基于贝叶斯不确定性的动态难度度量，在提升性能的同时加速训练超过 25%。

## 研究背景与动机

序列标注是 NLP 的核心任务，近年来许多研究通过整合外部知识（如 n-gram、词典、句法信息）来增强模型表现。然而，这种做法带来了两个问题：

1. **数据异质性**：引入外部知识使输入数据变得异构且复杂
2. **模型复杂度增加**：集成额外编码模块（如注意力机制、GNN）增加了参数量，使训练代价高昂

课程学习（CL）通过模拟人类的学习过程——从简单到困难地逐步引入训练样本——可以有效应对这些挑战。然而，现有的课程学习难度度量主要集中在句子级别，缺乏适用于序列标注任务的 token 级和词级难度度量。

## 方法详解

### 整体框架

DCL 框架由三个核心组件构成：教师序列标注模型、学生序列标注模型和双阶段课程学习训练策略。关键特点是 DCL 独立于具体的序列标注模型，可与任意模型结合使用。

**第一阶段：数据级课程学习**
- 在全部训练数据 $\mathcal{D}$ 上训练一个基础教师模型 $\theta_0$（训练 $E_0$ 个 epoch，少于收敛所需）
- 用教师模型对所有样本计算难度分数 $S(\theta_0)$
- 按难度升序排序，形成排序后数据集 $\mathcal{D}_r$

**第二阶段：模型级课程学习**
- 用排序后数据集的最简单子集（比例 $\lambda_0$）初始化学生模型训练集 $\mathcal{D}_s$
- 每个 epoch 后：用当前学生模型 $\theta_*$ 重新评估剩余数据难度，按更新后的难度排序，根据调度器增大 $\lambda$ 扩展训练集
- 当 $\lambda = 1$ 时，使用全部数据训练至收敛

### 关键设计

**1. 贝叶斯不确定性（BU）难度度量**

基于 Monte Carlo dropout 近似贝叶斯推断：
- 对每个 token $x_i$ 执行 $K$ 次随机前向传播，得到 $K$ 个预测分布
- 计算 token 级方差（对标签集上的概率取方差）
- 句子级难度 = 最大 token 方差 + 平均 token 方差：$S(\theta)^{BU} = var(\theta)_{max} + var(\theta)_{aver.}$

这一设计同时捕获了序列中最不确定的位置（max）和整体不确定性水平（avg），比仅用一个更全面。

**2. Root 函数训练调度器**

$$\lambda = \min\left(1, \sqrt{\frac{1-\lambda_0^2}{E_{grow}} \cdot t + \lambda_0^2}\right)$$

该函数确保模型有足够时间学习新引入的样本，同时逐步减少每轮新增样本数量。相比线性调度器，root 函数在初期增长更快，后期增长更慢，符合学习规律。

**3. 模型级动态难度重评估**

与静态课程学习不同，DCL 在模型级阶段会用当前学生模型动态重新评估剩余数据的难度。随着学生模型能力增强，之前"困难"的样本可能变"简单"，因此动态重评估能更好地适应模型的学习轨迹。

## 实验关键数据

### 主实验

在 CTB5/CTB6/PKU 三个中文数据集上（SynSemGCN + DCL(BU)）：

| 数据集 | CWS F1 | POS F1 |
|--------|---------|--------|
| CTB5 | 98.90 (+0.07) | 97.95 (+0.09) |
| CTB6 | 98.05 (+0.19) | 95.14 (+0.16) |
| PKU | 98.59 (+0.11) | 96.54 (+0.14) |

NER 泛化实验（BERT + DCL(BU)）：

| 数据集 | F1 (无CL → +DCL(BU)) |
|--------|------------------------|
| Weibo（中文） | 66.22 → 66.74 |
| OntoNotes4（中文） | 79.15 → 80.02 |
| CoNLL-2003（英语） | 90.94 → 91.77 |

### 关键发现

1. **BU 度量最优**：在大多数数据集上，BU 难度度量一致性地优于 TLC、MNLP、句子长度和随机排序
2. **训练加速 25%+**：DCL 不仅提升性能，还将训练时间从 393 分钟缩短到 287 分钟
3. **消融实验**：模型级 CL 的影响大于数据级 CL（模型级影响全训练过程，数据级主要影响初期）
4. **BU 两个组件同等重要**：去除 $var(\theta)_{max}$ 或 $var(\theta)_{aver.}$ 都会导致性能下降
5. **跨任务泛化**：DCL 方法在中文分词、词性标注和命名实体识别三类序列标注任务上均有效

## 亮点与洞察

1. **双阶段设计合理**：数据级 CL 解决冷启动问题，模型级 CL 实现动态自适应课程
2. **BU 度量的 token 级设计**：相比仅有句子级度量的先前工作，BU 能更精准地捕捉序列标注的特殊难度特征
3. **模型无关性**：DCL 作为训练策略可即插即用地与不同序列标注模型结合
4. **实际效果显著**：性能提升 + 训练加速的双重收益表明课程学习在复杂模型训练中的实用价值

## 局限性

- 难度度量涉及多个超参数（如 Monte Carlo dropout 次数 $K$、$\lambda_0$、$E_{grow}$ 等），调优成本较高
- 仅探索了由易到难的课程策略，未研究由难到易（anti-curriculum）的效果
- 训练调度器仅使用了 Root 函数，未对比其他调度策略（线性、阶梯等）
- 实验数据集规模相对较小（CTB5/6/PKU），在大规模序列标注任务上的效果有待验证
