# A Variational Approach for Mitigating Entity Bias in Relation Extraction

**会议**: ACL 2025  
**arXiv**: [2506.11381](https://arxiv.org/abs/2506.11381)  
**代码**: 待确认  
**领域**: 信息抽取 / 关系抽取  
**关键词**: 关系抽取, 实体偏差, 变分信息瓶颈, VIB, 去偏, 可解释性, 跨域泛化  

## 一句话总结

将变分信息瓶颈（VIB）应用于关系抽取的实体去偏——将实体映射到概率分布 $\mathcal{N}(\mu,\sigma)$，通过方差控制实体信息的压缩程度（高方差=更多依赖上下文），在 TACRED/REFinD/BioRED 三个领域（通用/金融/生物医学）的域内和域外设置上达到 SOTA，同时方差分析提供可解释性。

## 背景与动机

关系抽取（RE）中模型常过度依赖实体本身（如"Microsoft"→投资关系）而非上下文线索，导致域外泛化差。现有去偏方法：

1. **实体掩码**：用通用标记替换实体——但丢失必要的实体类型信息
2. **实体替换**：用邻近实体替换——启发式且改善有限
3. **SCM（结构因果模型）**：当前 SOTA，用邻居实体的凸包中心替换实体嵌入——但缺乏概率解释

## 核心问题

如何在压缩实体特异信息的同时保留任务相关特征，并提供对去偏程度的可解释度量？

## 方法详解

### VIB 实体压缩

1. 将实体 token 映射为高斯分布：$p(z|x,e) = \mathcal{N}(\mu, \sigma)$，$\mu$ 和 $\sigma$ 由单层感知机从实体嵌入计算
2. 重参数化采样：$z = \mu + \epsilon \cdot \sigma$，$\epsilon \sim \mathcal{N}(0,1)$
3. **VIB 损失**最小化 $I(X;Z|E)$ 的上界——即 $KL(p(z|x,e) || r(z|e))$，其中 $r(z|e) = \mathcal{N}(0,I)$
4. 通过实体掩码 M 选择性压缩：非实体 token 保持原始嵌入，实体 token 混合原始嵌入和采样 z（混合因子 β=0.5）

### 方差的可解释性

- **低方差 σ²**：模型对该实体信息依赖强，嵌入接近确定性
- **高方差 σ²**：模型对该实体知道得少，更多依赖上下文
- 通过方差可定量分析哪些实体被过度依赖

### 训练

总损失 = 交叉熵 BCE + α·VIB 损失，α 自适应权重（两损失之比）。RoBERTa-Large 和 LUKE-Large 作为骨干。

## 实验关键数据

### 主实验（LUKE-Large，Micro-F1）

| 数据集 | 方法 | ID↑ | OOD↑ |
|--------|------|-----|------|
| TACRED | LUKE base | 71.1 | 63.8 |
| TACRED | SCM (SOTA) | 68.6 | 64.8 |
| TACRED | **VIB (ours)** | **70.4** | **66.5** |
| REFinD | LUKE base | 75.0 | 73.4 |
| REFinD | **VIB (ours)** | **75.4** | **74.8** |
| BioRED | LUKE base | 56.9 | 51.8 |
| BioRED | SCM | 58.3 | 53.4 |
| BioRED | **VIB (ours)** | **61.2** | **58.7** |

- **所有三个领域+两个骨干上全面 SOTA**（LUKE-Large 上）
- BioRED 提升最大：OOD 从 51.8→58.7（+6.9），说明生物医学领域实体偏差最严重
- 实体掩码方法 ID 性能大幅下降——证明完全去除实体信息会损害性能

### 方差分析

- 高频实体（如"Microsoft"）方差更低——模型对它们更 confident→更 biased
- 罕见实体方差更高——模型被迫依赖上下文
- 通过 VIB，高频实体的方差被人为增大→减少对特定实体的依赖

## 亮点

- **理论优雅**：VIB 提供了信息论框架下的实体去偏方案，比启发式方法（掩码/替换/SCM）更有原则性
- **可解释性**：方差直接量化实体偏差程度——不只是去偏，还能诊断偏差
- **首次覆盖三个领域**：在通用/金融/生物医学上统一验证，证明实体偏差是普遍问题
- **简洁实现**：仅需额外两个 SLP（生成 μ 和 σ），计算开销极小

## 局限性 / 可改进方向

- **β 需要调参**：混合因子 β=0.5 是搜索得到的，敏感性分析有限
- **仅在 PLM 上验证**：未测试 LLM 骨干
- **r(z|e)=N(0,I) 假设可能过强**：更灵活的先验可能更好
- **OOD 测试集通过自动替换构造**：可能不完全反映真实域外场景

## 与相关工作的对比

- **vs 实体掩码**：掩码完全去除实体信息导致 ID 大幅下降；VIB 保留部分信息
- **vs SCM（当前 SOTA）**：SCM 用邻居凸包是确定性的；VIB 是概率性的且可解释
- **vs 反事实分析**：反事实方法需要构造反事实样本；VIB 直接在嵌入空间操作

## 启发与关联

- VIB 作为信息压缩的通用工具可推广到其他需要去偏的任务（如情感分析的 aspect 偏差）
- 方差作为偏差度量的思路非常实用——任何需要解释模型对哪些特征依赖多的场景都可用
- 概率性表示（而非确定性嵌入）是缓解偏差的有力工具
