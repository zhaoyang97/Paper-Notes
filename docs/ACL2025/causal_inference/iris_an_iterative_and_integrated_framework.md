# IRIS: An Iterative and Integrated Framework for Verifiable Causal Discovery

**会议**: ACL 2025  
**arXiv**: [2510.09217](https://arxiv.org/abs/2510.09217)  
**代码**: https://github.com/WilliamsToTo/iris  
**领域**: 其他  
**关键词**: causal discovery, LLM, knowledge graph, hybrid method, missing variable

## 一句话总结
提出 IRIS，从一组初始变量出发，自动收集文档→提取变量值→混合因果发现（统计方法+LLM关系提取）→缺失变量提议→迭代扩展因果图，放松了无环和因果充分性假设，在 6 个数据集上超越基线。

## 研究背景与动机
统计因果发现需要表格数据（昂贵）且假设不现实（无环、因果充分），LLM 因果发现只能回忆训练数据中的已知关系，无法发现新关系。IRIS 结合两者优势：LLM 提供已知关系和文档，统计方法发现新关系。

## 方法详解
(1) 用 Google API 按变量组合构造查询检索文档。(2) LLM 从文档提取变量值构建表格。(3) 混合因果发现：统计分支（GES/PC→因果图）+ LLM分支（将关系作为 claim 在学术文献中验证→因果图）→图合并。(4) 缺失变量提议：LLM 从文档抽象新变量 + PMI统计筛选 → 扩展变量集迭代。

## 实验关键数据
| 方法 | Cancer F1 | 扩展性 |
|------|:---:|:---:|
| 0-shot LLM | 0.43 | 小图 |
| CoT LLM | 0.48 | 小图 |
| RAG LLM | 更好 | 中图 |
| **IRIS** | **最优** | **4→27变量有效** |

### 关键发现
- **混合方法 > 纯统计 or 纯LLM**：统计发现新关系，LLM 验证已知关系，互补覆盖更广。
- **允许环路**更符合真实世界（贫困循环、捕食者-猎物循环等）。
- **从 4→27 变量均有效**——框架可扩展性好。

## 亮点与洞察
- **"仅需变量名"的范式转变**：传统因果发现需预收集表格数据，IRIS 全自动完成。
- **限制搜索域到学术网站**确保验证来源可信——简单有效的质量控制。
- **迭代扩展**使因果图从核心变量逐步增长——适合探索性研究。

## 局限性 / 可改进方向
- LLM 提取的变量值可能有噪声。依赖 Google API。因果方向在小样本下可能不稳定。
