# A Rose by Any Other Name: LLM-Generated Explanations Are Good Proxies for Human Explanations to Collect Label Distributions on NLI

**会议**: ACL 2025  
**arXiv**: [2412.13942](https://arxiv.org/abs/2412.13942)  
**代码**: [GitHub](https://github.com/mainlp/MJD-Estimator)  
**领域**: 标注分歧 / 人工判断分布  
**关键词**: 人工判断分布, NLI, LLM解释生成, 标签变异, 标注分歧, 解释作为代理  

## 一句话总结

研究 LLM 生成的解释能否替代昂贵的人工解释来近似 NLI 的人工判断分布（HJD）——发现在提供人工标签的条件下，LLM 生成的解释与人工解释在近似 HJD 方面效果相当（"名字不重要，玫瑰依然芬芳"），且方法可推广到无人工解释的数据集和域外测试集。

## 背景与动机

NLI 中标注者之间的分歧是真实的语义现象（而非噪声），可用人工判断分布（HJD）来捕捉。已有研究表明 LLM 可通过少量人工标签+人工解释有效近似 HJD，但收集人工解释成本远高于标签本身。LLM 在推理/情感分析等任务中能生成高质量解释——但在 NLI 标签变异的语境下尚未验证。

## 核心问题

LLM 生成的解释是否足以替代人工解释来帮助 LLM 近似 NLI 的人工判断分布？

## 方法详解

### 解释生成与选择

1. **模型解释生成**：提示 LLM 为给定前提-假设对的每个 NLI 标签（蕴含/中性/矛盾）列出所有可能解释
2. **无标签选择**：每个标签均匀选 1 条 → 3 条解释，不使用任何人工标签
3. **标签引导选择**：根据人工标签分布选择对应数量的解释（如 3 蕴含 + 1 中性 + 1 矛盾），优先选最长解释

### HJD 近似

将选中的标签-解释对输入 LLM，生成模型判断分布（MJD），与真实 HJD 比较。

### 推广实验

- 有人工解释的数据集（ChaosNLI）→ 验证模型解释可替代人工解释
- 无人工解释的数据集（MNLI/VariErr）→ 验证方法泛化
- 域外数据集（ANLI）→ 验证 HLV 信息对分类器的提升

## 实验关键数据

- LLM 解释 + 人工标签 ≈ 人工解释 + 人工标签（在 ChaosNLI 上 HJD 近似质量相当）
- 推广到无解释数据集：仅需少量人工标签即可有效近似 HJD
- ANLI 域外实验：建模 HLV 信息改善了 NLI 分类器性能
- 解释的多样性可作为 HLV 的潜在指标

## 亮点

- **"名字不重要"的核心发现**：解释的来源（人工 vs LLM）不如解释的存在重要
- **显著降低标注成本**：从需要人工解释降到只需人工标签+LLM 生成解释
- **推广性好**：从有解释数据集到无解释数据集到域外测试集均有效
- **实用的人类评估**：人工标注研究验证了 LLM 解释的质量

## 局限性 / 可改进方向

- **仅在 NLI 上验证**：其他标签变异显著的任务（如情感分析）未测试
- **仍需少量人工标签做引导选择**：完全无标签的场景效果较差
- **解释选择启发式简单**：最长解释不一定是最好的
- **HJD 基准依赖 ChaosNLI（100 标注者/实例）**

## 与相关工作的对比

- **vs Chen et al. 2024**：需要人工解释；本文用 LLM 生成解释替代，效果相当
- **vs 直接用 LLM 预测分布**：效果不如用解释引导的方法
- **vs 人工解释收集**：本方法大幅减少成本同时保持质量

## 启发与关联

- 标注分歧不是噪声——捕获并建模它对下游任务有价值（ANLI 实验证明）
- LLM 解释的代理价值暗示：在需要多样视角的任务中，LLM 可充当"多标注者"
- 解释的多样性作为 HLV 指标是一个有趣的方向

- 新颖性: ⭐⭐⭐⭐ LLM解释替代人工解释近似HJD是新颖且实用的发现
