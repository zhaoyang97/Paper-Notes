# G2S: A General-to-Specific Learning Framework for Temporal Knowledge Graph Forecasting with Large Language Models

**会议**: ACL 2025  
**arXiv**: [2506.00445](https://arxiv.org/abs/2506.00445)  
**代码**: 无  
**领域**: 时间序列 / 知识图谱  
**关键词**: 时序知识图谱, LLM微调, 泛化能力, 匿名化时序结构, 知识解耦

## 一句话总结
提出 G2S 框架，将时序知识图谱（TKG）预测中的通用模式（时序结构规律）与场景信息（具体实体/关系）解耦，先在匿名化时序结构上学习通用模式，再注入场景信息，有效提升 LLM 在 TKG 预测中的泛化能力。

## 研究背景与动机
1. **领域现状**：TKG 预测（根据历史事实预测未来）是时间敏感场景（金融、医疗、政治）的关键任务。传统方法（RE-GCN, TLogic）对每个 TKG 独立训练，泛化能力弱。近期 LLM 方法（GenTKG）通过微调提升泛化，但效果有限。
2. **现有痛点**：不同 TKG 描述不同场景（GDELT=地缘政治, WIKI=体育学术），实体和关系完全不同。LLM 同时学习通用时序模式和场景特定信息，两者学习过程互相干扰，限制泛化。
3. **核心矛盾**：通用模式（如"A对B做了X，之后A可能对C做Y"的时序规律）和场景信息（"Trump"、"visit"等具体实体关系名）纠缠在一起。
4. **本文要解决什么？** 如何解耦这两种知识的学习过程？
5. **切入角度**：匿名化——将实体、关系、时间戳替换为抽象 ID，创建场景无关的时序结构。
6. **核心idea一句话**：先在匿名化时序结构上学通用模式 → 再注入场景映射学特定知识 = 两阶段解耦学习。

## 方法详解

### 整体框架
G2S 分两阶段：(1) **通用学习阶段**：将多个 TKG 的实体/关系/时间戳匿名化为抽象 ID，在这些匿名时序结构上微调 LLaMA3-8B，学习跨场景的通用时序模式；(2) **特定学习阶段**：在目标 TKG 上，保留匿名结构同时提供 ID→实体名的映射表，通过 ICL 或 SFT 注入场景信息。

### 关键设计

1. **匿名化时序结构转换**:
   - 做什么：将 $(s, r, o, t)$ 四元组中的所有元素替换为抽象 ID
   - 核心思路：时间戳 ID = 查询时间 - 事实时间（$\mathcal{A}(t) = t_q - t$，查询为0）；实体/关系 ID 有三种策略——FID（频率排序）、GID（全局原始ID）、RID（随机分配）
   - 设计动机：消除场景特定信息，使模型只能从时序结构中学习通用模式

2. **三种匿名化策略的对比**:
   - FID（频率ID）：高频实体获得小ID，保留了频率信号
   - GID（全局ID）：用数据集原始编号，跨数据集编号不一致
   - RID（随机ID）：每个查询的实体独立随机编号，最纯粹的匿名化
   - 实验发现：RID 在零样本设置下最好（最干净的通用模式），FID/GID 在有监督设置下更好

3. **两种特定学习模式**:
   - ICL 模式：不允许使用训练数据，场景信息仅通过实体/关系名到ID的映射表在上下文中提供
   - SFT 模式：允许在目标数据集上继续微调
   - 设计动机：ICL 测试零样本泛化，SFT 测试带数据的适应能力

### 损失函数 / 训练策略
标准交叉熵 loss + LoRA 微调 LLaMA3-8B。通用阶段用 GDELT 10万 + WIKI 3万样本训练，特定阶段用目标数据集训练。

## 实验关键数据

### 主实验（标准设置 H@1）

| 数据集 | RE-GCN | TLogic | GenTKG | G2S (SFT+RID) |
|--------|--------|--------|--------|----------------|
| ICEWS14 | 31.3 | 33.2 | 37.3 | **39.1** |
| ICEWS18 | 22.3 | 23.7 | 25.3 | **28.5** |
| YAGO | 78.7 | 82.3 | 83.1 | **86.2** |

### 消融实验

| 设置 (ICEWS14 H@1) | 无G2S | +通用阶段 | +特定阶段 |
|---------------------|-------|----------|----------|
| Zero-shot ICL | 15.2 | **22.8** | - |
| 5% Low-resource | 28.1 | 33.4 | **35.7** |
| Standard SFT | 37.3 | - | **39.1** |

### 关键发现
- **通用阶段显著提升零样本性能**：H@1 从 15.2% 提升到 22.8%（+7.6pp），证明匿名化结构中确实存在可迁移的通用模式
- **RID 在零样本最佳**：随机编号最纯粹地捕捉时序结构，避免了频率/编号偏置
- **低资源场景收益最大**：5%数据下 G2S 将 H@1 从 28.1% 提升到 35.7%（+7.6pp），接近标准设置
- **解耦学习有效**：同时学两种知识 vs 先通用后特定，后者在所有设置下更优

## 亮点与洞察
- **知识解耦的洞察**：识别出 TKG 中通用模式和场景信息的纠缠是泛化瓶颈，匿名化是简洁有效的解耦手段。这种"先学结构后学具体"的范式可推广到其他需要跨域泛化的任务。
- **时间戳相对编码**：将时间戳编码为相对于查询的偏移量，消除了训练/测试时间不一致的偏置，简单但有效。
- **匿名化策略的影响**：RID/FID/GID 各有优劣，为 TKG+LLM 的实践提供了指导。

## 局限性 / 可改进方向
- LLM 无法为所有实体生成完整排名，因此无法计算 MRR 指标
- 通用阶段数据仅来自 GDELT+WIKI 两个来源，更多样的 TKG 可能进一步提升
- 匿名化丧失了实体名的语义信息（如"Trump"→"E5"），可能错过有用的名称关联
- 未探索通用阶段的最优训练策略（数据量、采样比例等）

## 相关工作与启发
- **vs GenTKG**：GenTKG 直接在目标 TKG 上微调 LLM，G2S 增加通用预训练阶段，标准设置下 H@1: 39.1 vs 37.3
- **vs ICL (Lee et al.)**：ICL 方法用 GPT-NeoX 等大模型做零样本，G2S 在相同设置下提升 7+pp
- **vs RE-GCN/TLogic**：传统方法无跨域泛化能力，G2S 在标准设置下也超越它们
