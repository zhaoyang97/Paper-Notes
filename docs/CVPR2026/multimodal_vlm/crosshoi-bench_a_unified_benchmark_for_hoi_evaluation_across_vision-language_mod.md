# CrossHOI-Bench: A Unified Benchmark for HOI Evaluation across Vision-Language Models and HOI-Specific Methods

**会议**: CVPR2026  
**arXiv**: [2508.18753](https://arxiv.org/abs/2508.18753)  
**代码**: [github.com/ChelsieLei/CrossHOI-Bench](https://github.com/ChelsieLei/CrossHOI-Bench)  
**领域**: 多模态VLM  
**关键词**: 人-物交互检测, VLM评估, HOI基准, 多选题, 跨范式比较

## 一句话总结
提出 CrossHOI-Bench，首个统一评估 VLM 和 HOI 专用模型的多选题 HOI 基准，通过精心策划的正负例避免不完整标注的错误惩罚，揭示了大型 VLM 零样本可比肩 SOTA HOI 方法，但在多动作识别和跨人归因上各有优劣。

## 研究背景与动机
1. HOI 检测长期由任务专用模型主导，但大型生成式 VLM 展示了强大的开放场景理解能力，引发"VLM 能否直接做 HOI 检测"的核心问题
2. 现有 HOI 基准（如 HICO-DET）依赖精确标签匹配+不完整标注——任何未标注但正确的预测被判错
3. 不完整标注导致系统性低估：HOI 专用方法 <50% mAP，VLM 在 HICO-DET 上仅约 15%
4. 歧义问题无法通过穷尽标注解决——单张图像常缺乏足够视觉证据区分动作（如"登机"vs "下机"）
5. HICO-DET 训练/测试集分布高度相似（KL=0.088），大量简单头部类场景难以真正评估模型能力
6. 需要一个统一协议，让 VLM 和 HOI 专用方法在同一标准下公平比较

## 方法详解

### 整体框架
将 HOI 检测重构为多答案多选题：每个问题明确定义正例和精心策划的负例，避免因标注不完整导致的错误惩罚。

### 关键设计

**数据集构建流程**：
1. **任务重构**：四选一格式，允许单题多正确答案（如同时 "hold knife" 和 "cut with knife"）
2. **粗筛**：用 GPT-4.1 初筛 → Qwen2.5-VL-32B + GPT-4o 双模型一致性验证，仅保留双方一致认定错误的负例
3. **人工精修**：移除过于简单场景，添加难正例（歧义动作）和难负例（周围人的动作、细粒度相似动作）
4. **重分布测试集**：与 HICO-DET 训练集 KL 散度从 0.088 提升到 0.629，减少数据先验偏置

**三种评估设置**：
- **Setting 1**：完整 HOI 检测（检测+识别），需先检测到目标人物（IoU≥0.5）
- **Setting 2**：给定目标人物框，仅评估交互识别（诊断定位瓶颈）
- **Setting 3**：图像级 HOI 识别，识别所有人的交互（评估全局理解）

**互补子基准**：
- V-COCO 扩展：多人场景（647 题）
- SWiG-HOI 扩展：人-人交互（1,852 题）

### 评估指标
Macro-F1、Instance-F1、Micro-F1、Exact Match (EM)、平均精确率/召回率

### VLM vs HOI 专用方法关键对比

| 维度 | VLM 优势 | HOI 专用优势 |
|------|---------|-------------|
| 整体 F1 | 大模型零样本超越 SOTA | 小模型与 HOI 方法持平 |
| 多动作识别 | 倾向只预测一个动作 | 更好识别共现多动作 |
| 跨人归因 | 常将周围人的动作归给目标人 | 更准确的人-动作绑定 |
| 精确率 | VLM 通常更高 | 较稳定 |
| 召回率 | 部分VLM极低 | 更均衡 |

### 关键发现
- Qwen2.5-VL-32B 零样本在 Instance-F1 上超越所有 HOI 方法 +5.18%
- 小型 VLM（7-8B）在仅识别设置下与 HOI 方法持平，但需要检测时性能下降
- VLM 的核心弱点：多动作预测不足（只预测一个交互）和跨人动作误归因
- Qwen3-VL-30B 出现极端行为：Avg.Prec=100% 但 Recall≈0%，几乎不做预测

## 亮点与洞察
- 首次在统一协议下比较 VLM 和 HOI 专用方法，结论有说服力
- 揭示了现有 HOI 基准系统性低估模型能力的根本原因
- 多选题格式+精心负例设计，方法论上可迁移到其他标注不完整的评估任务
- 三种互补设置和子基准提供了多维度的能力剖析

## 局限性
- 基准规模相对较小（1,274 主基准图片），可能不足以覆盖所有 HOI 类型
- 多选题格式可能简化了开放世界 HOI 理解的真实难度
- 负例策划依赖 VLM 一致性判断，可能引入特定模型偏见
- 未提供微调数据的实验深入分析

## 相关工作与启发
- 与 HICO-DET/V-COCO 的根本区别：多答案多选题 vs 精确匹配，避免不完整标注的错误惩罚
- 与 SWiG-HOI 的互补：CrossHOI-Bench 更关注跨范式比较而非开放词汇
- 启发：VLM 在 HOI 理解上的能力被严重低估，未来应更多关注多动作和跨人归因问题
