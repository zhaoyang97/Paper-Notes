# UniMMAD: Unified Multi-Modal and Multi-Class Anomaly Detection via MoE-Driven Feature Decompression

**会议**: CVPR 2026  
**arXiv**: [2509.25934](https://arxiv.org/abs/2509.25934)  
**代码**: [https://github.com/yuanzhao-CVLAB/UniMMAD](https://github.com/yuanzhao-CVLAB/UniMMAD) (即将开源)  
**领域**: 多模态VLM / 异常检测  
**关键词**: 多模态异常检测, MoE特征解压, 统一框架, 多类别, 稀疏门控  

## 一句话总结
提出 UniMMAD，一个统一的多模态多类别异常检测框架，核心是 MoE 驱动的特征解压机制——编码阶段将多模态输入压缩为通用特征并抑制潜在异常，解码阶段通过稀疏门控交叉 MoE 将通用特征解压为模态特定和类别特定的重建，配合分组动态滤波和 MoE-in-MoE 结构减少 75% 参数，在 9 个数据集（3领域12模态66类别）上达到 SOTA。

## 背景与动机
现有异常检测方法将模态和类别视为独立因素，导致解决方案碎片化和内存开销过大。基于重建的多类别方法通常依赖共享解码路径，但不同领域/类别的正常模式差异巨大，共享解码难以处理：(1) 正常边界被扭曲；(2) 领域间互相干扰；(3) 高误报率。

## 核心问题
如何在一个统一框架中处理多种模态（RGB、深度、点云等）和多个类别的异常检测，同时避免领域干扰和参数爆炸？

## 方法详解

### 关键设计

1. **通用到特定范式**: 编码阶段——多模态输入压缩为紧凑通用特征（特征压缩模块抑制潜在异常、促进跨模态交互、避免快捷学习）；解码阶段——通用特征解压为模态/类别特定形式。

2. **稀疏门控交叉 MoE**: 解码器核心——根据输入的模态和类别动态选择专家路径。每个专家专门处理特定领域的重建，避免了共享解码的领域干扰问题。

3. **效率优化**: 分组动态滤波机制 + MoE-in-MoE 结构，减少 75% 参数使用量，同时维持稀疏激活和快速推理。

## 实验关键数据
- **9 个数据集、3 个领域、12 种模态、66 个类别**上的 SOTA
- 参数减少 75% 的同时维持性能

## 亮点
- **统一框架覆盖面广**：一个模型处理 12 种模态 66 个类别
- **MoE 解决领域干扰**：稀疏专家选择使不同领域不互相干扰
- **高效设计**：MoE-in-MoE + 分组动态滤波大幅降低参数

## 局限性 / 可改进方向
- 仅基于摘要分析
- MoE 路由的稳定性在极多类别时可能成为问题
