# Beyond Static Artifacts: A Forensic Benchmark for Video Deepfake Reasoning in Vision Language Models

**会议**: CVPR2026
**arXiv**: [2602.21779](https://arxiv.org/abs/2602.21779)
**代码**: 待确认
**领域**: 多模态VLM
**关键词**: 深度伪造检测, 视频取证, VLM推理, 时序不一致, 多选题基准, 指令微调

## 一句话总结
提出 FAQ（Forensic Answer-Questioning），首个关注深度伪造视频中时序不一致性的多选问答基准，通过三层级任务体系（感知→定位→推理）逐步增强 VLM 的取证能力，微调后在域内和跨数据集检测中均取得显著提升。

## 研究背景与动机
1. 现有 VLM 深度伪造检测方法擅长识别空间伪影（如纹理、边缘），但忽略了时序不一致性这一关键维度
2. 传统深度伪造检测方法中，时序不一致性已被广泛验证为重要检测信号
3. 已有 QA 数据集（如 DD-VQA、VLFFD）仅从静态帧提取标注，无法引导 VLM 发现动态线索
4. 如何通过 QA 数据构建有效引导 VLM 发现和推理时序不一致性的训练范式，是一个开放问题
5. FakeForensics++ 等传统数据集只做二分类，缺乏可解释性和推理能力评估
6. 需要一个从感知到推理的渐进式基准，系统性地评估和提升 VLM 的取证能力

## 方法详解

### 整体框架
FAQ 基准包含约 33K QA 对、覆盖约 4,500 个被操纵视频，采用多选题格式。构建流程：视频收集 → 人工标注 → 时空聚类 → 关键点提取 → 描述解析 → QA 生成。

### 三层任务体系

**Level 1: 面部感知**（基础视觉感知）
- 区域感知：判断特定面部区域的清晰/模糊质量
- 边缘感知：区分面部边界的锐利与模糊程度

**Level 2: 时序深度伪造定位**（动态伪影定位）
- 类型理解：给定时间窗口和面部区域，确定伪影类型
- 区域定位：给定时间段和伪影类型，确定涉及的面部区域
- 时序定位：给定伪影区域和类型，定位发生的时间段

**Level 3: 取证推理**（综合推理）
- 伪造分析：顺序识别伪造类型 → 定位区域 → 确定时间段
- 最终评估：综合所有取证证据做出真伪判断

### 关键设计

**时空聚类**：用邻接函数 $f(c_i,c_j) = (\|c_i-c_j\|_2 \leq \tau_s) \land (\|c_i-c_j\|_1 \leq \tau_t)$ 将稀疏人工点击聚合为 14,392 个伪造视频片段（平均 2.1 秒）

**关键点提取**：用 dlib 提取面部关键点轨迹，通过空间相关性函数 $S_n = \sum_{k=1}^K d(\mathcal{P}_n^{(k)}, \bar{c})$ 确定与伪造最相关的面部部位

**干扰项设计**：LLM 在人工指导下生成四选一选项，干扰项经过精心设计使其视觉和时序上可信，迫使模型依赖动态视觉语义而非语言先验

### 关键发现
- 三层级训练的渐进设计至关重要，每一层级都为下一层提供能力基础
- 将时序不一致性转化为 QA 对是可行的 VLM 训练范式
- FAQ-IT 微调后模型在广泛使用的检测基准上均有性能增益

## 亮点与洞察
- 首个系统性关注时序维度的深度伪造 VLM 基准，填补了重要空白
- 三层级渐进设计符合认知学习规律：感知 → 定位 → 推理
- 从人工标注的稀疏点击到结构化 QA 的自动化流水线设计精巧

## 局限性
- 仅基于 FaceForensics++ 数据集，伪造方法有限
- 人工标注成本高，大规模扩展困难
- 多选题格式可能限制了开放式推理能力的评估

## 相关工作与启发
- 与 Forensics-Bench 的区别：FAQ 显式关注时序线索并提供伪造轨迹标注
- FakeShield/SIDA 等方法仅处理静态帧，FAQ 指出需要动态分析
- 启发：三层级任务设计可迁移到其他需要渐进式推理的视频理解任务
