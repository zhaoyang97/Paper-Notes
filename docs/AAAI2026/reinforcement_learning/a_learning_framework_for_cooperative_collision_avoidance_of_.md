# A Learning Framework For Cooperative Collision Avoidance of UAV Swarms Leveraging Domain Knowledge

**会议**: AAAI 2026 (Under review)  
**arXiv**: [2507.10913](https://arxiv.org/abs/2507.10913)  
**代码**: 无  
**领域**: 多智能体强化学习 / 无人机集群 / 避撞规划  
**关键词**: MARL, UAV swarm, cooperative collision avoidance, domain knowledge, active contour model  

## 一句话总结
提出 reMARL 框架，将图像处理中的主动轮廓模型（active contour）作为领域知识引入多智能体强化学习的奖励设计，使无人机集群仅通过最大化个体奖励即可学会协作避撞，在大规模集群（≤10架）中性能显著优于 COMA/VDN/QMIX/MAPPO 等 SOTA MARL 方法，反应时间比元启发式方法快 98.75%，能耗降低 85.37%。

## 背景与动机
无人机集群（UAV swarm）的协作避撞是核心挑战：既要保证安全（不撞障碍物、不互撞），又要节能（电池有限、变速耗能大）。现有方法分三类：

1. **传统方法**（VO/APF/元启发式）：缺乏长期规划导致能效低，且将别的 UAV 视为环境一部分而非合作者，难以实现真正的协作。APF 还存在局部最优陷阱。
2. **MARL 观测共享方法**（IQL/MADDPG）：观测空间维度随集群规模指数增长，扩展性差。
3. **MARL 信用分配方法**（COMA/VDN/QMIX）：显式方法（COMA）有中心化价值网络的可扩展性瓶颈；隐式方法（VDN/QMIX）基于 IGM 假设，可能导致无界发散和不可预测行为。

核心痛点：**已有 MARL 方法在集群规模增大时性能急剧退化**，因为观测共享的复杂度爆炸和信用分配机制的失效。需要一种新范式，让大规模集群在不依赖复杂网络结构的情况下实现协作。

## 核心问题
如何设计一种 MARL 框架，使无人机集群在**不需要观测共享机制和信用分配机制**的前提下，仅通过精心设计的个体奖励信号就能学到能安全、节能、协作的避撞策略，并且能够扩展到大规模集群（10+ UAV）？

## 方法详解

### 整体框架
reMARL 的核心思路是"**把协作的难题从网络结构转移到奖励设计**"。整体流程：
1. 将环境（障碍物+集群虚拟中心）映射为二维势场（potential field）
2. 利用主动轮廓模型设计个体奖励函数，引导每架 UAV 的轨迹逼近势场上的等值线（contour）
3. 用 PSO 调整各 UAV 的等值线层级，保证 UAV 之间的距离大于安全阈值
4. 每个 agent 独立运行 DDPG（策略网络+价值网络），仅最大化自己的个体奖励

关键洞察：等值线天然不穿过势场的峰值（障碍物位置），也不会交叉，且是光滑连续曲线。因此**让 UAV 沿等值线飞行自动就是安全且节能的**。

### 关键设计

1. **势场构建（Potential Field）**：每个障碍物建模为独立的排斥场（式1），集群整体建模为一个以"虚拟中心"为峰值的排斥场（式2），总势场为所有排斥场的叠加（式3）。排斥场强度与速度成正比，安全距离 $d_{safe}$ 以内为最大强度。集群用虚拟中心而非单个 UAV 来建场，避免 UAV 自身处于峰值点（峰值处无等值线）。

2. **基于主动轮廓模型的奖励设计（核心贡献）**：奖励函数由两部分组成：
   - **Contour 部分**：$-f(S, \Phi)$，其中 $f$ 是主动轮廓代价函数（式4），包含内部能量项（最大化轨迹光滑性 $|S''|^2$）和外部能量项（将轨迹拉向势场梯度最大处 $|\nabla\Phi|^2$，即安全距离边界）。最小化此代价 ≈ 轨迹逼近等值线。
   - **Swarming 部分**：$r_{form} \cdot r_{collide}$，其中 $r_{form}$ 是速度一致性（实际速度与预规划速度的余弦相似度），$r_{collide}$ 是碰撞惩罚（碰撞则为0，否则为1）。这部分确保编队和安全。

3. **PSO 等值线调整**：为保证 UAV 间距大于阈值 $\bar{d}_{U2U}$，用粒子群优化搜索 UAV 位置微调方案，最小化 $f_{PSO} = f_{thres} + f_{shift}$（保证间距约束的同时最小化位移代价）。PSO 仅在训练时使用，不影响推理实时性。

4. **极简观测设计**：每架 UAV 只需观测自身属性（位置、速度）、集群属性（虚拟中心位置、预规划速度）、障碍物属性（位置、速度），构成 $(2+m)\times4$ 数组。所有所需信息已嵌入个体观测中，无需额外的观测共享。

### 损失函数 / 训练策略
- 每个 agent 独立使用 DDPG：策略网络输出速度方向变化量 $a \in [-\pi/4, \pi/4]$（tanh 激活），价值网络估计 $Q(o,a)$
- 网络结构简单：FC-256-ReLU → 输出
- 使用经验回放和目标网络（Double DQN 思路）
- 对比方法仅用 Swarming 部分训练，reMARL 用完整奖励训练但评估时展示 Swarming 部分以公平对比

## 实验关键数据

| 指标 | 元启发式方法 | reMARL | 提升 |
|------|-------------|--------|------|
| 反应时间(s) | 0.48±0.05 | 0.006±0.03 | 98.75% |
| 能耗(曲率积分) | 134.88±8.3 | 19.72±31.8 | 85.37% |
| 最小 U2O 距离 | 38.70 | 24.78 | -35.96%（更近但仍安全） |
| 最小 U2U 距离 | 40.31 | 29.45 | -26.94%（更近但仍安全） |

测试场景涵盖 2U1O 到 10U1O 以及多障碍物（3U2O/5U2O/7U2O）。

### 消融实验要点
- **集群规模 ≤ 3 时**：传统 MARL 方法（COMA/VDN 等）可以与 reMARL 竞争甚至略优，因为小规模无需严格沿等值线飞行，存在更优的简单轨迹
- **集群规模 > 3 时**：reMARL 显著占优，所有 SOTA MARL 方法因维度灾难和信用分配失效而性能退化
- **10U1O 场景**：reMARL 是唯一能稳定训练的方法
- **Contour 项的自适应性**：UAV 能通过强化学习学会在等值线不可行时偏离等值线（如两障碍物之间的间隙），动作价值图证实 UAV "知道"穿越间隙是安全的

## 亮点
- **跨学科创新**：将图像处理的主动轮廓模型跨界用于 MARL 奖励设计，这种"从不相关领域借用数学工具"的 idea 非常巧妙
- **化解 MARL 核心难题的思路**：不在网络结构上做文章（复杂的信用分配和观测共享），而是把协作语义编码进奖励函数，让简单的 DDPG 就能学出协作行为
- **可扩展性**：代理间交互被最小化到只有奖励构建时需要集群信息，与传统 MARL 的 O(n²) 交互形成鲜明对比
- **等值线的物理直觉**：等值线不穿过峰值、不交叉 → 天然保证安全和不互撞

## 局限性 / 可改进方向
- **二维限制**：为节能只在二维平面避撞，未考虑三维空间；实际场景中高度变化可能是更优策略
- **速度恒定假设**：假设避撞期间速度大小不变，这在实际高动态场景中可能过于简化
- **PSO 在训练中的开销**：虽然不影响推理，但训练时每步都要跑 PSO 搜索，可能限制训练效率
- **静态/随机障碍物**：障碍物随机飞向 UAV，未考虑智能对抗性障碍或更复杂的动态环境
- **无真实飞行验证**：实验全在仿真中完成，sim-to-real gap 未讨论
- **对比不够全面**：未与最新的图注意力、Transformer-based MARL 方法对比

## 与相关工作的对比
1. **vs E²Coop / E²CoPre (Huang et al. 2021/2024b)**：同一组前序工作，也用图像处理领域知识让 UAV 沿等值线飞行，但用 PSO 元启发式搜索而非 RL 来优化轨迹。reMARL 通过 RL 训练获得了自适应能力（能在等值线不可行时偏离），且反应时间快两个数量级。
2. **vs COMA/VDN/QMIX**：这些方法通过复杂的信用分配实现协作，但随集群规模增大而退化。reMARL 不需要信用分配，直接在奖励层面编码协作，扩展性远优。
3. **vs MADDPG/IQL**：依赖观测共享，维度灾难严重。reMARL 的观测共享仅用于构建势场（奖励计算），不进入网络优化，避免了维度爆炸。

## 启发与关联
- 核心启发：**奖励工程 vs 网络工程** 的权衡。很多 MARL 论文在网络结构上下功夫（注意力、通信、分解），但本文证明精心设计的奖励函数可以让最简单的 agent 也学出复杂协作行为
- 跨学科方法论可迁移：可以探索其他领域的数学工具（如流体力学势流、电磁场理论）用于多机器人协调的奖励设计
- 与自动驾驶的关联：多车协作规划也面临类似的可扩展性问题，等值线思路是否可以用于交叉口多车协调？

## 评分
- 新颖性: ⭐⭐⭐⭐ 跨学科借用主动轮廓模型做奖励设计很有创意，但核心是在前序工作基础上加 RL
- 实验充分度: ⭐⭐⭐⭐ 多种集群规模和障碍物数量的实验覆盖全面，但缺少真实飞行验证和最新 MARL baseline
- 写作质量: ⭐⭐⭐⭐ 结构清晰、动机有说服力，但数学符号较多，部分推导可以更直观
- 价值: ⭐⭐⭐⭐ 提供了一种"不和网络结构死磕"的 MARL 协作新范式，对大规模多智能体系统有参考价值
