# Beyond Perplexity: Let the Reader Select Retrieval Summaries via Spectrum Projection Score

**会议**: AAAI 2026  
**arXiv**: [2508.05909](https://arxiv.org/abs/2508.05909)  
**代码**: https://zhanghao-aaai2026-sps.github.io/AAAI2026-SPS/  
**领域**: RAG / 信息检索  
**关键词**: 检索增强生成, 困惑度, 谱投影分数, 摘要压缩, 表示空间对齐

## 一句话总结
提出 Spectrum Projection Score (SPS) 这一无需训练的指标，通过衡量摘要 token 嵌入与 reader LLM 主子空间的对齐程度来评估检索摘要质量，替代传统困惑度指标。结合 xCompress 推理时控制器，在 5 个 QA 数据集上显著优于基于困惑度的方法（HotpotQA EM +3.6）。

## 研究背景与动机

1. **领域现状**：RAG 中需要将检索文档压缩/摘要为更紧凑的形式输入 reader LLM。现有方法常用困惑度 (PPL) 评估摘要质量，假设低困惑度=高质量。

2. **现有痛点**：PPL 衡量的是 token 的"典型性"而非语义对齐度——一个摘要可能流畅但与 reader 的表示空间不匹配。实验发现 PPL 与下游 QA 性能的相关性接近零甚至为负。

3. **核心矛盾**：好的摘要不等于 reader 容易生成的文本，而是能让 reader 的内部表示空间充分捕捉信息的文本。

4. **本文要解决什么？** 设计一个与下游 QA 性能高度相关的无训练摘要质量指标，并基于它构建推理时摘要选择/压缩框架。

5. **切入角度**：从 reader LLM 的嵌入矩阵出发，用 SVD 分解得到其"主表示子空间"，衡量摘要在此子空间外的残差。

6. **核心 idea 一句话**：摘要质量 = 其 token 嵌入在 reader 主子空间中的投影残差，残差越小越好。

## 方法详解

### 整体框架
xCompress 是推理时控制器，包含三步：(1) 用随机解码生成 $K$ 个候选摘要；(2) 用 SPS 打分选择最优；(3) 自适应范数过滤——检测摘要是否已足够好（无需额外采样）。

### 关键设计

1. **Spectrum Projection Score (SPS)**:
   - 做什么：衡量摘要与 reader 表示空间的对齐程度
   - 核心思路：对 reader 嵌入矩阵 $W \in \mathbb{R}^{D \times M}$ 做 SVD：$W = U\Sigma V^\top$，保留前 95% 特征值形成投影 $P = U\Sigma_p V^\top$。对摘要提取倒数第二层 token 表示，做逐元素最大池化得"边界向量" $\mathbf{x}$，计算 $\text{SPS}(\mathbf{x}) = \|(I - P)\mathbf{x}\|_2$
   - 设计动机：最大池化捕获 token 分布的凸包边界（"最显著"维度），残差范数度量主子空间外的信息量——越低说明摘要越在 reader 的"舒适区"

2. **自适应范数引导过滤**:
   - 做什么：避免对已足够好的摘要做多余采样
   - 核心思路：计算 $L2_{\text{mean}}/L1_{\text{max}}$作为集中度指标；若比值超过训练集 top-30% 阈值，直接使用初始摘要
   - 设计动机：减少计算开销同时保持准确率

### 损失函数 / 训练策略
无训练方法。SPS 仅依赖 reader 嵌入矩阵的 SVD 分解（一次性计算）和推理时的 token 嵌入提取。

## 实验关键数据

### 主实验

| 模型 + 方法 | HotpotQA EM/F1 | 2WikiMQA EM/F1 | NQ EM/F1 |
|------------|---------------|---------------|---------|
| LLaMA 3.1 8B + CompAct | 34.0/43.2 | 27.2/31.8 | 35.2/47.5 |
| LLaMA 3.1 8B + **xCompress** | **37.6/47.9** | **29.6/34.2** | **39.4/51.2** |
| Gemma 3 12B + CompAct | 19.2/29.7 | 23.8/29.0 | 27.6/40.9 |
| Gemma 3 12B + **xCompress** | **25.2/35.7** | **25.0/29.3** | **31.6/42.3** |

### 指标相关性分析

| 数据集 | PPL-PCC(EM) | SPS-PCC(EM) | PPL-PCC(F1) | SPS-PCC(F1) |
|--------|------------|------------|------------|------------|
| HotpotQA | 0.022 | **0.643** | -0.067 | **0.753** |
| NQ | 0.202 | **0.650** | 0.452 | **0.628** |
| 2Wiki | -0.318 | **0.557** | 0.295 | **0.503** |

### 关键发现
- PPL 与 QA 性能的 PCC 接近零或为负，SPS 的 PCC 达 0.4-0.75
- 最大池化优于均值池化和最后一个 token 的池化（HotpotQA EM 37.6 vs 36.2 vs 33.6）
- 倒数第二层嵌入最佳——末层过度特化于 token 预测
- 5 个候选摘要即可饱和性能

## 亮点与洞察
- **"困惑度对 RAG 无效"的发现本身就有重要意义**：挑战了一个广泛使用的假设，PCC 数据令人信服
- **SVD 主子空间的几何直觉**：把 reader 的"理解能力"形式化为一个线性子空间，摘要质量=在此空间中的"可表示性"

## 局限性 / 可改进方向
- 线性子空间假设可能过于简化——reader 的表示空间可能是非线性流形
- 仅在 QA 任务上验证，对摘要、翻译等其他 RAG 应用的效果未知
- 多候选摘要生成增加了推理时延和计算成本

## 相关工作与启发
- **vs CompAct**: 直接基于 CompAct 摘要做二次选择，无需改变压缩方法本身，即插即用
- **vs LongPPL**: LongPPL 也尝试改进 PPL，但仍基于困惑度框架，与 QA 性能的相关性同样接近零
