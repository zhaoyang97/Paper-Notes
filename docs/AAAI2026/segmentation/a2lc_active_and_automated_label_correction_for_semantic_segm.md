# A²LC: Active and Automated Label Correction for Semantic Segmentation

**会议**: AAAI 2026  
**arXiv**: [2506.11599](https://arxiv.org/abs/2506.11599)  
**代码**: 无（论文声称接收后开源）  
**领域**: 语义分割  
**关键词**: 主动标签校正, 自动标签校正, 伪标签, 类别不平衡, 基础模型  

## 一句话总结

提出 A²LC 框架，在传统主动标签校正（人工逐一纠错）的基础上增加一个自动校正阶段（Label Correction Module），利用标注员的反馈自动修正相似的错误mask，并设计自适应平衡采集函数缓解类别不平衡，在 Cityscapes 上仅用 20% 预算即超越前 SOTA，同等预算下 mIoU 提升 27.23%。

## 背景与动机

语义分割依赖大规模逐像素标注数据集，但人工标注成本极高且容易引入噪声标签。主动标签校正（ALC）通过迭代选择可能错标的样本让人工审核来构建干净数据集。最近 Kim et al. (ICML 2024) 将基础模型（Grounded SAM）引入 ALC pipeline：先用基础模型生成伪标签，再选不确定像素让标注员纠正。

然而现有 pipeline 存在两个核心问题：(1) **冗余查询**——基础模型在细粒度类别（如交通灯 vs 交通牌）上容易产生系统性偏差，导致大量特征相似的mask被反复选中并逐一人工纠正，浪费预算；(2) **类别不平衡**——传统采集函数倾向于采样头部类别，尾部类别长期得不到修正。

## 核心问题

如何最大化人工标注反馈的利用率？具体来说：当标注员纠正了一个"交通灯被错标为交通牌"的mask后，能否自动利用这个信号去修正数据集中所有类似的错误，而不是等标注员一个个纠正？同时，如何在有限预算下兼顾尾部类别的修正？

## 方法详解

### 整体框架

输入：无标注图像集。首先用 Grounded SAM 生成初始伪标签和mask集合。然后迭代 R 轮：每轮用分割模型对所有mask打分（通过采集函数），选 top-B 个mask交给标注员纠正真实标签（人工校正阶段）。接着，用人工校正的mask训练一个 Label Correction Module (LCM)，让 LCM 自动去修正未被选中的那些 mask（自动校正阶段）。两阶段校正完毕后，更新分割模型，进入下一轮。

### 关键设计

1. **Label Correction Module (LCM)**：这是全文最核心的贡献。LCM 是一个轻量级的四层全连接网络（256→128→64→类别数），输入为 mask 的平均特征向量（从主分割模型抽取），输出为类别预测。每轮用标注员纠正好的干净mask作为监督信号来训练 LCM，然后让 LCM 对未查询的mask做推理。为了避免错误传播，设计了三个保守的选择准则：(a) 预测置信度超过阈值 τ（初始0.99并递增），(b) 预测类别不是尾部类，(c) 原伪标签也不是尾部类。三者同时满足才执行自动修正。这样 LCM 可以在不增加人工成本的情况下额外修正大量 mask——例如 Cityscapes 上一轮额外修正了 22,105 个 mask，准确率从 19% 提升到 61%。

2. **Adaptively Balanced Acquisition Function (ABC)**：在基础的 CIL（Confidence in Label）采集函数上乘以一个自适应类别权重 w(x)。这个权重由两部分组成：class rarity score（按伪标签中各类别频率的比值，越少的类别越高）和 dataset imbalance score（用 KL 散度度量数据集分布与均匀分布的差异，不平衡越严重权重放大效果越强）。最终 w(x) = ŵ(x)^{KL³}，在严重不平衡的数据集上大幅提升尾部类别的采样权重，在轻度不平衡时则自动抑制。ABC 与 LCM 形成协同：ABC 引导标注员优先纠正尾部类别的mask，而这些纠正反馈又帮助 LCM 学到尾部类别的修正模式。

3. **Mask-level Correction + Non-redundant Correction**：将校正单元从像素级提升到 mask级（SAM 预测的mask作为单位），避免了像素级校正中单个像素标签传播导致的错误扩散问题。同时排除已查询过的 mask，避免冗余查询，随着轮次推进未查询mask越来越少，效率持续提升。

### 损失函数 / 训练策略

- LCM 训练使用加权交叉熵损失 ℒ_LCM = λ_{y(m)} · CE(y(m), ψ_r(m))，其中类别权重 λ_k = (N/N_k) / Σ(N/N_c) 处理类别不平衡
- 主分割模型为 DeepLab-v3+ + ResNet-101（ImageNet 预训练），每轮训练 30K 迭代，SGD + polynomial LR decay
- 自动校正的置信度阈值 τ 从 0.99 开始，逐轮递增，体现"越到后面越谨慎"的策略
- 尾部类别定义：α=0.5，即按频率排序后的后 50% 类别

## 实验关键数据

| 数据集 | 指标 | 本文 (A²LC) | 之前SOTA (ALC) | 提升 |
|--------|------|------|----------|------|
| Cityscapes (100k, 1轮) | Model mIoU | 72.37% | 70.71% | +1.66 |
| Cityscapes (10k×5轮) | Data mIoU | 85.26% | 67.01% | +18.25 |
| Cityscapes (10k×5轮) | Model mIoU | 70.51% | 58.59% | +11.92 |
| PASCAL (1k×5轮) | Data mIoU | 88.08% | 77.06% | +11.02 |
| PASCAL (1k×5轮) | Model mIoU | 68.42% | 65.48% | +2.94 |
| Cityscapes (50k, 采集函数对比) | Data mIoU | 84.92% (ABC) | 78.98% (SIM) | +5.94 |

效率亮点：Cityscapes 上仅用 ALC 预算的 20% 即超越其最终性能；PASCAL 上仅用 60%。

### 消融实验要点

- **LCM 贡献最大**：加入LCM后 5轮 Data mIoU 从 66.86% 提升到 80.59%（+13.73），是最核心的组件
- **ABC 也有独立贡献**：单独加 ABC 后 Data mIoU 从 66.86% 提升到 81.58%（+14.72）
- **LCM + ABC 协同效应显著**：两者组合后 84.15%，超过各自独立贡献之和，说明 ABC 引导的尾部类别采样为 LCM 提供了更好的训练数据
- **Mask-level correction** 从像素级提升到 mask级也有稳定增益
- LCM 在 Cityscapes 上一次自动修正了 22,105 个mask，准确率从 19.13% 提升到 60.99%；PASCAL 上修正 1,629 个 mask，准确率从 4.85% 到 82.26%

## 亮点

- **"一次标注，多次修正"的核心思路**：这是全文最巧妙的地方——将标注员的逐个纠错转化为可泛化的修正能力，LCM 用干净标注训练后自动修正特征相似的未查询 mask，极大提升了标注效率
- **ABC 权重的自适应设计**：用 KL 散度的三次方作为调节因子，数据集越不平衡越激进地偏向尾部类别，均衡数据集则自动退化为标准采集函数，无需手动调参
- **三条保守选择准则**：LCM 的自动修正只在高置信度 + 非尾部类才执行，有效避免了错误传播，实践中非常实用的设计
- **20% 预算超越100% baseline**：效率提升非常显著，说明自动修正真的在省钱

## 局限性 / 可改进方向

- **仍然依赖人工参与**：自动校正只是补充，不能完全替代人工，pipeline 的每一轮仍需标注员介入
- **LCM 模型过于简单**：四层FC网络，且只用了平均特征，丢失了空间信息，对形状/位置敏感的错误可能效果有限
- **数据集规模有限**：只在 Cityscapes (2,975张) 和 PASCAL VOC (1,464张) 上验证，未在大规模数据集（如ADE20K）或更复杂场景上测试
- **伪标签天花板效应**：初始伪标签质量受限于 Grounded SAM 的能力，后期精度提升趋于饱和；论文自身也承认10轮后饱和
- **类别间混淆模式的利用不够系统**：LCM 隐式学习了类别混淆模式，但没有显式建模类别间的混淆矩阵，可能有进一步提升空间
- **超参数设置**：τ从0.99递增、α=0.5、KL散度用三次方等都是经验设定，鲁棒性未充分验证

## 与相关工作的对比

- **vs ALC (Kim et al., ICML 2024)**：ALC 是直接前驱，同样用 Grounded SAM 生成伪标签+主动选择像素纠正。A²LC 的核心差异是加了自动校正阶段（LCM）和自适应平衡采集函数（ABC），从"只人工修"变成"人工修+自动扩展修"。实验上 A²LC 全面碾压 ALC
- **vs 超像素主动学习方法 (Spx/MerSpx/MulSpx)**：这些方法关注"选什么区域标注"，但没有利用标注反馈做自动校正，也没有显式处理标签噪声。A²LC 在同等 budget 下 mIoU 高出 5.77~8.60
- **vs 主动学习 (一般)**：传统主动学习侧重"选最有价值的未标注样本"，而A²LC侧重"修正已有的错误伪标签"，属于不同的问题设定

## 启发与关联

- **LCM 的"标注反馈泛化"思路可迁移到其他场景**：比如医学影像分割中，专家标注一个器官边界的修正后，能否自动修正同一CT序列中相似slice的标注？这个思路非常有启发性
- **自适应权重设计 (KL散度调节) 值得借鉴**：在任何涉及类别不平衡的采样/加权策略中，用分布差异自动调节权重强度是一个优雅的设计
- 该工作暗示了一个更广泛的研究方向：在 foundation model 生成伪标签后，如何用最少的人工干预把伪标签质量提升到接近真实标注——这对任何依赖伪标签的半监督/弱监督方法都有价值
- **相关 idea**: [混淆矩阵引导的层级式自动标签校正](../../../ideas/segmentation/20260317_confusion_guided_label_correction.md) — 在 LCM 基础上引入显式混淆矩阵建模，用混淆簇替代单一 LCM，并设计混淆覆盖率采样策略
