# Do Not Merge My Model! Safeguarding Open-Source LLMs Against Unauthorized Model Merging

**会议**: AAAI 2026  
**arXiv**: [2511.10712](https://arxiv.org/abs/2511.10712)  
**代码**: 无  
**领域**: LLM安全 / 模型知识产权保护  
**关键词**: 模型合并窃取, MergeBarrier, 正交投影, 权重重参数化, 线性模态连通性

## 一句话总结

提出MergeBarrier，一种即插即用的防御方法，通过对注意力层施加正交投影、对FFN层进行激活函数展开重参数化，破坏受保护模型与同源模型之间的线性模态连通性（LMC），从而在不损失模型性能的前提下主动阻止未授权的模型合并。

## 研究背景与动机

1. **领域现状**: 模型合并（Model Merging）作为一种高效扩展LLM能力的技术日益流行，通过整合多个专家模型即可获得综合能力，无需额外数据收集或高性能GPU。

2. **现有痛点**: 模型合并引入了新的知识产权威胁——"模型合并窃取"。搭便车者可以未授权地将受限开源模型合并到自己的模型中用于商业目的，且现有水印/指纹等被动方法在合并后可能失效，难以追溯。

3. **核心矛盾**: 现有方案无法同时满足三个关键属性：(1) 主动预防未授权合并（而非事后检测）；(2) 兼容通用开源环境（无需额外授权组件）；(3) 高安全性且性能损失可忽略。被动方法不能预防，授权方法不兼容开源，模拟器方法牺牲安全或效用。

4. **本文要解决什么**: 首次正式定义模型合并窃取防御问题，设计满足上述三个属性的主动防御方案。

5. **切入角度**: 从模型合并的底层机制（线性模态连通性LMC）入手——同源模型因共享低损失盆地而可被平滑插值合并，如果破坏这个连通性，合并后的模型将性能大幅下降。

6. **核心idea**: 通过等价权重变换将受保护模型移出低损失盆地，使其与同源模型之间的线性路径穿过高损失区域，从而使合并结果无法保持性能；同时由于变换的数学等价性，单独使用时模型功能不受影响。

## 方法详解

### 整体框架

MergeBarrier分别处理Transformer的注意力层和FFN层：对注意力层应用正交投影变换，对FFN层进行激活函数展开式重参数化。两种变换都保证模型推理时功能等价，但合并时权重偏移导致性能崩溃。

### 关键设计

1. **注意力权重投影 (Attention Weight Projection)**:
   - 核心思路：利用注意力机制中Q和K的配对线性结构，对两者施加共享正交投影矩阵P
   - 数学保证：由于$PP^\top = I$，投影在注意力计算中自行抵消，输出不变
   - 优化目标：最大化合并模型与原始低损失区域的Frobenius范数距离
   - 对MHA/GQA的扩展：通过分块对角正交矩阵适配多头/分组查询注意力
   - 加速：使用随机SVD（RSVD）将特征分解复杂度从$O(n^3)$降至$O(n^2k)$

2. **FFN权重重参数化 (FFN Weight Reparameterization)**:
   - 核心思路：FFN缺乏对称结构，改用Taylor展开近似激活函数，将原始权重替换为多项式系数
   - 设计动机：新参数与原始权重处于不同空间，攻击者无法在原始权重空间进行插值合并
   - 展开点选择：取训练集特征表示的中点，最小化近似误差
   - 安全保证：Taylor余项引入的噪声使权重反演退化为NP困难的LWE（Learning With Errors）问题
   - 额外收益：余项噪声满足差分隐私（DP），可防御成员推理攻击

3. **正交矩阵优化**:
   - 理想情况$P=-I$可最大化偏移但暴露真实权重
   - 松弛策略：对最大特征值方向翻转（-1），其余保持（+1），在安全性和隐蔽性间取平衡

### 损失函数 / 训练策略

- 优化目标：最大化合并后权重与原始低损失区域的Frobenius范数距离
- 等价于最大化 $\|W_q(P-I)\|_F^2 + \|W_k(P-I)\|_F^2$
- FFN部分通过Taylor展开在选定点附近近似，展开阶数N控制精度-安全权衡
- 整体为后处理方法，不需要额外训练

## 实验关键数据

### 主实验

基于LLaMA-2-13B，保护三个专家模型（WizardLM, WizardMath, Code Alpaca），测试Task Arithmetic、TIES、DARE三种合并方法：

| 合并方法 | 无保护性能 | MergeBarrier | PaRaMS | TaylorMLP |
|---------|-----------|-------------|--------|-----------|
| Task Arithmetic | 正常 | 合并后性能显著下降 | 部分有效 | 仅保护单层，安全性弱 |
| TIES | 正常 | 合并后性能显著下降 | 可被解码逆转 | 可被逆转 |
| DARE | 正常 | 合并后性能显著下降 | 可被解码逆转 | 仅单层保护 |

MergeBarrier保护的模型单独使用时准确率损失可忽略。

### 消融实验

- 正交投影和FFN重参数化分别有效，组合效果最优
- 翻转特征方向的数量k影响安全-隐蔽权衡
- RSVD加速在保持效果的同时大幅降低计算成本

### 关键发现

- MergeBarrier是唯一同时满足主动性、兼容性、安全性与效用性的方案
- PaRaMS的置换保护可通过权重对齐逆转，缩放保护可通过基模型参数还原
- TaylorMLP仅保护每个Transformer块中的单个线性层，其余权重暴露
- Taylor余项的NP困难性质为FFN重参数化提供了理论安全保证

## 亮点与洞察

- 从LMC机制出发设计防御，理论动机清晰优美
- 正交投影的自抵消性质保证无损性能，数学上完美
- Taylor余项→LWE问题的安全分析，将密码学难度引入模型保护，开创性思路
- 即插即用设计，无需外部授权组件，完全兼容开源发布场景

## 局限性 / 可改进方向

- 仅在LLaMA-2-13B上验证，缺乏更大规模模型和不同架构的实验
- 自适应攻击者（知道MergeBarrier机制后的逆向尝试）的分析不够充分
- FFN重参数化修改了模型结构，可能带来推理延迟（虽然声称可并行化）
- 展开点依赖训练集分布，分布外输入可能增大近似误差

## 相关工作与启发

- PaRaMS 通过MLP置换和注意力缩放防御，但两种保护都可被逆转
- TaylorMLP 用Taylor展开保护单层权重，安全性不足
- 本文将LWE密码学难度引入深度学习模型保护，这一思路可推广到其他模型安全场景
- LMC概念对理解模型合并机制有重要价值

## 评分

⭐⭐⭐⭐ (4/5)

理论动机清晰，方法设计精巧，数学推导严谨。正交投影的无损特性和LWE安全保证是亮点。但实验规模偏小（仅13B），自适应攻击分析可以更充分。对开源模型知识产权保护有重要实际价值。
