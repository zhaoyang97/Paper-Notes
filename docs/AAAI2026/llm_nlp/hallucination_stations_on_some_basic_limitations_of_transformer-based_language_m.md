# Hallucination Stations: On Some Basic Limitations of Transformer-Based Language Models

**会议**: AAAI 2026  
**arXiv**: [2507.07505](https://arxiv.org/abs/2507.07505)  
**代码**: 无  
**领域**: NLP理解 / LLM理论  
**关键词**: 幻觉, 计算复杂性, Transformer局限, LLM agent, 可验证性

## 一句话总结
从计算复杂性角度分析LLM幻觉和能力局限，论证超过特定计算复杂度后LLM不仅无法正确执行任务，甚至无法验证其输出的正确性，为幻觉问题划定理论边界。

## 研究背景与动机
1. **领域现状**：LLM幻觉是广受关注的问题，但多数研究从数据/训练角度分析，缺乏从计算复杂性理论的根本性视角。
2. **现有痛点**：(a) RAG、事实核查等方案是工程补丁而非根本解答；(b) 不清楚幻觉的理论边界——哪些任务在理论上就不可能被LLM正确完成？
3. **核心矛盾**：Transformer的计算能力存在根本性上限（对应特定复杂度类），但业界倾向于用更大模型"暴力解决"。
4. **本文要解决什么？** 从复杂性理论为LLM的能力和幻觉划定理论边界。
5. **切入角度**：计算复杂性分析——Transformer的表达能力对应特定复杂度类，超出此类的任务理论上不可能正确完成。
6. **核心idea一句话**：LLM幻觉的根源之一是计算复杂性的基本局限——超Transformer复杂度类的任务连验证正确性都做不到。

## 方法详解

### 整体框架
纯理论分析工作。从Transformer的计算理论模型出发，分析其对应的复杂度类，推导出不可执行和不可验证的任务边界。

### 关键设计

1. **Transformer复杂度类分析**：
   - 做什么：刻画Transformer能表达的计算类
   - 核心思路：分析attention和前馈网络的组合计算能力与TC0/AC0等复杂度类的关系
   - 设计动机：建立严格的理论上限

2. **不可验证性论证**：
   - 做什么：证明超过阈值的任务LLM连验证都做不到
   - 核心思路：如果验证器也是LLM，则验证本身受同样的复杂度限制
   - 设计动机：意味着self-consistency等自验证策略也有理论局限

3. **Agent能力限制推导**：
   - 做什么：分析多步推理中复杂度的累积效应
   - 核心思路：agent的多步操作可能使得总体复杂度超过单步阈值
   - 设计动机：agent场景下幻觉问题更为严峻

### 损失函数 / 训练策略
纯理论工作，无训练过程。

## 实验关键数据

### 理论结果

| 命题 | 说明 |
|------|------|
| LLM计算能力上界 | 对应特定复杂度类（与TC0相关） |
| 幻觉不可消除性 | 超阈值任务必然产生幻觉 |
| 验证不可能性 | LLM无法自我验证超阈值任务的正确性 |
| Agent累积复杂度 | 多步操作加剧问题 |

### 关键发现
- 幻觉不仅是数据/训练问题，还有**计算复杂性的根本边界**
- "让LLM检查自己的输出"在某些任务上理论不可行
- LLM agent面临更严峻限制——多步推理中复杂度累积

## 亮点与洞察
- **将幻觉问题提升到理论层面**：在实用主义主导的LLM研究中，这种基础理论分析提供了清醒视角。不是所有问题都能靠scaling解决。
- **不可验证性的重要发现**：意味着self-consistency、self-critique等策略在某些任务上有理论天花板。
- **对agent的额外警示**：agent的多步推理使得复杂度累积，幻觉风险更大。
