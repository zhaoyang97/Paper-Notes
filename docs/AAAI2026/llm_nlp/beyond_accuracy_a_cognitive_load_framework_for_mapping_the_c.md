# Beyond Accuracy: A Cognitive Load Framework for Mapping the Capability Boundaries of Tool-use Agents

**会议**: AAAI 2026  
**arXiv**: [2601.20412](https://arxiv.org/abs/2601.20412)  
**代码**: 无  
**领域**: LLM Agent / 工具使用评估  
**关键词**: 认知负荷理论, 工具使用Agent, 能力边界, Tool Interaction Graph, 基准测试  

## 一句话总结
借鉴心理学的认知负荷理论（CLT），将工具使用任务的复杂度分解为内在负荷（任务解题路径的结构复杂度）和外在负荷（问题表述的歧义性），构建可参数化调节认知负荷的 ToolLoad-Bench 基准，用指数衰减模型 $\text{Acc} \approx e^{-(k \cdot CL + b)}$ 精确刻画不同 Agent 的能力边界。

## 研究背景与动机
1. **领域现状**：工具使用是 LLM Agent 的核心能力，Berkeley Function Calling Leaderboard 等评测集广泛使用。现有基准提供最终准确率排名，但不分析任务何时、为何失败。
2. **现有痛点**：当前评测是"黑盒"——只知道模型能做什么，不知道它的认知瓶颈在哪里。缺少对任务难度的细粒度分解，无法诊断具体的失败模式（是结构太复杂，还是问题太模糊？）。
3. **核心矛盾**：评测的终极目标不仅是排名，更应理解每个模型的"能力边界"——在什么复杂度下开始显著退化。但"复杂度"本身是多维度的，单一指标无法捕捉。
4. **本文要解决什么？** 提供一个原理性框架，将工具使用任务的复杂度量化为可度量的组分，并用统计验证的模型刻画每个 Agent 的能力极限。
5. **切入角度**：将心理学中人类工作记忆容量有限的认知负荷理论映射到 LLM 的推理上下文——把模型的计算上下文和推理能力类比为人类工作记忆，用认知负荷量化任务对"推理容量"的需求。
6. **核心idea一句话**：用认知负荷理论的内在/外在负荷分解，量化工具使用任务的复杂度，并通过指数衰减模型精确映射每个 Agent 的能力边界。

## 方法详解

### 整体框架
输入：多轮工具使用任务 $(Q, T)$（用户查询序列 + 可用工具集）。方法：(1) 用 Tool Interaction Graph (TIG) 形式化解题路径；(2) 从 TIG 提取内在认知负荷 $CL_I$（结构复杂度）；(3) 从任务表述提取外在认知负荷 $CL_E$（歧义度）；(4) 总负荷 $CL_{\text{Total}} = CL_I + \omega_E \cdot CL_E$；(5) 拟合 $\text{Acc} \approx e^{-(k \cdot CL_{\text{Total}} + b)}$ 得到模型认知画像 $(k, b)$。

### 关键设计

1. **Tool Interaction Graph (TIG)**:
   - 做什么：将任务的 ground-truth 解题过程建模为有向无环图（DAG），节点是用户查询和函数调用，边是数据依赖或执行依赖。
   - 核心思路：每条依赖边 $e = (v_i, v_j)$ 的认知负荷由两个因子决定：(1) **记忆负荷（注意距离）** $\delta(v_i, v_j)$——两个操作之间的对话轮数，距离越远越难记忆；(2) **选择负荷（干扰）** $I(v_i, v_j)$——上下文中同类型但错误实体的数量（如多个用户ID中选正确的）。边权 $w(e) = \delta(v_i, v_j) \cdot (1 + \lambda \cdot I(v_i, v_j))$。
   - 设计动机：将抽象的"任务复杂度"操作化为图结构指标，使得复杂度可计算、可调节（增加边 = 增加内在负荷）。

2. **认知负荷分解**:
   - 做什么：将总负荷分为内在负荷 $CL_I$（解题路径固有的）和外在负荷 $CL_E$（任务呈现方式导致的）。
   - 核心思路：$CL_I = \sum_{v_f} \sum_{e \to v_f} w(e)$，是所有函数调用节点的所有依赖边权之和。$CL_E = \sum_{q_i} CL_E(q_i, T)$，由 Gemini-2.5-pro 对每个查询评估歧义度和干扰工具的分心度（各归一化到 [0,1]）。总负荷用经验校准的加权和 $CL_{\text{Total}} = CL_I + \omega_E \cdot CL_E$，其中 $\omega_E$ 由两种负荷对准确率的实际影响比值确定。
   - 设计动机：不同模型对两种负荷的敏感度不同（如有些模型擅长处理结构复杂但语义清晰的任务），分解可以诊断具体弱点。

3. **指数衰减预测模型**:
   - 做什么：用 $\text{Acc}(Q,T,G) \approx \exp(-(k \cdot CL_{\text{Total}} + b))$ 建模性能随认知负荷的衰减。
   - 核心思路：基于两个公设——(1) 单个操作的成功概率是其认知负荷的指数函数；(2) 全任务成功概率是各操作成功概率之积（独立性假设）。由此推导出总认知负荷的可加性。参数 $b$ 反映基线能力（对应零负荷时的准确率 $e^{-b}$），$k$ 反映负荷敏感度（越小越鲁棒）。
   - 设计动机：指数衰减是信息论和心理学中的经典模型，且可以用 Hosmer-Lemeshow 检验严格验证拟合质量。

### 损失函数 / 训练策略
本文是评测框架，无需训练。ToolLoad-Bench 通过图生成和边插入策略从 BFCL v3 的 200 个实例扩展到 500 个，覆盖 10 个工具类别、106 个独立工具。

### 主实验
ToolLoad-Bench 上的总体准确率：

| 模型 | 准确率(%) |
|------|----------|
| xLAM2-32B (微调) | **78.8** |
| GPT-4o | 68.0 |
| Claude 3.7 Sonnet | 64.8 |
| GPT-4o-mini | 62.2 |
| Gemini 2.5 Pro | 60.0 |
| Qwen3-235B | 58.0 |
| Qwen3-32B | 55.2 |
| Qwen3-8B | 38.6 |
| Llama3.3-70B | 17.0 |

认知画像参数：

| 模型 | $k$（负荷敏感度）↓ | $b$（基线负荷）↓ |
|------|---------------------|-------------------|
| xLAM2-32B | **0.034** | 1.22 |
| GPT-4o | 0.067 | 1.71 |

| Qwen3-8B | 0.085 | 1.12 |

### 消融实验
Hosmer-Lemeshow 拟合优度检验（p-value > 0.05 表示模型与现实无统计差异）：

| 模型 | H-L 统计量 | p-value | 说明 |
|------|-----------|---------|------|
| xLAM2-32B | 3.59 | **0.89** | 最佳拟合 |
| GPT-4o | 4.87 | 0.77 | 良好拟合 |
| Qwen3-235B | 5.19 | 0.74 | 良好拟合 |
| Llama3.3-70B | 13.21 | 0.10 | 仍通过检验 |

### 关键发现
- xLAM2-32B 以 32B 的规模达到最高准确率（78.8%）且最低负荷敏感度（$k=0.034$），证明针对性微调是提升工具使用能力的高效策略，比简单扩大模型规模更有效。
- 所有模型都展现"性能悬崖"：在低认知负荷下表现良好，超过特定阈值后急剧下降。Llama3.3-70B 即使在低负荷下也仅 23%，暴露了基本能力缺陷。
- 外在负荷（问题歧义性）和内在负荷（结构复杂度）对性能的影响模式高度相似，证实了问题表述的清晰度与结构复杂度同等重要。
- 框架的预测能力经过 Hosmer-Lemeshow 检验统计验证：所有模型的 p-value 均远高于 0.05，指数衰减模型与实际观测高度一致。

## 亮点与洞察
- 用心理学的认知负荷理论来量化 AI Agent 的"认知极限"这一跨学科类比非常精彩。将模型的上下文窗口和推理能力类比为人类工作记忆，既直观又有理论支撑。
- 两参数认知画像 $(k, b)$ 比单一准确率数字信息量大得多——同一个准确率可能对应不同的 $(k, b)$（如高基线+高敏感度 vs 低基线+低敏感度），这对模型选择和任务路由有直接应用价值。
- ToolLoad-Bench 的参数化设计（可调节认知负荷）是评测方法论的贡献——传统基准的难度是固定的，这个可以持续挑战更强的模型。

## 局限性 / 可改进方向
- 外在负荷 $CL_E$ 的度量依赖 Gemini-2.5-pro 的主观评估，不够客观可重复，作者也承认这一点。
- ToolLoad-Bench 仅 500 个实例，领域覆盖有限（基于 BFCL v3 的 8 个工具类别扩展到 10 个）。
- 独立性假设（每个函数调用的成功概率独立）在实际中不完全成立——前一步的错误会影响后续步骤。
- 没有提供代码，可复现性存疑。

## 相关工作与启发
- **vs BFCL**: BFCL 提供准确率排名但不分析复杂度。本文的认知负荷框架可以作为 BFCL 的诊断增强层——对同一基准的结果做更深入的分析。
- **vs ToolLLM/API-Bank**: 这些是更大规模的工具使用基准，但同样缺少对任务复杂度的系统化分析。认知负荷框架可以迁移到这些基准上。
- **启发**：认知画像的思路可以推广到其他 Agent 能力的评测——如多轮对话的对话负荷、代码生成的代码复杂度、甚至多模态推理的感知负荷。
