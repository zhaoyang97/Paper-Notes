# Where Norms and References Collide: Evaluating LLMs on Normative Reasoning

**会议**: AAAI 2026  
**arXiv**: [2602.02975](https://arxiv.org/abs/2602.02975)  
**代码**: https://github.com/TheSittingCat/SNIC  
**领域**: LLM推理 / 具身AI  
**关键词**: 社会规范推理, 参考消解, 具身Agent, 诊断测试台, 隐式知识

## 一句话总结
提出 SNIC 诊断测试台（9,000 实例/51 场景），评估 LLM 能否利用隐式社会规范来解决歧义参考消解（如"递给我杯子"时存在多个杯子）。发现 LLM 在仅看场景描述时平均准确率仅 44%，加上 Prolog 形式逻辑无显著改善（44.2%），但显式提供规范列表后猛升到 70.5%（GPT-4.1 达 99.6%），证明 LLM 缺乏隐式物理规范知识但能有效利用显式规范。

## 研究背景与动机

1. **领域现状**：具身 AI（如服务机器人）在物理世界中需要基于社会规范理解模糊指令。如"递给我杯子"——当存在一个干净杯和两个脏杯时，社会规范暗示"应优先选干净的"，无需明说。

2. **现有痛点**：
   - LLM 能否像人类一样推断这类隐式规范完全未知
   - 现有 NLU 研究聚焦语言消歧而非社会规范消歧
   - 缺乏系统化的基准来评估 LLM 的规范推理能力

3. **核心矛盾**：物理世界中的参考消解往往依赖于未显式表达的社会规范（整洁性>便利性、安全性>速度等），而 LLM 是否编码了这些物理接地的社会规范？

4. **本文要解决什么？** 设计诊断测试台评估 LLM 的隐式规范推理能力。

5. **切入角度**：51 个人工验证的规范场景 → 规则化增广到 9,000 实例（系统变化物体类型/属性保持规范结构）→ 三种条件评估（描述/+Prolog/+显式规范）。

6. **核心 idea 一句话**：LLM 不懂隐式社会规范，但能用好显式规范——具身 AI 需要外部规范知识库。

## 方法详解

### 整体框架
SNIC 基准：51 个种子场景（人工创建+5人标注验证，κ=0.22-0.66）→ 规则化程序增广（系统变化物体类型/属性，保持规范结构）→ 9,000 实例。三种评估条件：(1) 仅场景描述；(2) +Prolog 一阶逻辑；(3) +显式规范列表。11 个模型（Granite/Phi/Llama/Qwen/GPT-4o）。

### 关键设计

1. **规范场景设计**:
   - 做什么：构建需要社会规范推理的参考消解场景
   - 核心思路：每个场景包含一个请求（"递给我 X"）和多个候选物体，正确选择取决于隐式规范（如"公共厨房选干净的"、"给老人选轻的"）
   - 设计动机：选择物理接地的（非文化/社会的）规范——这些是相对普遍且对具身 AI 最关键的

2. **程序增广**:
   - 做什么：从 51 个种子扩展到 9,000 个覆盖多种物体/属性组合的实例
   - 核心思路：系统替换物体类型（杯子→碗→盘子）和属性（干净/脏→新/旧），保持规范结构不变
   - 种子质量是关键——增广不增加新规范，只增加同一规范的实例化

3. **三条件对照实验**:
   - 做什么：区分 LLM 是缺乏规范知识还是缺乏规范应用能力
   - 条件 1（仅描述）：测试隐式知识
   - 条件 2（+Prolog）：测试形式化推理能否帮助
   - 条件 3（+显式规范）：测试规范应用能力

### 损失函数 / 训练策略
- 评估型——零样本，无训练

## 实验关键数据

### 主实验

| 条件 | 平均准确率 | GPT-4.1 | Phi-3 14B |
|------|----------|---------|-----------|
| 仅描述 | 44.08% | 56.98% | ~45% |
| + Prolog | 44.22% | 59.62% | ~45% |
| **+ 显式规范** | **70.51%** | **99.6%** | **77.12%** |

### 消融

| 发现 | 具体数据 |
|------|---------|
| Prolog 几乎无帮助 | 44.08% → 44.22%（+0.14%） |
| 显式规范大幅提升 | 44.08% → 70.51%（+26.4%） |
| GPT-4.1 有规范时近乎完美 | 99.6% |
| 规范冲突处理 | 整洁性偏好 > 安全性偏好（偏向特定规范优先级） |
| 一致性差 | 相同规范下微小条件变化导致不同预测 |

### 关键发现
- **LLM 根本缺乏隐式社会规范知识**：仅场景描述时平均 44%，接近随机
- **Prolog 形式逻辑无用**：仅 +0.14%——模型不理解形式化规范规格
- **显式规范大幅提升**（+26%）：LLM 能「应用」规范但不能「推断」规范
- **规范优先级偏见**：模型偏好整洁性规范而忽视安全性规范——缺乏规范层次的深层理解
- **一致性问题**：相同规范在略有不同的条件下给出不同预测

## 亮点与洞察
- **"缺乏隐式知识但善于使用显式知识"**的结论对具身 AI 设计有直接指导：机器人需要外部规范数据库
- **Prolog 无用**的负面结果有价值——说明形式化规格不是解决办法
- 规范冲突分析揭示了 LLM 的浅层规范处理

## 局限性 / 可改进方向
- 纯文本评估，无视觉/多模态输入（真实具身场景需要）
- 仅测试现成模型，未探索规范微调或 RLHF
- 人类标注一致性有限（κ ≤ 0.66）
- 增广的 9,000 实例未独立验证

## 相关工作与启发
- **vs SocialBias (Howard et al.)**：社会偏见探测。本文探测的是物理规范，不同维度
- **vs Theory of Mind 基准**：ToM 测试心理归因。本文测试社会规范推理
- 对具身 AI 的规范注入策略有指导
