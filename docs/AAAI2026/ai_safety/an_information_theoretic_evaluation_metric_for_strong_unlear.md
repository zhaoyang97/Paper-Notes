# An Information Theoretic Evaluation Metric for Strong Unlearning

**会议**: AAAI 2026  
**arXiv**: [2405.17878v3](https://arxiv.org/abs/2405.17878v3)  
**代码**: 无  
**领域**: AI安全 / 机器遗忘 / 隐私保护  
**关键词**: machine unlearning, mutual information, evaluation metric, information bottleneck, white-box metric  

## 一句话总结
提出 Information Difference Index (IDI)，一种基于信息论的白盒评估指标，通过度量中间层特征与遗忘标签之间的互信息来衡量机器遗忘的彻底程度，揭示了现有黑盒指标（MIA、JSD等）无法捕捉的中间层残留信息问题，并提出 COLA 方法在特征层面消除残余信息。

## 背景与动机
机器遗忘（Machine Unlearning）旨在从已训练模型中移除特定数据的影响，以满足 GDPR "被遗忘权"等法规要求。理想的"强遗忘"要求遗忘后的模型与从头在保留数据上重新训练的模型不可区分。然而当前评估遗忘效果的主流方法（如 MIA 成员推断攻击、JSD 散度等）都是黑盒指标，只关注模型输出层面的相似性，忽略了中间层特征中可能残留的遗忘数据信息。

现有痛点：
- 黑盒指标容易被简单的输出层修改所"欺骗"——只改最后一层就能通过所有黑盒测试
- 缺乏可靠的白盒指标来验证近似遗忘算法在模型内部是否真正移除了数据影响
- 随着开源模型（Stable Diffusion、LLaMA）的普及，中间层残留信息可能被恶意利用

## 核心问题
如何评估机器遗忘算法是否真正在模型内部移除了遗忘数据的影响，而不仅仅是在输出层面伪装成重新训练的模型？

这个问题的关键在于：现有黑盒指标只衡量输出层行为，但一个仅修改了最后一层分类头的模型就能在所有黑盒指标上表现优异，而其编码器仍完整保留着遗忘数据的特征表示。这意味着这些数据仍可通过简单的头部重训练被恢复，构成严重的隐私风险。

## 方法详解

### 整体框架
论文分两大部分：（1）提出 IDI 评估指标；（2）提出 COLA 遗忘方法。评估流程是：对模型每个中间层，用 InfoNCE 损失估计该层特征与"样本是否属于遗忘集"这个二元标签之间的互信息，然后汇总各层的互信息差异得到 IDI 值。

### 关键设计

1. **Head Distillation (HD) 实验揭示黑盒指标的脆弱性**: 设计了一个极其简单的"伪遗忘"方法——冻结编码器，只用 KL 散度蒸馏重训最后一层分类头（将遗忘类的 logit 设为负无穷）。这个仅用 6.2 秒的方法在 MIA、JSD 等所有黑盒指标上超越了 SALUN、ℓ1-sparse 等近期方法，但编码器与原始模型完全相同，所有中间层信息完整保留。这个发现有力地证明了黑盒指标的不可靠性。

2. **Information Difference Index (IDI)**: 核心指标。定义信息差 $\text{ID}(\theta_u) = \sum_{\ell=1}^{L} (I(\mathbf{Z}_\ell^{(u)}; Y) - I(\mathbf{Z}_\ell^{(r)}; Y))$，其中 $\mathbf{Z}_\ell^{(u)}$ 和 $\mathbf{Z}_\ell^{(r)}$ 分别是遗忘模型和重训模型第 $\ell$ 层的特征，$Y$ 是标识样本是否属于遗忘集的二元标签。归一化后得到 IDI = ID(θ_u) / ID(θ_o)，值域解释为：0 表示完全遗忘（等同重训），1 表示无遗忘（等同原始模型），负值表示过度遗忘。

3. **互信息估计**: 采用 InfoNCE 损失作为互信息下界估计器。对每层 $\ell$，设计两个 critic 函数：$f_{\nu}^\ell$ 将中间特征映射到 $d$ 维嵌入空间（直接复用 $\ell+1$ 到 $L$ 层的网络结构加一个线性投影层），$g_{\eta}^\ell$ 将二元标签 $Y$ 映射为两个可学习的 $d$ 维向量。这种设计具有模型无关性，无需为不同架构重新设计 critic 网络。

4. **COLA (COLlapse and Align) 遗忘方法**: 两阶段方法。**Collapse 阶段**：用监督对比损失在保留集上训练编码器，使保留集特征紧密聚类，遗忘集特征被迫坍缩进保留集的聚类中，实现"灾难性遗忘"。**Align 阶段**：用交叉熵损失在保留集上微调整个模型，重新对齐编码器和分类头。COLA+ 变体额外处理随机数据遗忘场景：对遗忘样本分配伪标签（预测概率第二高的类别），使其特征坍缩到其他类的聚类中。

### 损失函数 / 训练策略
- IDI 计算只需最后 $n$ 层（$n \ll L$），因为早期层在原始模型和重训模型间差异极小，符合信息瓶颈理论。ResNet-18 用最后 2 个 block，ResNet-50 和 ViT 用最后 3 个 block。
- 用 10% 的保留集就能获得稳定的 IDI 估计，进一步降低计算成本。
- COLA 的 collapse 阶段用 supervised contrastive loss，align 阶段用标准交叉熵。

## 实验关键数据
| 数据集/架构 | 指标 | COLA | SALUN | SCRUB | ℓ1-sparse |
|-------------|------|------|-------|-------|-----------|
| CIFAR-10/R18 | IDI↓ | ~0 (最优) | ~0.8 | ~0 | ~0.1 |
| CIFAR-10/R18 | MIA→Retrain | 竞争力 | 好 | 竞争力 | 好 |
| ImageNet-1K/R50 | IDI↓ | ~0 (最优) | 高 | 中等 | 低 |
| CIFAR-100/R18 (多类) | IDI↓ | ~0 (最优) | 高 | 变化大 | 中等 |

关键发现：
- HD 在所有黑盒指标上排名第一或第二，但 IDI=1（完全没有遗忘中间层信息）
- CF-5 在 ImageNet-1K 上 MIA 接近 Retrain (10.21 vs 9.41)，但 IDI 高达 0.701
- SALUN 和 RL 在 t-SNE 中显示遗忘类特征仍然紧密聚集，与 IDI 高值一致
- 冻结遗忘模型编码器、仅用 2% 数据重训分类头后，Bad-T/SALUN/RL 可恢复 82%+ 的遗忘集准确率，而 Retrain 仅 41%

### 消融实验要点
- IDI 在不同层数 $n$ 选择下保持稳定，验证了只用最后几层估计的有效性
- 用 10% 数据 vs 全量数据计算 IDI，趋势一致，可显著节省计算
- IDI 在不同参考模型下（Retrain、COLA、EU-10、FT）的排序保持一致
- 与白盒 MIA（基于激活/梯度）相比，IDI 在 CIFAR-100 等更复杂数据集上更稳定，白盒 MIA 在随机初始化模型上会给出与 Retrain 相当的 MIA 值（不合理）
- CIFAR-100 上 ResNet-18 IDI 计算仅需约 5 分钟

## 亮点
- **HD 实验极具说服力**：用一个看起来"荒谬"的方法（只改最后一层）居然打败了所有精心设计的遗忘算法的黑盒指标，直观地暴露了当前评估体系的根本缺陷
- **IDI 的设计既优雅又实用**：基于信息瓶颈理论、通过 InfoNCE 估计互信息、复用现有网络结构作为 critic、只需少量层和少量数据，计算开销可控
- **IDI 与 t-SNE 和头部重训精度高度一致**，提供了多角度的交叉验证
- **COLA 方法简洁有效**：利用对比学习的"灾难性遗忘"特性，反过来用于遗忘目的，思路巧妙

## 局限性 / 可改进方向
- IDI 计算仍需多次训练 critic 网络，ResNet-50/CIFAR-100 上约 40-50 分钟，对大规模模型可能仍较慢
- IDI 依赖 Retrain 作为参考模型（虽然论文提出了无 Retrain 的变体，但此时 IDI 的绝对值解释会改变）
- 实验限于图像分类任务，未验证在生成模型（如扩散模型）、NLP 模型上的适用性
- COLA 在随机数据遗忘场景需要额外的伪标签步骤（COLA+），增加了复杂性
- 未讨论 IDI 如何处理遗忘集与保留集高度重叠的边界情况
- → 可扩展到视觉基础模型（CLIP、DINOv2）的遗忘评估，参见 [结构保持遗忘 idea](../../../ideas/ai_safety/20260316_structure_faithful_unlearning.md)

## 与相关工作的对比
- **vs NeurIPS 2025 "A Reliable Cryptographic Framework for Empirical Machine Unlearning Evaluation"**: 后者从密码学角度设计评估框架（SWAP test），关注黑盒层面的统计严谨性；本文从信息论角度提出白盒指标，直接检测中间层信息。两者互补——SWAP test 保证输出层评估的可靠性，IDI 保证特征层的完整遗忘
- **vs SALUN/ℓ1-sparse 等遗忘方法**: 这些方法在黑盒指标上表现好，但 IDI 揭示它们保留了大量中间层信息；只有 SCRUB 和 ℓ1-sparse 在部分设置下 IDI 接近 0
- **vs 参数距离/Fisher 信息等已有白盒指标**: 参数距离受训练随机性影响大且计算昂贵，Fisher 信息结果与理论直觉不一致。IDI 基于信息论有清晰的理论支撑，且在多种架构和数据集上表现稳定

## 启发与关联
- IDI 可直接用于 [结构保持遗忘 idea](../../../ideas/ai_safety/20260316_structure_faithful_unlearning.md) 的评估环节，验证在 CLIP/DINOv2 等基础模型上遗忘是否真正在特征层面生效
- "用对比学习实现灾难性遗忘"的思路（COLA）可迁移到其他需要精确移除特定知识的场景
- IDI 的 critic 网络设计（复用原网络层 + 投影层）可用于其他需要估计中间层特征与特定属性互信息的分析任务
- HD 实验的方法论——设计一个"反面教材"来暴露评估指标的弱点——是一种通用的研究策略，值得学习
