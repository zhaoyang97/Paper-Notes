# FinRpt: Dataset, Evaluation System and LLM-based Multi-agent Framework for Equity Research Report Generation

**会议**: AAAI 2026  
**arXiv**: [2511.07322](https://arxiv.org/abs/2511.07322)  
**代码**: https://github.com/jinsong8/FinRpt  
**领域**: LLM Agent / 金融NLP  
**关键词**: 股票研报生成, 多Agent框架, 金融数据集, 评估体系, 报告增强

## 一句话总结
首次系统化地定义股票研究报告（ERR）自动生成任务——构建 FinRpt 数据集（6,825篇中英文高质量研报，整合7类金融数据），提出11指标评估体系和9 Agent协作的FinRpt-Gen生成框架（含评级修正/专家审查/润色三阶段增强），人类评估显示生成报告质量接近专家撰写。

## 研究背景与动机

1. **领域现状**：LLM在金融情感分析、问答等任务上有成功应用，但自动生成完整股票研究报告（数千字结构化深度分析文档）尚未被系统研究。

2. **现有痛点**：
   - 缺乏高质量研报数据集——现有金融数据集聚焦摘要/短文本，不含结构化长报告
   - 缺乏研报专用评估指标——通用NLG指标（BLEU/ROUGE）无法衡量金融分析准确性
   - 研报包含6个关键章节（财务分析/新闻分析/管理层评估/风险分析/投资评估/综合评级），任何一个出错都可能误导投资者

3. **核心矛盾**：研报生成需要整合多种异构金融数据（时序股价、财报、公告、新闻、市场指数），且输出必须准确、全面、可操作——远超一般文本生成。

4. **本文要解决什么？** 定义ERR生成任务 + 构建数据集和评估体系 + 设计多Agent生成框架。

5. **切入角度**：将研报生成分解为9个专门化Agent的协作——每个Agent负责一种数据源或分析维度。

6. **核心idea一句话**：7类金融数据 + 9 Agent分工 + 三阶段增强 = 接近人类专家的研报自动生成。

## 方法详解

### 整体框架
公司代码+研究日期 → 自动收集7类数据（公司信息/财务指标/公告/新闻/股价/市场指数/宏观数据）→ 9个Agent分别处理 → 初始报告 → 三阶段增强（评级修正→专家审查→语言润色）→ 最终研报。

### 关键设计

1. **数据集构建管线**:
   - 做什么：自动生成高质量研报训练数据
   - 核心思路：从公开金融API收集数据 → LLM初始生成 → 三阶段增强：评级修正（比对市场走势）、专家风格审查、语言润色
   - 规模：6,825篇研报（中英双语），人类专家评分接近真实研报

2. **11指标评估体系**:
   - 做什么：全面衡量研报生成质量
   - 覆盖：内容准确性、逻辑连贯性、数据引用正确性、行业分析深度、风险评估全面性、投资建议可操作性、语言专业性、格式规范性、信息时效性、同行比较质量、评级合理性
   - 设计动机：BLEU/ROUGE在金融场景几乎无意义

3. **9 Agent协作框架 (FinRpt-Gen)**:
   - 做什么：分工协作生成完整研报
   - Agent分工：数据收集/财务分析/新闻分析/行业对比/风险评估/管理层评估/投资评级/报告整合/质量审核
   - 设计动机：单个LLM无法同时处理时序数据分析+文本理解+投资推理

### 损失函数 / 训练策略
- 数据集可用于SFT微调
- 评估使用LLM judge + 人类金融分析师评分

## 实验关键数据

### 主实验

| 指标 | FinRpt生成 | 专家撰写 | 说明 |
|------|----------|---------|------|
| 内容准确性 | 4.1/5 | 4.3/5 | 接近专家 |
| 逻辑连贯性 | 4.0/5 | 4.2/5 | 接近专家 |
| Fleiss kappa | 0.67-0.82 | - | 标注者一致性好 |

### 消融：增强阶段

| 配置 | 质量评分 |
|------|---------|
| 初始生成 | 3.5/5 |
| + 评级修正 | 3.8/5 |
| + 专家审查 | 4.0/5 |
| **+ 润色（完整）** | **4.1/5** |

### 关键发现
- 三阶段增强缺一不可，累计从3.5→4.1
- 数据集质量与专家撰写接近（kappa 0.67-0.82）
- 财务分析Agent是最关键组件

## 亮点与洞察
- 首次定义ERR生成任务的完整框架——输入格式、输出格式、评估体系
- 三阶段增强管线是数据质量的关键
- 9 Agent框架可推广到其他结构化长文档（审计报告、ESG报告）

## 局限性 / 可改进方向
- 评级修正依赖事后市场数据（非因果预测）
- 未在实际投资决策中验证报告的投资回报价值
- 金融数据时效性问题——训练数据可能在部署时已过时
- 6,825篇对大模型微调可能不够充分
- 9 Agent的编排开销和错误级联传播未分析

## 相关工作与启发
- **vs BloombergGPT**：金融预训练但不做报告生成
- **vs 金融摘要**：仅段落级。FinRpt是完整多章节报告
- 对金融AI有方法论贡献
