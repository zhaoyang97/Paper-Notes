# SoMe: A Realistic Benchmark for LLM-based Social Media Agents

**会议**: AAAI 2026  
**arXiv**: [2512.14720](https://arxiv.org/abs/2512.14720)  
**代码**: 无  
**领域**: LLM Agent / 社交媒体 / Benchmark  
**关键词**: 社交媒体Agent, MCP工具, 用户行为预测, 事件检测, 虚假信息检测  

## 一句话总结
提出 SoMe，首个全面评估 LLM 社交媒体 Agent 的 benchmark：8 个任务覆盖帖子分析、用户理解、综合推理，917 万帖子 + 6591 用户 + 1.7 万标注查询，配套 8 个 MCP 兼容工具，评估 13 个主流 LLM 发现最强模型仅 54.33 分（满分 100），揭示了推理能力≠Agent能力、工具调用幻觉普遍存在等关键发现。

## 研究背景与动机

1. **领域现状**：LLM Agent 在网页浏览、代码编写等领域已有成熟 benchmark，但社交媒体领域缺乏全面的评估标准。现有社交媒体相关工作多聚焦于单一任务（如情感分析），不涉及 Agent 式的工具调用和多步推理。
2. **现有痛点**：社交媒体 Agent 需要处理海量动态数据（千万级帖子）、理解用户画像、进行实时事件检测，这远超简单的 NLP 任务。目前没有 benchmark 能系统评估 LLM 在这些真实社交媒体场景中的表现。
3. **核心 idea 一句话**：构建覆盖 32 个平台的大规模真实社交媒体数据+8 个 MCP 工具+8 个任务的完整评估平台，首次全景式评估 LLM Agent 在社交媒体场景的能力边界。

## 方法详解

### Benchmark 设计

**三类八任务**：
- **帖子中心**：实时事件检测(RED)、流式事件摘要(SES)、虚假信息检测(MID)
- **用户中心**：用户行为预测(UBP)、用户情绪分析(UEA)、用户评论模拟(UCS)
- **综合**：媒体内容推荐(MCR)、社交媒体问答(SMQ)

**8 个 MCP 工具**：DataFolder（读数据）、SearchPost（按地点时间搜索）、SearchTopic（按话题搜索）、SearchUser（搜用户）、RetrievePost（向量检索帖子）、RetrieveKnowledge（知识库检索）、PostClustering（帖子聚类）、PostSummarization（LLM 摘要）

**数据规模**：9,164,284 帖子 + 6,591 用户 + 25,686 报告 + 17,869 标注查询，来自 32 个平台

## 实验关键数据

### 主实验（满分 100）

| 模型 | RED | MID | UBP | MCR | SMQ | **平均** |
|------|-----|-----|-----|-----|-----|---------|
| Gemini-2.5-Flash | 54.9 | 45.6 | 57.5 | 62.8 | 71.0 | **54.33** |
| Qwen3-32B | 44.3 | 47.4 | 67.0 | 63.3 | 80.3 | **53.98** |
| DeepSeek-V3 | 35.9 | 51.0 | 55.1 | 56.3 | 75.9 | **51.37** |
| GPT-4o | 47.6 | 50.2 | 55.2 | 61.6 | 64.2 | **49.88** |
| DS-R1-Qwen3-8B | 17.7 | 26.5 | 43.5 | 34.3 | 51.8 | **31.87** |
| Llama-3.1-8B | 3.4 | 40.5 | 34.2 | 47.4 | 31.1 | **31.65** |

### 任务完成率(TCR)

| 模型 | 平均 TCR |
|------|---------|
| DeepSeek-V3 | **98.60%** |
| Qwen3-32B | 98.45% |
| Llama-3.1-8B | 67.20% |
| DS-R1-Qwen3-8B | 69.73% |

### 关键发现
- **所有模型表现不佳**：最高平均分仅 54.33（Gemini），开放式任务（RED/SES/MID）大多低于 50 分
- **推理能力 ≠ Agent 能力**：DS-R1-Qwen3-8B（推理增强版）在所有 8 个任务上都**不如**原版 Qwen3-8B（降幅 29-56%）——过度推理反而不利于工具调用
- **工具调用幻觉普遍**：即使大模型也有 7-13% 的幻觉率（Kimi-K2: 7%, DeepSeek-V3: 13%）；小模型高达 28-29%
- **模型规模效应显著**：Qwen3 系列 32B→14B→8B 逐步下降；Llama 70B 比 8B 高 52.7%
- **工具链规划是关键能力**：大模型（Qwen3-32B/DeepSeek-V3/GPT-4o）几乎 100% 一次成功率，小模型显著挣扎

## 亮点与洞察
- **"推理增强模型做 Agent 反而更差"**是最令人意外的发现——DeepSeek-R1 的增强推理链在需要快速工具调用的场景下成为负担，说明推理和行动是两种不同的能力
- **MCP 工具标准化**使得不同模型在相同工具集上的对比公平且可复现
- **9 百万级帖子的真实数据规模**远超以往社交媒体 NLP 数据集——Agent 需要在海量数据中找到相关信息，更贴近实际

## 局限性 / 可改进方向
- 数据偏向中文社交媒体（微博），英文覆盖有限
- 评估主要依赖 LLM 判分（Qwen3-32B），可能有偏差
- 缺少多轮交互和长期记忆的评估
- 8 个任务仍未覆盖所有社交媒体场景（如趋势预测、社区管理）
