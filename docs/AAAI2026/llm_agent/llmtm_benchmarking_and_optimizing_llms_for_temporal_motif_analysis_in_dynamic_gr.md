# LLMTM: Benchmarking and Optimizing LLMs for Temporal Motif Analysis in Dynamic Graphs

**会议**: AAAI 2026  
**arXiv**: [2512.22266](https://arxiv.org/abs/2512.22266)  
**代码**: https://github.com/Wjerry5/LLMTM  
**领域**: LLM Agent / 图分析  
**关键词**: 时序图, 时序motif, 动态图, 结构感知路由, LLM基准

## 一句话总结
提出 LLMTM——首个评估 LLM 处理动态图中时序 motif 分析能力的综合基准，包含 6 类任务覆盖 9 种时序 motif 类型，评估 9 个模型后发现 LLM 对时序 motif 的识别能力随 motif 复杂度快速下降。提出结构感知分派器（Structure-Aware Dispatcher），根据图的结构属性和认知负荷智能路由查询到标准 LLM 提示或工具增强 Agent，在维持高准确率的同时降低计算成本。

## 研究背景与动机

1. **领域现状**：LLM 被用于各种图推理任务（节点分类、链接预测等），但在动态图的时序分析上几乎未被探索。时序 motif（在短时间内按特定顺序发生的交互模式）是动态图的基本单元，反映实时异常和独特现象。

2. **现有痛点**：
   - 没有专门评估 LLM 时序 motif 分析能力的基准
   - 不清楚 LLM 能否理解事件的时序顺序和结构模式
   - 缺乏针对图结构特性的查询路由策略——简单问题用重 Agent 是浪费

3. **核心矛盾**：时序 motif 分析需要同时理解图结构和时间顺序，但 LLM 本质上处理的是序列化的文本——如何有效地将图的时空信息编码为文本？

4. **本文要解决什么？** (a) 建立时序 motif 的 LLM 评估基准；(b) 设计高效的查询路由策略。

5. **切入角度**：6 类划分任务涵盖 motif 分析的不同方面（识别/计数/时间推理/节点角色/演化模式/异常检测），配合结构感知的智能路由。

6. **核心 idea 一句话**：6 任务 × 9 motif 类型基准 + 结构感知分派器 = 时序图上的高效 LLM 分析。

## 方法详解

### 整体框架
LLMTM 基准构建：定义 9 种时序 motif 类型（三角形/星形/链形的各种时序变体）→ 6 类分析任务（motif 识别/计数/时间窗口推理/节点角色判断/motif 演化趋势/异常 motif 检测）→ 在 9 个 LLM 上评估。优化方案：Structure-Aware Dispatcher 根据图结构属性（密度/度分布/时间跨度）和查询复杂度将查询路由到适当的推理模式。

### 关键设计

1. **6 类任务设计**:
   - Motif 识别：给定子图判断是否构成特定 motif
   - Motif 计数：统计图中某类 motif 的数量
   - 时间窗口推理：判断 motif 内事件的时序关系
   - 节点角色判断：识别节点在 motif 中的功能角色
   - 演化趋势：分析 motif 频率随时间的变化
   - 异常检测：识别偏离正常模式的 motif 实例
   - 设计动机：从基础感知到复杂推理的渐进难度

2. **结构感知分派器**:
   - 做什么：智能路由查询到合适的推理模式
   - 核心思路：分析图的结构特征（节点数、边密度、时间跨度）和查询复杂度，简单查询→标准 LLM 提示（低成本），复杂查询→工具增强 Agent（高精度但贵）
   - 设计动机：不是所有时序图查询都需要重量级工具——稀疏小图上的 motif 识别标准提示就够

3. **图文本序列化**:
   - 做什么：将动态图编码为 LLM 可处理的文本
   - 核心思路：按时间排序的边列表 + 节点属性描述 + 时间戳标注
   - 设计动机：保持时序信息的同时兼顾 LLM 的文本处理习惯

### 损失函数 / 训练策略
- 基准评估——无需训练
- 9 个 LLM 包括 GPT-4o、Claude-3.5、Llama-3 等

## 实验关键数据

### 主实验

| 任务 | 最佳模型准确率 | 难度趋势 |
|------|-------------|---------|
| Motif 识别 | ~85% | 较易 |
| Motif 计数 | ~60% | 中等 |
| 时间窗口推理 | ~55% | 较难 |
| 异常检测 | ~45% | 最难 |

### 消融：分派器效果

| 策略 | 准确率 | 计算成本 |
|------|--------|---------|
| 全部标准提示 | 较低 | 最低 |
| 全部 Agent | 最高 | 最高 |
| **结构感知路由** | **接近全Agent** | **显著低于全Agent** |

### 关键发现
- **LLM 对时序 motif 的理解随复杂度急剧下降**：识别 ~85% 但异常检测仅 ~45%
- **计数是普遍弱项**：所有模型在 motif 计数上表现最差——数数对 LLM 本来就困难
- **结构感知路由有效**：在准确率仅损失 1-3% 的情况下大幅降低计算成本
- **图的密度和大小显著影响性能**：稀疏小图上 LLM 表现好，密集大图急剧下降

## 亮点与洞察
- **首个时序 motif 的 LLM 基准**填补了动态图分析评估的空白
- **结构感知路由**的思路可推广到任何图 LLM 任务——不是所有图查询都需要同等重量级处理
- 6 类任务的渐进设计为分析 LLM 时序推理能力提供了细粒度诊断

## 局限性 / 可改进方向
- 仅在小规模图上评估（LLM 上下文限制）
- 时序 motif 的类型限于已知的 9 种，未覆盖更复杂的模式
- 分派器的结构特征选择可能需要领域知识调优
- 工具增强 Agent 的 tool 接口设计可能限制泛化性

## 相关工作与启发
- **vs NLGraph / GraphQA**：静态图基准。LLMTM 扩展到动态时序图
- **vs 传统 motif 算法（SNAP）**：精确但无 LLM 可解释性。LLMTM 为 LLM 提供了评估框架
- 对时序图上的 Agent 系统设计有参考

## 评分
- 新颖性: ⭐⭐⭐⭐ 首个时序 motif LLM 基准 + 结构感知路由
- 实验充分度: ⭐⭐⭐⭐ 6 任务 × 9 motif × 9 模型
- 写作质量: ⭐⭐⭐⭐ 任务分类系统
- 价值: ⭐⭐⭐ 对动态图 LLM 分析有基准贡献但应用场景较窄
