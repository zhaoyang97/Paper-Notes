# From Biased Chatbots to Biased Agents: Examining Role Assignment Effects on LLM Agent Robustness

**会议**: AAAI 2026  
**arXiv**: [2602.12285](https://arxiv.org/abs/2602.12285)  
**代码**: 无  
**领域**: LLM Agent / AI Safety  
**关键词**: LLM Agent, 隐性偏见, persona, 人口统计偏差, 鲁棒性

## 一句话总结
首个系统性案例研究，揭示基于人口统计学的 persona 分配会导致 LLM Agent 在 5 个操作领域的任务执行中出现最高 26.2% 的性能下降，证明 persona 诱导的偏见从文本生成延伸到了行动决策层面。

## 研究背景与动机

1. **领域现状**：LLM 正从聊天机器人转变为能执行实际操作的自主 Agent（代码部署、OS 自动化、医疗决策等）。Persona（角色设定）被广泛用于塑造 Agent 的行为风格和角色定位。
2. **现有痛点**：在纯文本生成场景中，persona 诱导的偏见已有充分研究——包括显性偏见（触发有害输出）和隐性偏见（相同任务因 persona 不同而产生不同输出）。但在 Agent 执行任务的场景中，persona 对**实际行动和决策**的影响几乎未被研究。
3. **核心矛盾**：Agent 的行为直接影响真实世界（执行命令、操作数据库、交易决策），如果 persona 能改变 Agent 的决策质量，这比文本偏见危害更直接——一个被分配了某种人口统计身份的 Agent 可能会在完全无关的任务上表现更差。
4. **本文要解决什么？** 系统性验证：人口统计学 persona（性别、种族/地域、宗教、职业）是否会影响 LLM Agent 在多步任务执行中的表现。
5. **切入角度**：用 23 个 persona 覆盖 4 个人口统计维度，在 5 个操作领域的 benchmark 上评估 3 个主流 LLM，量化 persona 对 Agent 性能的影响。
6. **核心发现**：与任务完全无关的 persona 线索可以导致高达 26.2% 的性能下降，且这种影响在不同模型和任务类型间普遍存在。

## 方法详解

### 整体框架
这是一项实证研究（case study），不提出新方法，而是设计严格的实验框架来测量 persona 对 Agent 性能的影响。整体流程：(1) 选择 persona 集合 → (2) 通过固定前缀注入 persona → (3) 在 5 个 benchmark 上评估 → (4) 对比有/无 persona 的性能差异。

### 关键设计

1. **Persona 选择与注入方式**:
   - **做什么**：覆盖 4 个人口统计维度共 23 个 persona，通过两轮对话前缀注入。
   - **核心思路**：使用标准化的 prompt 模板 `"From now on, you are a [ROLE]."` 加上角色确认回复，在所有下游指令之前插入。所有 persona 使用一致的措辞结构，仅在人口统计内容上不同。Baseline 完全省略前缀。
   - **设计动机**：保证唯一变量是 persona 内容，排除 prompt 格式差异的干扰。选择的 4 个维度（性别、种族/地域、宗教、职业）均有先前文本生成偏见的证据。

2. **评估 Benchmark 覆盖 5 个操作领域**:
   - **ALFWorld**：家庭任务规划（模拟环境中导航、定位物品、多步交互），衡量任务成功率
   - **WebShop**：电商决策（搜索、筛选、购买），衡量奖励分数
   - **Card Game**：策略推理（竞争性卡牌游戏），衡量胜率和得分
   - **OS Interaction**：系统级操作（解析指令执行 shell 命令），衡量命令准确率
   - **Database**：SQL 生成（从简单过滤到多表联接），衡量查询正确率

3. **模型选择**:
   - GPT-4o-mini（商业模型）、DeepSeek-V3（开源）、Qwen3-235B（MoE 模型）
   - 所有模型使用确定性解码和默认配置，反映真实部署条件

### 损失函数 / 训练策略
- 纯评估研究，不涉及训练。所有模型使用零温度确定性解码。

## 实验关键数据

### 主实验

| 模型 | 任务 | Baseline | 最差 Persona | 最大降幅 |
|------|------|----------|-------------|---------|
| DeepSeek V3 | Card Game | 71.2% | from Africa: 45.0% | **-26.2%** |
| Qwen3 235B | Card Game | 61.7% | White: 37.9% | **-23.8%** |
| GPT-4o-mini | Card Game | 78.2% | from America: 59.1% | **-19.1%** |
| GPT-4o-mini | ALFWorld | 52.0% | Asian: 46.0% | -6.0% |
| DeepSeek V3 | ALFWorld | 86.0% | Baseline最低 | +6.0% (persona反而提升) |

技术型任务（OS、Database）相对稳定，波动 2-5%；需要多步推理和规划的任务（Card Game、ALFWorld）受影响最大。

### 消融实验（按人口统计维度分析）

| 维度 | 关键发现 |
|------|---------|
| 种族/地域 | 影响最显著：DeepSeek V3 Card Game 在 "from Africa" 下降 26.2%，"Asian" 下降 23.4% |
| 性别 | GPT-4o-mini 中 Male 角色在家务任务上下降至 88%，Female 提升至 108%，反映刻板印象 |
| 职业 | GPT-4o-mini 在 Laborer 角色下 ALFWorld 最低；DeepSeek V3 在 Doctor 角色下显著提升 |
| 宗教 | DeepSeek V3 中 Christian 从 71.2% 降至 48.5%，但 GPT-4o-mini 中 Christian 反而提升 |

### 关键发现
- **高层推理最脆弱**：Card Game（策略推理）受 persona 影响最大，OS/Database（执行型任务）相对稳定
- **偏见方向因模型而异**：同一宗教 persona 在不同模型上效果可能相反，说明偏见来源于训练数据和对齐差异
- **性别偏见反映社会刻板印象**：Male 角色在家务类任务中表现差，暗示模型内化了"男性不擅长家务"的刻板印象
- **职业偏见呈阶层化**：高社会地位职业（Doctor、CEO）通常提升性能，低社会地位职业（Laborer、Farmer）降低性能

## 亮点与洞察
- **从文本偏见到行动偏见的跨越**：首次系统性地将 persona 偏见研究从文本生成扩展到 Agent 任务执行，揭示了更直接的操作风险。这一研究视角对 Agent 安全部署非常重要。
- **偏见的隐蔽性**：persona 与任务完全无关（给 Agent 设定"亚裔"身份不应影响卡牌游戏能力），但模型确实表现出了系统性差异，说明偏见深深嵌入在模型的推理过程中。
- **实验设计简洁有力**：仅通过两行 prompt 前缀就能引发最高 26.2% 的性能波动，方法论简单但发现有力。

## 局限性 / 可改进方向
- 仅评估 3 个模型和 5 个 benchmark，覆盖范围有限
- 缺乏机制分析——为什么 persona 会影响推理？信息处理的哪个环节被干扰？
- 没有提出缓解方案（debiasing）——只发现了问题，没有解决
- 确定性解码（temperature=0）可能不完全反映随机采样下的行为
- 未考虑多 Agent 协作场景中 persona 的交互效应

## 相关工作与启发
- **vs 文本偏见研究（Bai et al. 2025, Gupta et al. 2024）**: 先前工作证明 persona 影响文本生成和推理质量，本文将这一线索延伸到 Agent 行为。关键区别是：文本偏见影响输出质量，Agent 偏见影响实际决策和行动。
- **vs Agent 安全（DoomArena 等）**: Agent 安全研究通常关注对抗攻击和越狱，本文揭示了一种更隐蔽的脆弱性——完全正常的 persona 设定就能破坏性能。
- **启发**：开发 Agent 时需要进行 persona 鲁棒性测试，确保在各种身份设定下性能稳定。

## 评分
- 新颖性: ⭐⭐⭐⭐ 首次系统研究 persona 对 Agent 任务执行的影响，视角新
- 实验充分度: ⭐⭐⭐⭐ 3 个模型 × 5 个 benchmark × 23 个 persona，覆盖全面
- 写作质量: ⭐⭐⭐⭐ 结果呈现清晰，分维度分析系统
- 价值: ⭐⭐⭐⭐ 对 Agent 安全部署有重要警示意义，但缺乏解决方案

## 补充说明
- 角色分配放大偏见的发现对 multi-agent 系统设计有直接指导：应避免使用带有社会刻板印象的角色描述，或在角色提示中加入去偏指令
