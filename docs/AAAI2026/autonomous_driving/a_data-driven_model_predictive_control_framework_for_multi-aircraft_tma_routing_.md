# A Data-Driven Model Predictive Control Framework for Multi-Aircraft TMA Routing Under Travel Time Uncertainty

**会议**: AAAI 2026  
**arXiv**: [2511.19452](https://arxiv.org/abs/2511.19452)  
**代码**: [GitHub](https://github.com/zy2025github)  
**领域**: 空中交通管理 / 运筹优化  
**关键词**: 终端区管理, STAR路径调度, 模型预测控制, 混合整数线性规划, XGBoost预测, ADS-B数据, 新加坡樟宜机场  

## 一句话总结

提出面向终端管制区（TMA）多机冲突解脱和着陆调度的闭环 MPC 框架——集成 XGBoost 到达时间预测、MILP 优化模型（路径选择+速度调整+等待约束）和交通仿真器，在樟宜机场 50 海里 STAR 网络上实现实时无冲突调度，高峰期计算时间比一次性优化降低 7 倍，Monte Carlo 仿真验证鲁棒性。

## 背景与动机

航班延误带来巨大成本（旅客、航空公司、环境）。樟宜机场月商业航班量已恢复到疫情前水平（月均 3 万架次），管制员频繁使用雷达引导处理进场拥堵。现有研究的不足：

1. **跑道调度为主**：现有 TMA 优化主要关注跑道层面（ASP），对完整 STAR 路径的空域优化研究不足
2. **简化地图**：已有方法使用简化的 TMA 地图，航路点少、不含真实 STAR 结构
3. **开环优化**：一次性优化无法处理实时扰动和到达时间不确定性
4. **缺乏闭环验证**：未与交通仿真器集成做闭环验证

## 核心问题

如何设计一个可扩展的闭环系统，在完整的 STAR 网络上实时为多架飞机选择冲突解脱路径、调速和等待策略，在到达时间不确定性下保持高吞吐量？

## 方法详解

### 整体框架

四组件闭环系统：历史数据库 → XGBoost 预测器 → MPC 优化控制器 → 交通仿真器。MPC 滚动时域执行：优化→仿真执行→状态更新→重新优化。

### 关键设计

1. **数据驱动到达预测**：用 XGBoost 预测飞机到 50nm TMA 边界的到达时间，输入特征包括 ADS-B 位置/速度、跑道方向、出发机场、尾流等级、天气 METAR 数据。预测 TMA 边界到达而非跑道 ETA——因为 TMA 内等待/引导操作多，边界到达更可预测。

2. **MILP 优化模型**：建模 50nm 完整 STAR 网络（有向无环图），决策变量包括路径选择 $x_{ij}^f$、离场时间 $DP_i^f$、速度选择 $\delta_{ijs}^f$、等待时间 $HT_i^f$。约束包含：
   - 源/汇约束：每架飞机从初始航路点出发、最终着陆
   - 路径守恒：类似 VRP 的流守恒
   - 安全间隔：ICAO 安全间隔要求（航路点到达/离场间隔）
   - 防超车约束：同一边上先进先出
   - 速度离散化：管制员指令式速度等级
   - 目标：最小化平均着陆时间

3. **滚动时域 MPC**：每个 MPC 周期从仿真器获取当前状态、从预测器获取新飞机到达信息，重新求解 MILP，将优化指令发送回仿真器执行。

### 训练策略

XGBoost 在新加坡樟宜机场 ADS-B 数据集上训练；优化器为 MILP 精确求解（无学习组件）。

## 实验关键数据

| 指标 | 结果 |
|------|------|
| XGBoost 预测绝对误差 | 170.35 秒 |
| MPC vs 一次性优化计算加速 | **7 倍**（高峰期 36 架次/小时） |
| Monte Carlo 鲁棒性 | 在到达时间扰动下保持高可行性 |
| TMA 网络覆盖 | 完整 50nm STAR（樟宜机场 RWY20） |

### 关键观察

- TMA 内旅行时间方差（204 秒）远大于航路段（81 秒）——验证了 TMA 等待/引导操作频繁
- 预测 TMA 边界到达比预测跑道 ETA 更准确——正确的框架设计选择
- MPC 闭环在高峰期特别高效——滚动时域每次只优化有限飞机子集

## 亮点

- **完整 STAR 网络建模**：首次在完整 50nm STAR 结构上做 TMA 优化，而非简化地图
- **数据驱动 + 运筹优化的闭环整合**：XGBoost 预测 → MILP 优化 → 仿真验证的完整管道
- **预测 TMA 边界而非跑道 ETA**：避免 TMA 内操作引入的巨大方差
- **实际数据验证**：使用樟宜机场真实 ADS-B 数据

## 局限性 / 可改进方向

- **仅考虑进场**：离场航班和进/离场交互未建模
- **2D 模型**：假设连续下降，未考虑复杂三维冲突
- **单机场**：多机场协同 TMA 管理未涉及
- **MILP 求解器依赖**：高峰期仍需商业求解器

## 与相关工作的对比

- **vs Beasley MILP**：Beasley 仅关注跑道调度，本文扩展到完整 STAR 网络
- **vs Samà 等**：用简单 TMA 地图 + 有限航路点；本文用完整 AIP STAR 地图
- **vs Ng 优化模型**：预设跑道顺序无法全局优化；本文同时优化路径选择和顺序

## 启发与关联

- 数据驱动预测 + 运筹优化 + 仿真的三层闭环架构是工业控制问题的通用范式
- "预测容易预测的量而非困难的量"是系统设计的智慧
- MPC 滚动时域将大规模 MILP 分解为可管理的子问题
