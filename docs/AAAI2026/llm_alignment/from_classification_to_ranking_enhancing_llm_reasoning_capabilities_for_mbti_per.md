# From Classification to Ranking: Enhancing LLM Reasoning for MBTI Personality Detection

**会议**: AAAI 2026  
**arXiv**: [2601.18582](https://arxiv.org/abs/2601.18582)  
**代码**: 无  
**领域**: 对齐RLHF / 人格检测  
**关键词**: MBTI, 排序学习, GRPO, NDCG, 人格检测

## 一句话总结
将MBTI人格检测从传统的四维二分类重构为listwise排序任务，通过SFT冷启动+GRPO强化学习（NDCG+维度相似度双奖励），在Kaggle和PANDORA数据集上以7B模型达到SOTA。

## 研究背景与动机
1. **领域现状**：MBTI人格检测主流做法是将16类人格拆分为4个独立的二分类（E/I, S/N, T/F, J/P），使用BERT/RoBERTa等预训练模型编码用户帖子后分类。近期PsyCoT等方法尝试用LLM的CoT推理做人格分析。
2. **现有痛点**：(a) 四维独立分类忽略了维度间的心理学交互关系——INTJ和ENTJ虽然只差一个字母，但主导认知功能完全不同（Ni vs Te）；(b) Prompt-based方法过度依赖专家设计的提示，且大模型推理延迟高、成本大，不适合实时应用。
3. **核心矛盾**：人格特质本质上是连续光谱上的相对偏好，而非离散类别。分类范式无法捕捉类型间的细粒度亲和度差异。
4. **本文要解决什么？** 如何让LLM理解16种MBTI类型之间的相对亲和度关系，而非简单判断属于哪一类。
5. **切入角度**：心理测量学认为人格通过相对特质偏好而非绝对类别来表征，排序天然比分类更适合建模这种关系。
6. **核心idea一句话**：将人格检测重新定义为对16种MBTI类型的listwise排序任务，用GRPO+NDCG奖励训练LLM学习最优排序。

## 方法详解

### 整体框架
输入是用户的社交媒体帖子集合 $P=\{p_1, ..., p_n\}$，输出是16种MBTI类型按亲和度从高到低的Top-K排序列表。采用两阶段后训练流程：Stage 1 SFT冷启动 → Stage 2 GRPO强化学习。

### 关键设计

1. **训练数据构造（排序数据蒸馏）**：
   - 做什么：为SFT阶段构造带推理轨迹的排序训练数据
   - 核心思路：使用Qwen-plus作为教师模型，给定用户帖子生成`<think>...</think>`推理过程和Top-K排序答案`<answer>[type1, type2, ...]</answer>`。通过拒绝采样过滤掉ground-truth不在Top-K预测列表中的低质量样本，最终精选1000条高质量SFT数据
   - 设计动机：1000条精选数据足够冷启动排序能力，同时避免SFT阶段过拟合。拒绝采样确保推理链不仅正确且可读

2. **SFT冷启动**：
   - 做什么：让7B模型学会排序输出格式和基本推理模式
   - 核心思路：在Qwen2.5-7B-Instruct上用LoRA（rank=32）微调3个epoch，学习教师模型蒸馏的比较推理和格式化输出能力
   - 设计动机：为后续RL阶段提供合理的策略初始化；如图4所示，没有SFT，后续RL训练的性能提升非常微弱

3. **GRPO强化学习 + 排序奖励函数**：
   - 做什么：通过RL进一步优化排序质量，增强泛化能力
   - 核心思路：使用GRPO算法（无需价值网络），核心在于奖励函数设计。总奖励 = NDCG奖励 + 维度相似度奖励（DS Reward）。NDCG@k 通过位置折扣衡量整个排序列表质量（公式4-6）。维度相似度 $s(\hat{t}, t^*) = \sum_{i=1}^{4}(1+\epsilon \cdot i) \cdot \delta(\hat{t}^{(i)}, t^{*(i)})$ 逐字符比较预测与ground-truth的4个维度匹配度，DS Reward取Top-1预测的维度相似度以强化首位答案准确性
   - 设计动机：分类奖励信号稀疏且无差异（组内方差极小），无法为GRPO提供有效梯度。NDCG+DS双奖励互相稳定——单独使用任何一个都会在训练中崩溃（图4红线），两者联合使用才能避免reward hacking并产生稳定的性能提升

### 损失函数 / 训练策略
- GRPO目标函数（公式3）：对组内G个采样做相对优势归一化 $A_i = (r_i - \mu_r)/\sigma_r$，PPO-clip式更新 + KL正则
- RL阶段用排除SFT数据的原始数据集训练2000步，batch size 128，组内生成16个响应
- 每个帖子限128 token，每用户最多50帖，匹配人格标签的词替换为`<MASK>`防止信息泄露

## 实验关键数据

### 主实验

| 方法 | Kaggle Macro-F1 | Kaggle F1(16类) | PANDORA Macro-F1 | PANDORA F1(16类) |
|------|----------------|-----------------|------------------|------------------|
| D-DGCN | 71.35 | 30.32 | 61.40 | 29.69 |
| ETM | 77.79 | 32.55 | 65.77 | 30.09 |
| Qwen-plus | 76.43 | 38.63 | 61.34 | 25.08 |
| **PerDet-R1** | **80.57** | **41.34** | **66.10** | **35.08** |

- 比最强baseline ETM：Macro-F1提升2.78%/0.83%，16类F1提升8.79%/4.99%
- 16类多分类F1优势尤为显著，因为排序范式将4个MBTI维度作为整体建模

### 消融实验

| 配置 | Binary F1 | Multi F1 | NDCG@3 |
|------|-----------|----------|--------|
| Full (SFT+GRPO) | 80.57 | 41.34 | 74.31 |
| w/o GRPO | 65.03 | 15.82 | 64.09 |
| w/o SFT | 68.11 | 17.21 | 67.58 |
| w/o NDCG reward | 68.90 | 27.28 | 66.92 |
| w/o DS reward | 70.06 | 28.64 | 69.38 |
| Classification替代Ranking | 63.07 | 33.67 | - |

### 关键发现
- 去掉GRPO后NDCG降10.22%，去掉SFT+GRPO后降24.85%——两阶段训练缺一不可
- 分类替代排序后Binary F1降17.5%，证实排序范式的显著优势
- NDCG和DS奖励单独去掉任一个都导致另一个在训练中崩溃，验证双奖励的协同稳定性
- Top-5 vs Top-3：更大答案空间在多分类和NDCG上小幅提升(+2.24%/+0.60%)，但Binary F1下降4.13%

## 亮点与洞察
- **分类→排序的范式转换**是核心创新：排序自然地建模了16种MBTI类型间的4维特质交互关系，不再人为拆分维度。这种思路可迁移到其他多维度分类任务（如情感分析的多维度打分）
- **双奖励互稳定机制**巧妙：NDCG关注列表整体排序质量，DS关注Top-1的维度对齐，两者互为正则化避免reward hacking，解决了RL训练不稳定的实际问题
- **1000条SFT数据冷启动**的实用性：通过教师蒸馏+拒绝采样，少量高质量数据即可完成冷启动，降低了应用门槛

## 局限性 / 可改进方向
- 仅使用文本帖子，未利用头像、表情等多模态信息（作者已在Future Work提到）
- Kaggle数据集来自MBTI论坛，用户帖子中隐含人格类型缩写（如"xNTP"），可能造成数据偏差导致该数据集上结果偏高
- PANDORA上的提升相对较小(+0.83% F1)，说明在非MBTI专属论坛数据上泛化能力有待加强
- 仅在7B模型上验证，更大模型是否能进一步放大排序范式优势未知

## 相关工作与启发
- **vs ETM**：ETM利用LLM的文本嵌入和生成能力增强用户向量表示，仍是分类范式；本文直接改为排序，在16类F1上优势巨大(+8.79%)
- **vs PsyCoT**：PsyCoT用多轮对话模拟心理问卷填写，依赖专家设计的CoT提示；本文通过RL自主学习推理模式，无需专家知识
- 启发：分类→排序的范式转换思路可推广到其他主观评估任务，如情感分析、审美评分、产品推荐等
