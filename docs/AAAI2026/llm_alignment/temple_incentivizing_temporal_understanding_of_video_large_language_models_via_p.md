# TEMPLE: Incentivizing Temporal Understanding of Video LLMs via Progressive Pre-SFT Alignment

**会议**: AAAI 2026
**arXiv**: [2503.16929](https://arxiv.org/abs/2503.16929)
**代码**: [GitHub](https://github.com/lscpku/TEMPLE)
**领域**: LLM Alignment
**关键词**: Video LLM, 时间理解, DPO, 偏好学习, 课程学习

## 一句话总结
提出 TEMPLE，通过自动化的视频时间偏好数据生成管线（视频筛选→时间扰动→对比响应）和创新的 Progressive Pre-SFT Alignment 策略（课程学习 + DPO 先于 SFT），用少量自生成 DPO 数据显著提升 Video LLM 的时间推理能力，在 VideoMME、MLVU、Vinoground 等多个基准上一致改进。

## 研究背景与动机

1. **领域现状**：Video LLM 采用"大规模预训练 + SFT "范式取得了显著成功，但在时间推理上仍表现不佳——初步分析显示 Qwen2-VL-7B 的详细描述精度仅 74.4%、召回仅 35.0%，遗漏大量事件且引入幻觉。
2. **现有痛点**：(a) 视频数据集中时间信息稀缺——很多视频较静态，视频-文本对的时间对应性弱；(b) 预训练和 SFT 都依赖 next-token prediction，不显式强制动态时间理解；(c) 模型频繁忽略微妙的时间细节，过度依赖局部视觉/文本线索。
3. **核心矛盾**：高质量视频时间标注极其昂贵，但 SFT 需要完美标注；DPO 只需要相对比较，更适合视频任务。
4. **切入角度**：(a) 通过视频扰动（丢帧、乱序、时间反转）自动生成时间偏好对，无需人工标注；(b) 将 DPO 放在 SFT 之前而非之后（逆常规顺序），先建立时间理解再学指令遵循。
5. **核心 idea**：三阶段自动化管线生成时间偏好数据 + 渐进式 Pre-SFT 对齐（先易后难的课程学习 + DPO 前置于 SFT）。

## 方法详解

### 整体框架
两大组成部分：(1) 自动化偏好数据生成管线；(2) Progressive Pre-SFT Alignment 训练策略。

### 关键设计

1. **视频偏好数据生成管线**：
   - **视频筛选**（3步）：TransNetV2 场景检测筛选时间复杂度够高的视频 → SigLIP 相似度分组去除重复场景 → Laplacian 算子提取清晰关键帧。保留 4-32 个独特场景组的视频
   - **时间扰动策略**（3种）：
     - 随机片段丢弃：删除部分片段，迫使模型在缺失信息下生成响应
     - 随机片段乱序：打乱片段顺序，保留视觉信息但破坏时间结构
     - 片段时间反转：翻转片段内的时间顺序
   - **对比响应生成**：对干净/扰动输入分别生成响应，干净为 chosen、扰动为 rejected
   - 难度因子 $r$ 控制扰动强度：$r$ 越大扰动越强

2. **Progressive Pre-SFT Alignment**：
   - **课程学习**：训练过程中逐步降低 $r$（从易到难），让模型先学清晰区分再处理微妙差异
   - **Pre-SFT 对齐**（核心创新）：逆转常规的"先 SFT 再 DPO"顺序，先做 DPO 建立时间对齐再做 SFT
   - 为什么先 DPO？Video LLM 的主要瓶颈是无法建立视觉输入和文本描述之间的精确对应——应先解决这个基础问题，再学通用指令遵循
   - 实验证据：Pre-SFT 对齐后的模型在后续 SFT 中展现更低的 loss 和更稳定的梯度

3. **无外部依赖的自给自足设计**：
   - 视频筛选不依赖外部 LLM（如 GPT-4），全部用开源工具
   - 偏好数据由模型自身生成（self-generated），不需要人工标注
   - 与 Tarsier2/TPO 等依赖商业 API 的方法形成对比

## 实验关键数据

### 主实验

| 模型 | VideoMME | MLVU | Vinoground_txt |
|------|---------|------|---------------|
| Qwen2-VL-7B (baseline) | 低 | 低 | 低 |
| + TEMPLE | 显著提升 | 显著提升 | 显著提升 |
| GPT-4o | 77.2 | 64.6 | 54.0 |

### 消融实验

| 配置 | 效果 |
|------|------|
| 仅 SFT（无 DPO） | 基线 |
| 先 SFT 再 DPO（常规顺序） | 有提升但不如 Pre-SFT |
| **先 DPO 再 SFT（Pre-SFT）** | **最佳** |
| 无课程学习（固定难度） | 次于渐进式 |

### 关键发现
- **Pre-SFT Alignment 优于传统的 Post-SFT DPO**：先建立时间理解基础再学指令遵循，效果更好。表现为 SFT 阶段的 loss 更低、梯度更稳定
- **课程学习有效**：从易到难的渐进式训练比固定难度更高效
- **少量 DPO 数据即可产生显著改进**：不需要大规模标注，管线的自给自足特性很关键
- **DPO 数据在不同模型架构和规模间可迁移**：说明学到的时间对齐是通用的

## 亮点与洞察
- "先 DPO 再 SFT"的逆序策略是一个很有启发性的发现——传统观点认为需要先学基础能力再做对齐，但对于视频时间理解这种需要精确感知的能力，先通过偏好学习建立正确的感知基础反而更有效。这个洞察可能适用于其他需要"先学会看再学会说"的多模态任务
- 三种时间扰动策略（丢帧、乱序、反转）精确针对 Video LLM 的已知弱点设计，且通过难度因子统一控制，简洁优雅

## 局限性 / 可改进方向
- 仅用 detailed captioning 作为 DPO 任务，未扩展到 QA、时间定位等其他视频理解任务
- 扰动策略相对简单（clip 级而非 frame 级），可能无法捕获更细粒度的时间依赖
- 视频筛选管线对场景检测工具（TransNetV2）的质量有依赖
- Pre-SFT 策略增加了训练阶段，总训练成本未报告

## 相关工作与启发
- **vs POVID**：启发了通过扰动生成偏好对的思路，但 POVID 是图像级的，TEMPLE 扩展到视频的时间维度并增加了课程学习
- **vs Tarsier2/TPO**：依赖商业 LLM API 做数据筛选且在 SFT 后做 DPO，TEMPLE 完全自给自足且采用 Pre-SFT 策略
- **vs 标准 DPO/RLHF**：传统先 SFT 再 DPO，TEMPLE 逆转顺序并证明对视频任务更有效
