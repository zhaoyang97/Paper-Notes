# Differentiated Directional Intervention: A Framework for Evading LLM Safety Alignment

**会议**: AAAI 2026  
**arXiv**: [2511.06852](https://arxiv.org/abs/2511.06852)  
**代码**: 无  
**领域**: LLM Alignment  
**关键词**: LLM安全, 越狱攻击, 激活操控, 安全对齐, 可解释性

## 一句话总结
将 LLM 安全对齐的内部表征从传统的"单一拒绝方向"解构为功能独立的"危害检测方向"和"拒绝执行方向"，在此基础上提出 DBDI 框架，分别用自适应投影消除和直接引导两种策略精准干预两个方向，在 Llama-2 上实现 97.88% 的越狱成功率。

## 研究背景与动机

1. **领域现状**：LLM 经过 RLHF 等安全对齐训练后会拒绝有害请求，但对齐并未真正消除模型的有害能力，只是压制了它们。白盒越狱攻击利用模型内部状态来绕过安全机制。
2. **现有痛点**：主流的激活层攻击方法（如 Arditi 等人的 Directional Ablation）将安全机制建模为激活空间中的"单一线性方向"——通过计算有害/无害 prompt 激活的均值差得到"拒绝方向"。但这个单一方向混淆了两个不同的神经过程。
3. **核心矛盾**：单一方向的干预缺乏精度——可能同时破坏检测能力和执行能力，导致输出不连贯或安全绕过不彻底。
4. **切入角度**：作者假设安全机制由两个功能独立的方向构成——**危害检测方向**（模型识别请求是否有害）和**拒绝执行方向**（模型执行拒绝回复的动作），需要用不同策略分别干预。
5. **核心 idea**：用 SVD + 分类器引导稀疏化提取这两个方向，然后对拒绝执行方向做自适应投影消除（去掉"执行拒绝"的能力），对危害检测方向做直接引导（压制"感知危害"的能力），两步联合实现精准越狱。

## 方法详解

### 整体框架
DBDI 分为离线校准和在线推理两个阶段：
- **离线**：从对比 prompt 对中提取"拒绝执行向量" $\vec{v}_{refusal}$ 和"危害检测向量" $\vec{v}_{harm}$，并用分类精度选择最佳干预层 $l^*$
- **在线**：在推理时对 $l^*$ 层的隐状态执行两步修改，使模型生成不拒绝的回复

### 关键设计

1. **双方向向量提取（SVD + 分类器稀疏化）**：
   - 做什么：分别提取拒绝执行方向和危害检测方向
   - 核心思路：(a) **拒绝执行向量**用最小差异 twin prompt 对（语义几乎相同但一个有害一个无害），SVD 取第一主方向；(b) **危害检测向量**用有害 prompt vs 中性 prompt，同样 SVD 取第一主方向
   - 关键细节：原始 SVD 方向噪声多，训练线性分类器识别最具判别力的神经元，只保留前 k% 的重要维度（稀疏化），大幅提升方向的纯度
   - 设计动机：两个方向使用不同的 prompt 对比集合，确保捕获的是不同功能的信号——twin prompt 差异微小，捕获的是"拒绝 vs 不拒绝"；有害 vs 中性差异大，捕获的是"有害 vs 无害"

2. **差异化双向干预（推理时）**：
   - 做什么：在关键层修改隐状态以绕过安全对齐
   - 两步策略完全不同：
     - **Step 1 — 自适应投影消除**：$h^{(1)} = h - \alpha \cdot \text{proj}_{\vec{v}_{refusal}}(h)$，去掉隐状态中沿拒绝执行方向的分量。是自适应的——投影量依赖于当前隐状态在该方向上的投影大小
     - **Step 2 — 直接引导**：$h' = h^{(1)} - \beta \cdot \vec{v}_{harm}$，沿危害检测方向直接偏移固定量，压制模型对危害的感知
   - 完整公式：$h' = h - \alpha \cdot \text{proj}_{\vec{v}_{refusal}}(h) - \beta \cdot \vec{v}_{harm}$
   - 设计动机：两个方向的干预策略必须不同——拒绝执行是"动作"，用投影消除精准移除；危害检测是"感知"，用固定偏移更稳健。消融实验证实对称干预效果大幅下降

3. **最佳层选择**：
   - 做什么：用线性分类器的 5 折交叉验证精度选择拒绝执行方向最显著的层
   - 设计动机：只在单一最佳层干预，计算开销极小（仅一个 forward hook）

## 实验关键数据

### 主实验

| 模型 | 方法 | AdvBench ASR | HarmBench ASR |
|------|------|-------------|--------------|
| Llama-2 7B | Directional Ablation | 最高 ~60-70% | - |
| Llama-2 7B | **DBDI** | **95.96% (97.88%)** | **92% (95%)** |
| Qwen 7B | **DBDI** | **83.4%** | - |
| Llama-3.2 3B | **DBDI** | **87.5%** | - |

括号内为简化 template 的结果。

### 消融实验（AdvBench / HarmBench）

| 变体 | AdvBench ASR | HarmBench ASR |
|------|-------------|--------------|
| Full DBDI | 95.96% | 92% |
| 对称投影（两方向都投影消除） | 62.88% | 86% |
| 对称引导（两方向都直接引导） | 9.42% | 16% |
| 仅拒绝执行方向 | 1.34% | 73% |
| 仅危害检测方向 | 11.34% | 35% |

### 关键发现
- **两个方向功能确实不同**：仅干预一个方向效果远不如两个联合（仅拒绝方向 AdvBench 仅 1.34%，仅危害方向 11.34%），证实了双方向假设
- **不对称干预策略至关重要**：对称投影 AdvBench 62.88%，对称引导仅 9.42%，而 DBDI 的差异化策略 95.96%。说明两个方向需要不同的处理方式
- **简化 chat template 效果更好**：Llama-2 上从 95.96% 提升到 97.88%，说明复杂 template 中的安全相关 token 起到了额外的防护作用
- **计算开销极小**：仅在单层添加一个 forward hook，推理时几乎无额外延迟

## 亮点与洞察
- 将"拒绝方向"分解为"检测"和"执行"两个功能独立方向的想法很有机制性洞察价值——这不仅对攻击有用，更重要的是为理解 LLM 安全对齐的内部机制提供了新的分析框架。如果防御者也采用这个双方向视角，可能设计出更鲁棒的对齐方案
- 消融实验的设计很有说服力：对称 vs 不对称、单方向 vs 双方向、投影 vs 引导，系统性地验证了每个设计选择的必要性

## 局限性 / 可改进方向
- 完全依赖白盒访问——需要完整的模型权重和激活，对闭源 API 模型无效
- 需要构造对比 prompt 数据集来提取方向向量，这些数据集的质量和多样性会影响提取结果
- $\alpha$ 和 $\beta$ 两个超参需要在验证集上网格搜索，缺少自适应选择方法
- 作为一种攻击研究，论文未讨论针对 DBDI 的防御方案（如是否可以通过检测隐状态修改来发现攻击）
- 只在 7B-8B 规模的模型上实验，更大规模模型的安全机制是否也是双方向结构尚未验证
- 分类器引导稀疏化的 percentile 阈值 $k$ 对结果的敏感性未充分分析
- 两个方向向量的正交性假设未被验证——如果危害检测和拒绝执行方向存在显著相关性，分别处理可能引入干扰
- 缺少对干预后模型输出质量（连贯性、困惑度）的定量评估

## 相关工作与启发
- **vs Directional Ablation (Arditi et al.)**：将安全建模为单一拒绝方向并消除，DBDI 将其分解为两个功能方向分别处理，AdvBench ASR 从 ~60-70% 提升到 95.96%
- **vs GCG/AutoDAN**：这些方法在 prompt 层面搜索对抗后缀，容易被输入过滤检测；DBDI 在激活层面直接操作，绕过所有输入级防御
- **vs 微调类攻击**：微调会永久修改权重，容易被权重检查发现且可能损害通用能力；DBDI 仅在推理时临时修改激活，不留痕迹且完全可逆

## 评分
- 新颖性: ⭐⭐⭐⭐⭐ 将安全方向解构为检测和执行两个独立方向的洞察非常精彩，为理解 LLM 安全机制提供了新框架
- 实验充分度: ⭐⭐⭐⭐ 消融实验设计系统，多模型验证，但缺少 70B+ 大模型测试
- 写作质量: ⭐⭐⭐⭐ 数学形式化清晰，方法动机反复强调
- 价值: ⭐⭐⭐⭐⭐ 对攻防双方都有启示——攻击者获得更精准的工具，防御者获得更细粒度的安全理解

## 补充说明
- 该方法揭示了安全对齐不是单一维度的，危害检测和拒绝执行可以被独立操控——未来的安全训练应考虑多维度鲁棒性
