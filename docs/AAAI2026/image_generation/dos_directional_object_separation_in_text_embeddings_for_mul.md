# DOS: Directional Object Separation in Text Embeddings for Multi-Object Image Generation

**会议**: AAAI 2026  
**arXiv**: [2510.14376](https://arxiv.org/abs/2510.14376)  
**代码**: 未公开  
**领域**: 图像生成  
**关键词**: 多物体图像生成, 文本嵌入分离, 属性绑定, 对象融合, 扩散模型  

## 一句话总结
提出DOS方法，在文本嵌入空间中通过方向性分离解决多物体图像生成中的属性混淆和对象融合问题，使不同对象在嵌入空间中占据不同方向，从而生成结构清晰的多物体场景。

## 背景与动机
文本到图像扩散模型在生成包含多个对象的复杂场景时常出现属性混淆（把A的颜色给了B）和对象融合（两个对象合并为一个）。这源于文本编码器将多个对象的信息纠缠在同一嵌入空间中，扩散模型无法正确区分各对象的控制信号。

## 核心问题
如何在文本嵌入空间中有效分离多个对象的表示，使每个对象的属性信息独立可控，避免属性混淆和对象融合？

## 方法详解
### 整体框架
在文本编码后、输入扩散模型前，通过方向性分离操作将不同对象在嵌入空间中推向不同方向。

### 关键设计
1. **Directional Object Separation**: 基于对象解析，在嵌入空间中对不同对象的token施加方向性偏移
2. **属性保持**: 分离的同时保持各对象内部的属性绑定一致性

## 亮点
- 直接在嵌入空间操作，不需要修改模型——类似DICE对CFG的嵌入空间操作思路
- 解决了多物体生成的核心痛点

## 局限性
- 对象数量增多时分离效果可能退化
- 仅基于abstract信息

## 与相关工作的对比
- vs Attend-and-Excite: A&E通过attention引导，DOS在嵌入空间直接分离——更早期干预
- 与DICE(AAAI2026)异曲同工——都在text embedding空间做关键操作
