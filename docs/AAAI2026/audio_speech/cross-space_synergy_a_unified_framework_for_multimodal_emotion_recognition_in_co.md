# Cross-Space Synergy: A Unified Framework for Multimodal Emotion Recognition in Conversation

**会议**: AAAI 2026  
**arXiv**: [2512.03521](https://arxiv.org/abs/2512.03521)  
**代码**: 无  
**领域**: Audio & Speech / 多模态情感识别  
**关键词**: 多模态情感识别, 高阶融合, 梯度冲突, Pareto优化, 对话情感

## 一句话总结

提出 Cross-Space Synergy（CSS）框架，通过表示空间的协同多项式融合（SPF）和梯度空间的 Pareto 梯度调节器（PGM）双管齐下，同时解决多模态对话情感识别中融合表达力不足和多目标梯度冲突两大难题。

## 研究背景与动机

1. **领域现状**：多模态对话情感识别（MERC）需要整合文本、语音和视觉信号来推断说话者的情绪状态。现有方法包括基于注意力的模型（DialogueRNN、Ada2I、SDT）、多阶段融合框架（LMF、MulT）和图方法（DialogueGCN）。
2. **现有痛点**：浅层融合（如拼接、双线性）无法捕捉非线性高阶跨模态交互；而加深网络或采用更复杂的交互机制又会带来梯度冲突和训练不稳定。
3. **核心矛盾**：表达力与训练稳定性之间的根本性权衡——浅层交互导致建模不充分，深层融合引入混乱的优化动态。
4. **本文要解决什么？** 在一个统一框架中同时增强跨模态表示的表达力和多目标训练的稳定性。
5. **切入角度**：将 MERC 重新概念化为跨空间协同问题——表示空间需要更强的非线性建模能力，梯度空间需要协调多个学习目标的冲突。
6. **核心idea一句话**：用低秩张量分解实现高阶跨模态融合（表示空间），用 Pareto 最优梯度方向平衡多目标冲突（梯度空间），两者协同工作。

## 方法详解

### 整体框架

CSS 由三个组件组成：
1. **表示编码（Representation Encoding）**：说话者感知 + 上下文感知的两阶段编码器
2. **表示空间融合（SPF）**：基于低秩张量组合的高阶多模态融合
3. **梯度空间优化（PGM）**：基于 Pareto 最优的动态多目标梯度协调

### 关键设计

1. **两阶段表示编码**：
   - **模态感知编码（Modality-aware）**：对每个模态独立使用 Transformer 自注意力编码器，捕获模态内的上下文依赖 $\mathbf{h}_{\text{Ma}}^{(m)}$
   - **交互感知编码（Interaction-aware）**：引入门控跨注意力机制。对每个查询模态 $m$，通过 sigmoid 门控调节其他模态的影响：$\mathbf{g}^{(\tilde{m})} = \sigma(\mathbf{W}^{(\tilde{m})}\mathbf{q}^{(m)} + \mathbf{b}^{(\tilde{m})})$，然后通过 Hadamard 积聚合非查询模态信息。设计动机：门控机制可以自适应地过滤无关或噪声模态信号
   - 两阶段的输出拼接并投影到统一空间：$\tilde{\mathbf{h}}^{(m)} = \mathbf{W}^{(m)}[\mathbf{h}_{\text{Ma}}^{(m)}; \mathbf{h}_{\text{Ia}}^{(m)}] + \mathbf{b}^{(m)}$

2. **协同多项式融合（SPF）**：
   - 受多项式张量池化（PTP）和 CP 分解启发，模拟 $p$ 阶多路乘法交互
   - **模态特异性投影**：不使用共享投影（前人做法），而是为每个模态 $m$ 和每个交互阶 $j$ 使用独立的线性投影 $\mathbf{z}_j^{(m)} = \phi(\mathbf{W}_j^{(m)}\tilde{\mathbf{h}}^{(m)})$
   - **静态门控 + 偏置锚定**：通过可学习标量权重 $\lambda_j^{(m)}$ 调节每个模态贡献，加上偏置向量 $\mathbf{w}_j$ 作为稳定锚点，补偿高阶交互的乘法放大效应
   - 门控融合：$\mathbf{z}_j = \sum_m \lambda_j^{(m)}\mathbf{z}_j^{(m)} + \mathbf{w}_j$，然后元素级乘积聚合 $\mathbf{z} = \mathbf{z}_1 \odot \mathbf{z}_2 \odot \cdots \odot \mathbf{z}_p$
   - **符号平方根变换**：$\hat{\mathbf{z}} = \text{sign}(\mathbf{z}) \odot \sqrt{|\mathbf{z}|}$，控制融合张量的动态范围，减小幅度方差同时保留方向信息，促进更稳定的梯度传播

3. **Pareto 梯度调节器（PGM）**：
   - 将三个监督信号视为竞争目标：多模态分类损失 $\mathcal{L}_1$、单模态正则化损失 $\mathcal{L}_2$（硬标签监督各单模态分支）、蒸馏损失 $\mathcal{L}_3$（用融合分支的软标签指导各单模态分支）
   - 计算各任务梯度的 L2 归一化版本，构建 $3 \times 3$ 内积矩阵 $Q = GG^\top$
   - 求解 QP 问题：$\min_\gamma \frac{1}{2}\gamma^\top Q \gamma$，约束 $\gamma \geq 0, \mathbf{e}_3^\top \gamma = 1$，得到 Pareto 最优任务权重
   - 与 MGDA 不同，PGM 在归一化梯度上工作，QP 维度固定为 3，开销极小

### 损失函数 / 训练策略

最终损失为动态加权组合：$\mathcal{L} = \gamma_1 \cdot \mathcal{L}_1 + \gamma_2 \cdot \mathcal{L}_2 + \gamma_3 \cdot \mathcal{L}_3$

其中：
- $\mathcal{L}_1$：mask-aware 交叉熵（多模态融合分类）
- $\mathcal{L}_2$：各单模态分支的交叉熵（硬标签）
- $\mathcal{L}_3$：KL 散度蒸馏损失（温度缩放后的融合分支 → 单模态分支）
- 权重 $\gamma_1, \gamma_2, \gamma_3$ 由 PGM 在每个 mini-batch 动态计算

## 实验关键数据

### 主实验

**IEMOCAP 数据集**（6类情绪）：

| 方法 | ACC | w-F1 |
|------|-----|------|
| DialogueRNN | 65.43 | 64.29 |
| MM-DFN | 67.84 | 67.85 |
| M3NET | 69.56 | 69.52 |
| GraphSmile | 70.98 | 71.00 |
| SDT | 74.12 | 74.34 |
| **CSS (Ours)** | **75.42** | **75.66** |

相比最强基线 SDT，准确率提升 1.30%，加权 F1 提升 1.32%。

**MELD 数据集**（7类情绪）：

| 方法 | ACC | w-F1 |
|------|-----|------|
| DialogueRNN | 60.27 | 57.95 |
| M3NET | 67.32 | 65.91 |
| MPT-HCL | 65.86 | 65.02 |
| SDT | — | — |
| **CSS (Ours)** | **最优** | **最优** |

### 消融实验

- 移除 SPF → 性能下降，证明高阶融合的必要性
- 移除 PGM（用固定权重替代） → 训练不稳定，性能波动大
- 共享投影替代模态特异投影 → 性能下降，说明保留模态特性的重要性
- 移除符号平方根变换 → 梯度传播不稳定

### 关键发现

- SPF 在低融合复杂度（低秩 $r$）下就能实现高性能，说明高阶交互比网络深度更重要
- PGM 显著提升训练稳定性，减少不同运行之间的性能方差
- happy 和 frustrated 等较难情绪类别的提升最为显著

## 亮点与洞察

- 将 MERC 问题从"如何融合"升级为"表示空间 + 梯度空间"的双空间协同问题，提供了一个更完整的视角
- SPF 的模态特异投影 + 静态门控 + 偏置锚定设计，在保持紧凑的同时实现了非对称、噪声感知的高阶交互
- PGM 利用任务数量固定为 3 的特点，将多目标优化的开销降到极低

## 局限性 / 可改进方向

- 仅在 IEMOCAP 和 MELD 两个数据集上验证，缺乏更多数据集的泛化性验证
- SPF 的交互阶数 $p$ 是超参数，未提供自适应选择机制
- PGM 假设三个固定目标，不易扩展到更多辅助任务
- 各模态的特征提取器（如文本用预训练语言模型）的选择对结果影响未充分讨论

## 相关工作与启发

- SDT（自蒸馏）是最直接的基线：CSS 在其基础上增加了高阶融合和梯度协调
- MGDA（多目标梯度下降）是 PGM 的理论基础，但 CSS 通过固定维度 QP 大幅降低了开销
- PTP（多项式张量池化）是 SPF 的灵感来源，CSS 将其扩展为模态特异的版本
- 启发：多目标优化框架可推广到其他多模态学习任务（如视觉问答、跨模态检索）
