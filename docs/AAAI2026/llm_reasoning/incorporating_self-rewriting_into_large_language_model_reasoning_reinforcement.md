# Incorporating Self-Rewriting into Large Language Model Reasoning Reinforcement

**会议**: AAAI 2026  
**arXiv**: [2511.16331](https://arxiv.org/abs/2511.16331)  
**代码**: 无  
**领域**: LLM推理  
**关键词**: 推理质量改进, 自我重写, GRPO, 推理长度控制, 过度思考  

## 一句话总结
提出Self-Rewriting框架，让LRM在RL训练中对"简单"样本（全部回答正确的query）重写自身推理文本并从中学习，仅增加约10%训练开销即可在保持准确率的同时将推理长度减少46%，内部推理质量（LLM-as-Judge）提升7.2分，有效缓解过度思考、冗余思考等问题。

## 研究背景与动机

1. **领域现状**：基于RL的大型推理模型（如DeepSeek-R1、o1）通过扩展测试时计算在复杂推理任务上取得成功，但训练仅使用结果正确性奖励，缺乏对内部推理过程质量的监督。

2. **现有痛点**：
   - **过度思考(over-thinking)**：模型在无关或琐碎部分大量推理
   - **欠思考(under-thinking)**：跳过或过度简化复杂相关部分
   - **冗余思考(redundant-thinking)**：重复本质相同的想法
   - **混乱思考(disordered-thinking)**：多个思路线索混杂交织
   - 现有长度控制方法只关注"短"而不关注"好"

3. **核心矛盾**：结果正确性奖励是粗粒度的——只要最终答案对就给正奖励，不管推理过程有多冗长混乱。

4. **本文要解决什么？** 在保持RL训练框架（GRPO）的基础上，引入细粒度的推理质量提升机制。

5. **切入角度**：借鉴LLM self-rewarding思路，但不是自己打分（discriminative），而是自己重写（generative）——让模型重写自己的推理文本，然后从重写版本中学习。

6. **核心idea一句话**：让LRM重写自己的推理、从重写中学习（而非只从正确/错误的反馈中学习）。

## 方法详解

### 整体框架
在GRPO训练循环中嵌入选择性重写：对每个query先采样G/2个response → 如果全部正确（"简单"query），则重写推理文本并继续生成答案 → 给重写版本更高奖励 → 如果不全对（"困难"query），正常采样剩余G/2个response → 按标准GRPO优化。

### 关键设计

1. **选择性重写策略 (Selective Rewriting)**:
   - 做什么：仅对模型已完全掌握的"简单"query（所有G/2个response均正确）执行重写
   - 核心思路：当 $\mathbf{r}_{1:G/2} = \mathbf{1}$ 时，重写推理文本 $\mathbf{t}_{G/2+1:G} \leftarrow \mathcal{M}(\mathbf{t}_{1:G/2})$，然后拼接原始query和重写推理继续生成答案。否则正常采样
   - 设计动机：(1)最小化对GRPO的干扰——全对样本在原始GRPO中提供不了梯度信号（advantage≈0），重写将其变为有用样本；(2)在简单问题上练习简洁表达，复杂问题上保留正常探索

2. **重写奖励设计**:
   - 做什么：给重写后仍正确的样本更高奖励
   - 核心思路：$\hat{r}_i = 1$（重写且正确），$\hat{r}_i = 0$（全对query中未重写的原始样本），$\hat{r}_i = r_i$（非全对query保持原始奖励）
   - 设计动机：建立"重写优于原始"的偏好信号，推动模型产生更精炼的推理

3. **高效实现**:
   - 做什么：将重写和正常采样编译在同一batch中，最小化额外开销
   - 核心思路：第一批G/2采样后，将需要正常采样的query和需要重写的query编入同一batch做推理。重写后的继续生成仅产生</think>后的短输出。总体仅增加约10%时间
   - 设计动机：如果重写单独做一轮推理，overhead会很大。batch编译保持了RL算法的可扩展性

### 损失函数 / 训练策略
- 基于GRPO的PPO-clip目标函数，advantage基于重写奖励计算
- 重写prompt极简——仅要求"重写以提高质量"，不显式要求缩短长度
- 对比6个baseline：GRPO, LenPen1/2, ShortBetter, LPO, TOPS

## 实验关键数据

### 主实验

Qwen3-4B在4个推理基准上的表现（Acc/推理长度tokens/LLM-Judge分数）：

| 方法 | Avg Acc | Avg Len | Avg Judge |
|------|---------|---------|-----------|
| Original | 76.1 | 4025 | 67.1 |
| GRPO | 75.8(-0.3) | 3681(-9%) | 68.6(+1.5) |
| ShortBetter | - | -26% | +4.3 |
| TOPS | - | -35% | +3.3 |
| **Rewrite** | 76.2(+0.1) | 2835(**-35%**) | **73.5(+6.4)** |

Qwen3-1.7B类似趋势：Rewrite实现 Acc -0.7, Len -35%, Judge +8.1。

### 消融实验

| 配置 | Acc 变化 | Len 变化 | Judge 变化 | 说明 |
|------|---------|---------|-----------|------|
| Self-Rewrite (full) | +0.6 | -46% | +7.2 | 完整方法 |
| 仅GRPO | +0.0 | -11% | +1.0 | 基线RL |
| LenPen1 (长度惩罚) | -1.6 | -29% | +4.8 | 显式惩罚长度 |
| TOPS (截断) | -2.3 | -35% | +3.3 | 截断长推理 |

### 关键发现
- **不需要显式长度控制**：重写prompt不含缩短要求，但模型自然学会用更少token表达同样推理——长度减少35-46%
- **推理质量全面提升**：四种内部缺陷（over/under/redundant/disordered-thinking）的LLM-Judge分数均显著提升
- **准确率不降反升**：与显式长度惩罚方法（LenPen, TOPS）牺牲准确率不同，Self-Rewrite保持甚至略提升准确率
- **在简单问题上的练习泛化到困难问题**：仅重写全对样本，但学到的简洁推理习惯泛化到所有query

## 亮点与洞察
- **生成式vs判别式自我改进**：self-rewarding是让模型打分（判别），self-rewriting是让模型重写（生成）。生成式方法提供更丰富的改进信号
- **利用GRPO的"废弃"样本**：全对query在GRPO中advantage≈0，是无用样本。Self-rewriting将其转化为有价值的训练信号
- **重写即隐式蒸馏**：重写过程本质上是让模型将冗长的"思维实验"压缩为精炼的推理链

## 局限性 / 可改进方向
- **重写prompt过于简单**：仅"提高质量"的通用指令，针对特定推理缺陷的prompt可能更有效
- **仅验证简单样本的重写**：复杂样本的推理质量可能更需改善，但当前策略回避了它们
- **LLM-as-Judge的可靠性**：推理质量评估依赖LLM打分，这种评估本身的可靠性未充分讨论
- **未测试更大模型**：仅在1.7B和4B上验证，大模型的效果待确认

## 相关工作与启发
- **vs L1/ShortBetter等长度控制方法**: 显式惩罚长度会牺牲准确率；Self-Rewrite通过质量改善间接实现长度控制，更优雅
- **vs Self-Rewarding (Yuan et al.)**: Self-Rewarding让模型自我打分+DPO训练；Self-Rewrite让模型自我重写+GRPO训练，更直接地改善推理过程
- **vs TOPS (截断法)**: TOPS截断长推理再训练短版本，Self-Rewrite让模型自主决定如何改写
