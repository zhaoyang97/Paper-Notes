# From Programs to Poses: Factored Real-World Scene Generation via Learned Program Libraries

**会议**: NeurIPS 2025  
**arXiv**: [2510.10292](https://arxiv.org/abs/2510.10292)  
**代码**: 待确认  
**领域**: 3D 场景生成 / 程序合成  
**关键词**: 3D场景生成, 程序合成, LLM, 布局程序库, 姿态预测, ScanNet

## 一句话总结
提出 FactoredScenes，将真实世界 3D 场景生成分解为五步因式分解——从合成数据学布局程序库、LLM 生成场景程序、执行程序获得轴对齐布局、程序条件化层次姿态预测、物体检索放置，在卧室上 FID 改善 38.3%、KID 改善 80.4%，人类仅 67% 能区分生成与真实 ScanNet。

## 研究背景与动机

1. **领域现状**：3D 场景生成方法（ATISS自回归、DiffuScene扩散、LayoutGPT-LLM布局等）大多生成轴对齐 bounding box，忽视物体朝向，无法反映真实"居住过"的场景。

2. **现有痛点**：（a）真实场景数据稀缺（ScanNet 仅707场景），直接学全分布不可行；（b）现有方法只处理轴对齐布局，但真实场景中椅子朝桌子、显示器朝座位——pose 变化是"居住感"核心；（c）LLM 生成依赖手工领域语言，难灵活泛化。

3. **核心矛盾**：场景既有底层结构（椅子围桌排列）又有 noisy pose 变化（每椅朝向略不同），需同时建模结构和变化但数据极少。

4. **本文要解决什么？** 从有限真实数据中学习生成带有逼真物体朝向的 ScanNet 级3D场景。

5. **切入角度**：分解为"结构"（从大规模合成数据学的程序）和"变化"（从少量真实数据学的pose），不同数据源和方法处理不同子问题。

6. **核心 idea 一句话**：合成数据学结构程序 + LLM 生成布局 + 真实数据学 pose = 因式分解的真实场景生成。

## 方法详解

### 整体框架
五步因式分解：$P(S) = P(objects|pose, program) \cdot P(pose|layout, program) \cdot P(layout=f(program)) \cdot P(program|library) \cdot P(library)$。每步用不同数据源和方法：P(library) 从 3D-Front 学习，P(program|library) 由 LLM 生成，P(layout) 确定性执行，P(pose) 在 ScanNet 训练，P(objects) 最近邻检索。

### 关键设计

1. **程序库学习（Wake-Sleep + LLM）**:
   - 做什么：从 3D-Front 合成数据自动发现房间结构的可复用函数
   - 核心思路：Wake 阶段 LLM 作识别模型（布局->程序），Sleep 阶段 LLM 提出新抽象函数。从仅 object_instantiation 和 parallel 两个初始函数，学到 align、grid、grid_with_offset、symmetrical、cluster_placement 等函数
   - 设计动机：不需手工设计领域语言；LLM 自动提出适当抽象层次的函数；3D-Front 函数直接迁移到 ScanNet（证明人类设计房间有跨数据集通用结构）

2. **程序条件化层次姿态预测**:
   - 做什么：预测每个物体的朝向角度 $\theta$（离散化为36类，5度/bin）
   - 核心思路：分两阶段——主物体（如桌子）从bounding box、最近墙壁等编码预测 $\theta$；从属物体（如围桌椅子）额外条件于依赖目标的已预测朝向和程序函数文本嵌入。Multi-head self-attention 处理所有物体
   - 设计动机：真实场景中"椅子朝向跟着桌子转"是层次依赖。去掉依赖条件后从属物体准确率从 44.2% 降至 39.7%

3. **LLM 程序生成 + 库正则化**:
   - 做什么：利用 LLM（GPT-o1）生成多样化场景程序
   - 核心思路：先将 ScanNet 解析为 few-shot 程序示例，LLM 在库函数约束下生成新程序（可执行 Python）
   - 设计动机：LLM commonsense 决定物体位置，library 约束结构合理性

### 损失函数 / 训练策略
- 朝向：交叉熵（36类分类）
- 上采样困难场景（高比例难预测朝向的场景）
- 推理：$\hat{\theta} = 5 \cdot \arg\max_k \hat{y}_{ij,k}$

## 实验关键数据

### 主实验

| 方法 | 卧室FID | 客厅FID | 卧室KID | 客厅KID |
|------|---------|---------|---------|---------|
| DiffuScene | 135.57 | 186.54 | 0.130 | 0.177 |
| ATISS | 120.27 | 176.95 | 0.117 | 0.166 |
| LayoutGPT | 109.40 | 157.69 | 0.102 | 0.142 |
| Ours w/o poses | 101.55 | 137.66 | 0.085 | 0.106 |
| Ours w/ random poses | 110.03 | 123.64 | 0.086 | 0.079 |
| **FactoredScenes** | **67.51** | **83.49** | **0.020** | **0.024** |

### 消融实验

| 配置 | 卧室FID | 说明 |
|------|---------|------|
| w/o poses（轴对齐） | 101.55 | 仅程序结构已超前SOTA |
| w/ random poses | 110.03 | 随机朝向反而更差 |
| **完整模型** | **67.51** | pose预测是关键 |

| 库类型 | ScanNet函数使用 | ScanNet mIoU |
|--------|-----------------|-------------|
| 推理式(inference-only) | 0.34 | 0.96 |
| **学习式(learned)** | **2.53 (+644.1%)** | **0.98** |

| Pose配置 | 主物体准确率 | 从属物体准确率 |
|---------|------------|--------------|
| w/o 依赖条件 | 0.537 | 0.397 |
| **完整** | **0.542** | **0.442 (+11.4%)** |

### 人类评估
400判断（20对x20参与者）：选择 ScanNet 真实场景的准确率仅 **0.67**——33% 时间无法区分。

### 关键发现
- **仅程序结构已超越所有前SOTA**：w/o poses 的 FID 101.55 < LayoutGPT 109.40
- **随机pose比无pose更差**：说明朝向不是简单噪声——错误朝向导致物体重叠和不合逻辑
- **层次依赖关键**：从属物体需知道主物体朝向（+11.4%）
- **跨数据集迁移成功**：3D-Front 库函数直接用于 ScanNet

## 亮点与洞察
- **因式分解是核心贡献**：将数据稀缺难题拆为五个各有数据/方法优势的子问题——每模块在最适条件下工作
- **Wake-Sleep + LLM 做库学习**很优雅：DreamCoder 范式的 LLM 升级版
- **"居住感"的定量刻画**：通过 pose 预测将"场景看起来自然"转化为可学习目标

## 局限性 / 可改进方向
- LLM 生成偶有不自然放置（如床头柜在床中间）
- ScanNet 部分物体不完整导致 GT 朝向标注有误
- 目前 pose 是确定性预测，未来可扩展为分布预测
- 仅处理卧室和客厅
- 未考虑物理碰撞检测

## 相关工作与启发
- **vs LayoutGPT**：只输出轴对齐 box。FactoredScenes 通过程序库约束 + pose 模型，FID 降低 38.3%
- **vs Holodeck**：依赖预定义领域模块。FactoredScenes 通过 library learning 自动发现函数
- **vs DreamCoder**：扩展到 3D 场景域，用 LLM 替代神经识别模型
