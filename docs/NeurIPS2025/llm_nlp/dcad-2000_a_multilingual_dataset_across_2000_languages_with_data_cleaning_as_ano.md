# DCAD-2000: A Multilingual Dataset across 2000+ Languages with Data Cleaning as Anomaly Detection

**会议**: NeurIPS 2025  
**arXiv**: [2502.11546](https://arxiv.org/abs/2502.11546)  
**代码**: https://github.com/yl-shen/DCAD-2000  
**领域**: LLM/NLP  
**关键词**: 多语言数据集, 数据清洗, 异常检测, Common Crawl, 低资源语言

## 一句话总结
构建覆盖2282种语言、46.72TB文本的多语言数据集DCAD-2000，提出将数据清洗重构为异常检测问题的语言无关框架，通过8维统计特征+Isolation Forest动态过滤噪声数据，在多个多语言benchmark上验证效果，尤其对低资源语言提升显著。

## 研究背景与动机

1. **领域现状**：多语言LLM需要大规模高质量多语言数据集。现有数据集（CulturaX, Madlad-400, Fineweb-2等）覆盖语言数持续增加，但存在三个重要限制。
2. **现有痛点**：(a) **数据陈旧**：多数数据集基于2022年前的Common Crawl快照，知识过时；(b) **高/中资源语言覆盖不足**：Fineweb-2虽支持1915种语言，但仅10种高资源+62种中资源语言；(c) **清洗不充分**：Sailor研究发现Madlad-400仍有31%数据可被更先进清洗方法移除。传统基于固定阈值的启发式清洗方法在多语言场景下泛化性差——不同语言的词数、重复率、困惑度分布差异巨大。
3. **核心矛盾**：手动为每种语言调阈值不可扩展（Fineweb-2为1000+语言微调阈值计算量巨大），但统一阈值跨语言不work。需要一种语言无关的自适应清洗方法。
4. **本文要解决什么？** (a) 构建最新、最全、直接可用于训练的多语言数据集；(b) 提出不需要手动调阈值的数据清洗框架。
5. **切入角度**：将数据清洗重构为异常检测——低质量文档在特征空间中是"异常值"，可用Isolation Forest等算法自动识别。
6. **核心idea一句话**：用异常检测替代固定阈值做数据清洗——对每种语言提取8维统计特征，让算法自动学习"什么是该语言的正常文档"，偏离的即为噪声。

## 方法详解

### 整体框架

三阶段流程：
1. **数据收集**：整合MaLA(939语言) + Fineweb(英语) + Fineweb-2(1915语言) + 新抽取的Common Crawl(2024.5-2024.11)
2. **特征提取**：对每个文档提取8维统计特征向量
3. **异常检测过滤**：标准化特征→Isolation Forest打分→按阈值分为clean/anomalous

### 关键设计

1. **数据收集策略**:
   - 做什么：整合四个互补来源，覆盖最新数据
   - MaLA提供非Common Crawl来源的多样性（尤其低资源语言）；Fineweb提供高质量英语；Fineweb-2提供多语言广覆盖；新抽CC（2024.5-11月，21.54TB）确保数据新鲜度
   - 最终：2282种语言，8.63B文档，46.72TB，155种高/中资源语言，159种书写系统

2. **8维文档特征**:
   - 词数 $n_w(t)$：文档长度
   - 字符重复率 $r_c(t)$：检测编码错误/垃圾内容
   - 词重复率 $r_w(t)$：检测模板化/循环内容
   - 特殊字符比例 $r_s(t)$：使用语言特定符号列表
   - 停用词比例 $r_{stop}(t)$：Fineweb-2多语言停用词
   - 标记词比例 $r_{flag}(t)$：有毒/脏话词汇
   - 语言识别置信度 $s_{lid}(t)$：GlotLID（支持2000+语言）
   - 困惑度 $s_{ppl}(t)$：KenLM（每种语言一个模型，在Wiki上训练）
   - 设计动机：这些特征跨语言可计算，且能捕捉不同类型的质量问题

3. **异常检测清洗**:
   - 做什么：用Isolation Forest自动识别低质量文档
   - 标准化特征 $\tilde{x}_j = (x_j - \mu_j) / \sigma_j$，消除量纲差异
   - Isolation Forest异常分数：$\phi(\tilde{\mathbf{x}}) = 2^{-h(\tilde{\mathbf{x}})/c(n)}$，h为平均隔离路径长度
   - 使用scikit-learn默认参数，全局适用（不按语言/特征单独调参）
   - 设计动机：异常检测天然适应不同语言的数据分布——每种语言的"正常"分布不同，算法自动适配

### 训练策略
- 清洗后数据用于continue pretraining
- 评估模型：LLaMA-3.2-1B, Qwen-2.5-7B, Aya-expanse-32B
- Benchmark：FineTask（9语言）, SIB-200, Glot500-c, FLORES-200
- 计算资源：32核+128GB服务器，Kubernetes编排最多100并行任务

## 实验关键数据

### 主实验：清洗方法对比（FineTask benchmark）

| 清洗方法 | 相对无清洗基线提升 | 相对阈值法提升 |
|----------|-------------------|---------------|
| 无清洗（原始数据） | 基线 | - |
| 固定阈值法 | +5~10% | 基线 |
| **DCAD异常检测** | **+5~20%** | **+3~10%** |

### 消融：不同异常检测算法

| 算法 | 表现 | 说明 |
|------|------|------|
| Isolation Forest | 最佳 | 对高维稀疏数据友好 |
| LOF | 次之 | 局部密度估计 |
| One-Class SVM | 一般 | 对参数敏感 |

### 关键发现
- **即使已清洗的数据集仍有大量噪声**：对MaLA、Fineweb、Fineweb-2数据再清洗，仍移除约7.69%文档，验证了已有清洗不充分
- **异常检测比阈值法显著更好**：阈值法需要为每种语言/特征手动调参，异常检测自动适配
- **低资源语言受益最大**：SIB-200和Glot500-c上低资源语言提升尤为显著
- **特征可视化**验证了清洗效果：异常/正常文档在8维特征空间中有清晰分离（尤其LID score和PPL score维度）
- 地理覆盖广：非洲(28.6%)、巴布亚-新几内亚(26.3%)、欧亚(23.8%)

## 亮点与洞察
- **"数据清洗=异常检测"的重构**是核心insight：简单但有效——不需要定义"干净数据长什么样"，只需要定义"什么是异常"。异常检测算法天然处理多模态分布，完美适配多语言场景
- **规模惊人**：2282种语言、46.72TB、155种高/中资源语言，是目前最全面且可直接训练的多语言数据集
- **工程贡献扎实**：整个pipeline开源，从数据收集到清洗到评估都有详细documentation

## 局限性 / 可改进方向
- Isolation Forest使用默认参数，可能对某些语言不是最优——可探索per-language或per-script调参
- 困惑度特征依赖Wiki数据——低资源语言Wiki数据不足时使用默认值500，可能影响清洗质量
- 8维特征是手工选择的，未探索更丰富的特征（如语义一致性、主题相关性）
- 清洗去除了7.69%文档，但没有分析false positive/false negative率——可能误删高质量数据或遗漏低质量数据
- 书写系统以拉丁字母为主(79.4%)，对CJK等复杂书写系统的特征适用性需进一步验证

## 相关工作与启发
- **vs Fineweb-2**: Fineweb-2覆盖1915语言但仅10种高资源；Fineweb-2对1000+语言微调阈值计算量大。DCAD-2000用异常检测替代阈值调参，更可扩展
- **vs CulturaX/Madlad-400**: 基于更旧的CC快照，且清洗不充分。DCAD-2000使用最新CC(2024-46)且清洗更彻底
- **vs MaLA**: MaLA覆盖939语言但只有1种高资源语言。DCAD-2000整合MaLA作为子集并扩展覆盖

- 实验充分度: ⭐⭐⭐⭐⭐ 三个LLM、四个benchmark、多种清洗策略和异常检测算法的全面对比
- 写作质量: ⭐⭐⭐⭐ 结构清晰，数据详实，但公式推导较基础
