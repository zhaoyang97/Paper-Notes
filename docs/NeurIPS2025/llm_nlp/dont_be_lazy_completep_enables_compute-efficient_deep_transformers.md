# Don't Be Lazy: CompleteP Enables Compute-Efficient Deep Transformers
**会议**: NeurIPS 2025
**arXiv**: [2505.01618](https://arxiv.org/abs/2505.01618)  
**代码**: [https://github.com/EleutherAI/nanoGPT-mup/tree/completep](https://github.com/EleutherAI/nanoGPT-mup/tree/completep)
**领域**: LLM 扩展律, 超参优化
**关键词**: 参数化, 深度缩放, 超参转移, 计算效率

## 一句话总结
CompleteP参数化通过α=1确保深度超参转移和完全特征学习，相比μP在深模型上实现12-34%FLOP节省。

## 研究背景与动机
微调深度时超参需重复调优，导致计算浪费。μP虽解决宽度缩放，但深度缩放下超参非稳定迁移。

## 方法详解

### 整体框架
扩展μP框架，针对深度缩放引入深度感知的超参重缩放规则，关键是残差块输出缩放因子α∈[0.5,1]的选择。

### 关键设计
- **残差缩放**: $h^{l+1} = h^l + L^{-α}F_l(h^l)$，其中α决定深度影响
- **LayerNorm学习率**: η_base·m_L^(α-1)
- **完全特征学习**: 要求所有层保持非惰性(non-lazy)学习，仅α=1满足

## 实验关键数据

### 主实验
| 模型大小 | 深度范围 | μP HP转移 | CompleteP HP转移 | FLOP节省 |
|---------|--------|---------|-----------------|---------|
| 75M-1.9B | L=2-128 | 否 | 是 | 11.8% (最优) |
| 1.5B | L=179 | 否 | 是 | 34.4% (深模) |
| 50M优化 | L=16-63 | - | 是 | 0-30% 范围内 |

### 下游任务评估
| 任务 | μP(最优) | μP(深模) | CompleteP(最优) | CompleteP(深模) |
|------|---------|---------|----------------|----------------|
| HellaSwag | 53.3% | 49.1% | 54.2% | 52.7% |
| LAMBADA | 54.3% | 51.8% | 54.9% | 53.3% |
| 平均(6任务) | 54.3% | 52.0% | 55.2% | 54.3% |

## 亮点与洞察
1. **HP转移成功**: CompleteP首次实现完整的深度HP转移（2-128层）
2. **宽深比灵活性**: 支持N:L从11.8到接近100的广泛范围保持接近最优
3. **完全特征学习**: 理论上证明只有α=1保证所有层的完全非惰性学习
4. **扩展参数化**: 补充的LayerNorm LR、bias LR、AdamW ε缩放规则至关重要

## 局限性
- 实验仅限7×Chinchilla最优token预算
- 未跨不同数据集和架构验证
- 下游任务信号较弱（小模型短训练）

## 相关工作
μP奠定基础，本工作扩展到深度维度。Yang et al.认为α≠0.5不可行，本文反驳并证明α=1优越性。
