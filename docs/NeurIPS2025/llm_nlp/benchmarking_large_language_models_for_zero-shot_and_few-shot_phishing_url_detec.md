# Benchmarking Large Language Models for Zero-Shot and Few-Shot Phishing URL Detection

**会议**: NeurIPS 2025  
**arXiv**: [2602.02641](https://arxiv.org/abs/2602.02641)  
**代码**: 有  
**领域**: 网络安全 / LLM评估  
**关键词**: 钓鱼URL检测, LLM, 零样本, 少样本, 网络安全benchmark

## 一句话总结
系统评估多个 LLM 在零样本和少样本钓鱼 URL 检测任务上的表现，发现 LLM 在零样本下表现参差不齐但少样本 ICL 显著提升，专用安全微调模型仍有优势，为 LLM 在网络安全中的应用提供了基准参考。

## 研究背景与动机

1. **领域现状**：钓鱼 URL 检测传统上用黑名单和 ML 分类器，但新型钓鱼 URL 层出不穷需要泛化能力。
2. **现有痛点**：传统方法对未见过的钓鱼模式泛化差；LLM 的零样本/少样本能力在安全领域研究不足。
3. **切入角度**：系统 benchmark 多个 LLM 在不同 prompt 策略下的检测能力。

## 方法详解

### 评估设置
1. **模型**：GPT-4、Claude、开源 LLM 等
2. **策略**：零样本、少样本（1/5/10-shot）、不同 prompt 模板
3. **指标**：准确率、精度、召回率、F1

## 实验关键数据

| 设置 | 最佳 LLM F1 | 专用 ML 模型 F1 |
|------|-----------|---------------|
| 零样本 | ~80% | ~95% |
| 10-shot | ~88% | ~95% |

### 关键发现
- 零样本差距大但少样本快速缩小
- LLM 在解释性方面有优势（能解释为什么认为是钓鱼）
- 不同 LLM 对 URL 特征的敏感度不同

## 亮点与洞察
- 为安全领域的 LLM 应用提供了系统性 baseline

## 局限性 / 可改进方向
- 钓鱼 URL 快速进化，benchmark 需要持续更新
