# Provable Scaling Laws for the Test-Time Compute of Large Language Models

**会议**: NeurIPS 2025  
**arXiv**: [2411.19477](https://arxiv.org/abs/2411.19477)  
**代码**: 无  
**领域**: LLM推理  
**关键词**: 测试时缩放律, 淘汰赛, 联赛, Best-of-N, 可证明保证  

## 一句话总结
提出两种具有可证明缩放律的测试时计算算法——Knockout（淘汰赛式：生成多个候选再两两比较淘汰）和 League（联赛式：用平均胜率选最优候选），证明在 LLM 生成正确解概率 >0 且比较能力优于随机的极弱假设下，失败概率随测试时计算增加呈指数或幂律衰减，且仅需黑盒 LLM 无需额外验证器。

## 研究背景与动机

1. **领域现状**：测试时计算（Best-of-N、多数投票、树搜索）已成为提升 LLM 推理的主流手段，但理论上这些方法为何有效、缩放速率如何，缺乏严格分析。
2. **现有痛点**：(1) 大多数测试时方法需要外部验证器或奖励模型，增加系统复杂性；(2) 经验缩放律缺乏理论保证，不同任务表现不一致；(3) Best-of-N 需要可靠的打分机制。
3. **核心矛盾**：需要理论上可证明的、实践中可操作的测试时缩放方法，且最好不依赖外部组件。
4. **本文要解决什么？** 能否给出两个条件极弱的假设下，具有**可证明**缩放律的测试时计算算法？
5. **切入角度**：利用 LLM 自身的"比较"能力——给它两个候选答案让它判断哪个更好——这比生成正确答案容易得多。通过锦标赛/联赛结构聚合比较结果。
6. **核心idea一句话**：生成 $N$ 个候选 + 用 LLM 自身两两比较筛选 = 无需外部验证器的可证明测试时缩放。

## 方法详解

### 整体框架
两阶段方法：(1) **生成阶段**：让 LLM 对同一问题生成 $N$ 个独立候选解；(2) **聚合阶段**：通过 Knockout（淘汰赛）或 League（联赛）方式用 LLM 自身比较候选对，选出最终答案。

### 关键设计

1. **Knockout 算法（淘汰赛）**:
   - 做什么：像淘汰赛一样两两比较，输的被淘汰，$\log N$ 轮后剩一个
   - 核心思路：假设 (A1) LLM 能以 $p > 0$ 概率生成正确解，(A2) 给定一对正确和错误的解，LLM 以 $> 1/2$ 概率选对。则失败概率 $P_{\text{fail}} \leq \exp(-\Omega(N))$（指数衰减）或 $O(N^{-\alpha})$（幂律衰减）
   - 设计动机：仅需黑盒 LLM 的两个极弱能力假设，无需外部验证器

2. **League 算法（联赛）**:
   - 做什么：每个候选与多个"对手"比较，用平均胜率排名选最优
   - 核心思路：不像淘汰赛中输一场就出局，联赛中每个候选有多次机会。更鲁棒假设下，同样证明指数级缩放律
   - 设计动机：淘汰赛中一次比较错误就可能淘汰正确答案，联赛通过多次评估降低这种风险

### 损失函数 / 训练策略
纯推理时方法，**无训练**。仅需黑盒 LLM 的生成和比较能力。

## 实验关键数据

### 主实验

在多个模型（GPT 系列、Llama、Qwen）和数据集上验证：
- 失败概率确实随候选数 $N$ 增加呈指数/幂律衰减
- Knockout 和 League 都优于标准 Best-of-N + 多数投票
- League 在比较噪声较大时比 Knockout 更鲁棒
- 无需任何外部验证器，仅用 LLM 自身比较即可

### 关键发现
- **极弱假设即可保证缩放**：只要 LLM 能偶尔生成正确解 ($p > 0$) 且比较 > 随机 ($> 0.5$)
- **指数级衰减**：在适当条件下，生成 $N$ 个候选的失败率以 $\exp(-\Omega(N))$ 衰减
- **黑盒 LLM 足矣**：不需要 reward model、verifier 或任何额外模型

## 亮点与洞察
- **首次给出测试时计算的形式化可证明缩放律**：不是经验拟合，而是理论证明。假设条件极弱，几乎所有有用的 LLM 都满足
- **"比较比生成容易" 假设具有深刻直觉**：判断两个解谁更好，远比独立生成正确解容易——这在计算复杂性理论中也有对应（验证 vs 搜索）
- **极简且实用**：只需 LLM 本身，无需训练/微调，即插即用

## 局限性 / 可改进方向
- 比较能力假设在实际中可能不总成立（LLM 比较也可能犯系统性错误）
- 计算开销较高：$N$ 个候选 + $O(N \log N)$ 次比较调用
- 理论分析假设比较判断相互独立，实际中可能存在相关性

## 相关工作与启发
- **vs Best-of-N**: Best-of-N 需要外部 reward model，Knockout/League 用 LLM 自身比较替代
- **vs 多数投票**: 多数投票仅利用答案频率，Knockout/League 利用 LLM 的比较推理能力
- **vs s1/DISC**: 这些方法关注单条 CoT 的优化，本文关注多候选的聚合策略
