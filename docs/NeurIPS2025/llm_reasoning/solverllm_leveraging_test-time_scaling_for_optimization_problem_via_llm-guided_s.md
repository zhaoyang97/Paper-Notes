# SolverLLM: Leveraging Test-Time Scaling for Optimization Problem via LLM-Guided Search

**会议**: NeurIPS 2025  
**arXiv**: [2510.16916](https://arxiv.org/abs/2510.16916)  
**代码**: 无  
**领域**: LLM推理  
**关键词**: 测试时缩放, 优化问题, MCTS, 动态扩展, 无训练

## 一句话总结
无需训练，通过 MCTS 引导 LLM 生成 6 元素优化表述并转化为求解器代码，在 NL4Opt 上达 97.0%（vs OptiMUS 78.8%），超越微调方法且跨域泛化强。

## 研究背景与动机

1. **领域现状**：优化问题求解需三步（表述→代码→求解），表述最需专业知识。
2. **现有痛点**：提示方法对设计敏感、泛化弱；学习方法需大量标注且难跨域。
3. **核心矛盾**：通用性、高效性和不依赖训练三者难兼得。
4. **本文要解决什么**：用测试时计算换通用性，无需训练即超越微调方法。
5. **切入角度**：测试时缩放在推理任务上展现巨大潜力，迁移到优化问题。
6. **核心 idea**：MCTS 探索 6 元素表述空间 + 双重反向传播 = 无训练跨域优化求解。

## 方法详解

### 整体框架
MCTS 探索 6 元素表述空间（Type/Sets/Parameters/Variables/Objective/Constraints），每个完整路径对应一个优化表述，最后转化为代码求解。

### 关键设计

1. **动态扩展**：可在非叶子节点扩展（vs 传统 MCTS 仅叶子），支持变量约束的递归依赖修正
2. **提示反向传播**：将评估阶段推理信号回传更新节点状态和知识库，含触发器和修正指导
3. **不确定性反向传播**：估计 LLM 评分方差，以衰减因子加权反向传播，早期剪枝低质节点

## 实验关键数据

### 主实验

| 方法 | NL4Opt | NLP4LP | ComplexOR |
|------|--------|--------|----------|
| OptiMUS | 78.8% | 72.0% | 66.7% |
| LLMOPT (微调) | - | - | 68.0% |
| **SolverLLM** | **97.0%** | **87.0%** | **77.8%** |

### 消融

| 配置 | MamoComplex |
|------|-------------|
| Full | 76.0% |
| w/o Prompt Backprop | 66-69% |

### 关键发现
- 无训练超越微调方法 10+%，体现测试时缩放威力
- 6 数据集一致性表现，跨域泛化强

## 亮点与洞察
- 6 元素 schema 比 5 元素多 Type 提供全局指导
- 双重反向传播（符号+统计）实现推理-奖励互反馈
- 完全无训练即超越微调

## 局限性
- 未开源代码，平均生成 3.85 分钟
- 依赖 GPT-4o 级模型，开源 LLM 适配未验证
