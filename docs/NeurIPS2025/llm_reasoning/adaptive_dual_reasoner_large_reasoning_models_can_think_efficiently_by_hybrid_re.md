# Adaptive Dual Reasoner: Large Reasoning Models Can Think Efficiently by Hybrid Reasoning

**会议**: NeurIPS 2025  
**arXiv**: [2510.10207](https://arxiv.org/abs/2510.10207)  
**代码**: 无（腾讯优图实验室）  
**领域**: LLM推理 / LLM效率  
**关键词**: 推理效率, 混合推理, 快思慢想, 熵引导, 强化学习  

## 一句话总结
提出 Adaptive Dual Reasoner (ADR)——让推理模型在 fast thinking（简单推理步骤压缩）和 slow thinking（复杂推理步骤保留深度）之间动态切换，通过 SFT 冷启动 + EHPO（熵引导混合策略优化）训练，在数学推理基准上准确率提升最高 6.1% 同时推理 token 减少 49.5%-59.3%。

## 研究背景与动机

1. **领域现状**：大型推理模型 (LRM) 如 DeepSeek-R1、Qwen3 通过长 Chain-of-Thought 推理取得显著效果，但存在严重的"过度思考"问题——对简单子问题也生成冗长的推理步骤。
2. **现有痛点**：(a) 长度驱动的压缩方法可能不足以探索复杂步骤；(b) 粗粒度的快/慢模式切换无法适应推理轨迹内子问题复杂度的变化；(c) 静态 rollout 策略限制了难题的深入探索。
3. **核心矛盾**：减少推理长度（提效）vs 保持推理深度（保准确率）——需要根据每个推理步骤的复杂度自适应分配"认知资源"。
4. **切入角度**：将推理轨迹分解为推理单元 (reasoning unit)，按熵值标注为 easy/hard，easy 单元压缩，hard 单元保留。
5. **核心 idea 一句话**：熵引导的动态分支——在从 easy 切换到 hard 模式时展开多个探索分支，增加推理广度弥补深度的减少。

## 方法详解

### 整体框架
两阶段训练：(1) SFT 冷启动——用混合推理数据教模型区分 easy/hard 推理单元；(2) EHPO 强化学习——优化推理分配，使用熵引导动态 rollout + 难度感知奖励。

### 关键设计

1. **混合推理数据构建**:
   - 做什么：自动将长推理轨迹改造为 easy/hard 混合格式
   - 核心思路：(a) 将 CoT 分解为推理单元；(b) 按 token 熵值标注——含反思/验证关键词（"Wait", "However", "Alternatively"）的高熵单元标为 hard，其余标为 easy；(c) easy 单元用 CoD-style 压缩，hard 单元保留原始深度
   - 输出格式：`<think> <easy> u1 </easy> <hard> u2 </hard> ... </think> a`
   - 构建了 300K 训练样本（来自 OpenMathReasoning）

2. **EHPO 奖励设计**:
   - 做什么：联合优化推理准确性和效率
   - 总奖励：$R = \mathcal{R}_{format} \times \mathcal{R}_{accuracy} \times \mathcal{R}_{unit} \times \mathcal{R}_{mode}$
   - **单元语义奖励** $\mathcal{R}_{unit}$：确保 easy 单元无反思关键词、hard 单元含反思关键词，维持两种模式的语义一致性
   - **模式控制奖励** $\mathcal{R}_{mode}$：难度感知——简单题鼓励多用 easy 模式，难题允许多用 hard 模式
     - $\mathcal{R}_{mode} = \beta + (1-\beta)(N_{pass}/N \cdot p_{easy} + (1-N_{pass}/N) \cdot p_{hard})$
     - 通过 $N_{pass}/N$（正确样本比例）估计问题难度

3. **熵引导动态 Rollout (EDR)**:
   - 做什么：在关键转换点（easy → hard）展开多个推理分支
   - 核心观察：easy 单元的末端熵通常比初始熵高（越推越不确定），hard 单元反之（越推越确定）→ easy→hard 转换处是高不确定性点
   - 核心思路：在 easy→hard 转换时，以概率 $SP = \alpha + \Delta H$ 生成分支（$\alpha=0.5$ 为基础概率，$\Delta H$ 为归一化熵差）
   - 设计动机：mode control reward 压缩了探索空间，EDR 通过增加广度（多分支）弥补深度的减少
   - 效果：EDR 将平均 AES 从 0.51 提升到 0.70

### 训练策略
- 基座模型：DeepSeek-R1-Distill-Qwen-1.5B
- 冷启动 SFT：300K OpenMathReasoning 样本
- EHPO RL：DeepScaleR-Preview 数据集，两阶段（8K/16K max tokens）
- EDR 仅在 16K 阶段启用
- 评估：pass@1（16 次采样平均）

## 实验关键数据

### 主实验

| 方法 | AIME2024 Acc. | AIME2024 Tokens | MATH500 Acc. | MATH500 Tokens | Avg AES |
|------|-------------|-----------------|-------------|----------------|---------|
| Baseline (R1-1.5B) | 30.4% | 12290 | 81.7% | 4802 | - |
| O1-Pruner | - | - | 84.3% | 2913↓39% | 0.35 |
| DRP | 33.3% | 6135↓50% | 82.0% | 2122↓56% | 0.68 |
| ACPO | 30.0% | 6670↓46% | 81.0% | 1679↓65% | 0.50 |
| **ADR** | **36.5%** | **6110↓50%** | 81.0% | **1955↓59%** | **0.70** |

### 消融实验

| 配置 | AIME2024 Acc. | Avg AES | 说明 |
|------|-------------|---------|------|
| ADR w/o EDR | 33.8% | 0.51 | RL 但无动态 rollout |
| **ADR (full)** | **36.5%** | **0.70** | 熵引导动态 rollout |

### 关键发现
- **AIME2024 提升 6.1%**：在最难的竞赛数学上，ADR 比基线提升最大，同时 token 减少 50%
- **EDR 是关键**：无 EDR 时 RL 训练效果有限（+3.4% AIME2024），加 EDR 后跃升到 +6.1%
- **推理 token 减少 50-60%**：通过将简单推理步骤压缩为 easy 模式
- **难度自适应**：简单题大部分用 easy 模式，难题自动增加 hard 比例

## 亮点与洞察
- **推理单元级别的粒度控制**：不是在问题级别决定快/慢，而是在推理过程中的每个单元动态切换——更精细的"认知资源分配"
- **熵作为不确定性信号**：用模型自身的 token 生成熵来检测何时需要"深入思考"，这是一个轻量且自然的信号，不需要外部标注
- **广度换深度**：当减少推理深度（压缩 easy 单元）可能损害性能时，通过在关键转换点增加分支数量来弥补——这是一个有趣的计算效率 trade-off
- **自动数据构建**：用熵 + 关键词匹配自动将现有推理数据转化为混合格式，可扩展到任何推理数据集

## 局限性 / 可改进方向
- **仅在 1.5B 模型上验证**：更大模型（7B, 32B）的效果未知
- **仅数学推理**：代码推理、逻辑推理等场景的适用性未验证
- **关键词匹配的局限**：用固定关键词（"Wait", "However"）标注 hard 单元可能不够精确
- **改进方向**：(1) 扩展到更大模型和更多推理任务；(2) 用学习的分类器替代关键词匹配；(3) 让 easy/hard 的边界也可学习而非预定义

## 相关工作与启发
- **vs O1-Pruner**：O1-Pruner 从完整推理中预采样+修剪，ADR 在生成时就区分快慢模式，更主动
- **vs ACPO**：ACPO 也做模式切换但用标准 GRPO，ADR 的 EHPO 加入了熵引导动态 rollout，效果更好（AES 0.70 vs 0.50）
- **vs DRP**：DRP 用蒸馏+修剪，ADR 用 RL + 自适应——ADR 在 AIME2024 上准确率更高
- **vs System 1/System 2 框架**：ADR 的 fast/slow 对应 Kahneman 的 System 1/System 2，但实现在推理单元级别而非问题级别
