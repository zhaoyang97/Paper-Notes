# Bits Leaked per Query: Information-Theoretic Bounds on Adversarial Attacks Against LLMs

**会议**: NeurIPS 2025  
**arXiv**: [2510.17000](https://arxiv.org/abs/2510.17000)  
**代码**: 待确认  
**领域**: AI安全 / 信息论  
**关键词**: 信息泄漏, 对抗攻击, LLM安全, 查询复杂度下界, 信息通道模型

## 一句话总结
将 LLM 对抗攻击建模为信息通道问题——定义每次查询的"泄漏比特数" $I(Z;T)$ 为攻击目标属性 $T$ 与可观测信号 $Z$ 的互信息，证明攻击达到误差 $\varepsilon$ 所需最少查询数为 $\log(1/\varepsilon)/I(Z;T)$，在 7 个 LLM 上验证：暴露 answer tokens 需 ~1000 次查询，加 logits 降到 ~100 次，加思维链降到 ~几十次，为透明性-安全性权衡提供首个原则性标尺。

## 研究背景与动机

1. **领域现状**：LLM 面临多种对抗攻击——系统提示泄漏、越狱、遗忘后重学攻击。为了透明性，很多 LLM 服务暴露思维链（CoT）或 token 概率（logits），但这些信号可能被攻击者利用。

2. **现有痛点**：
   - 攻击评估纯经验性——绘制"成功率 vs 查询次数"曲线，缺乏理论最优参照
   - 防御者无法量化"暴露 CoT 比只暴露 answer tokens 增加了多少安全风险"
   - 攻击者无法判断自己的方法离理论极限还有多远

3. **核心矛盾**：透明性（暴露更多信号提升可解释性和用户体验）与安全性（暴露更多信号加速攻击）之间存在根本性 trade-off，但缺乏量化这一 trade-off 的理论框架。

4. **本文要解决什么？** 给出"最强攻击者在给定信息泄漏率下最多能做到什么"的理论下界，让防御者能原则性地控制透明性水平。

5. **切入角度**：将 LLM 查询视为信息通道（输入=攻击提示，输出=可观测信号，隐变量=攻击目标属性），用互信息量化每次查询的泄漏量，推导查询复杂度的信息论下界。

6. **核心 idea 一句话**：LLM 安全 = 信息通道问题，查询复杂度 $N \geq \log(1/\varepsilon) / I(Z;T)$。

## 方法详解

### 整体框架
形式化：攻击者发出 prompt $X_n$（可自适应依赖前 $n-1$ 次回复），LLM 返回可观测信号 $Z_n$（answer tokens / logits / CoT），攻击者试图推断目标属性 $T$（如"是否越狱成功"的二值标志）。每次查询的泄漏量 $I_{max} = \sup_x I(Z; T|X=x)$。

### 关键设计

1. **信息论下界 (Theorem 1)**:
   - 做什么：证明攻击所需最少查询数的不可逾越下界
   - 核心思路：利用信息处理不等式（data processing inequality），$N$ 次自适应查询后攻击者获得的总信息 $I(Z_{1:N}; T) \leq N \cdot I_{max}$。要将 $T$ 的不确定性从 $H(T)$ 降到 $\varepsilon$，需要 $N \geq \log(1/\varepsilon) / I_{max}$
   - 设计动机：这个下界对**任何**攻击策略（包括自适应策略）都成立，包括尚未发明的攻击方法

2. **相变现象**:
   - 做什么：揭示信息泄漏从零到非零时查询复杂度的急剧跃迁
   - 核心思路：当 $I(Z;T) \approx 0$ 时（几乎不泄漏），查询需求为 $O(1/\varepsilon)$（线性/二次增长）；一旦泄漏量达到固定常数比特（如暴露 logits），查询需求骤降为 $O(\log(1/\varepsilon))$——这是平方到对数的相变
   - 设计动机：解释了为什么"稍微多暴露一点信息"就会导致攻击难度断崖式下降

3. **四种信号制度的实验对比**:
   - 做什么：量化不同透明性水平下的实际泄漏量和攻击效率
   - 核心思路：四种制度——(a) 仅 answer tokens；(b) tokens + logits；(c) tokens + 思维链；(d) tokens + 思维链 + logits。测量每种制度下的 $I(Z;T)$和实际攻击所需查询数 $N$
   - 设计动机：为 LLM 部署者提供直接可操作的指导——"如果你暴露 X，攻击成本会降到 Y"

### 损失函数 / 训练策略
- 无需训练——纯理论+实验分析
- 理论部分：Fano 不等式 + 信息处理不等式推导
- 实验部分：3 种攻击（paraphrase、GCG、PAIR）× 3 种场景 × 7 个 LLM × 4 种信号制度

## 实验关键数据

### 主实验
$\log N$ vs $\log I(Z;T)$ 的散点图拟合（7 个 LLM，3 个攻击场景）：

| 信号制度 | 每查询泄漏 $I(Z;T)$ (bits) | 所需查询数 $N$ | 相对风险 |
|---------|-------------------------|-------------|---------|
| 仅 answer tokens | ~0.001 | ~1000 | 基线（最安全） |
| tokens + logits | ~0.01 | ~100 | 10× 风险 |
| tokens + 思维链 | ~0.05 | ~几十 | 50× 风险 |
| tokens + CoT + logits | ~0.1 | ~几个 | 100× 风险 |

### 消融：理论 vs 实验对齐

| 验证 | 结果 |
|------|------|
| $\log N$ vs $\log I$ 斜率 | ~-1（理论预测为 -1，实验高度吻合） |
| Pearson 相关性 | 显著负相关（$p < 0.001$） |
| 跨模型一致性 | GPT-4、DeepSeek-R1、OLMo-2、Llama-4 均符合 |

### 关键发现
- **泄漏翻倍 → 查询需求减半**：实验完美验证了 $N \propto 1/I(Z;T)$ 的理论关系
- **CoT 是最大安全成本**：暴露思维链比暴露 logits 泄漏更多信息，因为 CoT 包含了模型是否在拒绝边缘的明确信号
- **所有攻击方法都未达理论下界**：说明现有攻击还有提升空间（对攻击者是坏消息，对防御者是好消息）
- **速率限制的定量指导**：如果 $I(Z;T) = 0.01$ bit/query，要达到 99% 攻击成功率需要 ~700 次查询——设置 500 次/小时的速率限制就足以防御

## 亮点与洞察
- **"LLM 安全是信息通道问题"**的概念化非常优美——把复杂的安全问题还原为信息论的基本定律，给出不可逾越的下界
- **相变现象**的发现有深刻含义：不是"泄漏一点点没关系"，而是"一旦泄漏非零，查询复杂度就从多项式跳到对数"——安全性的崩溃是断崖式的
- **为 API 设计提供了可操作的标尺**：部署者可以计算"暴露 logits 会让攻击成本降低 10 倍，是否值得？"
- 框架是通用的——适用于任何攻击场景，不限于越狱

## 局限性 / 可改进方向
- 假设 $(Z, T)$ 跨查询 i.i.d.，实际中攻击者的自适应策略可能打破这一假设（尽管信息论下界仍然成立）
- $I(Z;T)$ 的经验估计需要大量样本，对新型攻击可能不准确
- 未考虑模型服务端可以动态调整透明度的策略（如只在低风险查询时暴露 CoT）
- 理论假设攻击者和模型之间的通道是确定性的（给定 $X$ 和 $T$），实际中模型有采样随机性

## 相关工作与启发
- **vs GCG (Zou et al.)**：经验性攻击方法。本文给出了 GCG 效率的理论上界——GCG 远未达到理论极限
- **vs PAIR (Chao et al.)**：自适应攻击。本文的下界对包括 PAIR 在内的所有自适应攻击都成立
- **vs 信息论隐私文献**：将差分隐私/通道容量思想迁移到 LLM 安全，是跨学科的理论贡献
- 对 API 设计有直接启示：可以定量计算"暴露 logits 的安全代价"来做决策
