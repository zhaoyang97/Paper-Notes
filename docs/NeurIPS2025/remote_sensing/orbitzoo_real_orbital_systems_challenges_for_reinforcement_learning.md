# OrbitZoo: Real Orbital Systems Challenges for RL

**会议**: NeurIPS 2025  
**arXiv**: [2504.04160](https://arxiv.org/abs/2504.04160)  
**代码**: 有  
**领域**: 遥感 / 航天  
**关键词**: 轨道力学, 多智能体RL, 碰撞规避, Starlink, 高保真仿真

## 一句话总结
构建OrbitZoo，基于工业标准库Orekit的多智能体RL环境，支持碰撞规避和协同机动，经Starlink真实数据验证MAPE仅0.16%。

## 研究背景与动机
1. **问题**：太空碎片日增，需要自主智能系统进行碰撞规避。
2. **痛点**：现有RL环境从零手写或用简化模型，缺乏标准化多智能体支持。
3. **方案**：基于Orekit+PettingZoo构建高保真标准化RL环境。

## 方法详解

### 关键设计
1. **高保真动力学**：重力场、大气阻力、SRP、三体力等扰动
2. **多智能体支持**：POMDP结构，合作/竞争/混合场景
3. **真实数据验证**：Starlink星座MAPE 0.16%

## 实验关键数据

### 主实验
| 任务 | 场景 | 指标 | 结果 |
|------|------|------|------|
| Starlink验证 | 真实星座轨道 | MAPE | 0.16% |
| 碰撞规避 | 双航天器接近 | 成功率 | 演示成功 |
| 编队飞行 | 多航天器协作 | 队形保持 | 演示成功 |
| 轨道转移 | 单航天器 | 燃料效率 | 竞争力 |

### 环境特性
- 支持 Gymnasium 接口，兼容主流 RL 库
- 高保真轨道动力学（J2/J3摄动、大气阻力、太阳辐射压力、第三体引力）
- 多智能体 POMDP 结构：合作/竞争/混合场景
- 可配置的奖励函数、观测空间、动作空间

## 亮点与洞察
- 首个同时满足高保真+多智能体+标准化的航天RL环境

## 局限性
- 基准RL算法结果有限
