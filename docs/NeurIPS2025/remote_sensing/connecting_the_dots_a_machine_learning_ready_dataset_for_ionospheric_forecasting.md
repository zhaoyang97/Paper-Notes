# Connecting the Dots: A ML Ready Dataset for Ionospheric Forecasting

**会议**: NeurIPS 2025  
**arXiv**: [2511.15743](https://arxiv.org/abs/2511.15743)  
**代码**: 有  
**领域**: 遥感 / 空间天气  
**关键词**: 电离层预测, TEC, 空间天气, 多源数据融合, 时空预测

## 一句话总结
构建了首个ML-ready电离层预测数据集，整合SDO、太阳风、地磁指数和TEC观测等多源异构数据为统一的时间-空间结构，并基准测试了多种时空ML架构用于TEC预测。

## 研究背景与动机

1. **领域现状**：电离层扰动影响GNSS导航精度、无线通信、航空安全和卫星运行。随着LEO卫星星座（如Starlink）增多和GNSS依赖加深，准确预测电离层状态越来越重要。

2. **现有痛点**：
   - 电离层观测数据来自多个平台（NASA SDO、OMNI2、JPL、Madrigal等），格式、分辨率和时间频率各不相同
   - 现有数据产品不是为ML工作流设计的，需要大量预处理才能使用
   - 缺乏标准化的ML-ready数据集阻碍了模型开发和系统对比

3. **核心矛盾**：多源异构数据的丰富性与ML管线对统一、清洁数据的需求之间的矛盾。

4. **切入角度**：作为2025 NASA Heliolab的一部分，系统整合、清洗和对齐多源数据为统一结构。

5. **核心idea一句话**：构建一个模块化、时空对齐的ML-ready电离层数据集，支持物理和数据驱动建模。

## 方法详解

### 整体框架
从7个数据源采集数据 → 时间和空间对齐到统一结构 → 构建模块化数据管线 → 训练和基准测试ML架构。

### 关键设计

1. **多源数据整合**:
   - 做什么：统一7个数据源的时间和空间分辨率
   - 数据源一览：SDO EUV辐照度嵌入(15s) | OMNI2太阳风+IMF(1min) | Kp/Ap地磁指数(3hr) | JPL稠密TEC GIM(15min, 1°×1°网格) | Madrigal稀疏TEC(5min) | F10.7/S10.7太阳通量(日) | 轨道力学参数
   - 设计动机：太阳活动到电离层响应是一条因果链（太阳辐射→太阳风→磁层→电离层），需要全链路数据

2. **空间对齐**：所有数据在纬度-经度网格上对齐，配合准偶极坐标系支持高纬度分析

3. **ML基准测试**：在安静和地磁活跃条件下评估多种时空架构的TEC预测能力

## 实验关键数据

### 主实验
覆盖2010-2024年数据。基准测试了ConvLSTM等时空ML架构在全球TEC预报上的表现。

### 数据集统计
| 数据源 | 时间分辨率 | 时间跨度 |
|--------|----------|---------|
| SDO EUV嵌入 | 15秒 | 2010-2024 |
| OMNI2太阳风 | 1分钟 | 2010-2024 |
| JPL TEC | 15分钟 | 2010-2024 |
| Kp/Ap指数 | 3小时 | 1997-2025 |

### 关键发现
- 地磁活跃期TEC预测更困难，但多源数据输入有助于改善
- 数据集支持从安静到风暴级别的全条件评估
- 手机GNSS众包数据为增加空间覆盖提供新途径

## 亮点与洞察
- **首个综合性ML-ready电离层数据集**：不是简单拼接，而是系统性的时空对齐和模块化设计
- **手机众包GNSS数据的引入**：安卓手机的GNSS接收器可以提供大量额外TEC观测，显著增加空间覆盖
- **因果链数据集**：从太阳到电离层的完整因果链数据，支持因果分析和多步预测

## 局限性 / 可改进方向
- 作为Workshop论文，基准测试的模型和实验规模有限
- 数据集的地理分辨率受限于GNSS站分布
- 未包含精细的高层大气模型参数
- 可扩展到更长期的预测（太阳周期尺度）

## 相关工作与启发
- **vs 传统TEC图（JPL GIM）**：JPL提供TEC图但不是ML-ready格式，本数据集做了对齐和预处理
- **vs 单源ML方法**：以前工作通常只用一种数据源，本数据集支持全链路多源输入
- 可作为空间天气数字孪生的基础数据层
