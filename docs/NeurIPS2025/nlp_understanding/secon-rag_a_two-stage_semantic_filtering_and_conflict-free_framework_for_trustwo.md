# SeCon-RAG: A Two-Stage Semantic Filtering and Conflict-Free Framework for Trustworthy RAG

**会议**: NeurIPS 2025  
**arXiv**: [2510.09710](https://arxiv.org/abs/2510.09710)  
**代码**: [GitHub](https://github.com/) (mentioned in paper)  
**领域**: NLP理解 / RAG安全 / 对抗鲁棒性  
**关键词**: RAG, 语料库投毒防御, 语义过滤, 冲突感知推理, 实体-意图-关系

## 一句话总结
提出 SeCon-RAG 两阶段防御框架，第一阶段用聚类+语义图联合过滤毒化文档，第二阶段在推理时做冲突感知过滤，在5个LLM和3个QA数据集上全面超越现有RAG防御方法，在100%投毒率下仍保持高准确率和极低攻击成功率。

## 研究背景与动机

1. **领域现状**：RAG 通过引入外部文档增强 LLM 的事实准确性，已成为主流范式。但 RAG 系统高度依赖外部语料库，天然易受语料库投毒（corpus poisoning）和检索污染（retrieval contamination）攻击。

2. **现有痛点**：
   - 现有防御方法如 TrustRAG 用聚类来检测投毒文档，但仅依赖向量空间的统计特征，属于粗粒度过滤，容易"误杀"与攻击文档话题重叠但实际正确的文档
   - 多数投票（majority voting）机制在高投毒率下失效
   - AstuteRAG 等方法用启发式规则融合内外部知识，但未解决检索文档之间以及文档与模型内部知识之间的语义冲突
   - 推理阶段缺乏对冲突信息的显式检测和过滤

3. **核心矛盾**：激进过滤会丢失有价值信息降低召回，宽松过滤则保留毒化内容降低可靠性——在信息保留与安全过滤之间存在根本性的 trade-off。

4. **本文要解决什么**：(1) 用细粒度语义信息替代粗粒度统计过滤，精准识别投毒文档；(2) 在推理阶段增加冲突检测，确保最终生成基于一致且可信的知识。

5. **切入角度**：引入结构化语义提取模块 EIRE，从文档中抽取实体、意图和关系三元组，构建语义图。正常文档的语义图结构连通密集，投毒文档的语义图则稀疏断裂——这一结构性差异可用于精准识别。

6. **核心idea一句话**：将结构化语义信息（实体-意图-关系）引入 RAG 防御的检索和推理两个阶段，实现细粒度投毒检测与冲突消解。

## 方法详解

### 整体框架
SeCon-RAG 分两阶段工作：**第一阶段 SCF（Semantic and Clustering-Based Filtering）** 在检索阶段对候选文档进行聚类+语义图双重过滤，剔除投毒内容；**第二阶段 CAF（Conflict-Aware Filtering）** 在推理阶段对通过第一轮筛选的文档进行语义一致性检查，过滤残余冲突。两阶段均基于一个共享的语义提取模块 EIRE。

### 关键设计

1. **EIRE（Entity-Intent-Relation Extractor）**：
   - 做什么：从文档中提取结构化语义三元组 $(E_d, I_d, R_d)$，分别对应实体集合、意图、实体间关系
   - 核心思路：用 prompt 驱动 LLM 完成结构化信息抽取，将文档从自由文本转化为可比较的语义框架。例如对一个关于飞行员的文档，EIRE 会提取出 "Nungesser"（实体）、"回答谁驾驶了飞机"（意图）、"Nungesser → piloted → L'Oiseau Blanc"（关系）
   - 设计动机：为下游的语义图比较和冲突检测提供统一的结构化表示，使过滤不再依赖于不可解释的向量距离

2. **SCF 第一阶段：聚类+语义图联合过滤**：
   - 做什么：在 top-k 检索之前清洗语料库，移除投毒文档
   - **聚类过滤**：将文档嵌入向量空间，用 K-means 聚类，过滤掉与聚类中心余弦相似度超过阈值 $\tau_{\text{cluster}}$ 的文档（投毒文档通常因模板化生成而紧密聚集）
   - **语义图过滤**：用 EIRE 为每个候选文档构建语义图 $G_d = (V_i, E_{ij})$，节点是实体嵌入，边是语义关系。将候选文档的语义图与少量人工验证的正确文档语义图 $G_{\text{cor}}$ 比较，用 LLM 评估图结构相似度得分 $ssG(d, D_{\text{cor}}) \in [0,1]$。正确文档的语义图密集连通，投毒文档则稀疏断裂
   - **保守AND逻辑**：只有同时被聚类和语义两个过滤器标记的文档才被移除：$\widetilde{\mathcal{D}} = \mathcal{D}' \setminus (\mathcal{D}_{\text{cluster}} \cap \mathcal{D}_{\text{semantic}})$，最大化保留有价值信息
   - 设计动机：聚类捕获统计异常（模板化投毒），语义图捕获内容异常（语义不连贯的投毒），两者互补

3. **CAF 第二阶段：冲突感知过滤**：
   - 做什么：在推理阶段对通过 SCF 的 top-k 文档做最终过滤
   - 核心思路：用 EIRE 提取每个候选文档的语义信息，从三个维度评判可信度：
     - **Q（Query Consistency）**：文档与用户查询的意图和实体是否对齐
     - **C（Corpus Consistency）**：文档与其他检索文档之间是否存在关系矛盾
     - **M（Model Consistency）**：文档关键实体是否与 LLM 内部知识一致
   - LLM 对每篇文档做出判断：poisoned / conflicting / irrelevant / trustable，只保留 trustable 文档用于最终生成
   - 设计动机：SCF 移除了统计和结构异常，但可能保留语义相关但事实矛盾的文档（如与模型内部知识冲突），CAF 提供精细的事实层过滤

## 实验关键数据

### 主实验
在 HotpotQA、NQ、MS-MARCO 三个数据集上，用5个LLM（Mistral-12B、Qwen-7B、LLaMA-3.1-8B、GPT-4o、DeepSeek-R1），在 Clean / PIA / PoisonedRAG-20% / PoisonedRAG-100% 四种设置下评估。

| 模型 + 数据集 | 设置 | SeConRAG ACC/ASR | TrustRAG ACC/ASR | 提升 |
|--------------|------|-----------------|-----------------|------|
| GPT-4o + HotpotQA | 100%投毒 | 83.6% / 2.4% | 80.9% / 2.7% | +2.7% ACC |
| GPT-4o + NQ | 100%投毒 | 81.8% / 0.0% | 80.0% / 0.1% | +1.8% ACC |
| GPT-4o + MS-MARCO | PIA | 93.6% / 0.0% | 89.1% / 1.3% | +4.5% ACC |
| DeepSeek-R1 + NQ | 100%投毒 | 96.4% / 0.0% | 88.2% / 0.0% | +8.2% ACC |
| DeepSeek-R1 + MS-MARCO | Clean | 94.0% | 91.0% | +3.0% ACC |
| LLaMA-3.1-8B + MS-MARCO | 100%投毒 | 89.1% / 0.0% | 84.5% / 6.4% | +4.6% ACC |

### 消融实验

| 配置 | HotpotQA (100% 投毒) ACC/ASR | 说明 |
|------|------------------------------|------|
| Full SeCon-RAG | 74.0% / 8.0% | 完整模型 (Mistral-12B) |
| w/o SCF | 71.0% / 25.0% | 去掉第一阶段，ASR 暴涨 +17% |
| w/o CAF | 68.0% / 56.0% | 去掉第二阶段，ASR 暴涨 +48% |
| 仅聚类过滤 | 略低于联合 | 缺少语义图互补 |
| 仅语义过滤 | 略低于联合 | 缺少聚类统计互补 |
| w/o EIRE | ACC 下降 | 失去细粒度语义推理 |

### 关键发现
- **CAF 是核心模块**：去掉 CAF 后 ASR 从 8% 飙升到 56%，说明推理阶段的冲突过滤对防御至关重要
- **聚类和语义过滤互补**：单独使用任一过滤器都不如联合效果好，AND 逻辑在保留信息的同时有效过滤
- **跨嵌入模型鲁棒**：在 MiniLM、SimCSE、BERT、BGE 四种嵌入模型上均保持 >75% 准确率和 <10% ASR
- **阈值不敏感**：$\tau_{\text{cluster}} \in [0.86, 0.90]$、$\tau_{\text{semantic}} \in [0.2, 0.4]$ 范围内性能稳定，波动 ±2%
- **运行时开销适中**：每批 1.21-1.45 分钟，比 TrustRAG 慢约 10 秒但鲁棒性显著提升

## 亮点与洞察
- **语义图结构差异的观察非常巧妙**：正常文档的 EIRE 语义图密集连通，投毒文档稀疏断裂——这个结构性 signal 比向量距离更可解释、更难被攻击者绕过
- **保守 AND 逻辑**：只过滤两个过滤器同时标记的文档，既降低误杀率又保持高检出率，是实用的工程设计
- **三维一致性检查思路可迁移**：Q/C/M 三个维度的校验（查询对齐、文档间一致、模型知识一致）可以用于任何需要多源信息融合的 NLP 系统

## 局限性 / 可改进方向
- **计算开销**：EIRE 需要对每篇候选文档做 LLM 推理提取语义三元组，SCF 的语义图比较也需要 LLM 调用，在大规模语料库上可能成为瓶颈
- **依赖少量正确文档**：SCF 语义过滤需要 10 篇人工验证的正确文档作为参考基准，在冷启动场景下可能难以获取
- **未测试自适应攻击**：当前实验用的是标准投毒策略，如果攻击者知道 EIRE 的语义图结构并有意构造密集连通的语义图，防御效果可能下降
- **CAF 依赖 LLM 内部知识的可靠性**：当模型内部知识本身有误时，M 维度的检查可能产生反效果

## 相关工作与启发
- **vs TrustRAG**：TrustRAG 仅用聚类做粗粒度过滤 + 简单一致性投票，SeCon-RAG 在此基础上增加语义图过滤和推理阶段冲突检测，在高投毒率下优势明显
- **vs AstuteRAG**：AstuteRAG 用启发式融合内外部知识但不做显式冲突检测，SeCon-RAG 的 CAF 模块提供了更系统的冲突感知机制
- **vs InstructRAG**：InstructRAG 用自合成理据引导检索，但在高投毒下仍然脆弱，SeCon-RAG 的双阶段设计更鲁棒

## 评分
- 新颖性: ⭐⭐⭐⭐ 语义图结构差异的观察有启发性，但整体框架是多个已知技术的组合
- 实验充分度: ⭐⭐⭐⭐⭐ 5个LLM × 3个数据集 × 4种攻击设置 + 详细消融 + 嵌入模型分析 + 阈值敏感性，非常全面
