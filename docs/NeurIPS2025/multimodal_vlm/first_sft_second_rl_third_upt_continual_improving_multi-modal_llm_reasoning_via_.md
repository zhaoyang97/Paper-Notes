# First SFT, Second RL, Third UPT: Continual Improving Multi-Modal LLM Reasoning via Unsupervised Post-Training

**会议**: NeurIPS 2025  
**arXiv**: [2505.22453](https://arxiv.org/abs/2505.22453)  
**代码**: https://github.com/waltonfuture/MM-UPT  
**领域**: 多模态VLM / LLM推理  
**关键词**: 无监督后训练, 多模态推理, GRPO, 多数投票, 自我改进

## 一句话总结
提出 MM-UPT 框架，在 SFT 和 RL 之后引入第三阶段"无监督后训练"，通过多数投票作为伪奖励信号结合 GRPO 实现 MLLM 的自我改进，在 MathVista 上将 Qwen2.5-VL-7B 从 66.3% 提升至 72.9%。

## 研究背景与动机

1. **领域现状**：当前 MLLM 的后训练主要依赖两阶段——SFT（需要标注数据）和 RL（需要可验证的 ground truth 或人类偏好）。这两个阶段都需要大量高质量标注的多模态数据。

2. **现有痛点**：随着任务复杂度和数量增长，大规模标注多模态数据变得不可持续。尤其是多模态推理任务，需要精确的推理过程标注，成本极高。

3. **核心矛盾**：模型需要持续改进，但高质量标注数据终将耗尽。之前的无监督方法（如 Genixer、STIC）流程复杂、难以迭代扩展。

4. **本文要解决什么？** 在没有任何外部监督的情况下，如何让 MLLM 持续自我改进推理能力？

5. **切入角度**：多数投票是一个简单有效的信号——如果模型对同一问题的多次采样中大多数答案一致，那个答案很可能是对的，可以作为伪标签。

6. **核心idea一句话**：用 GRPO + 多数投票伪奖励实现完全无监督的多模态后训练。

## 方法详解

### 整体框架
MM-UPT 在已经完成 SFT 和 RL 的 MLLM 上运行。输入是无标签的多模态数据 $\{(I_i, q_i)\}$（图像+问题，无答案）。对每个问题采样 $G$ 个回复，通过多数投票确定伪标签，用 GRPO 更新模型。

### 关键设计

1. **多数投票伪奖励机制**:
   - 做什么：为无标签数据生成训练信号
   - 核心思路：对问题 $q$ 采样 $G=10$ 个回复 $\{o_i\}$，用规则提取答案 $\hat{Y} = E(O)$，选择出现频率最高的答案 $y^* = \arg\max_y \sum \mathbb{I}[y=\hat{y}_i]$。与 $y^*$ 一致的回复奖励 $r_i=1$，否则 $r_i=0$。
   - 设计动机：多数投票已被证明能有效提升模型性能，且不依赖外部标签。它鼓励模型向高置信度的一致答案收敛。

2. **基于 GRPO 的在线强化学习**:
   - 做什么：用伪奖励更新模型参数
   - 核心思路：标准 GRPO 目标函数，用组内标准化的优势估计 $\hat{A}_i = \frac{r_i - \text{mean}}{\text{std}}$，加上 KL 散度约束防止偏离参考模型过远。$\beta=0.01$ 控制 KL 惩罚。
   - 设计动机：GRPO 不需要额外的 value model，计算效率高；KL 约束防止模型在伪标签不准时过度偏移。

3. **合成数据策略**:
   - 做什么：让模型自己生成训练问题，进一步扩展数据规模
   - 核心思路：两种策略——(1) **上下文合成**：给模型原始的(图像,问题,答案)三元组，让它生成语义不同但相关的新问题；(2) **直接合成**：仅给图像，让模型自由生成问题。
   - 设计动机：当人工创建的数据用尽后，模型可以自主生成新数据持续训练。

### 训练策略
- 训练 15 个 episodes，学习率 $1 \times 10^{-6}$，rollout 温度 0.7
- 每个数据点采样 10 个回复（$G=10$）
- 视觉编码器也参与训练（不冻结）

## 实验关键数据

### 主实验（Scenario 1: 标准数据集去标签）

| 方法 | MathVision | MathVerse | MathVista | We-Math | Avg |
|------|------------|-----------|-----------|---------|-----|
| Qwen2.5-VL-7B (base) | 24.87 | 43.83 | 66.30 | 62.87 | 49.47 |
| + GRPO (有监督, MMR1) | 29.01 | 45.03 | 71.40 | 67.24 | 53.17 |
| + SRLM (无监督) | 25.33 | 45.08 | 67.00 | 64.66 | 50.52 |
| + Genixer (无监督) | 23.68 | 43.30 | 65.50 | 64.66 | 49.29 |
| + **MM-UPT (无监督, MMR1)** | **26.15** | **44.87** | **72.90** | **68.74** | **53.17** |

MM-UPT 作为无监督方法，与有监督 GRPO 打平（均为 53.17 avg），大幅超越其他无监督方法。

### 消融实验：不同模型

| 模型 | Base Avg | + MM-UPT Avg | 提升 |
|------|----------|--------------|------|
| Qwen2.5-VL-3B | 39.00 | 41.72 | +7.4% |
| Qwen2.5-VL-7B | 49.47 | 51.23 | +3.6% |
| MM-Eureka-7B | 53.10 | 53.78 | +1.3% |
| ThinkLite-VL-7B | 52.63 | 54.07 | +2.8% |

### 关键发现
- **合成数据有效**：直接合成策略在 Geo3K 和 GeoQA 上甚至超过原始问题的效果（+5.8% vs +3.6%），因为生成了更多样化的问题
- **与有监督 GRPO 兼容**：已经用有监督 GRPO 训练过的模型（MM-Eureka、ThinkLite）仍能从 MM-UPT 获益
- **存在权衡**：MM-UPT 提升准确率但降低回复多样性，且需要模型本身具有足够的初始能力（否则多数投票会放大错误）

## 亮点与洞察
- **极简但有效**：核心思想就是"GRPO + 多数投票"，没有复杂的数据过滤、多阶段流程或外部工具。简单性使其易于复现和迭代。
- **与有监督方法打平**：无监督的 MM-UPT 在 MMR1 数据集上与有监督 GRPO 达到相同的平均分（53.17），这说明对于已足够强的模型，内部知识的自洽性利用可以替代外部标签。
- **三阶段后训练范式的提出**：SFT → RL → UPT 的范式定义有前瞻性，为后训练研究开辟了新方向。

## 局限性 / 可改进方向
- **多数投票的准确性依赖模型能力**：如果模型在某类题目上初始正确率很低，多数投票会锁定错误答案，形成"强化错误"的恶性循环
- **仅验证了数学推理任务**：未在开放式生成、对话、感知等任务上验证，这些场景下多数投票可能不适用
- **MathVerse 上偶有下降**：在部分设置下 MathVerse 分数反而下降，说明方法不是所有场景都有效
- **可扩展性隐忧**：每个数据点需要采样 10 个回复，计算成本较高

## 相关工作与启发
- **vs TTRL**: 同期工作 TTRL 也用多数投票做 LLM 的 GRPO，MM-UPT 是其在多模态领域的拓展
- **vs Genixer/STIC**: 这些方法用 SFT/DPO 做无监督改进，流程复杂；MM-UPT 用在线 RL 更简洁有效
- **vs 有监督 GRPO**: MM-UPT 不需要标签就能达到相近效果，说明模型已有的内部知识是重要的训练信号来源
