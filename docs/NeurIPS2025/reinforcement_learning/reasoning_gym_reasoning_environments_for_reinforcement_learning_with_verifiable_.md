# Reasoning Gym: Reasoning Environments for Reinforcement Learning with Verifiable Rewards

**会议**: NeurIPS 2025  
**arXiv**: [2505.24760](https://arxiv.org/abs/2505.24760)  
**代码**: [GitHub](https://github.com/open-thought/reasoning-gym/)  
**领域**: LLM推理训练与强化学习  
**关键词**: RL环境, 可验证奖励, 过程生成, 课程学习, 推理任务

## 一句话总结
发布Reasoning Gym库，包含100+可验证推理任务的过程生成环境，支持动态难度调整和无限数据生成，可用于RLVR训练和推理评估。

## 研究背景与动机
1. **数据瓶颈**: 推理模型训练受限于固定规模的benchmark，容易过拟合和memorization
2. **难度控制**: 现有benchmark难度固定，无法动态调整以适应模型能力
3. **成本问题**: 依赖人工标注的QA对和互联网数据，不可持续
4. **研究机会**: 设计程序生成的推理任务，具有自动验证和无限可扩展性

## 方法详解

### 整体框架
Reasoning Gym采用模块化设计，覆盖8大推理领域：

1. **数学领域**: 代数、算术、几何
2. **算法领域**: 搜索、优化、动态规划
3. **逻辑推理**: 形式化证明、推论规则
4. **模式识别**: 序列、视觉类比
5. **约束满足**: 游戏、拼图、规划

### 关键设计
**参数化难度控制**:
- 难度参数(Difficulty Parameters): 直接控制复杂度（节点数、多项式次数等）
- 结构参数(Structural Parameters): 确定问题性质（维度、约束类型）
- 风格参数(Stylistic Parameters): 改变呈现不影响难度（变量名、数字格式）

**零样本性能评估**:
- DeepSeek-R1: 59.5%
- Llama 4 Maverick: 41.5%
- Claude 3.5 Sonnet: 40.3%
- 非推理模型与推理模型22%的性能差距

## 亮点与洞察
1. **程序生成的威力**: 突破固定数据集的限制，支持无限数据和动态难度
2. **推理能力差距**: 推理模型与非推理模型22%的性能差距，说明推理训练确实有效
3. **跨域迁移的惊人**: 算法训练对代数(+29.1%)、几何(+22.3%)有显著提升
4. **难度悬崖现象**: 模型基础能力脆弱，难度增加导致大幅下降（代码-71.9%）

## 局限性
1. **文本表示限制**: 视觉空间推理在纯文本表示下仍然困难
2. **规模限制**: 当前评估基于3B/4B模型，超大规模模型的扩展特性未知
3. **多样性**: 虽有100+任务，但仍聚焦于STEM，覆盖面有限
4. **用户研究**: 缺乏人类基准对比，无法判断任务难度的真实性

## 相关工作
- **推理基准**: MATH、GSM8K、BIG-Bench Hard
- **强化学习**: RLVR框架、GRPO、Group Relative Policy Optimization
- **RLVR训练**: OpenAI-o1、DeepSeek-R1
- **过程生成**: Gym环境、ARC等自适应基准
