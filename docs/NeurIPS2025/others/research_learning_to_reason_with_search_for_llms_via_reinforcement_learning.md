# 笔记5：ReSearch - 学习通过搜索推理
**会议**: NeurIPS 2025
**arXiv**: [2503.19470](https://arxiv.org/abs/2503.19470)
**代码**: [GitHub](https://github.com/Agent-RL/ReSearch)
**领域**: LLM推理, 多跳检索, 强化学习
**关键词**: 推理-搜索融合, GRPO, 多步检索增强, 无监督推理

## 一句话总结
ReSearch框架将搜索操作嵌入推理链中作为第一类原语，通过GRPO强化学习自动学习何时何如搜索，无需任何推理步骤的监督标注，在多跳QA任务上相对基线平均提升15.81%。

## 研究背景与动机
1. **多步RAG的困境**：现有方法依赖手工设计启发式策略或繁琐的推理步骤标注，难以扩展到复杂问题
2. **信息检索的自适应缺陷** ：智能决策"何时搜索、搜什么"通常依靠静态规则或不完美的启发
3. **RL机遇**：DeepSeek-R1验证纯RL可诱发复杂推理，但无监督情况下如何有效结合外部搜索？
4. **核心问题**：能否训练LLM自主学会在思考过程中交织搜索操作，无需标注推理链？

## 方法详解
### 整体框架
**三管齐下集成**：推理(`<think>`)、搜索(`<search>`)与结果(`<result>`)共存于一个统一链条：
$$\text{Rollout}: <think>...推理...</think> <search>查询</search> <result>检索结果</result> <think>...继续推理...</think> <answer>最终答案</answer>$$

### 关键设计
**1. GRPO强化学习**：
相对于PPO避免显式批评网络，从一组rollout自估baseline：
$$\mathcal{J}(\theta) = \mathbb{E}_{x,\{y_i\}^G}\left[\min\left(\frac{\pi_\theta(y_i|x)}{\pi_{old}(y_i|x)}A_i, \text{clip}(...)\right)-\beta\mathbb{D}_{KL}(\pi_\theta||\pi_{ref})\right]$$

其中advantage $A_i = (r_i-\text{mean}(r_j))/\text{std}(r_j)$ 组内归一化。

**2. 搜索感知Rollout**：
在GRPO采样过程中，遇`</search>`停止，执行实际搜索调用：
- 搜索工具返回结果→包裹进`<result>...</result>`
- 继续生成下一思考段(搜索结果作条件)
- 重复直到EOS

策略：搜索结果掩码处理，不反向传播检索tokens，仅优化模型生成的搜索/思考。

**3. 奖励函数简约而有效**：
$$r = \begin{cases} \text{F1}(a_{pred}, a_{gt}) & \text{if F1}>0\\ 0.1 & \text{if F1}=0 \text{ and format correct}\\ 0 & \text{otherwise} \end{cases}$$

- **答案奖励**：F1分数(对开放式检索结果容错)
- **格式奖励**：检验标签与$\boxed{}$完整性

## 实验关键数据
### 7B模型 - 多跳QA性能

| 模型 | HotpotQA EM% | HotpotQA LJ% | 2Wiki EM% | MuSiQue EM% | Bamboogle EM% |
|:---:|:---:|:---:|:---:|:---:|:---:|
| **基线** | | | | | |
| 无RAG | 19.18 | 30.64 | 25.76 | 3.76 | 10.40 |
| 朴素RAG | 31.90 | 49.59 | 25.78 | 6.21 | 20.80 |
| Iter-RetGen | 34.36 | 52.22 | 27.92 | 8.69 | 21.60 |
| **ReSearch** | | | | | |
| ReSearch-7B | **40.57** | **60.26** | **44.67** | **21.68** | **43.20** |
| 相对基线最佳 | +18.1% | +15.5% | +59.9% | +149% | **+100%** |

### 32B模型 - 泛化能力

| 数据集 | ReSearch-32B EM | ReSearch-32B LJ | 相对Iter-RetGen |
|:---:|:---:|:---:|:---:|
| HotpotQA | 45.16% | 66.32% | +17.2% |
| 2WikiMultiHop | 51.23% | 58.94% | +68.5% |
| MuSiQue(训练集) | 25.39% | 38.22% | +192% |
| Bamboogle(未见) | 48.31% | 60.58% | +122% |

### 关键发现
1. **超大幅泛化**：仅在MuSiQue训练，在其他三个基准表现突出(未见数据集Bamboogle +122%)，证明推理+搜索能力通用迁移
2. **指数级无标注学习**：无推理步骤标注条件下实现149%(MuSiQue)的绝对提升，证明奖励信号足以诱发复杂多步推理
3. **自发反思与自我纠正**：分析发现，RL过程自然涌现出reflection与self-correction无需显式编程

## 亮点与洞察
1. **哲学创新**：将搜索视为推理链一等公民，打破RAG与推理通常分离的范式
2. **最少监督**：仅需最终答案奖励与格式反馈，诱发多步推理与搜索决策自动协调
3. **泛化强度**：跨数据集迁移效果杰出，说明学到的是推理+搜索的通用能力而非任务特例
4. **实验严谨**：开放式检索环境(真实Wikipedia)测评，规避数据泄露与过拟合

## 局限性
1. 搜索环卷依赖Wikipedia内容质量，不同知识库上效果未知
2. 多跳推理最深测试仅~3步，超复杂多跳(5步+)下表现未清
3. 没有detailed ablation关键超参(搜索次数限制、结果截断长度)
