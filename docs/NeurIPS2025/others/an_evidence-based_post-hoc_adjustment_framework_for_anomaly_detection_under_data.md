# EPHAD: An Evidence-Based Post-Hoc Adjustment Framework for Anomaly Detection Under Data Contamination

**会议**: NeurIPS 2025  
**arXiv**: [2510.21296](https://arxiv.org/abs/2510.21296)  
**代码**: [GitHub](https://github.com/sukanyapatra1997/EPHAD)  
**领域**: 异常检测  
**关键词**: 异常检测, 数据污染, 测试时适应, CLIP, 后处理校正

## 一句话总结
EPHAD 提出一种测试时后处理框架来修正在被污染数据上训练的异常检测模型——在不接触训练流程/数据的前提下，用多模态基础模型（CLIP）或经典方法（LOF）等"证据"在测试时调整模型输出，在 8 个视觉+26 个表格 AD 数据集上有效提升性能。

## 研究背景与动机

1. **领域现状**：无监督异常检测假设训练数据干净，但真实数据常含未检出的异常（如工业数据中的隐藏缺陷）。
2. **现有痛点**：(a) 现有方法需修改训练流程或已知污染比例；(b) 实际部署中模型可能是黑箱（专有模型），无法修改训练。
3. **核心矛盾**：如何在不接触训练的前提下缓解训练数据污染对 AD 模型的影响？
4. **切入角度**：测试时适应（TTA）——用外部证据在测试时校对模型输出。
5. **核心idea一句话**：将 AD 模型的先验知识（可能被污染）与测试时的证据（CLIP/LOF/领域知识）通过贝叶斯式融合进行后处理校正。

## 方法详解

### 关键设计

1. **证据收集**：
   - CLIP 证据：利用多模态基础模型的零样本分类能力判断是否异常
   - LOF 证据：经典的局部离群因子作为辅助判断
   - 领域知识：特定场景的规则

2. **后处理融合**：
   - 将原始 AD 得分视为先验，证据作为似然
   - 贝叶斯式调整产生校正后的异常分数

## 实验关键数据

### 主实验

| 设定 | 原始 AD 模型 AUROC | + EPHAD (CLIP) AUROC |
|------|-------------------|---------------------|
| 视觉AD (8数据集) | 被污染后下降 | 恢复大部分性能 |
| 表格AD (26数据集) | 下降 | 改善 |
| 工业AD | 下降 | 显著恢复 |

### 关键发现
- EPHAD 在不同污染水平下均有效
- CLIP 证据在视觉场景效果最好，LOF 在表格数据更适合
- 计算开销极低（仅后处理）

## 亮点与洞察
- **"不碰训练只修测试"**的 TTA 范式对实际部署极有价值
- 证据来源的灵活性——可以是基础模型、经典方法或领域规则

## 局限性 / 可改进方向
- 依赖外部证据的质量——如果 CLIP 本身对该类异常不敏感则无法帮助
- 融合策略较简单（加权组合），更复杂的贝叶斯更新可能更优
