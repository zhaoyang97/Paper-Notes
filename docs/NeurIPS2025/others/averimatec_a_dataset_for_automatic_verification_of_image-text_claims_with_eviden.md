# AVerImaTeC: A Dataset for Automatic Verification of Image-Text Claims with Evidence from the Web

**会议**: NeurIPS 2025  
**arXiv**: [2505.17978](https://arxiv.org/abs/2505.17978)  
**代码**: https://github.com/abril4416/AVerImaTeC  
**领域**: 事实核查 / 多模态  
**关键词**: 图文核查, 证据检索, QA推理, 多模态验证, 数据集

## 一句话总结
AVerImaTeC 构建了首个带完整证据标注的图文事实核查数据集——1297 条真实图文声明 + 5 阶段标注流水线（提取→QA 推理→充分性检查→迭代精炼→二次检查）+ 时间约束证据（防止时间泄露），基线系统在有 ground truth 证据时准确率 82%，但自动检索证据后降至 15-25%，揭示了图文核查的巨大挑战。

## 研究背景与动机

1. **领域现状**：文本事实核查有 FEVER/AVeriTeC 等数据集，但图文事实核查缺乏带证据标注的基准。现有图文数据集要么是合成声明，要么无证据路径标注。
2. **现有痛点**：(a) 无证据标注的数据集只能训练判定者（是否虚假）而非推理者（为什么虚假）；(b) 合成声明不反映真实错误信息的传播模式；(c) 时间泄露——使用核查文章发布后的证据导致"作弊"。
3. **核心矛盾**：有效的事实核查需要推理路径（QA 链→判定），但标注推理路径极其昂贵——每条声明需要多轮验证和至少 3 个标注者。
4. **本文要解决什么？** 构建带推理路径的图文事实核查数据集，推动图文核查从"判定"到"推理"。
5. **切入角度**：5 阶段标注流水线确保推理路径的充分性和一致性——第 3、5 阶段用独立标注者仅从 QA 对判断（不看原始内容），验证推理路径是否自足。
6. **核心 idea 一句话**：真实图文声明 + QA 推理路径标注 + 充分性双审 + 时间约束证据 + MLLM 基线系统 = 首个完整的图文核查基准。

## 方法详解

### 整体框架
**标注流水线**: Phase 1 提取规范化单一图文声明 → Phase 2 生成 QA 对（问题可含图像）→ Phase 3 充分性检查（第三方仅从 QA 判断）→ Phase 4 不一致则迭代精炼 → Phase 5 第二次充分性检查。**基线系统**: 生成器→回答器→判定器→解释器管线

### 关键设计

1. **QA 推理路径标注**:
   - 做什么：为每条声明标注验证推理过程
   - 核心思路：标注者提出验证所需的问题（62.5% 涉及图像，53.9% 需要图像搜索），并找到回答。问题-答案对构成推理链，最终汇聚为判定
   - 设计动机：QA 对使核查过程可解释——不仅知道"假"还知道"为什么假"

2. **充分性双审机制**:
   - 做什么：确保 QA 对足以独立判断声明真假
   - 核心思路：Phase 3/5 的独立标注者仅看 QA 对（不看原始声明+图像），尝试判断。如果判断一致说明 QA 对是自足的。Phase 3 后若不一致进入 Phase 4 迭代精炼
   - 设计动机：防止标注者在 QA 中遗漏关键信息——充分性检查淘汰了 ~15% 需要重新标注

3. **时间约束证据**:
   - 做什么：确保证据在声明发布前就存在（防止时间泄露）
   - 核心思路：只允许使用声明发布日期之前的网页作为证据。5% 的声明无精确日期需估计
   - 设计动机：使用核查文章本身作为"证据"是作弊——真实系统无法预知核查结果

### 损失函数 / 训练策略
- 数据集标注工作，无模型训练
- 基线：MLLM 管线（Gemini/Qwen/LLaVA 作为判定器）
- 评估：准确率（判定）+ ROUGE-1（解释质量）+ LLM-based 参考评分

## 实验关键数据

### 数据集统计

| 指标 | 值 |
|------|-----|
| 声明数 | 1297 (793 train, 152 dev, 352 test) |
| 图像相关问题比例 | 62.5% |
| 需要图像搜索 | 53.9% |
| 标注者间 κ | 0.742 |
| QA 对重叠率 | 74.7% 召回, 67.2% 精确 |
| 标签分布 | ~78% 驳斥（测试集偏斯） |

### 基线性能

| 条件 | 判定准确率 | 解释 ROUGE-1 |
|------|----------|-------------|
| Ground truth 证据 | **82%** (Gemini) | 0.50 |
| 自动检索证据 | **15-25%** | 低 |
| 并行 QG (最优) | — | 0.39-0.43 |
| 动态 QG | — | 0.27-0.33 |

### 关键发现
- 有 ground truth 证据时 82% 准确率——说明 MLLM 能理解并使用证据
- 自动检索后降至 15-25%——**证据检索是瓶颈**（召回率 15-25%），不是推理
- 并行问题生成优于动态生成——简单策略反而更稳定
- MLLM 有 ~30% 的反向图像搜索偏差——倾向于过度使用图像搜索
- 标签偏斜（78% 驳斥）需要未来平衡

## 亮点与洞察
- **充分性双审是质量保证的核心**：其他 QA 数据集没有这一步，导致 QA 对可能遗漏关键信息
- **证据检索是核心瓶颈**：模型的推理能力足够，但找不到正确证据——这重新定义了图文核查的重点
- **时间约束**使评估更真实——防止了"先看答案再找理由"的泄露

## 局限性 / 可改进方向
- 规模小（1297 条）——手动标注的固有限制
- 标签严重偏斜（78% 驳斥）——可能偏向事实核查文章的选择
- 5% 声明无精确日期——时间约束有误差
- ROUGE-1 不适合评估解释质量——与 ground truth 相关性低
- 预训练 MLLM 可能见过核查文章——潜在数据泄露

## 相关工作与启发
- **vs AVeriTeC**: 纯文本核查数据集，AVerImaTeC 扩展到图文多模态
- **vs FEVER**: 基于 Wikipedia 的合成声明，AVerImaTeC 用真实世界流传的错误信息
- **vs Snopes 等**: 核查网站无标准化 QA 推理路径标注

- 新颖性: ⭐⭐⭐⭐⭐ 首个带完整推理路径的图文核查数据集
- 实验充分度: ⭐⭐⭐⭐ 多基线+充分性分析+错误分析
- 写作质量: ⭐⭐⭐⭐ 标注流水线描述详细
- 价值: ⭐⭐⭐⭐⭐ 定义了图文事实核查的评测标准

### 方法补充说明
- **5 阶段标注的人力成本**：每条声明需要 ~45 分钟标注（5 阶段合计），1297 条约 975 小时人工——这是高质量数据的代价
- **充分性检查的淘汰率**：~15% 的 QA 对在 Phase 3 被判不充分需要返回 Phase 4 精炼——这个质量门槛保证了推理路径的自足性
