# ContextAgent: Context-Aware Proactive LLM Agents with Open-World Sensory Perceptions

**会议**: NeurIPS 2025  
**arXiv**: [2505.14668](https://arxiv.org/abs/2505.14668)  
**代码**: https://github.com/openaiotlab/ContextAgent  
**领域**: Agent  
**关键词**: 主动式Agent, 情境感知, 可穿戴传感器, 工具调用, Benchmark

## 一句话总结
提出 ContextAgent，首个利用可穿戴设备多模态感知（视频+音频+通知）来理解用户意图并主动提供工具增强服务的 LLM Agent 框架，同时构建了包含 1000 个样本的 ContextAgentBench 基准，在主动预测准确率和工具调用上分别提升 8.5% 和 6.0%。

## 研究背景与动机

1. **领域现状**：当前 LLM Agent 主要处于"被动响应"模式——只有在用户明确下达指令时才能执行任务。少数前沿工作尝试构建"主动式Agent"（Proactive Agent），但主要局限在桌面 UI 环境（如监控屏幕截图、键盘输入）。
2. **现有痛点**：已有主动式 Agent 存在两大问题：① 环境感知范围狭窄，只能观察封闭环境（桌面界面），无法感知用户日常生活中的开放世界环境；② 功能有限，只能通过 LLM 直接推理给出文本回复，不能调用外部工具提供更丰富的服务。
3. **核心矛盾**：要实现真正的"主动个人助理"，Agent 需要像人类一样通过多种感官（视觉、听觉）持续感知周围环境，在不需要用户操作的情况下理解意图并自动提供帮助——而现有方案在感知范围和服务能力上都严重不足。
4. **本文要解决什么**：如何让 LLM Agent 利用可穿戴设备（智能眼镜、耳机等）的多模态传感数据，在开放世界中理解用户上下文和意图，判断是否需要主动提供服务，并自动调用合适的工具完成服务。
5. **切入角度**：作者观察到智能眼镜、耳机等设备具有"免手操作"的特性，其第一人称视角感知与用户共享同一视角，非常适合作为主动式 Agent 的感知前端。同时引入用户画像（persona）信息可以实现个性化服务判断。
6. **核心 idea 一句话**：通过可穿戴设备获取多维传感上下文 + 用户画像，用 CoT Fine-tuned LLM 进行"先思考再行动"的主动推理和工具调用。

## 方法详解

### 整体框架
ContextAgent 的输入是用户可穿戴设备采集的多模态感知数据 $\mathcal{S}$（第一人称视频 $\mathcal{S_V}$、音频 $\mathcal{S_A}$、手机通知 $\mathcal{N}$）以及用户画像 $\mathcal{P}$。系统经过两个阶段处理：① 面向主动服务的上下文提取（Context Extraction），将原始传感数据转化为结构化的感知上下文 $\mathcal{C}$；② 上下文感知推理（Context-aware Reasoning），使用经 SFT 训练的 LLM 进行推理，输出思维链 $\mathcal{T}$、主动性得分 $\mathcal{P_S}$（1-5分）、工具调用链 $\mathcal{T_C}$ 和最终回复 $\mathcal{R}$。

### 关键设计

1. **面向主动服务的上下文提取（Proactive-oriented Context Extraction）**:
   - 做什么：从原始传感数据中提取有助于判断是否需要主动服务的关键线索
   - 核心思路：使用 VLM（如 Qwen-2.5-VL）结合 ICL（in-context learning）对第一人称视频生成面向主动服务的视觉上下文 $\mathcal{C_V}$，使用语音识别模型生成音频上下文 $\mathcal{C_A}$，最终整合为 $\mathcal{C} = [\mathcal{C_V}, \mathcal{C_A}, \mathcal{N}]$
   - 设计动机：零样本 VLM 倾向于生成过于简单或冗余的描述，忽略对判断主动服务至关重要的线索（如用户在健身房的具体健身器材细节），ICL 方式可以引导 VLM 生成更有针对性的描述。消融实验显示零样本比 ICL 方式在 Acc-P 上低 3.0%

2. **用户画像上下文（Persona Context）**:
   - 做什么：整合用户身份、偏好和历史行为信息
   - 核心思路：从历史传感数据（如对话记录）中用 LLM 提取用户画像，并在推理时作为输入的一部分
   - 设计动机：主动服务的必要性高度依赖于用户个人偏好（如健康意识强的用户在餐厅需要营养建议，而普通用户可能不需要）。消融实验显示去掉 persona 后 Acc-P 下降 9.0%，F1 下降 12.3%

3. **上下文感知推理器（Context-aware Reasoner）**:
   - 做什么：综合感知上下文和用户画像进行推理，判断是否需要主动服务以及调用哪些工具
   - 核心思路：使用 SFT + CoT 训练，训练数据的思维链从 Claude-3.7-Sonnet 蒸馏得到。推理时模型先在 `<think>...</think>` 标签内生成显式思维过程，再输出主动性得分 $\mathcal{P_S}$ 和工具调用链。当 $\mathcal{P_S} \geq \theta$ 时触发主动服务。LoRA rank=8，lr=0.0001，训练 5 epochs
   - 设计动机：直接使用 LLM 推理在处理传感上下文到工具映射上表现不佳，"先思考再行动"的 CoT 训练可以显著提升主动预测和工具调用的准确性

### 损失函数 / 训练策略
使用标准的 SFT 损失函数，训练数据 $\mathcal{D}_{SFT} = \{(\mathcal{X}, \mathcal{T}, \mathcal{Y})\}$，其中 $\mathcal{X}$ 包含感知上下文和用户画像，$\mathcal{T}$ 是从高级 LLM 蒸馏的思维链，$\mathcal{Y}$ 包含主动性得分和工具调用链。采用 LoRA 高效微调，8 张 A6000 GPU，AdamW 优化器。

## 实验关键数据

### 主实验
在 ContextAgentBench 上使用 Llama-3.1-8B-Instruct 作为基础模型：

| 方法 | Acc-P↑ | MD↓ | FD↓ | RMSE↓ | F1↑ | Acc-Args↑ |
|------|--------|-----|-----|-------|-----|-----------|
| Proactive Agent | 0.676 | 0.017 | 0.306 | 1.915 | 0.318 | 0.081 |
| ICL-All | 0.757 | 0.229 | 0.012 | 1.872 | 0.582 | 0.270 |
| Vanilla SFT | 0.813 | 0.068 | 0.117 | 1.572 | 0.580 | 0.405 |
| **ContextAgent** | **0.874** | **0.030** | **0.095** | **1.408** | **0.626** | **0.448** |

ContextAgent 在 Qwen2.5-7B 上达到 0.894 Acc-P、0.645 F1、0.459 Acc-Args，甚至可媲美 GPT-4o（0.886 Acc-P、0.639 F1）和 70B 模型。

### 消融实验

| 配置 | Acc-P | F1 | Acc-Args | 说明 |
|------|-------|-----|----------|------|
| Full (DeepSeek-R1-7B) | 0.888 | 0.647 | 0.468 | 完整模型 |
| w/o vision | 0.709 | 0.414 | 0.163 | 去视觉后主动预测-17.9%, 工具调用-23.3% |
| w/o audio | 0.720 | 0.493 | 0.212 | 去音频影响略小于视觉 |
| w/o persona | ~0.78 | ~0.50 | ~0.32 | 去画像 Acc-P 降 9%, F1 降 12.3% |
| Zero-shot VLM | ~0.86 | ~0.61 | ~0.45 | 零样本上下文提取 Acc-P 降 3% |

### 关键发现
- **视觉模态最关键**：去掉视觉上下文导致最大性能下降（Acc-P 降 17.9%），说明第一人称视觉对理解用户环境至关重要
- **用户画像不可或缺**：去掉 persona 后 Acc-Args 下降 12.6%，证明个性化信息对判断服务必要性和工具选择至关重要
- **OOD 泛化良好**：在未见过的场景上仍能达到 90.9% Acc-P 和 68.9% F1，超越最佳 baseline 8.3% Acc-P
- **小模型也能竞争**：7B/8B 模型经 ContextAgent 训练后可达到甚至超过 70B 模型和 GPT-4o 的 baseline 表现

## 亮点与洞察
- **问题定义新颖**：首次将可穿戴设备传感数据引入主动式 Agent 范式，填补了"开放世界感知 + 工具增强主动服务"这一研究空白
- **面向主动的 ICL 上下文提取**：不是简单地让 VLM 描述画面，而是通过 ICL 引导 VLM 关注对主动服务有用的线索，这个思路可迁移到任何需要"任务导向感知"的场景
- **Benchmark 构建方法论完整**：200 条人工种子 + LLM 自动扩展 + 人工验证 + 格式校验的数据构建流水线值得参考

## 局限性 / 可改进方向
- **传感上下文通过文本中转**：实际部署时视频/音频需先转换为文本描述，这一步的信息损失难以避免——可考虑直接用多模态 LLM 端到端处理
- **Benchmark 规模有限**：仅 1000 个样本、9 个场景、20 个工具，距离真实世界的多样性还有差距
- **隐私与伦理问题未深入讨论**：可穿戴设备持续录制视频/音频存在严重隐私隐患，实际部署面临重大挑战
- **主动性阈值 $\theta$ 设定**：论文虽提到用户可调，但未给出自适应调整策略
- **工具集固定**：20 个预定义工具限制了服务范围，缺乏动态工具发现/组合机制

## 相关工作与启发
- **vs Proactive Agent [Li et al.]**: 仅限桌面 UI 环境，使用 LLM 直接推理，无工具调用能力。ContextAgent 拓展到开放世界传感、支持工具链，Acc-P 提升约 20%
- **vs CodingGenie**: 专注编程辅助的桌面主动 Agent，需要键盘/屏幕输入。ContextAgent 面向日常生活全场景
- **vs 传统规则式主动服务**（如 Apple Watch 跌倒检测）：规则触发 vs LLM 推理，ContextAgent 更灵活但计算成本高

## 评分
- 新颖性: ⭐⭐⭐⭐ 首次提出开放世界主动 Agent + 工具增强的系统性框架，但核心技术（SFT+CoT+ICL）较常规
- 实验充分度: ⭐⭐⭐⭐ 13个LLM、6种baseline、消融/OOD实验完整，但缺少真实物理部署实验
- 写作质量: ⭐⭐⭐⭐ 问题定义清晰，实验组织有条理，附录细节充分
- 价值: ⭐⭐⭐⭐ 开创了"感知增强主动Agent"的新方向，benchmark会推动后续研究，但实际应用距离尚远
