# Agentic Plan Caching: Test-Time Memory for Fast and Cost-Efficient LLM Agents

**会议**: NeurIPS 2025  
**arXiv**: [2506.14852](https://arxiv.org/abs/2506.14852)  
**代码**: 无  
**领域**: LLM Agent / LLM效率  
**关键词**: Agent缓存, 计划模板, 测试时记忆, 推理成本优化, Plan-Act范式  

## 一句话总结
提出 Agentic Plan Caching (APC)——从 agent 执行日志中提取结构化计划模板，通过关键词匹配缓存命中后用小模型适配复用，平均降低 50.31% 成本和 27.28% 延迟，同时保持 96.61% 的最优准确率。

## 研究背景与动机

1. **领域现状**：LLM-based agent 采用 Plan-Act 循环——大模型规划、执行模型行动。规划阶段（思维链推理、任务分解）消耗大量 token 和昂贵 API 调用。
2. **现有痛点**：(a) **上下文缓存** (KV cache reuse)：模型相关，不同模型的 KV cache 不可迁移——多模型 agent 不适用；(b) **语义缓存** (semantic caching)：存储 (input, output) 对——但 agent 输出依赖于外部数据/环境，同查询不同数据会产生不同输出；(c) 两者都缺乏适配能力，无法处理查询的微小变化。
3. **核心矛盾**：agent 的规划模式在语义上高度重复（"分析数据集"→类似分解计划），但具体细节因数据/环境不同而异。现有缓存无法分离"核心意图"和"动态上下文"。
4. **切入角度**：从完成的 agent 执行中提取可复用的**计划模板**（去掉上下文特定信息），新任务到来时用小模型适配模板。
5. **核心 idea 一句话**：将 agent 级计划（而非查询级响应）作为缓存单元，关键词匹配 + 小模型适配 = 大模型级计划质量、小模型级成本。

## 方法详解

### 整体框架
输入任务查询 → 小模型提取关键词（高层意图）→ 关键词精确匹配缓存 → **命中**：小模型适配模板为具体计划 → **未命中**：大模型生成新计划 → Actor LM 执行 → 执行完成后提取模板入缓存。

### 关键设计

1. **计划模板提取 (Plan Template Generation)**:
   - 做什么：从完成的 agent 执行日志中提取可复用的结构化模板
   - 核心思路：两步过滤——(a) 规则过滤器提取关键信息、去掉冗余推理步骤；(b) 轻量 LLM 过滤器去除上下文特定元素（公司名、数值等），生成泛化模板
   - 设计动机：原始执行日志太长且包含不相关细节，小模型无法直接利用

2. **关键词提取与匹配 (Keyword Extraction & Matching)**:
   - 做什么：用关键词（高层意图）代替查询相似度进行缓存查找
   - 核心思路：用 GPT-4o-mini 从查询中提取核心意图关键词（如"均值计算"），精确匹配缓存中的关键词
   - 设计动机：实验证明**查询级语义相似度不适合 agent 缓存**——会过度关注上下文细节（公司名等）而非核心意图，导致高误报/漏报率。关键词则能更好地捕获任务意图
   - 精确匹配而非模糊匹配：避免引入相似度阈值调参问题

3. **计划适配 (Plan Adaptation)**:
   - 做什么：用小模型将缓存的通用模板适配为包含具体上下文的计划
   - 核心思路：缓存命中后，小 planner LM（如 LLaMA-3.1-8B）接收模板 + 当前任务上下文 → 生成适配后的具体计划
   - 设计动机：模板已包含正确的任务分解逻辑，小模型只需填入具体细节，不需要昂贵的推理能力

4. **与缓存完整执行日志的对比**:
   - Full-History Caching（缓存完整日志作为 in-context example）效果差——小模型难以处理长上下文、未过滤的执行日志
   - APC 的结构化模板更紧凑、更易适配

### 系统架构
- 基于 Minion 架构（Plan-Act 循环）
- 大 planner LM：GPT-4o（仅 cache miss 时使用）
- 小 planner LM / Actor LM：LLaMA-3.1-8B（cache hit 时使用）
- 关键词/模板提取：GPT-4o-mini（开销仅占总成本 1.04%）

## 实验关键数据

### 主实验 (5 个 Agent 工作负载)

| 方法 | 平均成本 | 平均延迟 | 准确率保持 |
|------|---------|---------|------------|
| Accuracy-Optimal (大模型always) | 100% | 100% | 100% |
| Cost-Optimal (小模型always) | 最低 | 最低 | 低 |
| Semantic Caching (80%) | 有减少 | 有减少 | 不稳定 |
| Full-History Caching | 有减少 | 有减少 | 较低 |
| **APC** | **-50.31%** | **-27.28%** | **96.61%** |

### 各数据集细节

| 数据集 | 任务类型 | 成本降低 | 准确率保持 |
|--------|---------|---------|------------|
| FinanceBench | 金融数据推理 | ~50% | ~97% |
| QASPER | 长上下文推理 | ~45% | ~96% |
| TabMWP | 数学推理 | ~55% | ~97% |
| AIME 2024/2025 | 竞赛数学 | ~40% | ~95% |
| GAIA | 多步工具使用 | ~60% | ~97% |

### 关键发现
- **语义缓存对 agent 不适用**：查询相似度无法捕获任务意图，不同阈值下误报/漏报都很高
- **完整日志缓存效果差**：小模型无法有效处理长且嘈杂的执行日志
- **关键词 > 查询相似度**：关键词精确匹配在所有阈值下都比语义相似度更准确
- **缓存开销极低**：关键词提取 + 模板生成仅占总成本 1.04%
- **与现有 serving 框架兼容**：可与 vLLM, SGLang 等共同使用

## 亮点与洞察
- **从查询级到任务级缓存的范式转换**：现有缓存为 chatbot 设计（缓存 QA 对），APC 为 agent 设计（缓存计划模板）——这个抽象层级的提升是核心贡献
- **关键词 vs 语义相似度的实验分析**：Figure 3 的对比非常有说服力——关键词提取在所有阈值下都能实现更低的 false positive 和 false negative
- **分离意图与上下文**：将通用计划逻辑（意图）与具体数据（上下文）分离，使小模型适配成为可能
- **测试时学习**：APC 是一种 test-time memory——agent 在推理过程中自动积累经验，无需额外训练

## 局限性 / 可改进方向
- **精确关键词匹配**：可能错过语义相似但措辞不同的查询，模糊匹配留待未来工作
- **缓存正确性依赖**：只缓存成功执行的计划，错误执行不入缓存——需要准确的成功判断
- **模板泛化能力**：当新任务与缓存模板差异较大时适配可能失败
- **改进方向**：(1) 层次化关键词（多粒度匹配）；(2) 计划模板的渐进式更新/改进；(3) 跨 agent 架构的评估

## 相关工作与启发
- **vs GPTCache (语义缓存)**：GPTCache 缓存 (query, response) 对——对 agent 的数据依赖输出不适用
- **vs MemGPT**：MemGPT 用记忆增强 agent 能力（减少幻觉），APC 用记忆减少成本——目标不同
- **vs Case-Based Planning**：经典 CBP 用手工设计的符号化计划，APC 从 LLM 生成中自动提取模板
- **vs Context Caching**：KV cache 缓存模型内部状态，APC 缓存任务级计划——抽象层级不同

- 新颖性: ⭐⭐⭐⭐ "计划模板缓存"的概念新颖且实用，从 chatbot 缓存到 agent 缓存的范式转换
- 实验充分度: ⭐⭐⭐⭐ 5 个多样化工作负载，与多种缓存基线对比，开销分析详细
