# CAM: A Constructivist View of Agentic Memory for LLM-Based Reading Comprehension

**会议**: NeurIPS 2025  
**arXiv**: [2510.05520](https://arxiv.org/abs/2510.05520)  
**代码**: https://github.com/rui9812/CAM  
**领域**: Agent  
**关键词**: 智能体记忆, 长文本理解, 层次化记忆, 增量聚类, 建构主义

## 一句话总结
受皮亚杰建构主义理论启发，提出CAM——一种具有结构性（层次化schema）、灵活性（重叠聚类的同化）和动态性（增量适应）三大特征的智能体记忆系统，在6个长文本阅读理解任务上全面超越RAPTOR、GraphRAG等基线。

## 研究背景与动机

1. **领域现状**：LLM处理超长文档（小说、会议记录等）时面临信息过载问题，主流做法是为LLM配备外部记忆模块——将文档分块存储后检索。现有方法分为非结构化记忆（MemGPT、ReadAgent）和结构化记忆（RAPTOR、GraphRAG、MemTree）。
2. **现有痛点**：(a) 非结构化记忆（表格式存储）无法捕捉信息间的关联性，关键信息分散时召回困难；(b) 结构化记忆虽好，但RAPTOR/GraphRAG是离线方法，每次更新需全量重建；MemTree虽然在线，但只能逐条插入且不支持重叠聚类；(c) 缺乏系统性的记忆设计原则。
3. **核心矛盾**：如何同时实现记忆的结构化、灵活的信息同化（一个单元可归属多个上层抽象）、和动态的结构适应（增量更新而非重建）？
4. **本文要解决什么？** 提供一个记忆系统设计蓝图（三大特征），并实现一个同时满足三者的原型系统。
5. **切入角度**：从认知科学的皮亚杰建构主义理论出发——记忆是通过"同化"（将新信息纳入现有schema）和"顺应"（调整schema以适应新信息）持续发展的认知结构。
6. **核心idea一句话**：用增量重叠聚类算法统一实现建构主义的同化（灵活性）和顺应（动态性），构建支持批量在线更新的层次化记忆。

## 方法详解

### 整体框架

CAM包含两个阶段：**记忆构建**（Reading Phase）和**记忆检索**（Inference Phase）。

构建阶段将文本块组织为多层层次结构：底层是原始文本块构成的语义网络 $G_0$，上层节点是下层紧密相关节点的LLM摘要。通过增量重叠聚类算法实现在线批量更新。

检索阶段采用Prune-and-Grow策略：先全局语义匹配快速定位，再沿记忆结构递归关联探索。

### 关键设计

1. **基础语义网络构建（Foundational Network Expansion）**:
   - 做什么：将新文本块集成到底层语义网络 $G_0$
   - 核心思路：对每对文本块计算综合相似度 $s(v_i, v_j) = \alpha \cdot \cos(f_{emb}(v_i), f_{emb}(v_j)) + (1-\alpha) \cdot \exp(-\frac{(i-j)^2}{2\sigma^2})$，结合语义相似度和位置邻近性。为每个新块选择top-k相关节点（超过阈值θ）建立边
   - 设计动机：纯语义相似度忽略叙事连贯性（相邻段落可能语义不同但逻辑相关），位置邻近性弥补这一不足

2. **自我中心解纠缠（Ego-Centric Disentanglement）**:
   - 做什么：实现重叠聚类的**灵活同化**——让一个底层节点可以同时贡献给多个上层抽象
   - 核心思路：对每个节点v，提取其ego-network（邻居子图），将ego-network分解为连通分量 $\{C_v^1, ..., C_v^{t_v}\}$。为每个分量创建v的一个副本，构成副本网络 $\tilde{G}_0$。这样就把重叠聚类转化为副本网络上的非重叠聚类问题
   - 设计动机：非常巧妙——重叠聚类通常算法复杂度高，通过ego-net分解 + 节点复制，可以用简单高效的标签传播算法实现重叠效果。且天然支持增量更新（只需更新受影响节点的副本）和并行化

3. **在线聚类更新（Online Clustering Updates）**:
   - 做什么：在副本网络上增量更新聚类分配，实现**动态顺应**
   - 核心思路：用增量标签传播算法——新副本初始化为独立标签，受影响的现有副本保留旧标签。在受影响子图上迭代传播直到收敛。标签变化会递归影响邻居（涟漪效应）
   - 对修改过的聚类，用LLM重新生成摘要节点，更新上层连接，并递归触发上层的解纠缠和聚类更新
   - 设计动机：局部优先策略——只更新受影响的部分而非全局重建，实现 $O(\text{local})$ 而非 $O(n)$ 的更新复杂度

4. **Prune-and-Grow检索策略**:
   - 做什么：自适应地从记忆结构中检索与查询相关的信息
   - 核心思路：两阶段——(1) **快速定位**：全局计算查询与所有记忆节点的embedding相似度，选top-s形成候选集D；(2) **关联探索**：LLM从D中选出有用节点形成激活集P，收集P中节点的同层邻居和下层子节点作为新候选，迭代扩展P直到不再增长
   - 设计动机：结合了全局检索的覆盖性和局部结构探索的精确性，避免了纯层次遍历可能遗漏底层相关信息的问题

### 训练策略
- 无需训练——基于GPT-4o-mini作为LLM backbone，text-embedding-3-small作为编码器
- 文本块大小512 tokens
- 所有基线统一使用相同LLM和embedding模型，确保公平比较

## 实验关键数据

### 主实验

| 方法 | 类型 | NovelQA ACC-L | QMSum ACC-L | FABLES F1P | MH-RAG F1 | ODSum-S ACC-L | ODSum-M ACC-L |
|------|------|--------------|-------------|-----------|-----------|--------------|--------------|
| MemGPT | 非结构化 | 40.8 | 40.3 | 81.3 | 67.1 | 36.8 | 33.2 |
| ReadAgent | 非结构化 | 42.3 | 45.5 | 84.2 | 65.5 | 39.4 | 35.7 |
| RAPTOR | 结构化 | 47.8 | 50.7 | 86.5 | 73.6 | 48.7 | 44.3 |
| GraphRAG | 结构化 | 45.3 | 53.9 | 87.2 | 70.3 | 50.2 | 45.8 |
| MemTree | 结构化 | 43.5 | 48.3 | 83.1 | 71.2 | 46.0 | 42.5 |
| **CAM** | **结构化** | **52.3** | **57.6** | **91.5** | **77.5** | **54.6** | **50.7** |

CAM在所有6个数据集的所有指标上全面领先，平均提升3.0%。

### 消融/效率实验

| 配置 | 说明 |
|------|------|
| 无重叠聚类（去除灵活性） | 性能下降约2%，验证重叠聚类的必要性 |
| RAPTOR检索 vs CAM检索 | CAM的Prune-and-Grow比RAPTOR的全局检索/遍历更好 |
| 在线批量更新效率 | RAPTOR/GraphRAG需>1小时重建；MemTree随批量大小线性增长；CAM亚线性增长，4x快于离线方法 |
| 不同批量大小的推理稳定性 | CAM在不同batch size下性能稳定，接近离线水平 |

### 关键发现
- **结构 > 非结构**：结构化方法全面优于非结构化，ACC-L差距约5-10%
- **灵活性至关重要**：RAPTOR（支持重叠）始终强于MemTree（不支持重叠），平均差距2.1%
- **动态性使实用成为可能**：CAM是唯一同时支持批量在线更新且保持离线水平性能的方法
- 知识图谱建模（GraphRAG/HippoRAG）对叙事类文本效果不如层次聚类摘要

## 亮点与洞察
- **ego-centric disentanglement**是本文最巧妙的设计：把重叠聚类问题转化为副本网络上的非重叠聚类，既简洁又高效，还天然支持增量更新和并行。这一技术可迁移到任何需要增量重叠聚类的场景
- **建构主义理论框架**的引入不是空头conceptual——三个特征（结构性、灵活性、动态性）被精确映射为技术设计（层次聚类、重叠聚类、增量聚类），且实验验证了每个特征的独立贡献
- **Prune-and-Grow检索**兼具全局和局部优势，比单纯的全局检索或层次遍历更灵活

## 局限性 / 可改进方向
- 依赖GPT-4o-mini做摘要和检索判断，开源模型效果未验证
- 底层语义网络的相似度阈值θ和top-k需要调参，对不同文档类型的鲁棒性未充分分析
- 当文档极长（如百万token）时，"全局计算所有节点embedding相似度"的快速定位阶段可能成为瓶颈
- 高斯位置邻近性假设对非顺序文档（如网页集合）可能不适用
- 可探索将LLM调用替换为更轻量的摘要模型以降低成本

## 相关工作与启发
- **vs RAPTOR**: RAPTOR也用层次聚类+重叠，但是离线方法，每次更新需全量重建。CAM在RAPTOR基础上增加了增量和在线能力
- **vs GraphRAG**: GraphRAG用知识图谱建模，对实体丰富的文本友好但对叙事类文本效果一般。CAM的聚类摘要方式更通用
- **vs MemTree**: MemTree有在线能力但无重叠聚类且只能逐条插入。CAM全面超越
- 对Agent记忆系统设计有普适启发：结构性+灵活性+动态性可作为评价记忆系统的三维度框架
