# TabSTAR: A Tabular Foundation Model for Tabular Data with Text Fields

**会议**: NeurIPS 2025  
**arXiv**: [2505.18125](https://arxiv.org/abs/2505.18125)  
**代码**: https://github.com/alanarazi7/TabSTAR  
**领域**: 表格学习 / 基础模型  
**关键词**: 表格基础模型, 文本字段, 目标感知token, 跨数据集学习, 迁移学习

## 一句话总结
提出 TabSTAR，一个专为含文本字段的表格数据设计的基础模型：通过解冻文本编码器（e5-small-v2）端到端优化文本表征 + 目标感知 token 注入分类目标语义信息 + 无数据集特定参数的架构实现跨数据集迁移学习，在 350 个数据集上预训练后，分类任务上 14 个数据集中 12 个超越 CatBoost-Tuned（4h 调参），8/11 超越 TabPFN-v2。

## 研究背景与动机

1. **领域现状**：表格学习长期被 GBDT（XGBoost/LightGBM/CatBoost）主导。表格基础模型（TFM）如 TabPFN-v2 开始在中等规模数据集上超越 GBDT，但处理文本字段时仍依赖静态嵌入——与 GBDT 无异。  
2. **现有痛点**：（a）现实表格数据常含自由文本字段（如病历报告、产品描述），但主流 benchmark 忽视这类数据——一半数据集超过 20 年；（b）现有方法用冻结的嵌入模型编码文本，生成**与目标变量无关**的静态表征——预测"患者是否出院"和"治疗费用"用的是同一个文本嵌入；（c）TFM 通常需要数据集特定的输出层（如 TP-BERTa 为分类和回归各训一个模型），限制了跨数据集学习。  
3. **核心矛盾**：文本字段的语义信息对预测至关重要，但需要**目标感知**的（target-aware）表征才能有效利用——同一个医疗报告对不同预测任务应有不同的重要信号。现有方法忽视了这一点。  
4. **本文要解决什么？** 构建一个端到端处理含文本字段表格数据的基础模型，能跨数据集迁移学习且无需数据集特定参数。  
5. **切入角度**：解冻文本编码器让文本表征随预测目标端到端优化 + 将目标变量的每个可能值也作为 token 输入（target-aware tokens），让模型在特征-目标交互中构建目标感知表征。  
6. **核心 idea 一句话**：解冻文本编码器 + 目标感知 token + 消除数据集特定参数 = 首个在含文本表格分类任务上全面超越 GBDT 和 TabPFN-v2 的基础模型。

## 方法详解

### 整体框架
TabSTAR 包含两阶段训练：（1）预训练：在 350 个表格数据集（253 分类 + 97 回归）上多任务预训练；（2）微调：用 LoRA 在单个下游数据集上微调。架构五模块：Verbalization（特征文本化）→ Encoding（语义+数值编码）→ Fusion（每特征独立融合）→ Interaction（跨特征自注意力）→ Prediction（共享预测头）。

### 关键设计

1. **Verbalization（特征文本化）**：
   - 做什么：将每个特征转化为"列名: 值"的文本描述。数值特征额外做 z-score 标准化 + 分位数离散化（如 "Age: 40-50 (Quantile 50-60%)"）
   - 目标变量也被 verbalize：分类任务中每个可能的目标值都被当作一个独立 token 输入（如 "Target. Decision: Hospitalized" 和 "Target. Decision: Released"）
   - 设计动机：将列名信息注入表征（大多数方法忽略列名），数值分位数描述弥补语言模型对精确数字的 tokenization 弱点

2. **目标感知 Token（Target-Aware Tokens）**：
   - 做什么：将分类目标的 $C$ 个可能值各自作为一个 token 与特征 token 一起参与 Transformer 交互
   - 核心思路：在 Interaction 阶段，目标 token 与所有特征 token 做自注意力——每个目标值 token 的输出表征编码了"这个值有多大可能是正确答案"的信息。预测时，所有目标 token 共享同一个分类头，softmax 得到概率分布
   - 设计动机：（a）消除数据集特定参数——无论数据集有多少类、类名是什么，都用同一个共享预测头；（b）目标语义信息作为输入而非仅作为标签，让模型理解"Released"和"Hospitalized"的语义差异

3. **解冻文本编码器**：
   - 做什么：使用 e5-small-v2 作为语义编码器，解冻上半部分层，让文本表征随预测目标端到端优化
   - 核心思路：冻结编码器 → 静态表征 → 与目标无关。解冻 → 表征随预训练/微调优化 → 目标感知
   - 消融验证：解冻 6 层时性能最优，即使只解冻 1 层也显著优于完全冻结。分类 normalized score 从 ~0.3（冻结）到 ~0.6（解冻 6 层）

4. **无数据集特定参数**：
   - 做什么：整个架构从编码到预测完全共享参数，没有数据集特定的输出层
   - 设计动机：TP-BERTa 分类和回归各需一个独立模型，XTab 需要数据集特定参数。TabSTAR 消除这些限制，使得 350 个数据集可以在同一个模型中联合预训练

### 训练策略
- 预训练：350 数据集（手工去重），48 小时单 A40 GPU
- 微调：LoRA（仅在预训练时解冻的层上加适配器），默认超参无需数据集特定调参
- 5-fold 交叉验证评估，避免数据泄露：每个数据集只被未见过它的预训练变体评估

### 关键发现
- **解冻编码器是核心贡献**：冻结→解冻 6 层，分类 score 翻倍。这证明了文本字段需要目标感知的动态表征，而非静态嵌入
- **预训练数据集数量呈 scaling law**：16→64→256 数据集，性能持续提升——尤其回归任务受益更大，暗示更多数据可能让回归也达到 SOTA
- **目标感知 token 消除了数据集特定参数**：同一个分类头处理所有数据集的所有类数和类名，实现了真正的跨数据集共享
- **推理效率与 GBDT 相当**：TabSTAR 推理 3.0s vs CatBoost 2.0s（GPU），远快于 TabICL（13.4s）和 TabDPT（161.4s）
- **回归仍是短板**：GBDT 在回归上仍主导，TabPFN-v2 也不如 GBDT——说明文本表格回归是一个开放问题

## 亮点与洞察
- **"解冻文本编码器"看似简单但影响深远**：之前所有方法（CM2/CARTE/TP-BERTa）都冻结语言模型，TabSTAR 首次证明解冻+端到端优化对含文本表格数据的决定性作用
- **目标感知 token 是优雅的"零数据集特定参数"设计**：将分类类别作为 token 输入而非输出层参数，使得 2 类和 100 类的数据集可以共享完全相同的架构和参数
- **数值 verbalization 的分位数策略**：用自然语言描述数值范围（"Age: 40-50, Quantile 50-60%"），比 TP-BERTa 的特殊 bin token 更自然，且利用了语言模型对数值描述的理解能力

## 局限性 / 可改进方向
- 回归任务明显落后于 GBDT——可能需要 regression-via-classification 技巧（如 TabPFN-v2）或更丰富的数值编码
- 训练较慢（493s vs CatBoost 69s），但推理时与 GBDT 相当
- 特征数量很多时可能遇到内存瓶颈（Transformer 自注意力的 $O(n^2)$）
- 预训练仅 350 数据集，scale 潜力还未充分挖掘
- 未在纯数值表格和 few-shot 场景下充分评估

## 相关工作与启发
- **vs TabPFN-v2**：TabPFN-v2 是 ICL 范式（最多 10K 样本），处理文本用静态嵌入。TabSTAR 端到端优化文本+无 scale 限制，分类 8/11 数据集胜出
- **vs CARTE**：图结构建模高基数特征，但不处理长文本且无法 scale to >2K 样本
- **vs TP-BERTa**：冻结 RoBERTa 且需数据集特定输出层。TabSTAR 解冻+共享预测头
- **vs AutoGluon-Multimodal**：多模型集成系统，非单一模型。TabSTAR 是单一端到端模型
