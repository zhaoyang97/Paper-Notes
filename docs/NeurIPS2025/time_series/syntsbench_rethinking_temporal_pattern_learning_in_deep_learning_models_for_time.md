# SynTSBench: Rethinking Temporal Pattern Learning in Deep Learning Models for Time Series

**会议**: NEURIPS2025  
**arXiv**: [2510.20273](https://arxiv.org/abs/2510.20273)  
**代码**: [GitHub](https://github.com/TanQitai/SynTSBench)  
**领域**: time_series  
**关键词**: benchmark, time series forecasting, synthetic data, evaluation framework, temporal patterns  

## 一句话总结
提出 SynTSBench，一个基于合成数据的时序预测模型评估框架，通过可编程特征配置（趋势/周期/噪声/依赖/多变量）和理论最优基准，系统揭示当前深度学习模型在各类时序模式上的能力边界。

## 背景与动机
- 当前时序预测模型在标准 benchmark 上表现好但在真实应用中仍挣扎
- 现有评估框架的两大局限：
  1. 缺乏特征隔离：真实数据中趋势/季节性/依赖等交织，无法针对性评估
  2. 缺乏理论性能上界：不知道"最优解"是什么，优化如"盲盒调参"
- 需要一种可控的评估范式来精确归因模型的优劣势

## 核心问题
能否通过合成数据系统地隔离各类时序特征，建立理论最优基准，从而精确评估模型在每种模式上的真实能力？

## 方法详解
1. **数据集设计**（6 大维度）：
   - 趋势信号：11 种趋势函数（指数、高斯、对数、多项式等）
   - 周期信号：10 种周期模式（2/3/5/10 正弦叠加、指数调制正弦波等）
   - 噪声鲁棒性：不同 SNR + 多种噪声分布（高斯/均匀/Laplace/t/Lévy）
   - 异常鲁棒性：点异常、脉冲异常、均值漂移、趋势漂移
   - 短/长程依赖：ARMA 过程、随机游走、白噪声
   - 多变量关系：延迟关系、线性加性、条件关系、非线性变换、反馈系统
2. **理论最优基准**：为每种模式类型建立数学最优解
3. **评估协议**：滑动窗口，输入长度 96，预测长度 {10,24,48,96,192}

## 实验关键数据
- **评测模型**：15个微调模型 + 3个零样本模型（Chronos、TimeMoE、Moirai）
- **趋势任务**：DLinear 在指数趋势上 MSE 3.40e-08（接近理论最优 0），但 Autoformer 高达 8.45
- **周期任务**：PatchTST、DLinear 表现较好；多频率叠加时所有模型差距明显增大
- **噪声鲁棒性**：SNR 降低时性能急剧下降；不同噪声类型对模型影响差异大
- **多变量**：模型在条件关系和非线性变换上普遍表现差
- **关键发现**：没有单一架构在所有时序模式上达到最优——模型设计存在内在权衡
- **零样本模型**：泛化能力仍然不足，尤其在极端噪声和复杂依赖上

## 亮点
- 理论最优基准（绿色标注）：第一次为时序预测提供"上帝视角"评估
- 揭示了反直觉的发现：简单的 DLinear 在趋势任务上远超复杂 Transformer
- 可编程特征配置：研究者可以精确控制评估维度，做"控制变量实验"
- 为模型选择提供了定量指导：不同场景应选不同模型

## 局限性 / 可改进方向
- 合成数据仍然不能完全反映真实世界数据的复杂性（缺乏真实数据的杂糅效应）
- 当前评估以 MSE/MAE 为主，概率预测评估不足
- 未涵盖非平稳性的系统评估
- 与真实数据 benchmark 的性能排名对比分析可以更深入

## 启发与关联
- 核心 insight：模型评估不应只看最终排名，而应理解每种模式上的能力边界
- 对模型设计的启示：可以针对弱点模式（如多频率叠加、复杂多变量）专门改进架构
- 可扩展为时序模型的"体检报告"工具
