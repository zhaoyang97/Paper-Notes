# RiverMamba: A State Space Model for Global River Discharge and Flood Forecasting

**会议**: NeurIPS 2025  
**arXiv**: [2505.22535](https://arxiv.org/abs/2505.22535)  
**代码**: [Project](https://hakamshams.github.io/RiverMamba)  
**领域**: 科学计算 / 水文学  
**关键词**: 河流流量预测, 洪水预报, Mamba, 时空建模, 全球尺度, GloFAS

## 一句话总结
首个能在 0.05°（~5.5km）全球网格上做 7 天河流流量预报的深度学习模型——用空间填充曲线将 3D 时空点序列化后输入双向 Mamba block，结合 ECMWF HRES 气象预报，在 1.5-500 年重现期洪水检测上 F1 =0.459 超越 LSTM（0.358）和物理模型 GloFAS。

## 研究背景与动机
1. **领域现状**：物理水文模型（GloFAS by ECMWF）是全球洪水预报的标准，但计算极贵（需要在超算上运行完整水文循环）。AI 方法近年取得进展：Google 的 ED-LSTM 在局部流域上表现好，NeuralHydrology 系列工作证明了 LSTM 在水文上的有效性。
2. **现有痛点**：(a) LSTM 将每个流域独立建模，不考虑流域间的空间关系（上下游交互）；(b) GNN 在图上建模空间关系但扩展不到全球尺度（>100K 活跃格点）；(c) Transformer 的 $O(n^2)$ 复杂度对全球 0.05° 网格（数十万格点 × 多天）不可行。
3. **核心矛盾**：全球洪水预报需要同时建模：(i) 长时间序列（需 30+ 天历史）、(ii) 全球空间关系（河网拓扑）、(iii) 气象驱动（天气预报作为未来条件输入），但现有方法都不能同时做到这三点。
4. **本文要解决什么**：构建首个全球 0.05° 分辨率、7 天前瞻的 AI 河流流量预报系统。
5. **切入角度**：Mamba 的 selective scan 机制提供 $O(n)$ 线性复杂度处理超长序列；空间填充曲线将 2D 空间格点无损地编码为 1D 序列供 Mamba 处理。
6. **核心idea一句话**：空间填充曲线序列化全球格点 → 双向 Mamba 编码时空上下文 → ECMWF 气象预报逐天驱动预测。

## 方法详解

### 整体框架
两阶段 Encoder-Decoder：(1) **Hindcast layers**（编码器）：聚合过去 $T=4$ 天的 ERA5 再分析数据到时空表示；(2) **Forecast layers**（解码器）：逐天接入 ECMWF HRES 气象预报，连续生成 $L=7$ 天的前瞻预报。

### 关键设计

1. **空间填充曲线序列化**:
   - 做什么：将全球 3D 时空格点 $(lat, lon, time)$ 通过双射函数 $\Phi: \mathbb{Z}^3 \to \mathbb{N}$ 转为 1D 序列
   - 核心思路：使用 Sweep + Gilbert 组合曲线，不同 Mamba block 用不同曲线——确保全部空间邻域关系都被覆盖
   - 设计动机：不同曲线保持不同的空间局部性——Hilbert 保持 2D 邻近性，Sweep 保持经/纬方向连续性

2. **双向 Mamba + LOAN**:
   - 做什么：bidirectional Mamba block 在序列化的时空特征上做双向状态空间建模
   - LOAN（Location-Aware Adaptive Normalization）：将静态河流属性（流域面积、坡度等）通过 FiLM 机制注入每一层的归一化步骤
   - 设计动机：河流属性是不变的物理先验（流域大不大、坡度陡不陡影响响应时间），应作为条件信号而非输入特征

3. **洪水加权损失**:
   - 做什么：$\mathcal{L} = \text{MSE} \times w_{\text{flood}}$，按重现期加权——稀有洪水事件权重远高于常规流量
   - 数据处理：log 变换处理流量的巨大动态范围（<1 m³/s → >10,000 m³/s）
   - 预测增量：模型预测 $\Delta X$ 而非绝对值——减少分布偏移问题

### 损失函数 / 训练策略
训练数据：1979-2018 GloFAS 再分析。验证：2019-2020。测试：2021-2024。GRDC 实测数据做进一步验证。

### 关键发现
- 洪水 F1 从 LSTM 的 0.358 提到 0.459（+28%），是目前全球尺度 AI 洪水预报最好结果
- Lead time >48h 优势显著——空间建模使 RiverMamba 能利用上游信息做更长期预测
- GRDC 实测验证 R² 只有 0.506——因为实测数据受大坝/水库等人为调控影响，模型未建模这些因素

## 亮点与洞察
- **Mamba 在科学计算中的优美应用**：利用 Mamba $O(n)$ 复杂度处理全球尺度时空序列是非常自然的选择。空间填充曲线的引入解决了将 2D 空间编码为 1D 序列的问题。
- **物理先验的集成方式**：LOAN 将静态河流属性作为归一化条件而非拼接到输入——这比简单拼接更优雅，因为物理属性影响的是特征的分布而非值。
- **全球覆盖的实际意义**：0.05° 分辨率的 7 天全球洪水预报对发展中国家（缺乏密集观测站网）尤其有价值。

## 局限性 / 可改进方向
- 不建模人为水管理（大坝/水库调控），GRDC 实测上性能下降明显
- ERA5 再分析数据有 5 天延迟，实际运行需要 near-real-time 数据
- 仅预测增量 $\Delta X$，依赖前一天观测值的质量
- 空间填充曲线的选择较为启发式，最优组合未充分探索
- 极端洪水（500 年重现期）的训练样本极少，模型在尾部预测仍有不确定性

## 相关工作与启发
- **vs Google ED-LSTM (Nearing et al., 2024)**：ED-LSTM 在局部流域上优于 GloFAS 但无全球空间建模；RiverMamba 是首个全球空间相关的 AI 模型
- **vs GloFAS (ECMWF)**：物理模型有完整的物理约束但计算极贵；RiverMamba 推理速度快几个数量级
- **vs ClimaX/Pangu-Weather**：这些是全球天气预报模型；RiverMamba 是全球水文预报模型——使用天气预报作为驱动输入

### 补充技术细节
- 训练时全球采样 P 个点，推理时可使用任意点集或全球密集网格
- 流量值通过 $sign(\Delta\hat{x})\log(1+|\Delta\hat{x}|)$ 变换处理跨数量级的动态范围
- LOAN 层将静态河流属性（流域形态、坡度等）编码为位置感知的偏置
- Hindcast blocks 用 3 层（T=4, 每层时间下采样×2），编码维度 K=192
- Forecast blocks 序贯处理 L=7 天 HRES 预报，第 l 个 block 处理第 l 天
- Flash-Attention 基线在推理时间和准确率上均劣于 Mamba（附录验证）
- 训练期 1979-2018，验证 2019-2020，测试 2021-2024（时间外推）

- 洪水重现期从 1.5 年到 500 年，覆盖了从常见到极端洪水的全谱
