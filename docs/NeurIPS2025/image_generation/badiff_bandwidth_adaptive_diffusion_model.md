# BADiff: Bandwidth Adaptive Diffusion Model

**会议**: NeurIPS 2025  
**arXiv**: [2510.21366](https://arxiv.org/abs/2510.21366)  
**代码**: [GitHub](https://github.com/xzhang9308/BADiff)  
**领域**: 扩散模型 / 图像压缩 / 自适应生成  
**关键词**: bandwidth-adaptive, diffusion model, entropy conditioning, early stopping, cloud streaming  

## 一句话总结
提出 BADiff——首个带宽自适应扩散模型，通过将目标熵约束作为条件嵌入扩散过程，使模型根据实时带宽动态调整生成质量并自适应提前终止采样，在保持感知质量的同时减少计算开销，避免了传统"高质量生成+后压缩"流程中的压缩伪影和计算浪费。

## 研究背景与动机

1. **领域现状**：扩散模型能生成高质量图像但部署在云-设备场景时面临带宽瓶颈。传统方案是生成高质量图像后做有损压缩传输，但压缩会擦除扩散模型精心构建的细节纹理。

2. **现有痛点**：(a) 高质量生成+后压缩=冗余计算+质量损失；(b) 粗暴提前终止扩散步数效果差——模型未针对提前终止优化；(c) 缺乏生成过程中的带宽感知。

3. **核心矛盾**：生成过程不知道下游传输约束，导致计算和质量双重浪费。

4. **切入角度**：扩散模型逐步从粗到细精炼——天然适合按需终止。关键是训练时让模型学会在不同终止点都产生好的感知质量。

5. **核心 idea 一句话**：将目标熵（带宽代理量）作为条件注入扩散模型，配合熵正则化损失和自适应采样策略，实现按需质量生成。

## 方法详解

### 关键设计

1. **熵条件嵌入**：将目标熵值（代表可用带宽）编码为轻量嵌入注入 UNet 条件通道，训练时暴露于不同熵约束。
2. **熵正则化损失**：$\mathcal{L} = \mathcal{L}_{\text{denoise}} + \lambda \mathcal{L}_{\text{entropy}}$，确保生成图像的实际熵接近目标熵。
3. **自适应采样策略**：动态确定最优终止步，在计算效率和质量间取得最优平衡。

## 实验关键数据

| 方法 | 感知质量 | 计算效率 | 带宽适配 |

| **BADiff** | **优** | **高** | **主动** |

### 关键发现
- BADiff 在多种带宽约束下均优于标准扩散+压缩流程
- 自适应终止节省大量计算且不损失感知质量
- 轻量质量嵌入即可有效引导去噪轨迹

## 亮点与洞察
- **将带宽约束从事后处理前移到生成过程**——避免了"先精心生成再粗暴压缩"的矛盾
- **利用扩散模型粗到细的天然特性**——低带宽只需粗结构，高带宽再加细节
