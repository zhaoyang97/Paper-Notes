# Blameless Users in a Clean Room: Defining Copyright Protection for Generative Models

**会议**: NeurIPS 2025  
**arXiv**: [2506.19881](https://arxiv.org/abs/2506.19881)  
**代码**: 有  
**领域**: AI伦理 / 版权理论  
**关键词**: 版权保护, 净室原则, 生成模型, 差分隐私, NAF

## 一句话总结
重建生成模型可证明版权保护的理论基础——证明现有的 Near Access-Freeness (NAF) 定义不能防止逐字复制（"被污染"），提出"无辜用户"(blameless) 框架和净室版权保护 ($(\kappa,\beta)$-clean) 定义，其中用户在反事实"净室设置"中不会复制则在真实世界中也不太可能复制，并证明差分隐私训练在"黄金数据集"假设下蕴含净室版权保护。

## 研究背景与动机

1. **领域现状**：生成模型可能在输出中再现训练数据中的版权内容。Vyas et al. (2023) 提出 Near Access-Freeness (NAF) 作为"可证明版权保护"的形式化定义，要求模型的输出分布与未见版权数据时的输出分布相近。

2. **现有痛点**：
   - NAF 定义有根本缺陷——本文证明 NAF 模型仍可能被用于逐字复制训练数据（"tainted"模型）
   - NAF 不保护多次查询或数据依赖的提示场景
   - 缺乏在法律和数学上都站得住脚的版权保护框架
   - "差分隐私隐含版权保护"是常见直觉但缺乏严格证明

3. **核心矛盾**：生成模型输出有时会再现版权内容，但并非所有再现都应归咎于用户——用户可能完全不知道训练数据包含什么。需要区分用户主动引发的复制与模型自发的复制。

4. **本文要解决什么？** (a) 证明 NAF 的不足；(b) 提出更好的形式化定义；(c) 将 DP 与版权保护形式关联。

5. **切入角度**：借鉴软件工程的"净室开发"(clean-room design)——在不接触原始代码的环境中重新实现功能以避免版权侵犯。将此原则数学化：如果用户在不含版权数据的"净室模型"中的行为不会导致复制，那么在真实模型中也不应导致复制。

6. **核心 idea 一句话**：保护"无辜用户"——在反事实净室中不会复制的用户，在真实世界中复制的概率应有上界。

## 方法详解

### 整体框架
三层递进：(1) 证明 NAF 的失败（存在 NAF 但 tainted 的模型）；(2) 定义"tainted"概念和"blameless"框架；(3) 实例化为净室版权保护并与 DP 连接。

### 关键设计

1. **Tainted 模型的定义与 NAF 的反例**:
   - 做什么：证明 NAF 不是版权保护的充分条件
   - 核心思路：构造一个 NAF 模型，但用户可以通过精心设计的多轮查询序列提取完整的逐字复制——NAF 只约束单查询分布的统计距离，不约束多查询下的联合行为
   - 设计动机：如果基础定义有缺陷，在其上建立的所有声明都不可靠——必须先"打破"旧定义才能建立新基础

2. **Blameless 版权保护框架**:
   - 做什么：定义一个通用框架来形式化"保护无辜用户"
   - 核心思路：定义用户的"blame"（责任）——如果用户的使用模式在任何不含版权内容的模型上也会产生相似输出，则用户是无辜的。保护 = 无辜用户的复制概率有上界
   - 设计动机：将法律概念"自愿性"(volition) 和"因果性"(causation) 映射到数学定义

3. **净室版权保护 $(\kappa, \beta)$-clean**:
   - 做什么：blameless 框架的具体实例化
   - 核心思路：训练算法是 $(\kappa, \beta)$-clean 的，如果对于每个在反事实净室环境中复制概率 $\leq \beta$ 的用户，其在真实世界中复制概率 $\leq \kappa$。用户可选择风险容忍度 $\kappa$，相应调节 $\beta$（自己行为的约束程度）
   - 设计动机：类比软件净室开发——如果你在不接触原始代码的环境中自然写出的代码与原始不同，那么在接触原始代码后写出相似的代码更可能是巧合

4. **DP 蕴含净室版权保护**:
   - 做什么：形式化 "DP 隐含版权保护" 的直觉
   - 核心思路：如果训练算法是 $(\varepsilon, \delta)$-DP 的，且数据集是"黄金的"（golden：每个版权作品至多出现一次），则算法是 $(e^\varepsilon \cdot \beta + \delta, \beta)$-clean 的
   - 设计动机：DP 保证模型输出对单个训练样本不敏感——如果版权作品仅出现一次，DP 保证移除它不会显著改变模型行为

### 损失函数 / 训练策略
- 纯理论工作——定义 + 定理 + 反例构造

## 实验关键数据

### 主结果（理论定理）

| 定理 | 内容 | 意义 |
|------|------|------|
| Theorem 1 | 存在 NAF 但 tainted 的模型 | NAF 基础不牢 |
| Theorem 2 | $(\kappa,\beta)$-clean 排除 tainted 模型（温和假设下） | 净室保护比 NAF 更强 |
| Theorem 3 | $(\varepsilon,\delta)$-DP + 黄金数据集 → $(e^\varepsilon\beta + \delta, \beta)$-clean | DP 是净室保护的充分条件 |

### 消融：定义之间的蕴含关系

| 属性 | NAF | $(\kappa,\beta)$-clean | DP (黄金数据集) |
|------|-----|----------------------|---------------|
| 排除 tainted | ✗ | ✓ | ✓ |
| 排除逐字复制 | ✗ | ✓ | ✓ |
| 保护多查询攻击 | ✗ | ✓ | ✓ |
| 用户可调风险 | ✗ | ✓（调 $\beta$） | ✓（调 $\varepsilon$） |

### 关键发现
- **NAF 的根本缺陷**：NAF 模型可以被设计为允许逐字复制——因为 NAF 只约束单查询边际分布
- **净室保护的直觉对应**：$\kappa$ 是真实世界复制概率的上界，$\beta$ 是用户在净室中的"自然复制率"——二者解耦
- **DP 的黄金数据集要求**：仅 DP 不够——如果同一版权作品出现 100 次，DP 无法保护
- **严格责任 vs 过错责任**：论文不主张改变法律，而是提出"当严格责任不公正时"的技术缓解方案（赔偿政策）

## 亮点与洞察
- **"反驳 NAF"**是勇敢但必要的——NAF 是该领域唯一的先行定义，证明其不足为后续工作扫清了基础
- **净室隐喻**在法律和技术之间搭建了优美的桥梁——法律人可以理解"净室"概念，技术人可以理解 $(\kappa,\beta)$ 参数
- **DP→版权保护**的形式化填补了一个重要的理论空白——之前只是直觉性的类比
- 提出的赔偿政策（Section 9）将理论与实际部署连接

## 局限性 / 可改进方向
- 净室保护假设存在"净室模型"（不含版权数据的对照），实践中构造此对照可能困难
- "黄金数据集"要求可能在大规模网络爬取中难以满足
- 仅考虑逐字复制和近似复制，未覆盖"风格模仿"等更微妙的版权争议
- 论文明确不做法律分析——是否法院会采纳此框架是开放问题

## 相关工作与启发
- **vs Vyas et al. (NAF)**：本文的直接前驱和主要批判对象。证明 NAF 不足并提出替代
- **vs 差分隐私文献**：DP 的版权含义之前是非形式化讨论，本文给出首个严格证明
- **vs 水印方案（如 BitMark）**：水印是检测性的（事后追踪），本文的保护是预防性的（事前保证）——两者互补
- 对 AI 法规制定者有直接参考——提供了"版权保护"的可操作化数学定义
