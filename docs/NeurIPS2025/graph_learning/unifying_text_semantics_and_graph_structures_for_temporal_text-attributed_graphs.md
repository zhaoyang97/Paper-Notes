# Unifying Text Semantics and Graph Structures for Temporal Text-attributed Graphs with LLMs

**会议**: NeurIPS 2025  
**arXiv**: [2503.14411](https://arxiv.org/abs/2503.14411)  
**代码**: 待确认  
**领域**: 图学习 / NLP  
**关键词**: 时序文本属性图, LLM, 语义-结构协同编码, 链接预测, Cross框架

## 一句话总结
提出 Cross 框架——用 LLM 在策略采样的时间点上动态总结节点邻域的语义演变（Temporal Reasoning Chain），然后通过语义-结构协同编码器双向融合文本语义和图结构时序信息，在时序链接预测上平均 MRR 提升 24.7%，工业数据（微信）上 AUC 提升 3.7%。

## 研究背景与动机
1. **领域现状**：时序图（Temporal Graph）建模是推荐、社交网络、金融风控的核心任务。TGAT、TGN、DyGFormer 等方法用注意力/记忆模块捕获图的结构动态。
2. **现有痛点**：时序文本属性图（TTAG，如社交帖子/新闻/论文引用网络）中节点有丰富的文本属性，但现有 TGNN 将文本编码为静态嵌入，完全忽略了文本语义随时间的演变——一个用户发帖的话题在不断变化，一篇论文被引用的语境也在变。
3. **核心矛盾**：图的结构动态和语义动态是耦合的（用户发了新话题→吸引了不同的交互），但现有方法只建模了其中一维。
4. **本文要解决什么**：让 TGNN 同时利用文本语义的时序演变和图结构的时序动态。
5. **切入角度**：LLM 天然理解文本语义和时间上下文——用 LLM 在关键时间点总结节点邻域的语义状态，然后与 TGNN 的结构信息双向融合。
6. **核心idea一句话**：LLM 提取语义时序链 + TGNN 提取结构时序 + 跨模态混合器融合 = 时序文本图的全面建模。

## 方法详解

### 整体框架
两条并行通路：(1) 语义通路：在策略采样的时间点上用 LLM 生成邻域语义摘要 → Transformer encoder 编码语义时序；(2) 结构通路：标准 TGNN 编码结构动态。跨模态 MLP 混合器融合最终表示。

### 关键设计

1. **时序推理链（Temporal Reasoning Chain）**:
   - 做什么：在节点的交互历史中策略采样 $m$ 个时间点 $\hat{t}_1, ..., \hat{t}_m$，每个时间点用 LLM 多轮提示总结该节点邻域的语义状态
   - 采样策略：等间隔 $\lceil n/m \rceil$ 从 $n$ 个交互中选取
   - LLM 输出：$\hat{d}_u(\hat{t})$——节点 $u$ 在时间 $\hat{t}$ 的语义摘要
   - 设计动机：不可能对每个交互都调用 LLM（太贵），策略采样在效率和粒度间取平衡

2. **语义-结构协同编码器**:
   - **语义层**：Transformer encoder 处理 LLM 生成的摘要序列，time2vec 编码时间
   - **结构层**：任意 TGNN（TGAT/TGN/DyGFormer）编码特征——框架对具体 TGNN 架构无关
   - **跨模态混合器**：2 层 MLP 融合最新语义表示和结构表示
   - 最终表示：语义（mean-pooled）+ 结构 + 混合 三者拼接

3. **即插即用设计**:
   - 做什么：Cross 可叠加在任意 TGNN 上（TGAT+Cross, TGN+Cross, DyGFormer+Cross）
   - 设计动机：不需要设计新的 TGNN，只需在现有的基础上添加语义通路

### 损失函数 / 训练策略
标准链接预测二元交叉熵损失。LLM 推理是离线预处理步骤。

## 实验关键数据

### 主实验（时序链接预测 MRR %，Transductive）

| 基础模型 | 原始 MRR | +Cross MRR | 提升 |
|----------|---------|-----------|------|
| TGAT | 66.06 | **95.58** | +29.5 |
| TGN | 73.05 | **95.84** | +22.8 |
| DyGFormer | 79.93 | **95.31** | +15.4 |

| 数据集 | 最大提升 | +Cross 后 MRR |
|--------|---------|--------------|
| Enron (email) | +29.52 | 95.58 |
| GDELT (events) | +44.02 | 81.63 |
| 工业 (微信) | **+86.23%** | 86.97 |

### 消融实验
| 配置 | 关键发现 | 说明 |
|------|---------|------|
| Inductive 设置 | 平均 +40.23% | 语义信息对新节点帮助更大 |
| LLM-only (DeepSeek 零/少样本) | 远低于 TGNN | 纯语义不够 |
| 语义通路 only | 低于 Cross | 结构信息仍不可替代 |
| 结构通路 only | = 原始 TGNN | 没有语义损失 |
| 节点分类（工业） | AUC +3.7% | 金融风控任务验证 |
| 推理时间点数 $m$ | 5-10 最优 | 更多收益递减 |

### 关键发现
- 工业数据上 +86.23% 是最震撼的数字——说明真实场景中语义信息极其重要但被浪费
- Cross 在 Inductive 设置上的提升远大于 Transductive——语义信息帮助泛化到新节点
- LLM-only 基线表现差——纯语义不能替代结构信息，两者是互补的

## 亮点与洞察
- **语义-结构解耦再融合**：将两条信息通路解耦后再融合比端到端 jointly training 更有效——因为 LLM 和 TGNN 各自擅长不同类型的信息。
- **工业验证的说服力**：微信金融风控数据上 +3.7% AUC 是真实业务中的显著提升。
- **即插即用的实用性**：不需要更换 TGNN 架构，只需添加语义通路——降低了实际采用的门槛。

## 局限性 / 可改进方向
- LLM 推理在每个节点的多个时间点上都要调用，计算成本高
- 策略采样的粒度由 $m$ 控制，但最优 $m$ 是数据集依赖的
- Prompt engineering 可能对不同领域需要调整
- 假设 LLM 能准确捕获语义动态——对非英语文本可能需要多语言 LLM

## 相关工作与启发
- **vs TGAT/TGN/DyGFormer**：这些只用结构时序，Cross 添加语义时序维度
- **vs TAG 方法（TextSAGE 等）**：TAG 方法用静态文本嵌入，Cross 用 LLM 动态总结
- **vs GraphSAGE + LLM**：GraphSAGE 不处理时序，Cross 通过时间推理链处理语义演变
