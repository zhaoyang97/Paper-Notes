# Moscat: Mixture of Scope Experts at Test for Generalizing Deeper GNNs

**会议**: NeurIPS 2025
**arXiv**: [2409.06998](https://arxiv.org/abs/2409.06998)
**代码**: https://github.com/Hydrapse/moscat
**领域**: 图神经网络 / 泛化理论
**关键词**: GNN深度, PAC-Bayes界, 专家混合, 测试时门控, 同质性子群

## 一句话总结
通过 PAC-Bayes 界证明 GNN 深度变化导致不同同质性子群间的泛化偏好漂移，提出 Moscat——后处理注意力门控模型，在测试时自适应组合不同深度的独立训练 GNN 专家。

## 研究背景与动机

1. **领域现状**：GNN 加深可扩大感受野但面临过平滑/优化退化/泛化差三重困境。

2. **现有痛点**：Oracle 集成显示不同深度正确预测的节点集重叠很低（Jaccard < 0.3）。现有 Graph MoE 用软作用域导致浅层专家过拟合深层噪声。

3. **核心矛盾**：核心问题不是“深还是浅”而是“对哪些节点用多深”。

4. **本文要解决什么？** 利用不同深度 GNN 的互补优势提升整体性能。

5. **切入角度**：理论（PAC-Bayes 界证明泛化偏好漂移）+ 方法（解耦的专家-门控范式）。

6. **核心 idea**：各深度 GNN 独立训练（硬作用域），轻量注意力门控在测试时学习节点自适应的专家权重。

## 方法详解

### 整体框架
三步：(1) 独立训练 $L_{max}+1$ 个 GNN 专家；(2) 收集 logits + 增强后训练注意力门控；(3) 测试时用门控权重融合。

### 关键设计

1. **解耦的专家-门控范式（硬作用域）**:
   - 做什么：各深度专家完全独立训练，门控在别的数据集上学习
   - 设计动机：软作用域导致浅层专家过拟合深层噪声

2. **异质性偏置样本过滤**:
   - 做什么：清洗门控训练数据中的噪声样本
   - 设计动机：专家容易在异质性节点上过拟合

3. **作用域感知 Logit 增强**:
   - 做什么：增强专家 logits 帮助门控识别泛化模式
   - 核心思路：标签嵌入近似同质性 + 结构编码检测过平滑

## 实验关键数据

### 主实验

| 数据集 | 类型 | GNN+Moscat 提升 |
|--------|------|-------------|
| amazon-ratings | 异质 | 显著提升 |
| Penn94 | 异质 | 显著提升 |
| PubMed | 同质 | 小幅提升 |

### 消融

| 配置 | 效果 | 说明 |
|------|------|------|
| 软作用域 vs 硬作用域 | 硬更优 | 浅层专家在软作用域下过拟合 |
| w/o 标签嵌入 | 下降 | 同质性信息对门控重要 |
| w/o 异质性过滤 | 下降 | 训练噪声影响门控质量 |

### 关键发现
- Oracle 集成的提升潜力巨大
- 异质图上提升更大（符合 Theorem 3.3）
- 任何 GNN 架构都受益

## 亮点与洞察
- PAC-Bayes 界揭示了深度困境的本质：不是“深层差”而是“深层在某些子群好某些差”
- 后处理范式优于联合训练范式

## 局限性
- 需要预留 holdout 集训练门控
- 理论分析基于 CSBM 合成图

## 相关工作
- **vs Graph MoE**: 软作用域过拟合。Moscat 硬作用域解耦
- **vs Skip-connection**: 前向传播中混合。Moscat 后处理中混合

### 补充技术细节
- Theorem 3.3 中的 $\Gamma_{L-1} = \mathbb{E}[(p_o - q_o)^{L-1}]$ 度量 L 跟同质性系数。当 $p_S > p_m$ 和 $p_S < p_m$ 时，最优深度不同
- 标签嵌入：$\xi_{label}^{(L)} = [\tilde{A}^1 Z^{(L)} \| \cdots \| \tilde{A}^{L_{max}} Z^{(L)}]$，用专家的 logits 作为伪标签近似各阶同质性
- 结构编码：平滑距离 $\bar{\epsilon}_v^{(L)} = \|X_v^{(L)} - X_v^{(0)}\|_2$ 和 $\tilde{\epsilon}_v^{(L)} = \|X_v^{(L)} - X_v^{(\infty)}\|_2$ + PageRank
- 门控模型的注意力机制为每个专家使用独立的变换权重 $W_L, a_L$（因为不同专家过拟合/过平滑模式不同）
- 实验中 MLP 作为分类器 $f(\cdot)$ 效果最佳，优于更复杂的 GNN/Transformer
- 在 amazon-ratings/Penn94/ogbn-arxiv/PubMed/roman-empire 等多个数据集上超越 SOTA
- 对 GCN/GAT/GraphSage/CIN 等多种 GNN 架构均有效，证明方法的通用性
- 轻量级设计：门控模型的参数量和训练时间远小于专家模型本身
- Figure 3 (Bottom) 显示多深度集成的提升显著超过同深度多种子集成
- holdout 集可以是验证集的子集，实验中使用 25% 标注数据作为 holdout
- Moscat 在 Graph Transformer 和 Graph MoE 方法之上也取得了一致的提升
- 在同质图（PubMed）上提升较小，在异质图（amazon-ratings）上提升巨大——完美符合 Theorem 3.3 的预测
- 整个 Moscat 流程无需修改任何专家的训练过程，完全是后处理
- 对 $L_{max} = 6$ 的设置，需要训练 7 个专家模型（含 MLP 作为 0-hop 专家）
