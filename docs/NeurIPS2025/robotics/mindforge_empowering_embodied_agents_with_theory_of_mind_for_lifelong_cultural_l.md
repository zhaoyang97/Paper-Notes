# MindForge: Empowering Embodied Agents with Theory of Mind for Lifelong Cultural Learning

**会议**: NeurIPS 2025  
**arXiv**: [2411.12977](https://arxiv.org/abs/2411.12977)  
**代码**: https://github.com/tapri-lab/mindforge  
**领域**: LLM Agent / 具身智能  
**关键词**: Theory of Mind, 文化学习, 多智能体协作, Minecraft, 知识蒸馏

## 一句话总结
MindForge 为 LLM 驱动的具身智能体引入显式的心智理论（ToM）表征、自然语言通信和多组件记忆系统，使开源 LLM 智能体通过与专家协作对话（无需梯度更新）大幅提升任务完成率，在 Minecraft 中比 Voyager 多获得 3× 科技树里程碑和 2.3× 独特物品。

## 研究背景与动机

1. **领域现状**：Voyager 等基于 LLM 的终身学习智能体在 Minecraft 中展示了持续探索和技能获取的能力，但严重依赖 GPT-4 的代码生成能力。
2. **现有痛点**：用开源 LLM（Mistral-7B、Llama 3.1-8B）替代 GPT-4 后，Voyager 连收集泥土这种最基础的任务都无法完成（成功率仅 4-7%）。现有智能体都是孤立学习，忽略了人类智能中社会学习的核心地位。PEFT 微调也没用——用 GPT-4 执行轨迹微调 Mistral-7B 后成功率几乎没有提升。
3. **核心矛盾**：开源 LLM 代码生成和常识推理能力不足，但直接微调无法有效传递结构化知识。缺少一种无需梯度更新、通过交互就能提升弱模型能力的机制。
4. **本文要解决什么？** 让能力不足的开源 LLM 智能体通过社会交互（与专家对话）实现能力提升，同时验证弱-弱协作是否也能带来涌现式进步。
5. **切入角度**：借鉴发展心理学的文化学习理论——人类通过观察他人视角、推断他人信念/意图来内化技能。将此建模为智能体间的显式 ToM 推理 + 自然语言对话。
6. **核心 idea 一句话**：用 BDI（信念-欲望-意图）因果模板赋予智能体显式 ToM 表征，通过自然语言对话实现社会化知识蒸馏，无需梯度更新就能弥合开源 LLM 与 GPT-4 的能力差距。

## 方法详解

### 整体框架
MindForge 在 Voyager 基础上增加三大组件：(1) 基于 BigToM 因果模板的结构化 BDI 表征，链接感知-信念-欲望-行动；(2) 基于 Minecraft 聊天系统的自然语言通信接口；(3) 包含情景记忆、语义记忆和程序记忆的 Soar 认知架构式记忆系统。智能体任务失败后发起通信，专家通过推断伙伴心理状态提供针对性反馈。

### 关键设计

1. **结构化 ToM 表征（BDI + BigToM）**:
   - 做什么：为每个智能体维护关于自身和伙伴的结构化心理状态表征
   - 核心思路：信念分四类——感知相关（来自 Minecraft API）、任务相关（对目标的理解）、交互相关（从对话中提取）、伙伴相关（对伙伴心理状态的推断）。每类信念由独立 LLM 调用生成，形成因果图结构
   - 设计动机：无结构的 ToM 在复杂任务上效果差（消融实验 Table 8 证实）。结构化表征让专家能精确诊断新手的错误信念（如"挖泥土需要工具"的错误认知），并提供针对性纠正

2. **自然语言通信 + 视角转换**:
   - 做什么：智能体任务失败后通过游戏内聊天与伙伴交流，专家基于对伙伴心理模型的推理提供帮助
   - 核心思路：专家使用自己内部维护的伙伴 BigToM 图模拟对方知道什么、相信什么、需要什么，生成回应时显式考虑对方视角。随着对话进行，伙伴信念持续更新
   - 设计动机：修正两类主要失败：(a) 错误信念（如不知道泥土不需要工具）→ 通过对话直接纠正；(b) 代码生成错误（如 Mineflayer API 误用）→ 专家提供正确用法。视角转换使反馈针对性更强

3. **三组件记忆系统**:
   - 做什么：情景记忆存储过去失败经历和社交互动；语义记忆存储任务/环境通用知识；程序记忆（技能库）存储可复用代码
   - 核心思路：情景记忆存储与当前任务相似的历史失败场景，避免重复犯错。语义记忆从情景中抽取通用知识（如"挖掘铁矿需要石镐"）。三种记忆协同影响信念更新和行动生成
   - 设计动机：消融实验表明去掉情景记忆后成功率下降（Table 9），验证了失败经验复用的价值

### 训练策略
- 无梯度更新，纯推理时优化（test-time compute scaling）
- 通信协议：任务失败后触发对话，动作和通信轮次交替进行
- 默认 4 次行动机会，通信轮次穿插其中

## 实验关键数据

### 主实验

| 设置 | 模型 | 泥土收集成功率 | 木材收集成功率 |
|------|------|--------------|--------------|
| Voyager 单独 | GPT-4 | 100% | 100% |
| Voyager 单独 | Mistral-7B | 7% | 4% |
| Voyager 单独 | Mixtral-8x7B | 27% | 27% |
| Voyager + PEFT | Mistral-7B | 7% | 12.5% |
| MindForge 无交互 | Mistral-7B | 37.5% | 33.3% |
| MindForge + GPT-4 专家 | Mistral-7B | +16.16% | +12.52% |
| MindForge + GPT-4 专家 | Mixtral-8x7B | +37.51% | +20.8% |
| MindForge + 人类专家 | 各模型平均 | +57.85% | — |

### 关键发现
- 开源 LLM 通过 MindForge 协作后成功率大幅提升（Mixtral: 泥土 27% → 66.6%），但仍无法完全匹配 GPT-4
- 通信轮次越多性能越好——泥土收集随通信轮次从 ~30% 上升到 ~70%
- 弱-弱协作（两个 Mixtral 智能体）单独效果不好，但经过一轮 GPT-4 引导后再互相协作，群体成功率从 62% 升到 79%，呈现 Condorcet 陪审团效应
- 视角转换模块贡献高达 13% 的成功率提升（消融 Table 7）
- 协作后获得的知识可迁移到分布外任务（泛化性验证）

## 亮点与洞察
- **社会化蒸馏替代梯度蒸馏**：通过对话传递结构化知识比用 GPT-4 轨迹做 PEFT 微调更有效（微调几乎没改善，对话提升 40%+）。本质上是 test-time compute scaling 的一种形式
- **Condorcet 效应在弱智能体间涌现**：两个能力有限的智能体通过多轮对话也能产生群体级进步，暗示社会交互本身就是一种计算资源
- **ToM 的结构化很重要**：结构化 BDI 表征比非结构化的自然语言描述在复杂任务上效果更好，因为因果图结构使推理更精确

## 局限性 / 可改进方向
- Minecraft 环境相对简单，未在更复杂的真实世界具身任务中验证
- 即使有协作，开源小模型（7B/8B）仍远不及 GPT-4 单独运行的效果
- 通信轮次增多带来延迟增加，实时任务中可能不可行
- 伙伴建模基于 LLM 推理，可能产生二阶错误信念

## 相关工作与启发
- **vs Voyager**: Voyager 孤立学习，只通过自我反思改进；MindForge 增加社会交互维度，对开源 LLM 尤其有效
- **vs CoELA/COMBO**: CoELA 有多智能体通信但缺乏显式 ToM 推理和信念表征；MindForge 的结构化视角转换更精确
- **vs 知识蒸馏**: 传统蒸馏通过 logits/权重传递知识，MindForge 通过自然语言对话实现，无需访问模型内部

## 评分
- 新颖性: ⭐⭐⭐⭐ 将文化学习理论和显式 ToM 引入 LLM 智能体的首次系统性尝试
- 实验充分度: ⭐⭐⭐⭐ 多种设置（指导式/协作式/消融），但环境限于 Minecraft
- 写作质量: ⭐⭐⭐⭐ 理论动机充分，实验细致，略冗长
- 价值: ⭐⭐⭐⭐ 为开源 LLM 智能体提供了社会学习的新范式，减少对闭源模型的依赖
