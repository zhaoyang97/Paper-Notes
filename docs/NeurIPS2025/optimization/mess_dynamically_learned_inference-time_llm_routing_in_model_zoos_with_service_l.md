# MESS+: Dynamically Learned Inference-Time LLM Routing in Model Zoos with Service Level Guarantees
**会议**: NeurIPS 2025
**arXiv**: [2505.19947](https://arxiv.org/abs/2505.19947)
**代码**: https://github.com/laminair/mess-plus
**领域**: 模型选择、动态路由、SLA保证
**关键词**: 成本优化、学习路由、虚拟队列、约束优化

## 一句话总结
MESS+是首个成本最优的LLM路由框架，通过在线学习请求满足度预测和虚拟队列约束，动态选择模型同时保证SLA合规，相比现有方法实现平均2倍成本节省。

## 研究背景与动机
开源LLM生态（Llama、Qwen、Granite）提供多个相同系列不同规模的模型，形成"模型zoo"。使用者面临三重困境：
1. **用户需求**：希望获得准确、安全输出，通常无能力选择合适模型
2. **成本压力**：推理成本高，服务商需最小化运营成本
3. **质量保证**：商业应用需SLA（如90%请求满足度），需严格保证

现有路由方案（RouteLLM、Zooter、RouterDC）缺乏形式化的成本最优性和SLA保证理论。

## 方法详解

### 整体框架
**问题形式化**：
$$\min_{y_{m,t}} \quad \frac{1}{T}\sum_t\sum_m \mathbb{E}[y_{m,t}E_{m,t}]$$
$$\text{s.t.} \quad \frac{1}{T}\sum_t\sum_m \mathbb{E}[y_{m,t}s_{m,t}] \geq \alpha$$
其中：
- $y_{m,t} \in \{0,1\}$：对请求t选择模型m
- $E_{m,t}$：成本（API调用费、能耗）
- $s_{m,t} \in \{0,1\}$：模型m是否满足请求t（未知）
- $\alpha$：目标满足率（SLA）

**两个关键创新**：
1. 虚拟队列捕捉累积SLA违反
2. 在线学习预测器$\hat{s}_{m,t}$避免昂贵重训

### 关键设计

**1. 请求满足度预测器**
采用轻量级BERT骨干+多标签分类器，在线学习：
$$L(\mathbf{z}) = -\mathbb{E}[s_{m,t}\log\hat{s}(\mathbf{z},\mathbf{a}_t)_m + (1-s_{m,t})\log(1-\hat{s}(\mathbf{z},\mathbf{a}_t)_m)] + \frac{\mu}{2}\|\mathbf{z}\|^2$$

探索概率随时间衰减：$p_t = \min(1, c/\sqrt[4]{t})$，平衡精度和成本。

**2. 虚拟队列机制**
$$Q_{t+1} = \max\{0, Q_t + \alpha - \sum_m y_{m,t}s_{m,t}\}$$

捕捉所有时刻的SLA缺口。队列长度越长，越倾向于选择"安全"（高满足率）的模型。

**3. 每请求优化问题**
$$\min_{y_{m,t}} V \cdot \sum_m y_{m,t}E_{m,t} + Q_t(\alpha - \sum_m y_{m,t}\hat{s}_{m,t})$$

其中：
- V参数：权衡成本vs SLA达成速度（V大→成本优先，V小→快速达成）
- 惟一解：arg min的贪心选择

**预测准确性**：模型调用比例在20 ARC Challenge中：
- L70B: MESS+ 32% vs RouterDC 45% vs RouteLLM 70%
- 更多低成本模型调用同时保证满足度

**SLA合规性**：
- 所有基准+α配置中MESS+都满足目标，无违反
- 其他方法在严格α下存在违反

**参数V的鲁棒性**：
- V范围0.0001-0.001内，离线吞吐变化<15%
- 表明框架对超参不敏感

## 亮点与洞察

1. **首个理论保证的框架**：
   - Theorem 1：约束违反界 $\mathcal{O}(V/T + 1/\sqrt{T})$
   - Theorem 2：成本最优性 $\mathcal{O}(M/\sqrt[4]{T} + 1/V)$
   - 理论与算法无缝结合

2. **虚拟队列创新应用**：
   - 借鉴Lyapunov框架但针对LLM特性创新
   - 预测误差与队列长度的耦合分析新颖

3. **在线学习与决策解耦**：
   - 不直接用预测器路由（常见错误）
   - 而是将预测特征作为优化约束
   - 更鲁棒地处理预测误差

4. **现实可行性**：
   - 轻量级预测器（训练15ms，推理0.018ms）
   - 支持任意大小模型zoo
   - 无需昂贵重训

5. **平衡多维目标**：
   - 成本、满足度、公平性(防止饥饿)的统一框架
   - V参数提供直观的policy-level控制

## 局限性

1. **满足度定义简化**：
   - 二值标签（满足/未满足）可能粗粒度
   - 缺乏对多阶段反馈（部分满足）的建模

2. **i.i.d.假设紧张**：
   - Assumption 2假设请求和成本跨时间独立
   - 实际工作负载存在时间相关性、突增等
   - 附录虽有非i.i.d. ablation，但有限

3. **探索-利用权衡粗糙**：
   - $p_t = \min(1, c/\sqrt[4]{t})$ decay固定
   - 无自适应机制应对变化的预测准确性

4. **跨domain泛化未验证**：
   - 仅在8个目标任务（ARC、BoolQ等）评估
   - 模型zoo固定（Llama 1B、8B、70B）
   - 不同zoo配置下表现未知
