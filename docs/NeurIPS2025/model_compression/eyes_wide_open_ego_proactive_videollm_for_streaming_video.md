# Eyes Wide Open: Ego Proactive Video-LLM for Streaming Video

**会议**: NeurIPS 2025  
**arXiv**: [2510.14560](https://arxiv.org/abs/2510.14560)  
**代码**: [https://zhangyl4.github.io/publications/eyes-wide-open/](https://zhangyl4.github.io/publications/eyes-wide-open/)  
**领域**: LLM/NLP / 视频理解 / 流式处理  
**关键词**: streaming video, proactive response, ego-centric, just-in-time, dynamic compression, ESTP-Bench  

## 一句话总结
聚焦"主动式流式视频理解"新任务——给定第一视角流式视频输入，AI助手需要主动地在恰当时机回答多样化的、随事件演变的问题，同时保持感知和推理的同步。提出ESTP-Bench评估框架和ESTP-F1指标，以及包含数据引擎、多阶段训练和主动动态压缩技术的完整技术pipeline。

## 背景与动机
现有Video-LLM主要是被动响应模式——用户问了才答，且通常处理预录制的完整视频（离线）。但真实世界的AI助手（如智能穿戴设备、自动驾驶copilot）需要处理**实时流式视频**，并且**主动**地在关键时刻提供信息，不等用户提问。这需要三个关键能力：(1) 主动一致性——持续跟踪场景并识别需要响应的时刻；(2) 即时响应——在恰当的时间点回应；(3) 同步效率——感知和推理必须与视频流速度同步。

## 核心问题
如何让Video-LLM从被动的"问了才答+离线处理"升级为主动的"自主判断何时响应+实时流式处理"？

## 方法详解

### 关键设计
1. **ESTP-Bench和ESTP-F1指标**：设计了第一个针对流式视频主动理解的benchmark，评估模型是否在正确的时间点提供了正确的回答。ESTP-F1综合衡量了响应时机的准确性和内容的正确性。

2. **数据引擎**：构建了用于训练主动流式视频理解的数据——标注何时应该主动响应以及响应什么内容。这解决了现有视频QA数据都是被动格式的问题。

3. **多阶段训练策略**：渐进式训练模型的三个能力——先学会基础视频理解，再学会识别关键时刻，最后学会主动生成响应。

4. **主动动态压缩技术**：在流式处理中动态压缩历史帧的表示——重要的帧保留更多信息，不重要的帧高度压缩——确保模型能处理持续不断的视频流而不爆内存，同时保持关键时刻的感知精度。

## 实验关键数据
- 在ESTP-Bench上有效评估主动式流式视频理解
- 在多个在线和离线benchmark上超越baseline
- 主动动态压缩兼顾效率和精度

## 亮点
- **"主动式流式视频理解"是重要的新任务定义**——从被动→主动、离线→流式的范式转换
- **三个关键属性的形式化**（主动一致性、即时响应、同步效率）为这个方向建立了评估框架
- **ESTP-Bench是首个相关benchmark**，填补评估空白
- **动态压缩技术**对任何需要处理长视频流的模型都有参考价值
- 应用前景广阔——智能穿戴、自动驾驶辅助、安防监控等

## 局限性 / 可改进方向
- 主动响应的时机判断可能产生误报（不该响应时响应）
- 流式处理的延迟与精度trade-off需要更多探索
- 第一视角的数据收集和标注成本高

## 与相关工作的对比
- **vs. VideoChat/Video-LLaVA**：这些是被动式离线视频QA；Eyes Wide Open是主动式流式响应
- **vs. SAM2Long（ICCV2025）**：SAM2Long处理长视频分割的记忆管理；Eyes Wide Open处理流式视频的主动响应——不同任务但都面临长序列处理挑战
