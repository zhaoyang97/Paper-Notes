# DuoLens: A Framework for Robust Detection of Machine-Generated Multilingual Text and Code

**会议**: NeurIPS 2025  
**arXiv**: [2510.18904](https://arxiv.org/abs/2510.18904)  
**代码**: 有（含训练和评估脚本）  
**领域**: AIGC检测  
**关键词**: AI生成文本检测, 代码检测, 多语言, SLM, BERT

## 一句话总结
提出 DuoLens，一种基于 CodeBERT + CodeBERTa 双编码器融合的 AI 生成内容检测框架，在多语言文本（8 种语言）和源代码（7 种编程语言）检测上以极低计算成本（延迟降低 8-12×，VRAM 降低 3-5×）实现 AUROC 0.97-0.99，远超 GPT-4o 等大模型。

## 研究背景与动机

1. **领域现状**：LLM 生成内容泛滥，检测需求日益迫切。现有检测器要么计算成本高（基于 LLM 的方法），要么准确率不足（基于困惑度的方法），且大多只对英文有效。

2. **现有痛点**：
   - 基于 LLM 的检测方法（Binoculars, AGENT-X）推理慢、资源需求大
   - 现有代码检测数据集**语言不平衡**（Python 占主导），多语言文本数据集**领域单一**
   - LLM 做二分类可能有偏差（倾向认为 AI 生成文本质量更高）

3. **核心矛盾**：二分类任务不需要 LLM 级别的模型复杂度，SLM（小语言模型）在此任务上可能更优且更高效。

4. **切入角度**：微调 encoder-only 模型（RoBERTa/CodeBERT 变体），用双编码器融合互补表示。

5. **核心idea一句话**：对于 AI 内容检测的二分类任务，精心微调的 SLM 在准确率和效率上均远超 LLM。

## 方法详解

### 整体框架
双编码器架构：CodeBERT（NL+代码预训练）和 CodeBERTa（纯代码预训练）分别编码输入，通过学习门控融合两个 [CLS] 向量，送入线性分类器做二分类。

### 关键设计

1. **平衡数据集构建**:
   - 代码数据集：7 种语言 × 每语言 12,000 样本（人写/AI生成各半），共 84,000 样本
   - 多语言文本：8 种语言共 54,520 样本，跨 news/social/QA 等多领域
   - 设计动机：消除语言和标签不平衡的偏差

2. **DuoLens 双编码器融合**:
   - 做什么：融合 CodeBERT 的语义/语言对齐能力和 CodeBERTa 的句法/结构敏感性
   - 核心思路：门控机制学习降低冗余特征，强化编码器特异性线索
   - 设计动机：两个编码器预训练数据不同（NL+Code vs 纯 Code），捕获互补信号

3. **跨语言评估**:
   - 做什么：测试在特定语言上微调后对其他语言的泛化
   - 设计动机：实际部署中无法为每种语言训练专用模型

## 实验关键数据

### 主实验

| 模型 | 代码检测 AUC | 代码 F1 | 多语言 AUC | 多语言 F1 |
|------|------------|--------|-----------|----------|
| GPT-4o (few-shot) | 0.535 | 0.414 | 0.573 | 0.490 |
| CodeBERT (微调) | 0.980 | 0.930 | - | - |
| DuoLens (微调) | **0.985** | **0.937** | **0.975** | **0.911** |
| XLM-R-large (微调) | - | - | 0.974 | **0.924** |

### 消融/鲁棒性
| 攻击方式 | 性能保持 |
|---------|--------|
| 释义攻击 | ≥92% clean AUROC |
| 回译攻击 | ≥92% clean AUROC |
| 代码格式化/重命名 | ≥92% clean AUROC |

### 关键发现
- GPT-4o 在二分类上**大幅落后**于微调的 SLM（AUC 0.535 vs 0.985）
- 即使**未微调**，CodeBERT 的 AUC 也达 0.953，远超 GPT-4o
- DuoLens 延迟比 LLM 低 8-12×，VRAM 低 3-5×
- 对抗性变换下性能保持 ≥92%

## 亮点与洞察
- **SLM > LLM 的反直觉结论**：在二分类任务上，编码器模型的归纳偏置天然适配，LLM 的生成能力反而成了累赘。这对 AIGC 检测社区是重要提醒
- **数据集工程**：精心平衡语言和标签分布是关键——现有数据集的不平衡是造成检测器泛化差的主因

## 局限性 / 可改进方向
- BERT 模型 512 token 限制，长文本需要分块
- Encoder-only 模型无法提供逐句级别的检测解释
- 多语言文本数据集存在语言不平衡
- 基线中闭源 LLM 只用了 GPT-4o

## 相关工作与启发
- **vs AGENT-X / Binoculars**：基于 LLM 的方法计算成本高且准确率未必更好
- **vs GPT-Sentinel**：同为 RoBERTa 微调但泛化能力差，DuoLens 通过平衡数据集+双编码器改进
- AI 生成内容检测的实际部署应优先考虑 SLM 方案
