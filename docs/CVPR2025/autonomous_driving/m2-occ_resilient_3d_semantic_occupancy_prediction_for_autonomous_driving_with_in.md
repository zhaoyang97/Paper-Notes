# M²-Occ: Resilient 3D Semantic Occupancy Prediction for Autonomous Driving with Incomplete Camera Inputs

**会议**: CVPR 2025  
**arXiv**: [2603.09737](https://arxiv.org/abs/2603.09737)  
**代码**: [GitHub](https://github.com/qixi7up/M2-Occ)  
**领域**: 自动驾驶 / 3D占用预测  
**关键词**: 语义占用预测, 缺失视角鲁棒性, 掩码重建, 语义原型记忆, 传感器故障, 环视相机

## 一句话总结
M²-Occ 针对多相机输入不完整时的语义占用预测问题，提出多视角掩码重建（MMR）模块利用相邻相机重叠区域恢复缺失视角特征，以及特征记忆模块（FMM）通过类级语义原型精炼不确定体素特征，在缺失后视角设置下 IoU 提升 4.93%。
该工作的重要意义不仅在于方法本身，还在于首次系统研究了占用预测在传感器故障下的行为。

## 研究背景与动机
1. **领域现状**：3D 语义占用预测（如 SurroundOcc, TPVFormer）为自动驾驶提供密集体素级的几何+语义理解，多相机方案因成本优势受到关注。
2. **现有痛点**：现有方法隐式假设所有环视相机完好——但实际部署中相机遮挡、硬件故障、通信中断频繁发生。即使单个关键视角丢失（如后视），SurroundOcc 的 IoU 就从 32.38 骤降到 23.94（-26%）。
3. **核心矛盾**：环视相机之间有视野重叠但现有方法未利用这种冗余；缺失视角不仅导致几何空白，还引起跨视角对应关系不一致，破坏特征融合质量。
4. **本文要解决什么？** 如何在不牺牲完整视角性能的前提下，使占用预测模型对相机故障具有鲁棒性？
5. **切入角度**：借鉴人类从上下文和记忆推断不可见区域的能力——(a) 利用相邻相机重叠区域恢复缺失特征（空间冗余）；(b) 用全局语义原型作为先验知识精炼不确定特征（记忆）。
6. **核心idea一句话**：相邻重叠区域特征重建 + 类级语义原型记忆 = 传感器故障鲁棒的占用预测。

## 方法详解

### 整体框架
标准 2D→3D 流程上增加两个模块：(1) MMR 在特征提取阶段利用相邻视角重叠恢复缺失视角特征；(2) FMM 在体素精炼阶段用全局语义原型增强不确定体素。整体端到端训练。

### 关键设计

1. **Multi-view Masked Reconstruction (MMR)**
   - 做什么：利用相邻未掩码视角的重叠边界特征重建缺失视角的特征表示
   - 核心思路：建模环视相机为循环图 $\mathcal{N}(v_i) = \{v_{(i-1) \bmod N}, v_{(i+1) \bmod N}\}$ → 裁剪左右邻居的重叠边界区域 $w_{ov}$ → 与可学习 mask token 拼接 → 轻量 Transformer 解码器重建
   - 设计动机：在特征空间（而非像素空间）操作避免了生成式幻觉的不确定性；重叠区域提供了结构先验

2. **Feature Memory Module (FMM) - Single-Proto**
   - 做什么：维护每类一个全局质心原型，提供语义稳定性
   - 核心思路：动量移动平均更新 $\mathbf{m}_k^{(t)} = (1-\lambda)\mathbf{m}_k^{(t-1)} + \lambda \cdot \bar{\mathbf{f}}_k$，$\lambda=0.1$
   - 设计动机：Single-Proto 捕捉类别核心特征，在不完整观测下提供稳定的先验——确保"车"的特征即使视觉特征部分损坏仍保持车的语义

3. **Feature Memory Module (FMM) - Multi-Proto**
   - 做什么：每类维护 $N_p$ 个子原型建模类内变异
   - 核心思路：查询体素特征与所有子原型计算余弦相似度 → softmax + 温度参数 $\tau$ 加权 → 动态检索最相关的子原型用于特征精炼
   - 设计动机：Multi-Proto 捕捉类内多样性（如"卡车"包含皮卡和半挂车），实现更细粒度的语义精炼

### 损失函数 / 训练策略
$\mathcal{L} = \mathcal{L}_{occ} + \mathcal{L}_{MMR}$。
$\mathcal{L}_{MMR} = \frac{1}{|\mathcal{M}|}\sum_{i\in\mathcal{M}} \|\hat{\mathbf{f}}_i - \mathbf{f}_i^{gt}\|_2^2$，仅在掩码视角上计算 MSE 重建损失。
训练时随机掩码视角模拟故障，确保模型学习到各种视角缺失模式。
对于 FMM，动量系数 $\lambda=0.1$，确保原型更新平滑且过滤器噪声。
Multi-Proto 中温度参数 $\tau$ 控制检索分布的尖锐度，较低温度强调最相关原型。
SurroundOcc baseline 上采用 ResNet-101 + FPN 作为编码器，保持与原始实现一致的设置。

## 实验关键数据

### 主实验（nuScenes/SurroundOcc benchmark）

| 缺失设置 | SurroundOcc IoU | M²-Occ IoU | 提升 |
|---------|----------------|-----------|------|
| 标准（无缺失） | 32.38 | 31.08~31.25 | ~持平 |
| 缺失前视 | 25.03 | 30.40 | **+5.37** |
| 缺失后视 | 23.94 | 28.87 | **+4.93** |
| 缺失前右 | 30.56 | 31.17 | +0.61 |

### 极端缺失场景

| 缺失视角数 | Baseline IoU | M²-Occ IoU | 说明 |
|-----------|-------------|-----------|------|
| 5个缺失 | 13.35 | 18.36 | **+5.01** IoU |

### 关键发现
- 缺失前视和后视影响最大（IoU 分别降 7.35 和 8.44），因为它们覆盖自车前后关键区域且与相邻相机重叠最少
- M²-Occ 在缺失后视设置下 IoU 提升 4.93 但不损害完整视角性能——鲁棒性是免费获得的
- 侧面视角缺失影响较小（IoU 仅降 1.6~2.0），因为相邻视角有更大重叠
- 5 视角缺失的极端场景下 M²-Occ 仍维持 18.36 IoU（vs baseline 13.35），展现了 FMM 在几乎无观测时用先验知识维持基本结构的能力

## 亮点与洞察
- **首次系统研究多相机占用预测的传感器故障鲁棒性**：不仅提出方法，更重要的是建立了完整的缺失视角评估协议（单视角确定性故障+多视角随机丢弃）
- **MMR 的特征空间修复**：在特征空间而非像素空间操作——更高效且不依赖生成模型，利用循环图模型化环视配置的自然冗余
- **语义原型作为安全后备**：FMM 类似于"先验知识库"——即使所有视觉证据丢失，模型仍可用类级原型提供最低限度的语义理解
- **所有改进在完整视角下不退化**：说明 MMR 和 FMM 不是以牺牲正常性能来换取鲁棒性

## 局限性 / 可改进方向
- 基于 SurroundOcc 框架验证，未在更先进的占用预测模型（如 OccFormer, VoxFormer）上测试通用性
- MMR 依赖相邻相机有足够重叠——如果相机配置重叠极少则效果有限
- FMM 原型在训练集外的场景（如罕见物体）可能缺乏相关原型
- 未考虑时序信息（如利用前一帧的结果补偿当前帧缺失），可能是一个重要的改进方向

## 相关工作与启发
- **vs M-BEV**：M-BEV 用整视角掩码+跨视角重建但在 BEV 感知（2D）上；M²-Occ 将此思路扩展到 3D 占用预测并加入语义原型
- **vs MetaBEV**：MetaBEV 处理传感器损坏和模态缺失但面向联合 BEV 任务；M²-Occ 面向更密集的 3D 体素语义
- **vs SafeMap/FlexMap**：这些方法增强 BEV 地图构建的鲁棒性；M²-Occ 面向 3D 体素级语义
- **vs MAE 范式**：MAE 在单图像中随机掩码 patch 做自监督；MMR 掩码整个视角并用跨视角上下文重建，是 MAE 思想在多传感器场景的自然扩展
- **vs 多传感器融合方法**（如 OccFusion, FusionOcc）：这些方法通过融合 LiDAR+相机提供冗余，但增加了硬件成本；M²-Occ 仅用相机就实现了故障鲁棒性
- **对自动驾驶安全认证的启发**：M²-Occ 的缺失视角评估协议可作为感知系统鲁棒性认证的标准化测试框架
- **时序扩展方向**：利用前一时刻的占用预测结果补偿当前帧缺失视角，是一个自然且有潜力的改进方向
