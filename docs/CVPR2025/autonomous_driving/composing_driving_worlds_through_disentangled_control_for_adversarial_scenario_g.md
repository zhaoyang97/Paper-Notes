# CompoSIA: Composing Driving Worlds through Disentangled Control for Adversarial Scenario Generation

**会议**: CVPR 2025  
**arXiv**: [2603.12864](https://arxiv.org/abs/2603.12864)  
**代码**: [GitHub](https://github.com/Yifever20002/CompoSIA)  
**领域**: 自动驾驶 / 视频生成 / 场景仿真  
**关键词**: 驾驶世界模型, 解耦控制, 对抗场景生成, 身份注入, Flow Matching

## 一句话总结
CompoSIA 提出一种基于 Flow Matching DiT 的组合式驾驶视频生成框架，通过解耦结构（3D bbox）、身份（单参考图像）和自车动作（相机轨迹）三类控制信号的注入方式，实现精细独立控制和组合编辑，用于系统化合成对抗性驾驶场景，FVD 提升 17%，碰撞率增加 173%。

## 研究背景与动机
1. **领域现状**：端到端自动驾驶依赖大规模视频数据学习统一表示，但 nuScenes/Waymo 等数据集中安全关键的长尾场景严重不足。可控生成模型是合成这些场景的关键手段。
2. **现有痛点**：现有驾驶世界模型（MagicDrive-V2, DriveEditor, Vista）要么只能控制部分场景因素，要么控制信号耦合——多条件通过共享路径注入导致编辑时质量下降，无法独立操控结构、身份和动作。
3. **核心矛盾**：对抗场景往往来自"常见交通元素的异常组合"（如突然卡车变道+急刹车），需要独立精细控制场景结构、元素身份和自车动作——但耦合的生成器无法实现这种组合编辑。
4. **本文要解决什么？** 如何在一个统一框架中实现结构、身份、动作的解耦控制，使其可独立操控和自由组合，以系统化合成对抗驾驶场景？
5. **切入角度**：不同类型的控制信号应在扩散过程的不同层次注入——结构在数据流中注入、身份在噪声级别注入、动作通过 AdaLN 和 PRoPE 注意力注入。
6. **核心idea一句话**：三路独立条件分支（结构-零初始化投影、身份-噪声级别替换、动作-层级双分支）实现解耦控制，支持组合式对抗场景合成。

## 方法详解

### 整体框架
基于 Wan2.1-T2V-1.3B 的 Flow Matching DiT 骨干，三类控制信号通过独立路径注入：结构条件通过零初始化投影加到 latent token 上；身份条件在高噪声阶段替换对应区域的 latent；动作条件通过 AdaLN 局部调制和 PRoPE 全局注意力注入。训练模态比例 0.6:0.3:0.1 分配给[动作]、[结构+身份+动作]和无条件。

### 关键设计

1. **结构条件（时空布局）**
   - 做什么：通过 3D bbox 序列控制场景元素的位置和运动轨迹
   - 核心思路：3D bbox → 投影到 2D 图像平面 → VAE 编码 → 轻量卷积适配器 → 零初始化投影加到 latent token
   - 设计动机：3D bbox 在世界坐标系而 latent 在 2D 图像域，需要先投影对齐；零初始化确保训练初期不破坏预训练权重

2. **身份条件（噪声级别注入）**
   - 做什么：用单张参考图像控制场景元素的外观身份
   - 核心思路：select-and-repaint 策略构建训练对 → 将参考图像裁剪并填充到 bbox 序列形成 $\bm{r}_f$ → 在高噪声阶段（$t > T_\text{id}$）将 latent 对应区域替换为身份条件 latent → 模型学习从噪声恢复到几何一致序列
   - 设计动机：将身份控制建模为修复问题而非注意力条件——避免注意力机制中身份约束过强压制运动表达力的问题；仅高噪声阶段替换避免破坏低噪声阶段的生成路径

3. **动作条件（层级双分支）**
   - 做什么：通过帧级连续相机轨迹控制自车运动
   - 局部分支：提取相邻帧相对变换的 $(\Delta x, \Delta y, \Delta\text{yaw})$ → 正弦频率编码 → 零初始化投影 → AdaLN 门控注入（shift/scale/gate）
   - 全局分支：PRoPE 投影位置编码注入低维子空间注意力 → 零卷积加回主注意力
   - 设计动机：局部分支加速早期训练收敛，全局分支提升整体动作控制精度和稳定性

### 损失函数 / 训练策略
v-prediction loss（Flow Matching 标准损失）。第一帧背景区域保持干净 latent 锚定场景身份，前景区域可编辑。
训练时对第一帧施加局部噪声扰动增强修复能力。
基于 Wan2.1-T2V-1.3B 初始化，动作投影器学习率 $2 \times 10^{-4}$，其他组件 $1 \times 10^{-5}$。
混合分辨率训练策略（33×256×512 和 33×480×960）加速条件建模收敛。
数据采样为 10 Hz，16 张 NVIDIA A100 80GB GPU 训练。

### 关键发现
- 身份编辑 FVD 比 DriveEditor 低 17%（149.15 vs 179.57），跨元素姿态泛化性更强
- 动作控制旋转误差降 30%，平移误差降 47%
- 合成的对抗场景能暴露规划器的隐藏故障模式——碰撞率增加 173%
- 解耦设计使独立编辑和组合编辑都保持高质量

## 亮点与洞察
- **控制信号注入的层次化设计原则**：不同因素需要在扩散过程的不同位置注入——这是超越具体实现的设计原则，可迁移到其他条件生成任务
- **身份注入的修复范式**：将身份控制建模为噪声级别的区域替换+修复，而非注意力条件——巧妙避免了身份-运动的权衡问题
- **对抗场景合成的系统化方法**：不是随机合成，而是通过控制策略系统化构造"常见元素的异常组合"——更像是可控的压力测试工具
- **PRoPE + AdaLN 双分支动作控制**：融合局部残差信号和全局相机轨迹，前者加速收敛后者提升精度

## 局限性 / 可改进方向
- 基于 nuScenes + 100h 内部数据训练，场景多样性仍有限
- 只能控制已有 3D bbox 标注的元素，对无标注的场景元素（如临时施工区）无法控制
- 合成视频分辨率（256×512 或 480×960）可能不足以用于某些下游任务
- 未分析生成图像的真实感对下游规划器训练的实际提升效果

## 相关工作与启发
- **vs MagicDrive-V2**：MagicDrive-V2 通过共享路径注入多条件导致耦合和质量退化；CompoSIA 解耦注入路径避免冲突
- **vs DriveEditor**：DriveEditor 仅支持结构和身份编辑，无法控制自车动作，且身份编辑限于姿态对齐的图像
- **vs Vista**：Vista 支持动作和场景身份控制但无元素级结构和身份控制
- **vs GAIA-2**：GAIA-2 支持结构+场景身份+动作控制但无元素级身份控制；CompoSIA 进一步解耦了元素身份
- **vs TTM**：TTM 在噪声级别进行身份编辑但采用无训练策略，难以在准确运动下实现身份控制；CompoSIA 通过有训练的修复范式解决此问题
- **对自动驾驶安全测试的启发**：CompoSIA 可作为可控压力测试工具，系统化构造对抗场景暴露规划器弱点——比随机数据增强更有针对性
