# BoSS: A Best-of-Strategies Selector as an Oracle for Deep Active Learning

**会议**: CVPR 2025  
**arXiv**: [2603.13109](https://arxiv.org/abs/2603.13109)  
**代码**: 待确认  
**领域**: 主动学习 / 数据选择  
**关键词**: Active Learning, Oracle策略, 策略集成, 批量选择, 深度学习

## 一句话总结
提出BoSS——一种可扩展的主动学习oracle策略，通过集成多种选择策略生成候选批次、冻结backbone仅重训最后一层来评估性能增益，选择最优批次；在ImageNet等大规模数据集上首次展示了oracle性能，揭示SOTA主动学习策略仍有显著提升空间。

## 研究背景与动机

1. **领域现状**：主动学习（AL）通过迭代选择最有价值的样本标注来降低标注成本。现有选择策略分为基于不确定性的（Margin、BADGE）和基于代表性的（TypiClust），近年也有结合两者的方法（FastBAIT、DropQuery）。Foundation model时代，DINOv2等预训练特征使样本选择更容易。

2. **现有痛点**：(a) 没有单一策略在所有模型架构、标注预算、数据集上都表现最优——有的策略在前期好（TypiClust），后期差；有的反过来。(b) 现有策略基于启发式（不确定性/代表性），不直接优化模型性能。(c) 缺乏可靠的oracle基准来评估现有策略距离最优还有多远。

3. **核心矛盾**：理论上的oracle策略（用GT标签近似最优选择）可以作为评估基准，但现有oracle（SAS、CDO）无法扩展到大规模数据集和深度网络——SAS搜索步数太多计算量爆炸，CDO逐样本贪心选择需要频繁重训模型。

4. **本文要解决什么？**
   - 如何构建一个可扩展到ImageNet等大规模数据集的oracle策略？
   - 现有SOTA主动学习策略距离oracle性能到底有多大差距？
   - 不同策略在AL不同阶段的表现差异有什么规律？

5. **切入角度**：不搜索单个样本的最优组合（组合爆炸），而是搜索一批由现有策略生成的候选批次中的最优批次——把搜索空间从$\binom{|\mathcal{U}|}{b}$压缩到$T$个候选批次。

6. **核心idea一句话**：用多策略集成生成少量高质量候选批次，冻结backbone仅重训最后一层快速评估每个批次的性能增益，选择最优批次。

## 方法详解

### 整体框架
BoSS将最优批次选择问题$\mathcal{B}^* = \arg\min_{\mathcal{B} \subset \mathcal{U}} \mathbb{E}[\ell(y, p(y|x, \mathcal{L}^+))]$分解为三个可高效实现的子问题：候选批次生成 → 快速重训 → 性能评估选最优。

### 关键设计

1. **候选批次生成（Algorithm 1）**
   - 做什么：从多种AL策略集成生成$T$个候选批次
   - 核心思路：选择10种互补策略（Random、Margin、CoreSets、BADGE、FastBAIT、TypiClust等），每种策略生成$T/|S|$个批次。关键技巧：每次从$\mathcal{U}$中随机采样大小为$k$的候选池$\mathcal{C}$，在$\mathcal{C}$上运行策略。$k$在$[b, k_{max}]$之间随机变化，避免某些策略对大池不稳定
   - 设计动机：随机生成批次找到好批次的概率极低（$\binom{1000}{10} \approx 2.63 \times 10^{23}$），但现有AL策略生成的批次质量远高于随机，用少量候选批次就能覆盖解空间的高质量区域。不同策略覆盖不同AL阶段的需求——前期需要代表性（TypiClust），后期需要不确定性（BADGE）

2. **快速重训（冻结Backbone）**
   - 做什么：对每个候选批次，快速评估加入该批次后模型性能的提升
   - 核心思路：冻结预训练backbone $h^\phi$（如DINOv2-ViT），仅重训分类头$g^\theta$（线性层）。对每个候选批次$\mathcal{B}_t$，将其加入已标注集$\mathcal{L}^+$，重新拟合分类头，在评估集$\mathcal{E}$上计算准确率
   - 设计动机：重训整个DNN一次可能需要数小时，而仅训练线性层只需秒级。冻结backbone不影响特征提取质量（预训练特征已经足够好），大幅降低$T$次重训的总成本

3. **性能评估与选择**
   - 做什么：选择$T$个候选批次中使准确率提升最大的那个
   - 核心思路：$\mathcal{B}^* = \arg\min_{\mathcal{B} \in \{\mathcal{B}_1, ..., \mathcal{B}_T\}} \sum_{(x,y) \in \mathcal{E}} \mathbb{1}[y \neq \arg\max_c p(c|x, \mathcal{L}^+)]$
   - 注意：这是oracle策略，用了GT标签和测试集——实际AL中不可用，仅作为评估基准

### 损失函数 / 训练策略
- 外层：每个AL周期选择batch size $b$的样本进行标注（如CIFAR-10 $b=10$, ImageNet $b=1000$）
- 内层评估：仅训练线性分类头，用0-1 loss评估
- 默认$T=100$个候选批次（10种策略各10个）
- 滑动窗口式AL，固定$A$个周期

## 实验关键数据

### 主实验（BoSS vs SOTA AL策略，DINOv2-ViT-S/14）

在8个图像数据集上（CIFAR-10/100, STL-10, Snacks, Flowers102, Food101, Tiny-ImageNet, ImageNet），BoSS始终超越所有SOTA AL策略。

关键发现（以ImageNet为例）：
- BoSS的准确率提升约为最佳AL策略的**2倍**
- 在多类别大规模数据集（100+类）上差距最大

### BoSS vs 其他Oracle策略

| Oracle方法 | CIFAR-10 运行时 | DTD 运行时 | 可扩展性 |
|-----------|---------------|-----------|---------|
| BoSS | 10:07 | 22:07 | 可扩展到ImageNet |
| CDO (对齐) | 10:26 | 24:56 | 大batch时退化 |
| SAS (对齐) | 10:20 | 22:14 | 需极高搜索步数 |
| CDO (默认) | 10:26 | 2:46:14 | - |
| SAS (默认) | 17:22:39 | 62:34:13 | - |

### 消融实验

| 配置 | CIFAR-10 AULC | DTD AULC |
|------|-------------|---------|
| 随机生成候选批次 | 弱于策略生成 | 弱于策略生成 |
| T=10 (每策略1个) | 89.90 | 70.45 |
| T=50 (每策略5个) | 90.45 | 71.55 |
| T=100 (每策略10个) | **90.70** | **71.79** |
| T=200 (每策略20个) | 90.83 | 71.91 |

### 关键发现
- **SOTA策略距离oracle有显著差距**：尤其在大规模多类别数据集上（Food101、ImageNet），差距可达2倍。这说明AL策略还有很大改进空间
- **没有单一策略在所有AL阶段都最优**：BoSS的策略选频分析显示，前期偏好代表性策略（TypiClust*、DropQuery*），后期甚至Random也被选中（说明特化策略此时反而无效）
- **增加候选批次数$T$持续改善性能**，但边际效益递减（T=10→100提升明显，100→200提升微小）
- **加入更多策略总是有益的**，不会因为引入弱策略而退化——因为弱策略的批次不会被选中
- **冻结backbone对性能评估影响不大**：证明预训练特征质量已经足够做可靠的性能排序

## 亮点与洞察
- **从"选最优样本"到"选最优批次"的视角转换**：避免了逐样本贪心选择的组合爆炸，同时考虑了批次内样本的集体效应。这个insight可以迁移到其他需要子集选择的场景（如数据蒸馏、课程学习）
- **策略集成的元学习思想**：不预判哪种策略最好，而是让数据说话——每个周期自动选择此时最优的策略。这是一种无需学习的meta-AL
- **揭示了AL研究的方向**：前期需要更好的探索策略（当前策略对多类别数据集的cold-start表现差），后期需要更好的利用策略。ensemble-based AL是一个值得深耕的方向
- **可扩展性设计**：冻结backbone + 候选池采样两个技巧使BoSS首次能在ImageNet上运行，为大规模AL评估提供了基准

## 局限性 / 可改进方向
- **Oracle策略本身不实际可用**：依赖GT标签和测试集，仅作评估参考。未来方向是将BoSS的集成思想转化为实际可用的无标签版本（用验证集替代测试集？）
- **仅评估了图像分类任务**：未覆盖目标检测、分割、NLP等任务
- **冻结backbone的假设**：对于domain gap大、预训练特征不够好的场景，冻结backbone可能导致评估不准确
- **改进思路**：(1) 将oracle策略的策略选频分析转化为自适应权重，构建实用的ensemble AL策略；(2) 用少量标注验证集替代测试集做近似oracle；(3) 扩展到detection/segmentation等非分类任务

## 相关工作与启发
- **vs CDO (Werner 2024)**：CDO逐样本贪心选择，batch大时需频繁retrain，无法扩展。BoSS直接评估完整批次效果，batch越大优势越明显
- **vs SAS (Zhou 2021)**：SAS用模拟退火搜索整个标注集最优序列，计算量与搜索步数挂钩，默认配置在DTD上运行62小时。BoSS仅需22分钟达到更好效果
- **vs BADGE**：BADGE是最常被推荐的AL策略，但在BoSS揭示的oracle性能下，BADGE在大规模数据集上仍有显著差距
