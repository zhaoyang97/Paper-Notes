# One Token, Two Fates: A Unified Framework via Vision Token Manipulation Against MLLMs Hallucination

**会议**: CVPR 2025  
**arXiv**: [2603.10360](https://arxiv.org/abs/2603.10360)  
**代码**: 待确认  
**领域**: 多模态大模型 / 幻觉缓解  
**关键词**: MLLM幻觉, vision token, 对比解码, 训练无关, 隐表示校准, 视觉注意力衰减

## 一句话总结

提出首个统一的训练无关MLLM幻觉缓解框架，围绕vision token的双重角色——增强(SVC)与抑制(CRC)——在隐表示层协同操作，在LLaVA-1.5上POPE准确率提升约2%，仅增加1.06×推理延迟。

## 研究背景与动机

1. **MLLM幻觉的核心机制**: 多模态大语言模型(MLLM)生成文本时，视觉注意力随生成步骤逐渐衰减，而语言先验（text inertia）逐渐主导，导致生成内容与视觉证据矛盾——即"幻觉"
2. **现有两类训练无关方法的局限**:
   - 视觉增强类(如PAI)：通过放大注意力权重强化视觉信号，但面对强语言先验仍力不从心
   - 文本抑制类(如VCD)：通过对比解码在logit层抑制语言惯性，但依赖图像级扭曲(modality-gap)产生的负样本噪声大且不稳定
3. **朴素组合失败**: 直接组合PAI+VCD实验表明性能无提升——两种方法设计理念不同、干预层级不同（一个在attention层、一个在output logit），产生信号冲突
4. **核心洞察**: Vision token是视觉-语言交互的桥梁，可以同时承担两种角色：(1)通过增强图片提供互补视觉语义(F2)；(2)通过裁剪token(information-gap)精确隔离幻觉倾向(F3)
5. **核心idea**: 统一框架在隐表示层操作，从vision token派生所有校准信号——增强和抑制协同协作

## 方法详解

### 整体框架

统一的训练无关框架，包含两个核心模块：
- **SVC (Synergistic Visual Calibration)**: 在中间层注入增强视觉上下文对抗视觉衰减
- **CRC (Causal Representation Calibration)**: 在浅层通过隐空间负样本纯化模型内部偏差
- 两个模块均在隐表示层操作（非logit层），避免了信号冲突

### 三大发现 (Foundation Findings)

- **F1 反相关**: 随生成推进，视觉注意力急剧衰减，幻觉频率恰在视觉接地最弱处飙升
- **F2 语义互补**: 原始图像和增强图像（翻转+模糊+噪声）的注意力模式互补，融合可增强视觉接地
- **F3 信息差优于模态差**: 在隐空间裁剪vision token(information-gap)产生稳定的、分布内的幻觉探针；而像素级遮挡(modality-gap)产生噪声大、偏离分布的结果

### 关键设计 1: SVC — 协同视觉校准

- **做什么**: 在关键中间层注入丰富的视觉上下文，对抗视觉衰减
- **步骤**: (1) 对输入图像做随机翻转+高斯模糊+椒盐噪声生成增强图像 → (2) 拼接原始和增强vision token得协同视觉记忆库 $\mathbf{V}_{\text{syn}} \in \mathbb{R}^{2N_v \times d}$ → (3) 在第 $L_c$ 层用scaled dot-product attention将hidden state作为Query、 $\mathbf{V}_{\text{syn}}$ 作为Key/Value → (4) 通过插值融合注入: $\mathbf{H}'_t = (1-\lambda_s) \mathbf{H}_t + \lambda_s \mathbf{C}_t$
- **设计动机**: 利用F2的语义互补性，无需额外参数，在单一层干预即可有效增强视觉表示

### 关键设计 2: CRC — 因果表示校准

- **做什么**: 在浅层纯化隐表示，消除"幻觉方向"
- **步骤**: (1) 随机裁剪vision token至仅保留 $N_h=5$ 个，生成K=3个负样本 → (2) 在首步t=0并行前向传播获取原始和负样本的隐状态 → (3) 差值平均得稳定的幻觉方向向量: $\mathbf{v}_{\text{crc}}^{(l)} = \frac{1}{K}\sum_{k=1}^{K} (\mathbf{H}_{\text{org}}^{(l)} - \mathbf{H}_{\text{neg}}^{(l,k)})$ → (4) 后续每步在归一化空间做校正: $\mathbf{h}_{\text{crc}} = \mathbf{h}_{\text{norm}} + \lambda_c \mathbf{v}_{\text{norm}}$，再缩放回原始量级
- **因果理论支撑**: 基于结构因果模型(SCM)，差值向量精确捕获了"因视觉信息缺失而产生的偏差"，与text/query/bias的共享效应无关
- **关键参数**: $N_h=5$（保留5/576个token，精确触发偏差探测），$K=3$（性能与效率的最佳折中）

### 协同机制

- SVC在中间层 $L_c$ 干预（增强视觉），CRC在1至 $L_c$ 的浅层干预（抑制偏差）
- 两者在同一层级（隐表示）操作，信号兼容而非冲突
- 幻觉方向向量在t=0计算一次后缓存复用，开销极小

### 关键发现
- SVC和CRC各自有效，组合后进一步提升——验证协同设计
- Token裁剪负样本 >> 图像遮挡负样本（CRC: 86.11 vs 84.77），t-SNE可视化证实裁剪样本在分布内
- $N_h=5$ 是精确的操作点：≥20个token不足以触发偏差探测

## 亮点与洞察

- **首个统一框架**: 将增强和抑制两条独立路线统一为vision token的双重操作，解决了朴素组合失败的问题
- **Information-gap > Modality-gap**: 在隐空间裁剪token比像素级遮挡更稳定、更in-distribution，这一发现对后续工作有指导意义
- **极低开销**: 仅1.06×延迟，显著优于VCD的2.4×，因为幻觉方向向量只需计算一次
- **跨架构泛化**: 在linear projection(LLaVA/Shikra)和Q-Former(MiniGPT-4/InstructBLIP)两类架构上均有效

## 局限性 / 可改进方向

- 仅在~7B规模MLLM上验证，更大模型(如70B+)的效果未知
- SVC的增强策略（翻转+模糊+噪声）较固定，可能不是最优的语义互补组合
- CRC假设幻觉方向在生成过程中不变（t=0计算后缓存），长文本生成时方向可能漂移
- 未在video/3D等多模态场景中验证

## 相关工作与启发

- **vs VCD**: VCD在logit层用图像扭曲做对比解码，延迟2.4×；本文在隐表示层用token裁剪，延迟仅1.06×，且负样本质量更高
- **vs PAI**: PAI仅增强注意力权重，无法对抗强语言先验；本文SVC+CRC双路协同
- **vs VISTA**: VISTA也在中间层干预但只做视觉增强；本文增加了CRC抑制路径
- **启发**: Token裁剪作为因果探针的思路可推广到其他需要separating信号的场景（如bias检测、可解释性分析）
