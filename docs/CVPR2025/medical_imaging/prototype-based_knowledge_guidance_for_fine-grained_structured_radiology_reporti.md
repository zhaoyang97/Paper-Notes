# Prototype-Based Knowledge Guidance for Fine-Grained Structured Radiology Reporting

**会议**: CVPR2025  
**arXiv**: [2603.11938](https://arxiv.org/abs/2603.11938)  
**代码**: 待发布  
**领域**: medical_imaging  
**关键词**: structured radiology reporting, prototype learning, knowledge base, free-text mining, late fusion, VQA

## 一句话总结

ProtoSR 提出从大规模自由文本放射学报告中挖掘模板对齐的原型知识库，并通过原型条件化的后期融合残差模块注入结构化报告预测，在 Rad-ReStruct 基准上实现 SOTA，尤其在细粒度属性问题 (L3) 上获得 72.1% 的相对提升。

## 研究背景与动机

- **结构化报告 (SR) 的价值**: 相比自由文本，结构化报告使用预定义字段和标准化答案选项，能提升一致性、完整性，并支持质量监控和下游分析
- **自动化 SR 的挑战**: 细粒度模板包含大量罕见属性（如位置、外观、严重程度），而结构化数据集（如 Rad-ReStruct 仅 3597 个样本）规模有限，导致长尾属性的监督信号稀疏
- **自由文本报告的丰富性**: MIMIC-CXR 等公开数据集包含 22 万+ 配对的胸部 X 光和自由文本报告，隐含了丰富的图像关联细粒度信息
- **现有方法的差距**: 通用医学 VLM（MedGemma、CheXagent）在 SR 任务上仍落后于专用模型；知识集成方法主要面向非结构化输出，未解决细粒度离散决策的问题
- **核心动机**: 如何将丰富但格式不统一的自由文本报告转化为与 SR 模板对齐的知识，并有效注入结构化预测流水线

## 方法详解

### 1. 知识库构建 (Knowledge Base Construction)

将大规模自由文本报告语料 (MIMIC-CXR) 映射到结构化模板标签空间：

- **术语扩展 (Terminology Expansion)**: 使用 zero-shot LLM 为每个模板标签生成同义词、缩写和替代表述，构建变体 → 规范标签的映射词典
- **模板约束提取 (Template-Constrained Extraction)**: 对每份报告，逐层级查询 LLM 判断发现是否存在，若存在则提取属性值；使用约束解码确保只输出合法答案
- **后处理与原型构建**: 规则过滤去噪 + 层级一致性检查（无子标签支持则移除父标签）；每个标签均匀采样最多 K=5 张图像，用图像编码器提取特征后通过 element-wise max pooling 聚合为一个原型向量
- **提取模型选择**: 对比 Llama 3.1 8B / Mistral 8B / Qwen2.5 7B，Qwen2.5-7B-Instruct + 术语扩展效果最优

### 2. 知识增强的后期融合架构 (ProtoSR)

- **基础模型**: 沿用 Rad-ReStruct 的层级 VQA 架构，EfficientNet-B5 (图像) + RadBERT (文本) → Transformer 融合 → 分类头输出 base logits
- **原型条件化知识分支**:
  1. 将融合表征 S 与原型嵌入 P 投影到共享空间
  2. 计算余弦相似度权重 α（仅考虑当前问题合法答案对应的原型）
  3. 加权聚合得到原型特征向量 v 和答案支持向量 u
  4. 拼接 [v; u] 经 MLP 生成支持偏置 $b_{\text{sup}}$
- **后期融合**: $z_{\text{final}} = z_{\text{base}} + s \odot b_{\text{sup}}$，其中 s 为可学习的逐答案维度缩放向量
- **设计优势**: 保留基础模型决策路径，仅在原型证据有意义时做定向修正；无原型时 bias 为零不干扰
- **原型对齐**: 使用 EMA 副本的图像编码器每 10k 步刷新原型嵌入，保持训练中的对齐

### 3. 训练细节
- 端到端训练，使用与 Rad-ReStruct 相同的多标签目标
- 单张 RTX 3090 (24GB)，34 epochs，学习率 1e-5，batch size 8 + 梯度累积 4

## 实验关键数据

### Rad-ReStruct 基准性能
| 方法 | Overall F1 | L1-F1 | L2-F1 | L3-F1 |
|------|-----------|-------|-------|-------|
| MedGemma | 26.8 | 38.2 | 63.4 | 2.8 |
| CheXagent | 32.4 | 62.1 | 69.8 | 6.2 |
| hi-VQA (Rad-ReStruct) | 32.0 | 64.6 | 71.6 | 4.1 |
| Context-VQA | 32.9 | 67.2 | 71.8 | 3.2 |
| **ProtoSR** | **34.4** | 66.2 | **72.8** | **7.4** |

- L3（细粒度属性）上 ProtoSR 达 7.4，相对基线 4.3 提升 **72.1%**
- Overall F1 相对 Context-VQA 提升 1.5 绝对值

### 消融实验
| 变体 | Overall F1 | L3-F1 |
|------|-----------|-------|
| No knowledge | 32.5 | 4.3 |
| Early Fusion | 32.5 | 4.3 |
| 随机原型 | 32.7 | 4.4 |
| **ProtoSR** | **34.4** | **7.4** |

- 早期融合和随机原型均无效，证明增益来自原型内容而非额外参数容量
- 随机原型能力等同无知识基线，说明模型学会了忽略无信息信号

## 亮点

1. **自由文本 → 结构化知识的桥梁**: 将非结构化报告系统性转化为模板对齐的原型知识库，解决了 SR 数据稀缺问题
2. **优雅的后期融合设计**: 残差 + 可学习缩放的设计保守而有效，不破坏基础模型已学到的模式，仅做定向修正
3. **L3 细粒度属性上的突破**: 72.1% 相对提升集中在最难的长尾细粒度属性上，恰好是结构化报告最需要改进的地方
4. **完备的消融验证**: 早期融合、随机原型、无知识三个消融干净地证明了方法有效性
5. **轻量级扩展**: 知识分支仅增加一个 MLP 和一个缩放向量，几乎不增加推理开销
6. **术语扩展的重要性**: 简单的 LLM 同义词扩展在所有提取模型上均带来大幅改善

## 局限性

1. **绝对性能仍偏低**: L3-F1 虽大幅提升但仅 7.4，距离实用仍有巨大差距；Overall F1 34.4 也较低
2. **仅在单一基准验证**: 只在 Rad-ReStruct (ChestXR) 上测试，未验证在其他模态/解剖区域的泛化
3. **知识库提取质量受 LLM 限制**: 7B 模型的提取精度有限（L3-F1 约 80.6），噪声会传播到原型中
4. **原型构建较简单**: max pooling 聚合 K=5 个样本可能不足以代表标签的视觉多样性，更大规模可能受限
5. **Report Accuracy 未最优**: ProtoSR 的报告级准确率 (36.6%) 低于 Context-VQA (39.7%)，说明整体一致性有待改善
6. **未与最新 VLM 对比**: MedGemma/CheXagent 直接用于 SR 并不公平，缺少与更适配的多轮 VLM 的对比
