# GIIM: Graph-based Learning of Inter- and Intra-view Dependencies for Multi-view Medical Image Diagnosis

**会议**: CVPR2025  
**arXiv**: [2603.09446](https://arxiv.org/abs/2603.09446)  
**代码**: 待确认  
**领域**: medical_imaging  
**关键词**: graph neural network, multi-view, heterogeneous graph, missing view, medical image classification

## 一句话总结

提出 GIIM，一种基于多异构图（MHG）的多视图医学图像分类框架，同时建模视图内（intra-view）和视图间（inter-view）的病灶依赖关系，在肝脏 CT、乳腺 X 线和乳腺 MRI 三种模态上均显著优于现有多视图方法，并对缺失视图具有鲁棒性。

## 研究背景与动机

临床诊断需要综合分析多视图/多时相医学图像中异常的相互关系。例如，多相 CT 中不同时相的增强模式差异是肝脏病灶分类的关键线索；乳腺 X 线的 CC 和 MLO 视图提供互补信息。然而，现有多视图 CADx 方法存在两个核心不足：(1) 独立分析各病灶，忽略了同一视图内多个病灶的空间关系（intra-view）和同一病灶跨视图的动态变化（inter-view）；(2) 实际中常出现视图缺失（如协议差异、技术故障），严重影响预测可靠性。CNN 和 Transformer 需要固定大小输入，难以灵活建模可变数量的病灶及其复杂关系。

## 方法详解

### 整体框架（两阶段）

**阶段 1：单视图特征提取**
- 为每个视图 $v$ 独立训练 ConvNeXt 特征提取器 $f_v$
- 冻结训练好的骨干网络，提取各视图的病灶特征

**阶段 2：多异构图构建与推理**
- 以患者为单位构建异构图，通过消息传递进行分类

### 图结构设计

**节点类型**：
- **单视图节点** $N_{single}$：每个视图的病灶特征，$V$ 个视图各一个节点
- **多视图节点** $M_{multi}$：所有单视图节点特征的拼接，汇总全局信息

**四种边类型（核心创新）**：
1. **Intra-tumor, Inter-view（$E_{intra}$）**：连接同一病灶的不同视图节点，捕获时间/角度变化
2. **Single-to-Multi-view（$E_{s-m}$）**：连接单视图节点到对应的多视图汇总节点
3. **Inter-tumor, Single-view（$E_{inter-s}$）**：连接同一视图中的不同病灶，建模空间关系
4. **Inter-tumor, Multi-view（$E_{inter-m}$）**：连接不同病灶的多视图汇总节点，建模高层上下文

### 异构消息传递

对每个节点 $n$ 在第 $k$ 层分别从两类邻居聚合信息：
- 从单视图邻居聚合：$h_{N_{single}(n)}^k = \frac{1}{|N_{single}(n)|} \sum_{u} \mathbf{W}_{single}^k h_u^{k-1}$
- 从多视图邻居聚合：$h_{M_{multi}(n)}^k = \frac{1}{|M_{multi}(n)|} \sum_{u} \mathbf{W}_{multi}^k h_u^{k-1}$

最终更新：$h_n^k = \sigma(\mathbf{W}^k \cdot \text{CONCAT}(h_n^{k-1}, h_{N_{single}(n)}^k, h_{M_{multi}(n)}^k))$

架构使用 5 层 SAGEConv（512→256→128→64→32→类别数）。

### 缺失视图处理（四种策略）

1. **Constant**：零向量填充，简单有效
2. **Learnable**：随机初始化可学习张量 + Frobenius 归一化
3. **RAG-based**：基于检索，用余弦相似度从数据库找最相似样本的对应视图特征
4. **Covariance-based**：基于协方差相似度检索，利用视图间特征差异的统计关系

## 实验关键数据

**多视图分类（三个数据集）**：

| 数据集 | 方法 | Acc (%) | AUC (%) |
|--------|------|---------|---------|
| Liver (877例) | Attention-based | 73.41 | 88.53 |
| | **GIIM** | **78.20** | **91.05** |
| VinDr-Mammo | Attention-based | 68.09 | 81.00 |
| | **GIIM** | **71.17** | **82.54** |
| BreastDM (232例) | Attention-based | 85.1 | 76.37 |
| | **GIIM** | **87.23** | **89.02** |

**关键发现**：
- 多视图方法在肝脏数据集上比单视图提升约 12% Acc、8.3% AUC
- GIIM 比次优多视图方法在肝脏数据集提升 3% Acc、2% AUC
- 缺失视图场景：GIIM (constant) 在完全缺失测试集上仍保持 72-74% 准确率（肝脏），显著优于其他方法
- 完整视图 vs 缺失视图性能差距：肝脏约 4%，乳腺 X 线约 5%

## 亮点

- 四种边类型的异构图设计全面捕获了多视图医学图像中的所有关键关系
- GNN 天然适配不定数量病灶的建模，比 CNN/Transformer 更灵活
- 在 CT、X 线、MRI 三种不同模态上均验证有效，具有通用性
- 四种缺失视图处理策略提供了实际部署的灵活选择（完整视图用 RAG/Covariance，缺失视图用 Constant）
- 两阶段训练策略简单有效，特征提取与关系建模解耦

## 局限性

- 特征提取器和图模型分阶段训练，未端到端优化，可能限制整体性能
- 乳腺 X 线实验中 CC-MLO 视图的病灶对应关系处理简化（用均值池化代替精确匹配）
- 肝脏数据集为私有数据，可复现性受限
- 图构建依赖精确的病灶检测/标注，未讨论检测误差对下游分类的影响
- 仅验证分类任务，未扩展到分割或检测

## 相关工作

- **Phase Attention** (Wang et al., 2022)：注意力机制加权不同视图特征，但不建模病灶间关系
- **SSL-MNGCN/MMGCN** (Ibrahim et al., 2022)：GCN 用于乳腺 X 线，但仅建模纹理和空间特征
- **ConvNeXt** (Liu et al., 2022)：本文使用的单视图特征提取骨干
- **Multi-phase fusion**：传统方法通过对齐+融合组合多视图，但忽略病灶间依赖
