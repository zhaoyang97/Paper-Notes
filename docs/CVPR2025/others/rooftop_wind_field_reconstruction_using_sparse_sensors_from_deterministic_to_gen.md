# Rooftop Wind Field Reconstruction Using Sparse Sensors: From Deterministic to Generative Learning Methods

**会议**: CVPR2025  
**arXiv**: [2603.13077](https://arxiv.org/abs/2603.13077)  
**代码**: [GitHub](https://github.com/Yng314/windreconstruction)  
**领域**: others  
**关键词**: 风场重建, 稀疏传感器, 深度学习, PIV测量, 传感器优化

## 一句话总结

提出基于风洞PIV实验数据的学习框架，系统对比Kriging插值与三种深度学习模型（UNet、ViTAE、CWGAN）在稀疏传感器屋顶风场重建任务中的表现，并结合QR分解优化传感器布局。深度学习在混合风向训练下全面优于Kriging，SSIM提升最高33.5%。

## 研究背景与动机

- 屋顶风场的实时分布信息对无人机起降、风控系统和屋顶利用至关重要，但屋顶流场具有强非线性、分离流和方向依赖性
- 传统CFD方法计算成本高且缺乏实时性，现场测量受传感器数量限制
- 现有风场重建研究大多依赖CFD模拟数据训练，模拟数据可能包含湍流闭合模型和离散化方案引入的系统偏差
- 已有工作通常仅研究单一网络架构，缺少不同主流深度学习框架之间的系统对比
- 传感器放置策略多为预定义均匀网格，缺乏数据驱动的优化
- 方向特定训练策略（仅在相同风向上训练和测试）限制了跨方向泛化能力

## 方法详解

**整体框架**: 建立"从观测中学习"的实验框架，使用东京大学风洞PIV测量数据（几何比例1:200，模型高度H=0.2m），在15×15网格域上重建屋顶风场。风洞实验覆盖3个风向（0°、22.5°、45°），每个方向2-3次独立实验，每次实验约8秒数据（时间分辨率0.001s，7999个瞬时快照）。

**四种重建方法**:
1. **Kriging插值**: 经典地统计基线，采用高斯变异函数模型（相关长度0.5-10.0网格单元），假设空间平稳性，nugget设为零强制精确插值
2. **UNet**: 编码器-解码器结构+跳跃连接，输入15×15×3（u/v速度分量+传感器掩码），滤波器32→128，参数量47万
3. **CWGAN**: 条件Wasserstein GAN，生成器基于UNet架构处理传感器测量+噪声输入，判别器用带步长卷积+LeakyReLU+BatchNorm（64→512通道），参数量877万，计算量为UNet的45.6倍。生成器损失结合对抗目标+L1重建损失（权重100:1），每次生成器更新前做5次判别器更新
4. **ViTAE**: 视觉Transformer自编码器，3×3 patch嵌入→25个patch→64维特征，8层Transformer+8头注意力+MLP扩展比4.0，后接CNN解码器，参数量467K
- 所有深度学习模型输入为15×15×3（u速度、v速度、传感器掩码），输出为15×15×2（重建的u/v速度场）
- 训练均使用Adam优化器+自适应学习率衰减+early stopping

**两种训练策略**:
- **SDT（单方向训练）**: 仅用0°数据训练（3次实验），在22.5°/45°方向测试跨方向泛化能力
- **MDT（混合方向训练）**: 每个方向各取一次实验用于训练（0°+22.5°+45°），其余独立实验用于测试
- 数据按独立实验划分（非随机快照采样），避免时间泄露
- 训练内部再按80/20划分训练/验证集，early stopping patience=20

**传感器配置**: 评估6种传感器数量（5/10/15/20/25/30），基于Voronoi划分的均匀分布+±1网格位置扰动测试部署鲁棒性

**QR传感器优化**: 对风场数据矩阵做SVD提取前40个POD模式（保留84.6%能量），对POD基的转置做QR列主元分解，按列重要性排序选取最优传感器位置

**评估指标**: SSIM（结构相似度）、MG（几何均值偏差，理想值1.0）、NMSE（归一化均方误差）、FAC2（2倍因子率）

## 实验关键数据

| 指标 | Kriging vs 深度学习改进（MDT） |
|------|-------------------------------|
| SSIM | 深度学习优于Kriging 18.2–33.5% |
| FAC2 | 深度学习优于Kriging 3.5–24.2% |
| NMSE | 深度学习优于Kriging 10.2–27.8% |

- **SDT下5个传感器**: Kriging SSIM=0.502，深度学习仅0.194-0.237（Kriging更优）
- **MDT下5个传感器**: 深度学习SSIM 0.531-0.550 vs Kriging 0.415（深度学习反超）
- **MDT对深度学习SSIM的提升**: 5传感器时提升131.2-146.0%，30传感器时仍有4.0-7.4%
- **推理速度**: UNet ~0.109ms/snapshot，Kriging ~1.493ms（慢13.7倍）
- **QR优化鲁棒性**: MDT下90%指标获正向改进，平均提升3.7%
- **模型效率**: UNet参数量471K/0.0285 GFLOPs，ViTAE参数量467K/0.0210 GFLOPs，CWGAN参数量8.77M/1.301 GFLOPs
- **CWGAN鲁棒性最佳**: 传感器扰动下FAC2退化仅1.7%，优于UNet/ViTAE的3.6-4.4%
- **UNet几何精度最高**: MDT下MG值0.986-1.072（最接近理想值1.0）
- **中等传感器密度（15-25）对扰动最敏感**: 稀疏配置假设少不易违反，密集配置冗余足够
- **0°风向最难重建**: 因流场分离泡带来的高边界-中心差异和空间梯度

## 亮点

- 首个系统使用风洞PIV实验数据（而非CFD模拟）训练和评估风场重建模型的工作，训练数据包含真实噪声和自然湍流变化
- 全面对比传统方法与三种代表性深度学习架构（确定性UNet、生成式CWGAN、混合ViTAE），揭示不同场景下的最优选择
- 发现混合风向训练对深度学习至关重要（SSIM提升最高173.7%），而对Kriging反而有害——因为混合方向破坏了Kriging的空间平稳性假设
- QR传感器优化方法提供数学严格的布局优化，在扰动下鲁棒性提升最高27.8%
- MDT下深度学习全面优于Kriging，为实际部署提供明确的方法选择建议：
  - UNet: 平衡精度的可靠默认选择
  - CWGAN: 最高结构相似度但计算成本高45倍
  - ViTAE: 效率最优（仅UNet 73.7%计算量），适合边缘实时场景
  - Kriging: 仅在单方向+极稀疏传感器时有优势
- 实验设计严谨：按独立实验划分训练/测试、传感器扰动分析、时间平均策略对比
- 代码开源，可复现性好

## 局限性

- 仅使用单个矩形建筑在一个测量高度（z/H=1.05）的数据，几何泛化能力未验证
- 仅测试3个风向（0°、22.5°、45°），去掉22.5°训练后性能明显下降，说明所有方向都必不可少
- QR传感器优化基于特定建筑/风况的POD基，换场景需重新计算
- 风洞实验尺度（1:200）与实际部署条件存在差距，未在真实大气条件下验证
- 深度学习在极稀疏传感器（5个）+单方向训练时SSIM落后Kriging 52.7-61.4%
- 未探索物理信息约束（physics-informed）损失函数
- 时间预测（下一步/多步）能力未涉及，仅做空间重建

## 与相关工作的对比

- vs POD-LSE: POD-LSE需要大量训练数据且难以处理非线性特征，本文的深度学习方法在非线性流场上有明显优势
- vs 已有GAN重建: 已有GAN工作多用CFD模拟数据训练且仅研究单一架构，本文首次在实验数据上系统对比多种架构
- vs 传统Kriging: 在多方向训练下深度学习全面超越，但在极稀疏+单方向场景Kriging仍有优势
