## 研究背景与动机

### 领域现状
- 工业产品设计中，边界表示（BRep）是CAD模型的事实标准存储格式
- BRep包含层次化几何原语：实体壳 → 面（平面、圆柱面等）→ 闭合环 → 边（线、圆、B样条）→ 角点
- 从3D扫描逆向重建CAD模型是工业中的核心需求

### 现有痛点
- **设计历史方法受限于DeepCAD的17万简单模型**：DeepCAD数据集主要包含简单长方体形状，几何和拓扑多样性极低
- **BRep方法数据量极小**：Fusion-360仅~8K样本，ABCParts仅32K样本
- **ABC数据集缺乏配套标注**：虽有100万CAD模型但缺少3D扫描、文本描述、BRep标签等
- **现有方法使用的输入点云来自低真实度的随机采样**，而非真实扫描
- 这形成了一个**死锁局面**：方法需要数据进步，数据需要方法生成

### 核心矛盾
大规模、多模态、高质量的BRep标注数据极度匮乏，严重制约了CAD逆向工程、BRep重建、草图到BRep生成、文本到BRep建模等方向的研究进展。

### 切入角度
直接面向ABC数据集的100万复杂CAD模型，系统性地生成涵盖3D扫描、草图、文本和BRep拓扑的全方位多模态标注，填补数据空白。

## 方法详解

### 整体框架
A2Z数据集包含四大类标注，总存储量近5TB：
1. 高分辨率3D扫描网格（含BRep标签）
2. 多技能等级的3D手绘草图
3. 多视角图像、文本描述和标签
4. 额外25K电子外壳CAD模型

### 关键设计1：高分辨率扫描模拟（Sec. 3.1）
将ABC的低多边形网格转化为模拟真实3D扫描的高分辨率网格，经过四步处理：
- **Step-I 网格上采样**：两次中点三角细分，得到约15万顶点、38万三角面的高分辨率网格
- **Step-II 小孔附近切向收缩**：识别小环路，模拟传感器视锥限制导致的遮挡和缺失
- **Step-III 表面粗糙度**：用多八度Perlin噪声场沿法线方向位移顶点，注入毫米级误差
- **Step-IV 凹痕和凸起**：在平面BRep面上随机添加局部高斯凹痕和正弦波凸起，模拟制造瑕疵

### 关键设计2：基于近邻感知的平滑标注（Sec. 3.2）
- 替代传统的硬最近邻标注规则，提出**多尺度、BRep边长感知、SPH加权的成员分配策略**
- 使用SPH核在K个不同尺度上聚合权重，选择概率软标签π(x)
- 标注覆盖率极高：边界ID 99.37%、边界类型 97.67%、环 99.99%、面ID 99.93%

### 关键设计3：多技能等级3D草图生成（Sec. 3.3）
- 使用单一技能参数κ∈{1,...,5}控制草图质量，模拟从业余到专业画师的不同水平
- 对线段、圆弧、一般曲线分别设计不同的位移场
- 线段：均值回归随机游走 + 端点渐细 + 弓形弯曲
- 圆弧：径向谐波碰撞 + 有意开口 + 切向抖动
- 共生成约500万张带BRep标注的3D草图

### 关键设计4：VLM驱动的文本标注（Sec. 3.4）
- 首次使用**VLM评审系统**（Qwen3-14B + InternVL-26B）自动化生成高质量描述
- 输入12张多视角透明渲染的BRep图像（4×3网格），生成≤200词的描述和≤20个标签
- 借鉴ImageNet/WordNet建立层次化类别结构（深度最多4层），包含6大特征类

### 基础模型架构（Sec. 4）
- 基于DGCNN骨干网络 + 二头结构（边界检测 + 角点检测）
- 逐点二分类（边界/非边界），使用focal loss
- 在300K CAD模型子集上训练，使用两块Nvidia H100 GPU训练20个epoch（4天）

## 亮点与洞察

1. **规模空前**：100万+CAD模型、1000万+标注，是目前已知最大的CAD逆向工程数据集
2. **多模态完整性**：唯一同时提供BRep标签、3D扫描、草图和文本的大规模数据集
3. **真实感扫描模拟**：四步几何变换流程系统性地模拟真实扫描噪声，包括遮挡、粗糙度、凹痕等
4. **VLM评审系统**创新性地用于CAD文本标注，成本效益高且质量有保证
5. **标注覆盖率惊人**：边界和面标注覆盖率均>97%，通过严谨的多阈值SPH加权策略实现
6. **强零样本迁移能力**：在CC3D上未经训练即达到高性能，证明数据的泛化价值

## 局限性 / 可改进方向

1. **无设计历史**：A2Z不包含CAD参数化操作序列，对需要设计历史的方法不适用
2. **3D扫描为模拟而非真实扫描**：虽然引入了多种噪声，但与真实扫描仪数据仍有分布差异
3. **角点检测仍有较大提升空间**：即使最佳方法的角点Recall仅0.732，因角点极度稀疏而具挑战性
4. **基础模型架构相对简单**：使用DGCNN骨干，未探索更先进的transformer等架构
5. **文本标注依赖VLM质量**：VLM对复杂机械零件的描述准确性仍有限
