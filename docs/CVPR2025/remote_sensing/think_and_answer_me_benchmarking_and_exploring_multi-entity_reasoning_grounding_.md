# Think and Answer ME: Benchmarking and Exploring Multi-Entity Reasoning Grounding in Remote Sensing

**会议**: CVPR2025  
**arXiv**: [2603.12788](https://arxiv.org/abs/2603.12788)  
**代码**: [github.com/CV-ShuchangLyu/ME-RSRG](https://github.com/CV-ShuchangLyu/ME-RSRG)  
**领域**: remote_sensing  
**关键词**: 遥感视觉定位, 多实体推理, 强化学习, GRPO, 视觉语言模型

## 一句话总结

构建遥感多实体推理定位基准 ME-RSRG（首个显式标注主体-客体角色的遥感定位数据集），提出 Entity-Aware Reasoning (EAR) 框架，结合 SFT 冷启动与实体感知奖励驱动的 GRPO 优化，实现结构化推理链输出和主-客体联合定位，Qwen2.5-VL 系列在 EAR 优化后 mAcc@0.5 提升超 10%。

## 研究背景与动机

- 推理语言模型（OpenAI-o1、DeepSeek-R1）展现了强大的多步推理能力，结合强化学习可验证奖励进一步增强
- 遥感视觉定位（RSVG）场景特征：大尺度空间布局、密集目标分布、复杂语义关系 → 天然适合推理范式
- **现有局限**：
  - 数据层面：现有遥感定位数据集仅标注**单实体**，缺乏多实体及其关系的结构化标注
  - 方法层面：现有方法本质是**感知级匹配**（将文本与视觉区域对齐），缺乏显式的中间推理过程、实体角色建模和实体间关系推理
- **核心观察**：遥感中目标身份通常由其与其他实体的关系决定（如"停机坪旁边的飞机"），而非孤立外观

## 方法详解

### ME-RSRG 数据集构建（5步流水线）

1. **数据收集**：整合 RSVG-HR、DIOR-RSVG、OPT-RSVG 三个数据集
2. **实例筛选**：用 Gemini-2.5-Pro 过滤，仅保留包含显式主-客体结构的描述
3. **人工标注**：继承原主体框，标注员进一步标注客体实体框，支持"单主体-单客体"和"单主体-多客体"
4. **专家审核**：两位专家顺序审核每个实例的框坐标和实体角色分配
5. **CoT 生成**：对约 20% 训练集用 Gemini-2.5-Pro 生成 \<think\> 推理描述

**数据规模**：7,162 张图，12,091 个图-文实例；训练 10,305 / 测试 1,786；2,149 条含 CoT 标注

### EAR 框架

**整体架构**
- 策略模型（视觉语言基础模型，如 Qwen2.5-VL）：输入图像+描述，输出 \<think\>推理链+\<answer\>主客体框
- 参考模型（策略模型冻结副本）：用于 KL 散度正则化
- 每个输入采样 G 个候选输出，由实体感知奖励模型评分

**两阶段优化**

**Stage I: SFT 冷启动**
- 用 2,149 条 CoT 标注数据训练模型学会输出结构（\<think\>+\<answer\>）、基本实体识别和粗定位

**Stage II: 实体感知奖励驱动 GRPO**
- 扩展到全部 10,305 训练样本（去除 CoT 标注），通过三种奖励信号优化：

1. **两级格式奖励 $R_{fmt}$**（最大 0.6）
   - 结构标签奖励（0.3）：\<think\>…\</think\>\<answer\>…\</answer\> 格式正确
   - 实体格式奖励（0.3）：\<answer\> 内实体格式（subject/object + 坐标）正确

2. **实体感知定位精度奖励 $R_{ent}$**
   - 分角色加权（主体 1.5，客体 1.25）
   - IoU 分级评分：>0.75→1.0, >0.5→0.8, >0.25→0.4, 否则→0

3. **关系一致性奖励 $R_{rel}$**（奖金机制）
   - 主-客体同时正确匹配 → +0.3
   - 多个客体正确匹配 → 额外 +0.3

## 实验关键数据

### 主实验（ME-RSRG 数据集）

| 模型 | Baseline mAcc | +SFT mAcc | +EAR mAcc | 提升 |
|------|--------------|-----------|-----------|------|
| Qwen2.5-VL-3B | 9.18% | 28.79% | **40.41%** | +31.23% |
| Qwen2.5-VL-7B | 10.35% | 33.34% | **46.72%** | +36.37% |
| Qwen3-VL-4B | 17.25% | 36.13% | **37.65%** | +20.40% |
| InternVL3.5-4B | 0.51% | 32.80% | **36.62%** | +36.11% |
| InternVL3.5-8B | 2.33% | 33.78% | **36.92%** | +34.59% |

- Qwen2.5-VL 系列在 EAR 优化后 mAcc@0.5 提升均超 10%（SFT→EAR）
- 最强结果：Qwen2.5-VL-7B 达 46.72% mAcc@0.5

### 消融实验
- GRPO-only（无 SFT）：Qwen2.5-VL-3B 仅 15.49% mAcc → SFT 是 GRPO 的必要前提
- 去除第一级格式奖励：推理链易崩溃，训练不稳定
- 去除关系一致性奖励：模型倾向仅预测单个客体，主体和客体精度均下降

## 亮点

1. **首个遥感多实体推理定位基准**：显式标注主-客体角色与关系，将遥感定位从"匹配"提升为"推理"
2. **奖励设计精细**：三层奖励（格式/定位/关系）覆盖从结构到语义的多层面，分级 IoU 评分提供密集反馈
3. **两阶段策略合理**：SFT 解决格式和初始化问题，GRPO 解决精度和推理质量问题，消融证明缺一不可
4. **广泛基准测试**：涉及 Qwen-VL、InternVL、LLaVA 等多系列模型，提供全面的基线参考

## 局限性

1. 数据集当前仅支持"单主体-单/多客体"设置，未覆盖"多主体"场景
2. CoT 标注依赖 Gemini-2.5-Pro 生成（仅 20% 训练集），可能引入偏差
3. 最优结果（46.72%）仍较低，说明多实体推理定位仍极具挑战
4. 用户实例筛选依赖 LLM，可能遗漏部分有效多实体样本或引入错误
5. 仅评估了 mAcc@0.5 单一 IoU 阈值，缺少不同阈值下的分析
