# milliFlow: Scene Flow Estimation on mmWave Radar Point Cloud for Human Motion Sensing

**会议**: ECCV2024  
**arXiv**: [2306.17010](https://arxiv.org/abs/2306.17010)  
**代码**: https://github.com/Toytiny/milliFlow  
**领域**: 3d_vision  
**关键词**: 毫米波雷达, 场景流, 人体运动感知, 点云, 自动标注

## 一句话总结
提出首个毫米波雷达点云场景流估计方法 milliFlow，通过多尺度特征提取、全局聚合、GRU 时序传播和约束回归，在自建数据集上将 EPE3D 从次优 0.107m 降至 0.046m（cm 级精度），并展示场景流特征对人体活动识别（+7.9%）、人体部位解析（+3.6%）、人体追踪等下游任务的增强效果。

## 研究背景与动机

1. **领域现状**：毫米波雷达因隐私保护（不拍摄图像）、全天候全光照工作成为人体感知研究热点。已有工作用雷达点云做活动识别、手势识别等，但缺乏底层运动表征。
2. **现有痛点**：雷达点云极度稀疏（~100 个点表示整个人体）、噪声大（多径反射产生鬼影点）、无纹理信息。LiDAR 场景流方法（FlowNet3D、PV-RAFT 等）直接迁移效果极差，因为它们针对密集点云设计。
3. **核心矛盾**：雷达提供的运动信息（径向 Doppler 速度）分辨率低且缺少切向分量，无法直接作为场景流；现有自动驾驶场景流方法假设密集点云和丰富几何结构。
4. **本文要解决**：(1) 如何在极稀疏、高噪声的雷达点云上估计高质量场景流？(2) 如何自动生成训练标签（避免人工标注）？(3) 场景流能否增强下游人体感知任务？
5. **切入角度**：利用共位安装的 RGB-D 相机做自动标注——OpenPose 提取 2D 关键点 + 深度提升到 3D → 构建骨架 → 为每个雷达点分配最近骨架的运动。
6. **核心 idea**：针对雷达特性设计五模块场景流网络（多尺度局部特征 + 全局聚合 + 时序 GRU + 约束回归），配合 RGB-D 相机自动标注方案构建首个雷达场景流数据集。

## 方法详解

### 整体框架
输入连续两帧雷达点云 $\mathcal{X}, \mathcal{Y}$，输出每个点的 3D 位移向量 $\mathcal{F} = \{f_i\}_{i=1}^N$。五个级联模块：局部特征提取 → 全局聚合 → 流嵌入生成 → 时序信息传播 → 约束场景流回归。

### 关键设计

1. **多尺度局部特征提取**:
   - 做什么：四个并行 Set Abstraction (SA) 层，分组半径分别为 [0.05, 0.1, 0.2, 0.4m]
   - 核心思路：不同半径捕获从精细肢体运动到粗粒度躯干运动的多尺度局部几何
   - 设计动机：雷达点云极稀疏，单尺度特征容易遗漏远距离点的关联，多尺度确保信息完整

2. **全局特征聚合 + GRU 时序传播**:
   - 做什么：用 MLP 注意力权重将局部特征聚合为全局特征向量；GRU 跨帧传播全局状态 $h^t = \text{GRU}(h^{t-1}, g^{\mathcal{B}})$
   - 核心思路：全局聚合让每个点"知道"整个身体的状态（如手臂挥动时躯干静止），GRU 利用时间连续性平滑运动估计
   - 设计动机：雷达丢点严重（某些身体部位帧间消失），时序信息可补偿单帧信息缺失

3. **约束场景流回归**:
   - 做什么：MLP 回归每点 3D 位移后，对每个轴做 clamping 约束 $f \in [-\epsilon, \epsilon]$，$\epsilon = 0.1$m
   - 核心思路：人体运动在相邻帧间位移有上限（~13Hz 帧率下人体最大速度对应约 10cm/帧）
   - 设计动机：约束防止异常大位移预测（由噪声或鬼影点导致），消融实验显示这是贡献最大的单一组件

4. **RGB-D 自动标注方案**:
   - 做什么：用共位 RGB-D 相机获取骨架运动作为伪标签
   - 核心思路：OpenPose 提取 2D 关键点（14 个，置信度 >0.5），用深度值提升到 3D，构建 13 段骨架。每个雷达点分配到最近骨架，其伪标签 = 该骨架端点的运动
   - 关键过滤：丢弃置信度 <0.5 的关键点和位移 >0.5m 的异常标签

### 损失函数 / 训练策略
加权 L2 loss 区分大/小幅运动：$\mathcal{L} = \alpha_l \mathcal{L}_{large} + \alpha_s \mathcal{L}_{small}$，$\alpha_l=2, \alpha_s=1$，阈值 $\zeta=0.1$m。有效掩码 $\mathcal{M}$ 过滤噪声标签。

## 实验关键数据

### 主实验

| 方法 | EPE3D All(m)↓ | Moving(m) | Static(m) | Acc3D Strict↑ | Acc3D Relax↑ |
|------|-------------|-----------|-----------|--------------|-------------|
| PV-RAFT | 0.161 | 0.170 | 0.107 | 0.179 | 0.292 |
| RaFlow | 0.107 | 0.115 | 0.094 | 0.271 | 0.427 |
| **milliFlow** | **0.046** | **0.051** | **0.009** | **0.406** | **0.703** |

EPE3D 从次优 0.107m 降至 0.046m（降幅 57%），Acc3D Relax 从 42.7% 提升至 70.3%。

### 消融实验

| 配置 | EPE3D(m)↓ | Acc3D Strict↑ | Acc3D Relax↑ |
|------|-----------|--------------|-------------|
| 完整模型 | 0.046 | 0.406 | 0.703 |
| w/o 约束回归 | 0.083 | 0.286 | 0.490 |
| w/o 上下文特征 | 0.071 | 0.315 | 0.536 |
| w/o 全局聚合 | 0.061 | 0.361 | 0.628 |
| w/o 时序传播 | 0.053 | 0.382 | 0.676 |

### 下游任务

| 任务 | 基线 | +场景流特征 | 增益 |
|------|------|-----------|------|
| 活动识别（HAR均值） | 48.14% | 56.01% | **+7.87%** |
| 人体解析（mIoU） | 49.09% | 52.72% | **+3.63%** |
| 身体部位追踪（mJE 1帧） | — | 0.025m | — |

### 关键发现
- **约束回归贡献最大**：去掉后 EPE3D 从 0.046 升至 0.083（+80%），说明噪声抑制对雷达至关重要
- **泛化到未见活动**：在训练集未包含的坐下/蹲下/点头上 EPE3D 仅 0.034m（比训练集上还好），说明学到了通用运动模式
- **实时推理**：74ms/帧，满足 13.5Hz 的雷达帧率要求，GPU 显存仅 134MB

## 亮点与洞察
- **首次将场景流引入毫米波雷达人体感知**：填补了一个重要空白——之前雷达人体感知要么直接用点云做分类，要么用 Doppler 做粗粒度分析。场景流提供了 per-point 的细粒度运动表征
- **自动标注方案巧妙实用**：利用共位 RGB-D 相机在训练阶段提供标签，推理时只需雷达即可。这种"训练用相机辅助、部署只用雷达"的范式解决了雷达数据缺乏标注的核心痛点
- **约束回归的简单有效**：就一个 clamp 操作，但对噪声点云的效果极其显著（消融最大），可泛化到任何稀疏传感器的运动估计

## 局限性 / 可改进方向
- **仅单人、正面、室内**：未处理多人场景、非正面朝向、室外环境（阳光干扰深度测量）
- **固定平台假设**：雷达静止安装，无自运动补偿；移动机器人/无人机场景需要额外处理
- **骨架假设限制**：将人体运动建模为刚性骨架运动，无法处理衣物飘动等非刚性形变
- **可改进**：(1) 用 4D 雷达（带高程信息）替代 3D 雷达提升点云密度；(2) 引入 Transformer 替代 GRU 做更长程时序建模

## 相关工作与启发
- **vs RaFlow**: 目前唯一的雷达场景流方法，也用 RGB-D 辅助标注。milliFlow 在 EPE3D 上提升 57%，因为 RaFlow 缺乏多尺度特征和时序建模
- **vs LiDAR 场景流方法（PV-RAFT 等）**: 这些方法在密集 LiDAR 点云上设计，直接用于稀疏雷达效果差（EPE3D > 0.15m）。milliFlow 针对雷达稀疏性做了专门优化
- **vs 纯 Doppler 方法**: Doppler 只提供径向速度且分辨率低，milliFlow 估计完整 3D 位移，信息量大幅提升
