# Genixer: Empowering Multimodal Large Language Model as a Powerful Data Generator

**会议**: ECCV2024  
**arXiv**: [2312.06731](https://arxiv.org/abs/2312.06731)  
**代码**: [zhaohengyuan1/Genixer](https://github.com/zhaohengyuan1/Genixer)  
**领域**: multimodal_vlm  
**关键词**: 多模态大语言模型, 视觉指令微调, 数据生成, 数据过滤, VQA, 视觉定位

## 一句话总结

提出 Genixer 数据生成流水线，训练 MLLM 自身作为数据生成器，无需依赖 GPT-4V 即可自动生成高质量视觉指令微调数据，生成的 915K VQA 数据和 350K REC 数据分别提升 LLaVA1.5 和 Shikra 在多个基准上的表现。

## 研究背景与动机

当前多模态大语言模型（MLLM）的训练高度依赖视觉指令微调数据，而数据来源主要有两条路径：

1. **现有 VL 数据集转换**：如 InstructBLIP 将传统 VQA/Caption 数据集转为指令格式，但图像多源自 COCO，多样性受限，泛化能力不足。
2. **调用 GPT-4 生成**：如 LLaVA、Shikra、ShareGPT4V 依赖 GPT-4 生成指令数据。存在两大问题：(a) 大规模生成的经济成本高昂；(b) GPT-4V 在复杂任务（如 REC 视觉定位）上生成质量不佳，无法输出正确的 bounding box。

**核心动机**：既然现有 MLLM 已具备强大的多模态理解能力，能否训练 MLLM 自身成为数据生成器，摆脱对 GPT-4V 的依赖？这样既零额外成本，又能灵活为任意无标注图像生成高质量指令数据。

## 方法详解

### 整体框架

Genixer 流水线包含四个关键步骤：

1. **指令数据收集**：选取 9 类代表性 VL 任务，分为通用任务（Common VQA、Adv VQA、MC VQA、Multi-turn Dialogue）和定位任务（REC、REG、PointQA、Q→CBoxA、Referential Dialogue）。
2. **指令模板设计**：提出两级指令模板，支持 Task-agnostic（模型自由生成任意类型数据）和 Task-specific（通过特定指令控制生成任务类型）两种模式。
3. **赋能 MLLM**：以 LLaVA1.5 为骨干训练 GenixerL（通用任务生成器），以 Shikra 为骨干训练 GenixerS（定位任务生成器）。
4. **数据生成与过滤**：用 1.4M 无标注图像喂入生成器，再通过自动过滤框架筛选高质量样本。

### 关键设计

**两级指令模板**：

- **通用指令（Generic Instruction）**：从 58 条手写指令中随机采样，如"Please provide a clear and direct question and answer after examining the image"，让模型自由决定生成何种任务类型。
- **特定指令（Specific Instruction）**：如"This is a Common VQA task"，精确控制输出任务类型。
- **控制常数 τ**：训练时控制仅使用通用指令的样本比例（Common VQA τ=0.2, MC VQA τ=0.5 等），平衡不同数据规模的任务。

**两阶段训练（GenixerS）**：由于 Referential Dialogue 等任务数据量极小（仅 1.8K），GenixerS 采用两阶段训练——第一阶段专注 REC/REG 数据生成；第二阶段加入 PointQA、Q→CBoxA、RD，同时削减 REC/REG 数据体量以实现平衡。

**数据过滤框架**：

- **Fuyu-driven 过滤（通用任务）**：将生成的 QA 对喂入 Fuyu-8B，计算模型预测"Yes"的概率 P(Yr)，设定阈值 λ=0.7 过滤低质量样本。1.4M → 915K。
- **CLIP-driven 过滤（定位任务）**：三步粗到细过滤——(1) 正则表达式检查格式；(2) 移除宽/高 < 50 的 box；(3) 用 OpenCLIP-L 计算文本表达与图像区域的相似度，阈值 0.6。1.4M → 350K。

### 损失函数

标准自回归语言建模损失，训练目标为给定图像和指令，生成问题-答案对：

$$\max \sum_{i=1}^{L} \log p_\theta(x_i | X_G, X_S, X_I, X_{o,<i})$$

其中 $X_G$, $X_S$ 分别为通用/特定指令，$X_I$ 为图像，$X_o$ 为完整输出序列（问题+答案），$\theta$ 为可训练参数。GenixerL 使用 AdamW, lr=1e-5, batch=128, 训练 1 epoch（约 14 小时）。GenixerS 两阶段分别 lr=3e-5/1e-5, batch=128/64。

### 关键发现

1. **GenixerL 生成数据质量高**：经 Fuyu-8B 评估，Common VQA 准确率 82.4%，MC VQA 87.8%，MD 82.5%，平均概率均 > 0.8。
2. **GenixerS 超越 GPT-4V**：在 REC 数据生成上，GPT-4V 无法正确输出 bounding box，而 GenixerS 可以。用户偏好研究中，REC 任务上用户更偏好 Genixer。
3. **合成数据缓解幻觉**：POPE 基准从 85.9 提升至 87.3，说明合成数据有助于抑制模型幻觉。
4. **GenixerL 混合训练后全面提升**：与 LLaVA1.5 微调数据混合训练后，6 个基准全面大幅提升（VizWiz +4.1%, POPE +1.6%）。

## 亮点与洞察

1. **首次证明 MLLM 可自举生成数据**：不依赖 GPT-4 等商业模型，当前开源 MLLM 经过适当训练即可成为有效的数据生成器，打开了"自我进化"的可能性。
2. **两级指令模板设计巧妙**：通过控制常数 τ 和两级指令，优雅地实现了 task-agnostic 和 task-specific 两种生成模式的切换。
3. **两种互补的数据过滤框架**：Fuyu-driven（基于概率阈值）和 CLIP-driven（基于视觉-文本相似度）分别适配通用和定位任务，自动化且有效。
4. **实验设计严谨**：同时涵盖统计分析、人工评估、训练评估、消融、可视化、用户研究六个维度，说服力强。
5. **在 GPT-4V 失败的地方成功**：REC 任务上 GenixerS 生成质量明显优于 GPT-4V，展示了任务特化训练的优势。

## 局限性

1. **LLM 规模受限**：仅在 7B 模型上验证，未测试 13B/34B 等更大规模模型，更大模型可能获益更多。
2. **图像规模受限**：仅使用 1.4M 图像语料，未扩展到 LAION-2B 等更大图像库，而消融实验已显示数据越多提升越大。
3. **开放式任务过滤困难**：对 Referential Dialogue 等复杂开放式任务，自动过滤质量评估仍具挑战。
4. **部分指标微降**：LLaVA-Bench 和 MM-Vet 略有下降（约 1 分），可能与 GPT-4 作为评估指标引入偏差有关。
5. **Genixer-350K 数据集均来自 REC 任务**，未覆盖全部 5 类定位任务的大规模生成。

## 相关工作与启发

- **与 LLaVA 系列的关系**：Genixer 基于 LLaVA1.5 架构，但训练目标从"理解"转为"生成数据"。生成的数据又反哺 LLaVA1.5 提升性能，形成自我增强闭环。
- **与 Self-Instruct 的异同**：LLM 领域的 Self-Instruct 用模型生成文本指令数据，Genixer 将此思路首次扩展到多模态领域，但额外引入了图像条件生成和自动过滤。
- **与 ShareGPT4V 的对比**：ShareGPT4V 仍依赖 GPT-4V 生成 caption 数据，Genixer 完全不依赖商业模型。
- **数据过滤启发**：Fuyu-driven 概率阈值方法是一种通用范式——用更强/不同模型验证生成模型的输出质量，可扩展到更多场景。
- **对定位任务数据生成的启发**：GPT-4V 无法可靠生成 bounding box 坐标，而任务特化微调可以解决，说明通用大模型不一定在所有下游任务上最优。
