# Reprojection Errors as Prompts for Efficient Scene Coordinate Regression

**会议**: ECCV 2024  
**arXiv**: [2409.04178](https://arxiv.org/abs/2409.04178)  
**代码**: [https://tingru0203.github.io/egfs](https://tingru0203.github.io/egfs)  
**领域**: 3D Vision / Visual Localization  
**关键词**: scene coordinate regression, visual localization, SAM, feature selection, confidence map

## 一句话总结

本文提出 EGFS（Error-Guided Feature Selection）机制，利用低重投影误差区域作为 SAM 的 point prompts 扩展为语义掩码，迭代地筛选可靠训练样本，在 Cambridge Landmarks 和 Indoor6 数据集上以更小模型和更少训练时间超越现有无 3D 信息依赖的 SCR 方法。

## 研究背景与动机

1. **领域现状**: 视觉定位（Visual Localization）需要从图像估计 6-DoF 相机位姿，是 AR/VR、自动驾驶的核心技术。场景坐标回归（SCR）方法通过 DNN 直接预测像素对应的 3D 坐标，再用 PnP+RANSAC 估计位姿，具有模型小、训练快的优势。ACE 方法可在 5 分钟内训练完成。
2. **现有痛点**: SCR 方法在训练时对所有图像区域均匀采样，包括动态物体（行人、车辆）和无纹理区域（墙壁、天空），这些区域的特征不稳定，会产生高重投影误差，导致模型权重更新不稳定。
3. **核心矛盾**: 虽然 RANSAC 可以在推理时过滤离群点，但训练阶段仍然在这些有害区域上浪费计算资源并引入噪声梯度。端到端优化方法试图下调离群点权重，但未考虑区域的语义信息。
4. **关键发现**: 作者通过深入分析发现——低重投影误差区域通常有更高的 inlier 比率，但低误差并不总对应特定语义类别（如"tree"在不同场景中误差差异很大），因此不能简单使用预定义的语义类别来选择区域。
5. **切入角度**: 不依赖预定义语义类别，而是用**重投影误差本身**作为信号，选择低误差点作为 SAM 的 prompt，自动扩展为语义一致的掩码区域。
6. **核心 idea**: 用低重投影误差点 → SAM prompt → 语义掩码 → 置信度精炼 → 迭代更新训练区域。

## 方法详解

### 整体框架

每次迭代训练 $k$ 个 epoch → 计算重投影误差 → 选前 $\tau$% 低误差点作为 SAM prompts → SAM 扩展为语义掩码 → 置信度图精炼掩码 → 仅在掩码内采样特征训练场景特定 MLP → 重复迭代。推理时不需要掩码，仅用置信度图过滤并估计位姿。

### 关键设计

1. **Error-Guided Feature Selection (EGFS)**:
   - 做什么：从重投影误差中选择可靠训练区域
   - 核心思路：选择前 $\tau$%（默认 10%）低重投影误差的像素点作为 point prompts，输入 EfficientViT-SAM-L0，将稀疏点扩展为完整的语义掩码。掩码每 $k=5$ 个 epoch 更新一次，实现从粗到细的迭代精炼。
   - 设计动机：直接按误差阈值采样点不如按语义区域采样——因为低误差点可能分散且不覆盖完整物体，而 SAM 能根据少数种子点推断出完整的语义区域边界

2. **Confidence Map Refinement**:
   - 做什么：进一步过滤 EGFS 掩码中不可靠的点
   - 核心思路：在场景特定 MLP 末端增加置信度预测头，输出每像素置信度 $c_i$。训练损失为：
     $$\ell(p_i, y_i, h_i^*) = \begin{cases} c_i \cdot \hat{r}(p_i, y_i, h_i^*) - \alpha \log c_i & \text{if } y_i \in \mathcal{V} \\ \|y_i - \bar{y_i}\|_0 - \alpha \log(1 - c_i) & \text{otherwise} \end{cases}$$
     在掩码内，低于中位数置信度的点被剔除；推理时也用中位数置信度过滤
   - 设计动机：EGFS 掩码虽然选出了语义一致的区域，但区域内仍可能有难以准确预测的局部位置（如遮挡边缘），置信度图提供了更细粒度的过滤

3. **Iterative Training**:
   - 做什么：每隔固定 epoch 动态更新训练掩码
   - 核心思路：总共训练 20 个 epoch，每 5 个 epoch 重新计算重投影误差并更新 EGFS 掩码。初始迭代使用随机全图采样以获得首批误差信号
   - 设计动机：随着训练进展，模型对场景的理解不断加深，低误差区域会动态变化，因此掩码需要持续更新而非一次性确定

### 损失函数 / 训练策略

- 基于 ACE 架构，场景特定 MLP 使用 $1 \times 1$ 卷积独立处理每个采样点
- 置信度正则化参数 $\alpha = 10$
- SAM 使用轻量化 EfficientViT-SAM-L0，保证高效
- 模型大小仅 4.5MB（ACE 为 4MB + 0.5MB 置信度头）

## 实验关键数据

### 主实验 — Cambridge Landmarks (cm/°)

| 场景 | EGFS (4.5MB) | ACE (4MB) | DSAC* (28MB) | ACE quad (16MB) |
|------|-------------|-----------|-------------|----------------|
| King's College | **14/0.3** | 28/0.4 | 18/0.3 | 18/0.3 |
| Great Court | 31/0.1 | 43/0.2 | 34/0.2 | **28/0.1** |
| Old Hospital | **21/0.4** | 31/0.6 | 21/0.4 | 25/0.5 |
| Shop Facade | **5/0.3** | 5/0.3 | 5/0.3 | 5/0.3 |
| St Mary's Church | 15/0.5 | 18/0.6 | 15/0.6 | **9/0.3** |
| **Average** | **17/0.3** | 25/0.4 | 19/0.4 | 17/0.3 |

### 主实验 — Indoor6 (5cm/5° 准确率 %)

| 场景 | EGFS (4.5MB) | ACE (4MB) | DSAC* (28MB) | ACE quad (16MB) |
|------|-------------|-----------|-------------|----------------|
| scene1 | 46.4% | 26.0% | 23.0% | 52.9% |
| scene2a | **60.6%** | 32.3% | 33.9% | 52.5% |
| scene4a | **78.7%** | 62.0% | 67.1% | 69.6% |
| **Average** | **56.1%** | 35.6% | 35.1% | 58.6% |

### 消融实验

| 配置 | Cambridge (cm/°) | Indoor6 (%) | 说明 |
|------|---------|------|------|
| 无 EGFS + 无置信度 | 25/0.4 | 35.6% | ACE 基线 |
| EGFS 掩码 only | 19/0.3 | 41.6% | 掩码选择即有显著提升 |
| 置信度 only | 19/0.4 | 50.9% | 置信度过滤效果更大 |
| EGFS + 置信度 | **17/0.3** | **56.1%** | 两者互补，最佳 |

### 关键发现

- **语义掩码 vs 纯误差阈值采样**: 在 Indoor6 上，纯按 $Q_{0.5}(R)$ 阈值采样仅达 44.3%，EGFS 掩码达 56.1%，提升 26.6%——证明语义一致的区域选择远优于逐像素过滤
- 低重投影误差和特定语义类别之间**没有稳定的对应关系**：同一类别在不同场景中的误差差异显著
- EGFS 掩码在训练过程中从粗到细逐步精炼，最终聚焦于对定位最有价值的区域
- EGFS 训练时间 12 分钟，略长于 ACE 5 分钟但远短于 DSAC* 15 小时

## 亮点与洞察

- **将重投影误差转化为 SAM prompts 的思路非常巧妙**：既不需要预定义语义类别，又能利用 foundation model 的泛化能力自动发现有益区域
- **置信度自适应加权的损失设计**可迁移到其他需要处理噪声标签/不可靠区域的任务
- **迭代式掩码更新**形成了一个 curriculum learning 的效果：模型先学简单区域，再逐步扩展到更困难的区域
- 展示了 **foundation model (SAM) 作为工具模型**在下游任务中的即插即用价值

## 局限性 / 可改进方向

- **仅在 ACE 架构上验证**: 未扩展到 DSAC* 或其他 SCR backbone
- **SAM 运行开销**: EfficientViT-SAM 虽然轻量，但每 5 epoch 需要对所有训练图生成掩码
- **室外大场景泛化性**: Cambridge 场景相对简单，缺少大规模城市级场景验证
- **prompt 策略单一**: 仅使用点 prompt，未探索 box prompt 或其他交互方式
- **可扩展到 3D 高斯/NeRF 场景表示**的重投影误差引导

## 相关工作与启发

- **vs ACE**: EGFS 在 ACE 基础上仅增加 0.5MB 和 ~7min 训练时间，但在 Indoor6 上准确率从 35.6% 提升至 56.1%
- **vs DSAC***: DSAC* 需要 15 小时训练且使用 28MB 模型，EGFS 12 分钟 + 4.5MB 即超越之
- **vs FocusTune**: FocusTune 通过权重微调来关注特定区域，EGFS 则通过数据采样策略来实现类似效果，更轻量
- **vs SACReg (w/ 3D model)**: 使用 3D 模型的方法天然有优势，EGFS 在不使用 3D 信息的前提下逼近其性能

## 评分

- 新颖性: ⭐⭐⭐⭐ 将重投影误差作为 foundation model 的 prompt 是新颖且优雅的交叉创新
- 实验充分度: ⭐⭐⭐⭐ 室内外两个数据集，丰富的消融和定性分析，但缺少更大规模场景验证
- 写作质量: ⭐⭐⭐⭐ 动机分析详细（Section 4 的误差-语义关系分析很有说服力），整体逻辑清晰
- 价值: ⭐⭐⭐⭐ 方法轻量、即插即用、思路可迁移，但应用场景偏窄（仅视觉定位）
